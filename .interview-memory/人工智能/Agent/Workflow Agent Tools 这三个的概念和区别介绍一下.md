# Workflow、Agent、Tools 这三个的概念和区别介绍一下？

## 场景触发词
Workflow vs Agent vs Tools、工作流和智能体区别、Agent 和工具关系、Workflow 和 Agent 怎么选

## 速记卡片
`流 | 流固灵，工被动 | Workflow vs Agent vs Tools`

## 原始知识

| 维度 | Workflow | Agent | Tools |
|------|----------|-------|-------|
| **本质** | 预设的执行流程（剧本） | 自主决策的执行实体（演员） | 被调用的外部能力（道具） |
| **灵活性** | 低，按固定步骤执行 | 高，根据环境动态调整 | 无，被动等待调用 |
| **决策权** | 无，人类编排好 | 有，LLM 自主判断下一步 | 无，只执行具体动作 |
| **典型例子** | 审批流、定时任务、ETL | ReAct Agent、AutoGPT | 搜索、代码执行、发邮件 |

**三者关系**：
```
Workflow（导演剧本）
    ↓ 编排
Agent（演员）
    ↓ 调用
Tools（道具）
```

- **Workflow** 是**编排层**：定义先做什么、后做什么，通常由人类预先设计
- **Agent** 是**决策层**：在 Workflow 的框架内（或独立）自主决定每一步怎么做
- **Tools** 是**能力层**：提供具体的功能接口，被 Agent 或 Workflow 调用

## 顺口溜
**流固灵，工被动**

## 单字锚点 + 防崩推导
- **锚点字**：流
- **扩展口诀**：流固灵，工被动
- **为什么选它**：“流”= Workflow，三者中偏“流程化”的代表，用它串起整个对比
- **防崩推导**：
  1. 流 → Workflow（工作流）：**固**定步骤、预设路径
  2. 固 → Agent：**灵**活决策、动态调整（与 Workflow 的“固”形成对偶）
  3. 灵 → Tools：被调用的“工”具，完全**被动**
  4. 工被动 → Tools 没有自主权，等 Agent 来调用

## 一字一画面
**流** —— 像 **工厂流水线**：[灰色传送带] 每个工位按固定顺序执行，不能跳过也不能回头，效率稳定但缺乏弹性。
**固** —— 像 **剧本**：[黄色纸张] 导演写好了每一场戏的对白和动作，演员照本宣科。
**灵** —— 像 **即兴演员**：[聚光灯下] 没有固定台词，根据观众反应随时调整表演，能应对突发状况。
**工** —— 像 **工具箱**：[红色铁箱] 里面放着锤子、螺丝刀、电钻，本身不会干活，等工人来拿。
**被动** —— 像 **遥控器**：[黑色] 不按按钮就不动，完全依赖外部触发。

## 面试话术
Workflow、Agent、Tools 是三个不同层次的概念。**Workflow 是编排层**，像导演写的剧本，定义固定的执行步骤；**Agent 是决策层**，像即兴演员，能根据环境自主判断下一步做什么；**Tools 是能力层**，像工具箱里的工具，本身不会主动行动，等 Agent 来调用。简单说：**Workflow 管顺序，Agent 管决策，Tools 管执行**。

## 防混补丁
**别混**：不要把 Agent 和 Workflow 混为一谈。口诀：**有剧本是流，会即兴是 Agent，干活的是工具**。
