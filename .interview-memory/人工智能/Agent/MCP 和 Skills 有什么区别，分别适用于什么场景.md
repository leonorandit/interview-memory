## 单字锚点 + 防崩推导

**锚点字：协**

- **扩展口诀**：协封装，两区别
- **为什么选它**："协"=协议，MCP本质是协议（Protocol），"协"字一听就是标准化协作的规则。
- **推导链**：
  1. 协 → 协议 → MCP是Model Context Protocol
  2. MCP是什么 → 标准化Agent与外部工具的通信协议
  3. 它的作用 → 跨平台互操作，解决"连接问题"
  4. 另一边 → Skills不是协议，是能力封装/实现模块
  5. Skills的作用 → 解决"能力问题"，面向业务功能复用
  6. 区别总结 → MCP=接口协议（怎么连），Skills=实现模块（做什么）
  7. 场景 → MCP适合标准化接入，Skills适合业务复用

## 一字一画面

**协** —— 像**国际通用电源转接头**：白色外壳，全球各国插孔整齐排列，任何电器插上就能通电（本质：MCP是统一接口，让不同工具能跨平台对话）。

## 面试话术

"MCP和Skills最大的区别在于，MCP是协议标准，解决的是'怎么连'的问题，而Skills是能力模块，解决的是'做什么'的问题。你可以把MCP理解成国际通用电源转接头，它定义了插头和插孔的标准，让任何电器都能插上就用；而Skills则是具体的电器本身，比如一个电饭煲或者吹风机。适用场景上，MCP更适合做标准化工具接入，Skills更适合做业务功能的复用。"

## 防混补丁

容易把MCP和Skills当成**一回事**？记住：**协是插头，技是电器**——没有插头电器没法通电，但插头本身不会煮饭。
