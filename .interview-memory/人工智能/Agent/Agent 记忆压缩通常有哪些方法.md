# Agent 记忆压缩通常有哪些方法？

## 场景触发词
Agent 记忆压缩、上下文压缩、对话摘要、Memory Compaction、Prompt 压缩、长期记忆压缩

## 速记卡片
`缩 | 缩摘滑失，四方法 | 记忆压缩`

## 原始知识

- **摘要法（Summarization）**
  - 将多轮对话生成一段摘要，保留关键信息
  - 实现：每 N 轮调用 LLM 生成摘要，替换原始对话
  - 优点：信息密度高，语义完整
  - 缺点：摘要本身消耗 Token，可能丢失细节

- **滑动窗口（Sliding Window）**
  - 只保留最近 K 轮对话，丢弃更早的内容
  - 实现：固定长度队列，超出即弹出
  - 优点：简单高效，零额外计算
  - 缺点：可能丢失关键历史上下文

- **重要性筛选（Importance Filtering）**
  - 评估每轮对话的重要性，只保留高价值信息
  - 实现：用模型打分或规则判断（如包含用户偏好、关键决策）
  - 优点：精准保留有价值记忆
  - 缺点：需要设计合理的评分机制

- **实体提取/结构化压缩（Entity Extraction）**
  - 从对话中提取实体-关系-事实，结构化存储
  - 实现：提取"用户喜欢 Python"这类三元组，丢弃冗余表达
  - 优点：压缩率极高，便于检索
  - 缺点：丢失情感、语气等非结构化信息

- **混合策略（Hybrid）**
  - 近期用滑动窗口保连贯，远期用摘要+实体提取保知识
  - 实际项目中常用分层压缩：原始 → 摘要 → 实体图谱

## 面试话术
Agent 记忆压缩主要有四种方法：**摘要法**（用 LLM 把多轮对话压缩成摘要）、**滑动窗口**（只保留最近 N 轮）、**重要性筛选**（打分保留高价值信息）、**实体提取**（把对话抽成结构化三元组）。实际项目中常用**混合策略**：近期用滑动窗口保证连贯，远期用摘要+实体提取保证知识不丢，分层压缩逐步提炼。

## 防混补丁
**别混**：不要把"记忆压缩"和"记忆裁剪"搞混。裁剪是暴力截断，压缩是保留语义。口诀：**裁剪是砍，压缩是榨**。

## 追问预演
1. **问**：摘要法会不会丢失关键细节？
   **答**：会。解决方式：摘要中保留关键决策点，细节放向量库长期记忆，需要时检索。
2. **问**：滑动窗口的 N 怎么定？
   **答**：看模型上下文窗口和任务类型。通常 10-20 轮，复杂任务多留，简单任务少留。
3. **问**：实体提取用什么方法？
   **答**：可以用 LLM 直接抽取（Prompt 指定格式），或用 NER 模型预提取关键实体关系。

## 手撕代码（伪代码）
```python
class MemoryCompressor:
    def __init__(self, strategy="hybrid"):
        self.strategy = strategy
    
    def compress(self, messages):
        if self.strategy == "sliding_window":
            return messages[-10:]  # 只保留最近10轮
        
        elif self.strategy == "summarization":
            old = messages[:-5]
            recent = messages[-5:]
            summary = llm_summarize(old)
            return [summary] + recent
        
        elif self.strategy == "entity_extraction":
            entities = llm_extract_triples(messages)
            return [{"role": "system", "content": f"已知事实：{entities}"}]
        
        elif self.strategy == "hybrid":
            # 近期滑动窗口 + 远期摘要
            recent = messages[-5:]
            old_summary = llm_summarize(messages[:-5])
            return [old_summary] + recent
```
