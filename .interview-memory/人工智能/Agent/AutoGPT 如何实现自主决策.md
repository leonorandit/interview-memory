## 原始知识

- **目标输入**：用户给出高层目标
- **任务分解**：LLM将目标拆分为子任务列表
- **自主执行**：逐个完成任务/调用工具/读写文件/浏览网页
- **结果反思**：评估执行结果/发现错误自我修正
- **记忆管理**：长期记忆存储上下文/避免重复
- **循环迭代**：Thought-Action-Observation循环

## 顺口溜

目分反，三循环，记忆存，迭代转。

## 单字锚点 + 防崩推导

- **锚点字**：目
- **扩展口诀**：目分反，三循环
- **为什么选它**："目"是AutoGPT的起点，所有自主决策都围绕目标展开
- **推导链**：
  1. 目 → 目标输入（用户给高层目标）
  2. 目要拆解 → 分（任务分解，LLM拆成子任务）
  3. 分完要执行 → 逐个完成/调用工具/读写文件/浏览网页
  4. 执行完要复盘 → 反（结果反思/评估/自我修正）
  5. 反思需要依据 → 记忆管理（长期记忆存上下文）
  6. 整个过程 → 三循环（Thought-Action-Observation）

## 一字一画面

- **目** —— 像射箭靶心：箭支最终要飞向的红心，所有动作围绕它展开。（本质：用户输入的高层目标）
- **分** —— 像厨师切菜：一根胡萝卜被切成均匀薄片，每片就是一个子任务。（本质：LLM将目标拆分为可执行的子任务）
- **反** —— 像照镜子：做完一件事立刻照镜子检查妆容，歪了立刻补。（本质：执行后的自我评估与修正）
- **记** —— 像随身笔记本：走到哪里记到哪里，避免重复问同一个人。（本质：长期记忆存储上下文信息）
- **循** —— 像跑步机：跑完一圈又回到起点继续，但累计里程在增加。（本质：Thought-Action-Observation的持续迭代）

## 面试话术

1. "AutoGPT实现自主决策的核心是'目分反'：先接收用户目标，然后用LLM拆解成子任务，接着自主执行并调用工具，最后反思结果并自我修正。"
2. "支撑这个循环的是记忆管理和Thought-Action-Observation循环，记忆避免重复劳动，循环保证不断逼近目标。"

## 防混补丁

- **容易和什么搞混**：容易把AutoGPT和BabyAGI搞混，也容易漏掉"反思"和"记忆"这两个关键差异点。
- **补丁口诀**：Auto自己会反思，Baby只会排任务，记忆是Auto的笔记本。

## 追问预演

1. **Q：AutoGPT的反思是怎么实现的？** A：LLM评估执行结果与目标的差距，发现错误后重新规划子任务或修正执行路径。
2. **Q：记忆管理具体用什么技术？** A：通常用向量数据库存储历史上下文，通过相似度检索避免重复执行。
3. **Q：AutoGPT为什么会陷入死循环？** A：目标过于模糊或反思机制失效，导致在相似任务间反复横跳。
