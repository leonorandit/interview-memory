## 单字锚点 + 防崩推导

**锚点字：限**

- **扩展口诀**：限深资，三保护
- **为什么选它**："限"=限制，题目问的就是"边界问题"和"限制保护"，限字直接点题。
- **推导链**：
  1. 限 → 限制 → OpenClaw的限制和保护机制
  2. 边界问题有哪些 → 嵌套层级过深（深）、资源耗尽（资）、循环spawn死锁、子Agent失控（控）、Context传递混乱
  3. 提取关键 → 深、资、控
  4. OpenClaw保护 → 最大嵌套深度限制（限深）
  5. 资源保护 → Token预算配额、资源隔离沙箱（限资）
  6. 失控保护 → 超时自动终止、父Agent强制停止、防止无限递归（限控）
  7. 总结 → 三保护：限深、限资、限控

## 一字一画面

**限** —— 像**游乐园身高限制杆**：黄色横杆上标着"1.2m"，工作人员手持秒表计时，超时的过山车自动刹停，旁边还有紧急制动红色按钮（本质：限制机制是安全运行的底线，防止失控和资源透支）。

## 面试话术

"父Agent spawn子Agent时，最大的边界风险有三个：嵌套层级过深导致栈溢出、资源耗尽拖垮系统、以及子Agent失控或循环spawn造成死锁。OpenClaw针对这些做了三层保护：第一，设置最大嵌套深度限制防止无限递归；第二，引入Token预算配额和资源隔离沙箱，防止资源耗尽；第三，子Agent超时自动终止，父Agent也可以强制停止子Agent，防止失控。"

## 防混补丁

容易和**普通进程fork**搞混？记住：**限有Token，进有PID**——OpenClaw特别针对LLM的Token消耗做了预算配额，这是传统进程管理没有的。

## 追问预演

| 追问 | 简要答案 |
|------|----------|
| Token预算配额怎么分配？ | 按任务复杂度预估，支持动态调整和子Agent间配额借用。 |
| 父Agent强制停止会丢失数据吗？ | 会触发优雅终止，已产生的结果回传，未完成的标记为取消。 |
| 循环spawn怎么检测？ | 调用链追踪+哈希检测，发现重复spawn模式立即阻断。 |
