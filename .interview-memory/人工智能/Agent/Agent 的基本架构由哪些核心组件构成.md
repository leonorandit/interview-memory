# Agent 的基本架构由哪些核心组件构成？

## 场景触发词
Agent 架构组成、Agent 核心组件、智能体由哪些部分组成、构建 Agent 需要什么模块

## 速记卡片
`组 | 组感记工，四组件 | Agent 架构`

## 原始知识
Agent 基本架构通常由四大核心组件构成：

1. **感知（Perception）**
   - 接收外部环境输入：用户消息、传感器数据、API 返回等
   - 负责理解意图、提取关键信息

2. **记忆（Memory）**
   - **短期记忆**：当前对话上下文、最近几步执行记录
   - **长期记忆**：用户画像、历史偏好、知识库
   - 解决大模型上下文窗口有限的问题

3. **工具（Tools）**
   - 外部能力接口：搜索、代码执行、数据库、API 调用等
   - Agent 能完成复杂任务的关键扩展

4. **规划/执行（Planning & Action）**
   - 将目标拆解为可执行步骤
   - 循环执行：观察 → 思考 → 行动 → 反馈
   - 常见模式：ReAct、CoT、Plan-and-Execute

## 单字锚点 + 防崩推导
- **锚点字**：组
- **扩展口诀**：组感记工
- **为什么选它**：“组”= 组成、组件，直接对应题目问的“由哪些核心组件构成”
- **防崩推导**：
  1. 组 → 组件/组成
  2. 感 → 感知（接收输入）
  3. 记 → 记忆（短期+长期）
  4. 工 → 工具（调用外部能力）
  5. 四组件 → 最后加上规划/执行，把目标落地

## 面试话术
Agent 的基本架构可以概括为四大组件：**感知、记忆、工具、规划执行**。感知负责接收和理解输入；记忆分短期和长期，解决上下文限制和个性化问题；工具扩展 Agent 的能力边界，让它能操作外部系统；规划执行则把大目标拆成小步骤，通过观察-思考-行动的循环把任务完成。

## 防混补丁
**别混**：不要把 Agent 组件和 LLM 本身混为一谈。口诀：**大脑是核心，四件是骨架**。
