# 什么是 Multi-Agent？

## 场景触发词
Multi-Agent、多智能体、Agent 协作、多 Agent 系统、Agent Swarm、Multi-Agent 架构

## 速记卡片
`多 | 多分协争，四模式 | Multi-Agent`

## 原始知识

- **定义**：Multi-Agent 系统（MAS）是由多个自主 Agent 组成的系统，它们通过协作、竞争或通信来完成单个 Agent 无法完成的复杂任务
- **核心特征**
  - **自治性**：每个 Agent 独立决策，不依赖中央控制
  - **协作性**：Agent 之间可以分工合作，共享信息
  - **分布式**：没有单点故障，任务并行处理
  - **涌现性**：整体能力大于个体之和

- **四种协作模式**
  - **分层模式（Hierarchical）**：上层 Agent 分配任务，下层 Agent 执行
  - **对等模式（Peer-to-Peer）**：Agent 之间平等协商，共同决策
  - **市场模式（Market-based）**：Agent 像拍卖一样竞标任务，价高者得
  - **竞争模式（Competitive）**：多个 Agent 尝试同一任务，取最优结果

- **典型框架**
  - **AutoGen**（微软）：对话式多 Agent，通过聊天协作
  - **MetaGPT**：模拟软件公司角色（PM、架构师、工程师）
  - **CAMEL**：角色扮演式协作，两个 Agent 互相挑战
  - **CrewAI**：流程化多 Agent，像团队一样分工

- **适用场景**
  - 复杂系统需要多角色分工（如软件开发、研究报告）
  - 需要多角度验证的任务（如代码评审、辩论）
  - 分布式并行处理（如数据采集、多源分析）

## 一字一画面
**多** —— 像 **足球队**：[绿茵场] 11 个人各自有位置，前锋进攻、后卫防守、门将守门，互相传球配合进球。

## 面试话术
Multi-Agent 就是**多个自主 Agent 组成的协作系统**。核心特点是自治、协作、分布式、涌现性。常见的四种协作模式是**分层**（上级分配下级执行）、**对等**（平等协商）、**市场**（竞价分配任务）、**竞争**（多 Agent 试做取最优）。典型框架有 AutoGen（对话协作）、MetaGPT（模拟软件公司角色）、CrewAI（流程化团队）。适合复杂多角色任务，比如软件开发需要 PM、架构师、工程师分工配合。

## 防混补丁
**别混**：不要把 Multi-Agent 和单 Agent 的多次调用搞混。Multi-Agent 是每个 Agent 有独立角色和记忆，不是同一个 Agent 反复调用。口诀：**多人分工是 Multi，一人多跑是循环**。

## 追问预演
1. **问**：Multi-Agent 之间怎么通信？
   **答**：常见三种：直接消息传递（点对点）、共享黑板（公共消息板）、广播/订阅（发布事件）。
2. **问**：Multi-Agent 会不会有冲突？怎么解决？
   **答**：会有目标冲突或资源竞争。解决方式：协商机制、优先级仲裁、引入协调者 Agent（Orchestrator）。
3. **问**：和单体大模型相比，Multi-Agent 优势在哪？
   **答**：分工专业化（每个 Agent 擅长不同领域）、并行效率高、可解释性强（每个角色输出清晰）、容错性好（单点失败不影响全局）。

## 手撕代码（伪代码）
```python
class MultiAgentSystem:
    def __init__(self):
        self.agents = {
            "pm": Agent(role="产品经理", skills=["需求分析"]),
            "arch": Agent(role="架构师", skills=["系统设计"]),
            "dev": Agent(role="工程师", skills=["代码实现"]),
        }
    
    def run(self, task):
        # 分层协作：PM 分解 -> 架构师设计 -> 工程师实现
        requirements = self.agents["pm"].execute(task)
        design = self.agents["arch"].execute(requirements)
        code = self.agents["dev"].execute(design)
        return code
```
