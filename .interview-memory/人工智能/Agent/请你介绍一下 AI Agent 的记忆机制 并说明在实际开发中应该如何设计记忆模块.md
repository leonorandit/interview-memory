# 请你介绍一下 AI Agent 的记忆机制，并说明在实际开发中应该如何设计记忆模块？

## 场景触发词
Agent 记忆机制、LLM Agent 记忆、长期记忆、短期记忆、记忆模块设计、Agent 记忆怎么实现

## 速记卡片

## 原始知识

- **短期记忆（Short-term Memory）**：当前对话上下文，存在 Prompt 里，窗口有限
- **长期记忆（Long-term Memory）**：跨会话持久化，通常用向量数据库存储
- **知识记忆（Knowledge Memory）**：外部知识库、文档、RAG 检索
- **工作记忆（Working Memory）**：当前任务执行中的中间状态、临时变量

**实际开发设计要点**：
- **分层存储**：短期放内存/Redis，长期放向量库，知识放文档库
- **记忆检索**：用 Embedding 做语义检索，按相关度召回历史记忆
- **记忆压缩**：对话太长时做摘要（Summarization），提取关键信息
- **记忆更新**：每次交互后评估是否值得存入长期记忆，避免垃圾信息
- **记忆遗忘**：设置 TTL、重要性评分、定期清理低价值记忆

## 面试话术
Agent 的记忆机制分为四个层次：**短期记忆**（当前对话上下文，存在 Prompt 里）、**长期记忆**（跨会话持久化，通常用向量数据库）、**知识记忆**（外部知识库，通过 RAG 检索）、**工作记忆**（任务执行中的中间状态）。实际开发中，记忆模块要分层设计：短期放内存或 Redis，长期放向量库；用 Embedding 做语义检索召回历史；对话太长时做摘要压缩；每次交互后评估重要性，有价值的才存入长期记忆，同时设置 TTL 定期清理。

## 防混补丁
**别混**：不要把长期记忆和知识库搞混。长期记忆是 Agent 自己的经历，知识库是外部文档。口诀：**经历是日记，知识是百科**。

## 追问预演
1. **问**：短期记忆窗口满了怎么办？
   **答**：做滑动窗口保留最近 N 轮，或做摘要压缩历史对话。
2. **问**：长期记忆怎么检索？
   **答**：把历史对话 Embedding 化存入向量库，查询时用语义相似度召回 Top-K。
3. **问**：怎么避免记忆爆炸？
   **答**：设置重要性评分，只存高价值信息；定期归档或删除；做记忆摘要合并。

class MemoryModule:
    def __init__(self):
        self.short_term = []  # 短期记忆：当前对话历史
        self.vector_store = VectorDB()  # 长期记忆：向量数据库
    
    def add(self, message):
        self.short_term.append(message)
        if self._is_important(message):
            embedding = embed(message)
            self.vector_store.add(embedding, message)
    
    def retrieve(self, query, k=5):
        query_emb = embed(query)
        return self.vector_store.search(query_emb, k)
    
    def compress(self):
        # 对话太长时做摘要
        if len(self.short_term) > 20:
            summary = llm_summarize(self.short_term)
            self.short_term = [summary]
