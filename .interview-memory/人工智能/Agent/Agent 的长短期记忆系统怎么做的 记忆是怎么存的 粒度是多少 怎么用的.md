# Agent 的长短期记忆系统怎么做的？记忆是怎么存的？粒度是多少？怎么用的？

## 场景触发词
Agent 长短期记忆、记忆存储、记忆粒度、Memory 系统设计、Agent 记忆怎么存、短期记忆长期记忆实现

## 速记卡片
`存 | 存快慢细，四维度 | 长短期记忆系统`

## 原始知识

- **短期记忆（Short-term）**
  - **存储位置**：Prompt 上下文窗口 / 内存 / Redis
  - **存储格式**：对话历史列表 `[{role, content}, ...]`
  - **粒度**：单轮对话（Message 级别）
  - **用法**：直接拼接到 Prompt 里，供模型理解当前语境
  - **容量**：受限于模型上下文窗口（通常保留最近 10-20 轮）

- **长期记忆（Long-term）**
  - **存储位置**：向量数据库（Milvus / Chroma / Pinecone）
  - **存储格式**：Embedding 向量 + 原始文本 + 元数据（时间、类型、重要性）
  - **粒度**：三种可选
    - **粗粒度**：整段对话摘要（每次会话结束后生成）
    - **中粒度**：关键事件/事实片段（实时提取重要信息）
    - **细粒度**：实体-关系三元组（如"用户喜欢Python"结构化存储）
  - **用法**：查询时先将问题 Embedding，语义相似度召回 Top-K 相关记忆

- **混合使用策略**
  - 短期保**最近 N 轮**（保证连贯性）
  - 长期补**相关历史**（保证知识性）
  - 超长时做**摘要压缩**（避免窗口爆炸）

- **记忆更新机制**
  - 每次对话后评估重要性，高价值记忆异步写入向量库
  - 定期合并重复记忆，更新实体状态
  - 设置 TTL 自动清理过期低价值记忆

## 单字锚点 + 防崩推导
- **锚点字**：存
- **扩展口诀**：存快慢细
- **为什么选它**：“存”= 存储，直接对应题目核心词"怎么存的"
- **防崩推导**：
  1. 存 → 存储
  2. 快 → **快**速访问 = 短期记忆（放内存/Redis，直接读）
  3. 慢 → **慢**速检索 = 长期记忆（放向量库，需语义搜索）
  4. 细 → **细**粒度 = 三种粒度（粗中细，按需选择）
  5. 四维度 → 存储位置、存储格式、粒度、用法

## 一字一画面
**存** —— 像 **抽屉柜+仓库**：[办公室] 桌面抽屉放常用文件（短期），楼下仓库放归档资料（长期），需要时查目录调阅。
**快** —— 像 **手机相册**：[屏幕滑动] 最近照片一滑就到，不用搜索。
**慢** —— 像 **图书馆检索**：[卡片目录] 先查索引号，再去书架上找，慢一点但能挖到很久以前的资料。
**细** —— 像 **俄罗斯套娃**：[多层嵌套] 外层是整段摘要（粗），中层是关键事件（中），内层是结构化三元组（细）。

## 面试话术
Agent 的长短期记忆系统分四维度设计。**短期记忆**存在内存或 Redis 里，粒度是单轮对话，使用时直接拼进 Prompt；**长期记忆**存在向量数据库，粒度分三种——整段摘要、关键片段、结构化三元组，使用时先把问题 Embedding 做语义检索召回 Top-K。**实战中**短期保最近 10-20 轮保连贯，长期补相关历史保知识，超长时做摘要压缩。每次交互后评估重要性，有价值的异步写入向量库，同时设 TTL 定期清理。

## 防混补丁
**别混**：不要把"记忆粒度"和"检索粒度"搞混。存储可以细粒度，检索时可以按主题聚合。口诀：**存得细，取得聚**。

## 追问预演
1. **问**：同一实体在不同时间状态冲突怎么办？（比如用户之前说喜欢 Java，现在说喜欢 Python）
   **答**：加时间戳和置信度，取最新或让用户确认；或存为"2024-01: Java, 2025-06: Python"时间线。
2. **问**：多用户记忆怎么隔离？
   **答**：向量库元数据加 user_id，查询时带过滤条件 `where user_id = 'xxx'`。
3. **问**：记忆写入向量库是同步还是异步？
   **答**：异步。对话响应优先，记忆入库放后台队列，避免阻塞。

## 手撕代码（伪代码）
```python
class MemoryStore:
    def __init__(self):
        self.short_term = []          # 短期：内存列表
        self.vector_db = VectorDB()   # 长期：向量库
    
    def add_turn(self, role, content):
        # 写入短期记忆
        self.short_term.append({"role": role, "content": content})
        if len(self.short_term) > 20:
            self.short_term = self.short_term[-20:]
        
        # 异步写入长期记忆（仅重要信息）
        if self._is_important(content):
            self._async_save_to_longterm(content)
    
    def _async_save_to_longterm(self, content):
        embedding = embed(content)
        self.vector_db.add(
            vector=embedding,
            text=content,
            metadata={"time": now(), "type": "fact", "importance": 0.8}
        )
    
    def retrieve(self, query, user_id, k=5):
        # 短期：直接拿最近 N 轮
        recent = self.short_term[-10:]
        
        # 长期：语义检索
        query_emb = embed(query)
        longterm = self.vector_db.search(
            query_emb, k=k, 
            filter={"user_id": user_id}
        )
        
        return {"recent": recent, "longterm": longterm}
```
