## 原始知识

- **OpenManus**：开源的通用AI Agent（Manus的开源替代）
- **核心组件**：规划器Planner/执行器Executor/工具集Tools
- **实现原理**：接收用户任务→LLM分解为子任务→调用浏览器/代码执行/文件操作等工具→逐步执行并反馈→生成最终交付物
- **与Manus区别**：开源免费/社区驱动/可本地部署

## 单字锚点 + 防崩推导

- **锚点字**：开
- **扩展口诀**：开器规，三核心
- **为什么选它**："开"直接对应OpenManus的Open（开源），也是与闭源Manus的最大差异点
- **推导链**：
  1. 开 → OpenManus（开源/开放）
  2. 开的是什么 → 器（通用AI Agent工具）
  3. 器的组成 → 规（规划器Planner）
  4. 三核心 → 规划器（规）+ 执行器（行）+ 工具集（器）
  5. 为什么能开 → 开源免费/社区驱动/可本地部署
  6. 工作原理 → 接任务→分解→调工具→执行→反馈→交付

## 一字一画面

- **开** —— 像打开上锁的宝盒：原本Manus是锁着的，OpenManus把钥匙交给所有人。（本质：开源开放，人人可用）
- **器** —— 像瑞士军刀展开：刀片、螺丝刀、剪刀同时弹出，对应浏览器、代码执行、文件操作。（本质：丰富的工具集）
- **规** —— 像建筑蓝图：拿到项目先画图纸，把大楼拆成地基、框架、装修。（本质：Planner将任务分解为子任务）
- **行** —— 像施工队按图施工：工人看着蓝图一砖一瓦垒起来，监理随时检查。（本质：Executor逐步执行并反馈）
- **替** —— 像开源安卓替代闭源塞班：功能相似，但一个封闭一个开放。（本质：OpenManus是Manus的开源替代方案）

## 面试话术

1. "OpenManus是Manus的开源替代方案，核心可以概括为'开器规'三核心：开源开放、工具集丰富、规划器分解任务。它由Planner、Executor和Tools三件套组成。"
2. "实现原理上，OpenManus接收用户任务后，由LLM分解子任务，然后通过浏览器自动化、代码执行、文件操作等工具逐步执行，最终生成交付物。最大的优势是开源免费、可本地部署。"

## 防混补丁

- **容易和什么搞混**：容易把OpenManus和Manus当成同一个东西，或者以为OpenManus是Manus的官方开源版。
- **补丁口诀**：Manus闭源要邀请，OpenManus开源本地装，社区驱动非官方。

## 追问预演

1. **Q：OpenManus和Manus的能力差距大吗？** A：核心架构相似，但Manus有更强的基础设施和优化，OpenManus作为开源版迭代很快，社区在不断追赶。
2. **Q：OpenManus支持哪些工具？** A：主要包括浏览器自动化、Python代码执行、文件读写、Shell命令等。
3. **Q：本地部署OpenManus需要什么配置？** A：需要LLM API Key，本地环境支持Python，部分功能需要配置浏览器驱动。
