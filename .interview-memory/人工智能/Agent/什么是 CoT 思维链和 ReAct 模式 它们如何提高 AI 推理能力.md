## 原始知识

- **CoT**：Chain-of-Thought/让LLM一步步推理/通过中间步骤提升复杂问题准确率/Zero-shot CoT用"Let's think step by step"触发
- **ReAct**：Reasoning+Acting/推理与行动结合/通过工具反馈修正推理
- **提高推理能力的方式**：CoT分解复杂问题/减少跳步错误/ReAct通过行动反馈验证推理/减少幻觉/两者结合先CoT规划再ReAct执行

## 单字锚点 + 防崩推导

- **锚点字**：链
- **扩展口诀**：链反工，两模式
- **为什么选它**："链"是CoT（思维链）的核心意象，也是题目两个知识点的共同锚点
- **推导链**：
  1. 链 → CoT（Chain-of-Thought，思维链）
  2. 链的作用 → 让LLM一步步推理/中间步骤/提升复杂问题准确率
  3. 链的触发 → Zero-shot用"Let's think step by step"
  4. 另一个模式 → 反（ReAct，Reasoning+Acting）
  5. 反的特点 → 推理+行动结合/工具反馈修正推理
  6. 两者关系 → 先链后反（先CoT规划，再ReAct执行）
  7. 共同目标 → 提高推理能力（分解问题/减少跳步/减少幻觉）

## 一字一画面

- **链** —— 像串珍珠：一颗颗珠子用线串起来，缺一颗链子就断。（本质：CoT把推理拆成连续中间步骤）
- **反** —— 像盲人拄拐杖走路：走一步用拐杖敲敲地面，确认安全再迈下一步。（本质：ReAct用行动反馈验证推理）
- **工** —— 像修理工查电路：先看图纸推理哪根线有问题，再用万用表测一测，测完再推理。（本质：推理与工具交替进行）
- **模** —— 像两种导航模式：一个只看地图推算（CoT），一个边开车边听实时路况（ReAct）。（本质：两种推理模式的差异）

## 面试话术

1. "CoT和ReAct是两种提升LLM推理能力的模式。CoT是思维链，让模型把复杂问题拆成一步步推理，比如加一句'Let's think step by step'就能触发；ReAct是推理加行动，每想一步就做一次行动，用行动结果来修正下一步推理。"
2. "实际应用中，两者可以结合：先用CoT做整体规划，再用ReAct逐步执行并反馈，这样既能保证全局思路清晰，又能通过行动反馈减少幻觉。"

## 防混补丁

- **容易和什么搞混**：容易认为CoT和ReAct是同一种东西，或者觉得ReAct取代了CoT。
- **补丁口诀**：CoT只动脑子不动手，ReAct边想边做，先做链规划再做反执行。

## 追问预演

1. **Q：CoT为什么能减少跳步错误？** A：因为强制模型输出中间推理步骤，不能直接从问题跳到答案，每一步都可检查。
2. **Q：Zero-shot CoT和Few-shot CoT有什么区别？** A：Zero-shot只需加一句触发词；Few-shot需要提供带推理过程的示例。
3. **Q：先CoT后ReAct具体怎么落地？** A：第一阶段用CoT生成整体任务规划，第二阶段把规划拆成ReAct循环逐步执行，每步有Observation反馈。
4. **Q：ReAct比CoT多了什么开销？** A：多了工具调用的延迟和成本，但换来的是更高的准确率和可解释性。
