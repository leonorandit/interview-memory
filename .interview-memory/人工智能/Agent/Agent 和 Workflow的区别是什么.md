# Agent 和 Workflow 的区别是什么？

## 场景触发词
Agent vs Workflow、智能体和工作流区别、Workflow 和 Agent 怎么选、什么时候用 Workflow 什么时候用 Agent

## 速记卡片
`差 | 差固灵，编决策 | Agent vs Workflow`

## 原始知识

| 维度 | Workflow | Agent |
|------|----------|-------|
| **本质** | 预设流程 = **剧本** | 自主决策 = **即兴演员** |
| **编排方式** | 人类预先编排步骤 | LLM 动态决定下一步 |
| **灵活性** | **固**定路径，不可偏离 | **灵**活应变，自适应调整 |
| **决策权** | 无，严格按剧本走 | 有，根据环境自主判断 |
| **适用场景** | 规则明确、步骤固定的任务 | 规则模糊、需要推理探索的任务 |
| **典型例子** | 审批流、ETL、定时报表 | 智能客服、代码生成、研究助手 |
| **失败处理** | 按预设分支处理 | 自主尝试替代方案 |

**核心区别**：
- **Workflow 是编排层**：解决「先做什么、后做什么」的问题
- **Agent 是决策层**：解决「面对未知，该怎么做」的问题

## 单字锚点 + 防崩推导
- **锚点字**：差
- **扩展口诀**：差固灵，编决策
- **为什么选它**：“差”= 区别/差异，直接对应题目问的“区别是什么”
- **防崩推导**：
  1. 差 → 区别
  2. 固 → Workflow **固**定步骤、预设路径
  3. 灵 → Agent **灵**活决策、动态应变
  4. 编 → Workflow 是**编**排好的流程
  5. 决策 → Agent 拥有自主**决策**权

## 一字一画面
**差** —— 像 **两条岔路**：[一左一右] 左边是铺好的石板路（Workflow），右边是野路（Agent）。
**固** —— 像 **火车轨道**：[钢铁轨道] 列车只能沿着固定路线行驶，不能随意变道。
**灵** —— 像 **越野车**：[红色吉普] 遇到障碍可以自己判断绕行、爬坡或涉水。
**编** —— 像 **导演剧本**：[黄色纸张] 每一句台词、每一个动作都写好了，演员照本宣科。
**决策** —— 像 **围棋手**：[黑白棋盘] 面对复杂局面，自己判断落子位置，没有固定套路。

## 面试话术
Agent 和 Workflow 的核心区别在于**灵活性和决策权**。Workflow 是预设好的流程，像火车轨道，按固定步骤执行，适合规则明确的任务；Agent 是自主决策的智能体，像越野车，能根据环境动态调整策略，适合需要推理和探索的复杂任务。简单说：**Workflow 管编排，Agent 管决策**。

## 防混补丁
**别混**：Workflow 和 Agent 不是互斥的，实际项目中常常结合使用——Workflow 定义整体框架，Agent 负责框架内需要灵活决策的节点。口诀：**大框用流，灵活处放 Agent**。
