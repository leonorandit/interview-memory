# 原始知识

- **初始化阶段**：加载Agent配置，初始化上下文，加载可用工具
- **思考阶段**：LLM**分析用户意图**，决定下一步行动，生成Thought
- **行动阶段**：**调用工具或生成回复**，执行外部操作
- **反思阶段**：**评估执行结果**，决定是否继续，更新记忆状态
- **循环**：重复**思考-行动**直到任务完成或达到最大轮次

# 单字锚点 + 防崩推导

## 锚点字：运

- **扩展口诀**：运（运行）思（思考）调（调度/行动）反（反思）四阶段
- **为什么选它**："运"是Agent Runner的核心动作，也是"运行"的题眼
- **推导链**：
  1. 看到"运" → 想到**Agent Runner运行**
  2. 第一阶段：**初始化** → 加载配置 + 初始化上下文 + 加载工具（赛前热身）
  3. 第二阶段：**思考** → LLM分析用户意图，生成Thought，决定下一步（中场分析）
  4. 第三阶段：**行动** → 要么调工具，要么生成回复，执行外部操作（场上调度）
  5. 第四阶段：**反思** → 评估结果，决定继续还是结束，更新记忆（赛后复盘）
  6. 关键：**思考+行动循环**，直到任务完成或达到最大轮次限制

# 一字一画面

- **运** —— 像[马拉松起跑枪响]：发令枪"砰"的一声，运动员从起跑线弹射出去，计时器开始跳动（本质：Runner启动Agent运行的发令枪）
- **思** —— 像[棋手托腮凝视棋盘]：木质棋盘上黑白子交错，棋手食指敲桌"哒哒"，突然眼睛一亮落子（本质：LLM分析局势生成Thought）
- **调** —— 像[交响乐指挥挥棒]：黑色燕尾服指挥棒上下翻飞，小提琴铜管齐声响应，高潮迭起（本质：Runner调度工具或生成回复执行行动）
- **反** —— 像[录像回放慢动作]：球场大屏幕播放刚才进球回放，解说员画圈标注"这脚传球是关键"（本质：反思阶段评估结果、更新记忆）

# 面试话术

"Agent Runner的工作分为四个阶段：初始化阶段加载配置和工具；思考阶段让LLM分析用户意图并生成Thought；行动阶段根据Thought决定调用工具还是直接回复；反思阶段评估执行结果并更新记忆。思考和行动会循环进行，直到任务完成或者达到最大轮次。"

# 追问预演

| 追问 | 简要答案 |
|------|----------|
| 最大轮次是多少？可以配置吗？ | 可以配置，默认通常3-5轮，防止无限循环消耗Token |
| 反思阶段具体更新什么记忆？ | 更新短期记忆中的执行结果，必要时触发长期记忆写入或Compaction |
| 如果行动阶段调用了工具，反思阶段怎么评估？ | 评估工具返回结果是否满足预期，不满足则继续循环，满足则进入最终回复生成 |
