# 速记卡片

| 锚点字 | 口诀 | 核心差异 |
|--------|------|----------|
| 链 | 链运调生，六步走 | 用户消息像快递包裹，六站分拣最终送达 |

# 原始知识

- **接收消息**：用户输入进入系统，**预处理**
- **构建上下文**：System Prompt + 历史对话 + 当前输入，**Context Manager组装**
- **调用LLM**：发送完整Prompt给模型，**获取模型输出**
- **解析决策**：判断是否**调用工具**，生成工具调用JSON
- **执行工具**：Tool Manager**调用对应工具**，获取结果
- **回传生成**：工具结果返回LLM，LLM**生成最终回复**，返回给用户

# 单字锚点 + 防崩推导

## 锚点字：链

- **扩展口诀**：链（链路）运（运行LLM）调（调用工具）生（生成回复）六步走
- **为什么选它**："链"是"链路"的题眼，也是流程题的天然锚点
- **推导链**：
  1. 看到"链" → 想到**完整链路/流程**
  2. 第一步：**接收**消息 → 用户输入 + 预处理
  3. 第二步：**构建上下文** → System Prompt + 历史 + 当前输入，Context Manager组装
  4. 第三步：**调用LLM** → 发送完整Prompt，拿到模型输出
  5. 第四步：**解析决策** → 模型想不想调工具？生成JSON
  6. 第五步：**执行工具** → Tool Manager出马，拿到工具结果
  7. 第六步：**回传生成** → 结果塞回LLM，生成最终回复给用户
  8. 口诀"链运调生"：链（链路）运（运行/调LLM）调（调用工具）生（生成回复），覆盖核心动作

# 一字一画面

- **链** —— 像[工厂流水线]：银色传送带六个工位，包裹依次经过扫描、装箱、贴单、分拣、装车、签收，绿灯逐一亮起（本质：消息处理是严格顺序的六步链路）
- **运** —— 像[快递员骑电动车飞驰]：蓝色外卖服小哥手拿手机导航，AI语音播报"前方右转到达目的地"（本质：LLM是运输大脑，决定往哪走）
- **调** —— 像[电话客服转接]：客服 headset 听到需求，按下转接键"为您转接技术部"，对方接听（本质：解析到工具需求，转给Tool Manager执行）
- **生** —— 像[打印机吐出纸张]：激光打印机"嗡嗡"出纸，纸上是排版整齐的最终答案（本质：LLM综合所有信息生成最终回复）

# 面试话术

"用户消息进入系统后，首先经过预处理，然后Context Manager会把System Prompt、历史对话和当前输入组装成完整上下文。接着发给LLM，LLM输出后会由系统解析，判断是否需要调用工具。如果需要，Tool Manager会执行对应工具并把结果回传给LLM，最后LLM生成最终回复返回给用户。"

# 防混补丁

- 容易和 **HTTP请求链路** 搞混？一句补丁：**不是网络七层，是Agent业务六步，第三步调LLM才是核心，不是发HTTP请求**

# 追问预演

| 追问 | 简要答案 |
|------|----------|
| 如果工具调用失败了怎么办？ | 错误信息会作为结果回传给LLM，LLM根据错误信息决定重试或告知用户 |
| 调用LLM后，模型怎么表达要调工具？ | 通常输出结构化JSON，包含工具名和参数，系统解析后路由到Tool Manager |
| 这个链路支持多轮工具调用吗？ | 支持，只要模型继续输出工具调用JSON，就会循环执行工具阶段直到生成最终回复 |
