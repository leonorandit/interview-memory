## 一字一画面

**插** —— 像**乐高积木插口**：彩色积木块底部有圆形凸起，顶部有凹槽，咔嗒一声严丝合缝拼上，既能随时拆换，又能搭出任意造型，旁边还有说明书标注兼容版本（本质：插件是标准化接口下的可替换扩展单元，安全插拔是关键）。

## 面试话术

"设计插件系统，我觉得最关键的是四个问题：安全性、隔离性、版本兼容性和性能影响。插件不能随便访问主系统资源，所以要有沙箱隔离；不同版本的插件不能冲突，所以要有依赖管理和兼容性检查；插件崩溃不能拖垮主系统，所以要有错误隔离；还要支持热加载，避免重启服务。OpenClaw的插件API定义了Plugin接口，包含Init、Start、Stop生命周期方法，支持配置注入和事件订阅，还暴露了Hook点让第三方可以扩展功能，并且支持动态加载和卸载。"

## 防混补丁

容易和**微服务扩展**搞混？记住：**插跑同进程，服跑不同机**——插件通常和主系统运行在同一进程内，而微服务是独立部署的进程。

## 追问预演

| 追问 | 简要答案 |
|------|----------|
| 插件沙箱隔离怎么实现？ | 通过进程隔离、容器化或WASM运行时限制系统调用。 |
| Hook点和事件订阅有什么区别？ | Hook点是预留的扩展点位，事件订阅是异步消息监听，Hook更侵入。 |
| 热加载会不会有内存泄漏？ | 需要完善的生命周期管理和类加载器卸载机制，配合监控告警。 |
