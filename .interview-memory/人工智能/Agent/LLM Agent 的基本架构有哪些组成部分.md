## 单字锚点 + 防崩推导

**锚点字**：架

**扩展口诀**：架感记工，四组件

**为什么选它**：面试问"架构组成"，首字就是"架"，天然锚定，脱口而出。

**推导链**：
1. 架 → 架构 → 组成部件 → 四个
2. 感 → **感**知？不对，是**规**划（谐音"感"→"规"的口误修正，记住"架感"其实是"架规"的变音，或理解为"感想"即思考规划）
   - 更稳的推导："感"音近"G"→ Planning 的 "P"？不，直接记：感 = 规划（做任务前要有"感"悟和计划）
3. 记 → 记忆 Memory（短期/长期）
4. 工 → 工具 Tools（API/函数）
5. 四组件还差一个 → "架感记工"的"架"本身隐含"行动"（架构落地即行动）
   - 更好的方式：四组件 = 规、记、工、行；口诀取"架感记工"，面试时补全为"规划、记忆、工具、行动"
   - "感"谐音"干"→ 行动Action；同时"感"也带"规划"之意。背的时候：架（架构）感（规划）记（记忆）工（工具）+ 行动（执行）
   - 最稳路径：记住"四组件"这个数字，锚点"架"起头，然后手指计数：1规划 2记忆 3工具 4行动。

## 一字一画面

**架** —— 像[工地脚手架]：[银灰色的钢管框架层层搭起，工人站在上面逐层施工，先搭哪层、后搭哪层都有图纸规划]（本质：架构是Agent的骨架，四根主梁撑起全部能力）

**感** —— 像[导航APP]：[蓝色路线上不断弹出"前方拥堵，建议绕行"，地图先规划再出发]（本质：Planning就是Agent的导航，任务分解相当于路线规划）

**记** —— 像[双格抽屉]：[上层透明玻璃放当天便签（短期），下层实木柜锁着历年账本（长期），用时各取]（本质：Memory分短长期，短期上下文窗口，长期向量库）

**工** —— 像[瑞士军刀]：[红色外壳弹出螺丝刀、剪刀、开瓶器，按需抽换]（本质：Tools是Agent的手，API/函数就是不同刀头）

**行** —— 像[机器人按下回车]：[绿色执行灯亮起，屏幕滚动输出结果，成功或报错都回传大脑]（本质：Action是闭环执行，反馈决定下一步）

## 面试话术

"LLM Agent的架构主要分四块：首先是Planning负责把大任务拆成小步骤；然后是Memory，分短期上下文和长期向量记忆；第三是Tools，也就是调用外部API或函数的能力；最后是Action，执行并拿到反馈。这四块搭起来，Agent才能从'只会聊天'变成'能干活'。"

"简单来说就是'规划—记忆—工具—行动'的闭环，规划决定做什么，记忆记得住，工具扩展能力边界，行动把结果带回来。"

## 追问预演

**Q1**：短期记忆和长期记忆具体怎么实现？
**A**：短期一般靠对话上下文窗口，长期用向量数据库比如FAISS、Chroma存历史对话的embedding，需要时做相似度检索。

**Q2**：Planning模块常见的任务分解方法有哪些？
**A**：思维链CoT、思维树ToT、ReAct模式，还有像HuggingGPT那种按能力路由到不同专家模型。

**Q3**：Tools和Action的区别是什么？
**A**：Tools是Agent的"手"，指可调用的外部能力；Action是"动手"这个动作本身，包含执行+获取反馈的完整步骤。

**Q4**：如果Agent没有记忆模块会怎样？
**A**：每次对话都是无状态，无法跨会话保持用户偏好，也做不了需要多步上下文积累的任务。
