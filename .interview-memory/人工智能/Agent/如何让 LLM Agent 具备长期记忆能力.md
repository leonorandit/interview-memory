## 单字锚点 + 防崩推导

**锚点字**：忆

**扩展口诀**：忆库存，三记忆

**为什么选它**：问题核心词是"长期记忆"，"忆"字直接锁定主题，且区别于"记"（可指短期），"忆"更带"回忆、久远"的意味。

**推导链**：
1. 忆 → 记忆 → 长期记忆 → 怎么实现？
2. 库 → 向量数据库（FAISS/Milvus/Chroma）→ 存对话历史的embedding
3. 存 → 存储 → 长期保留，跨会话可用
4. 三记忆 → 三种手段/三层结构
   - 手段一：向量库存储（原始对话的海量存储）
   - 手段二：总结压缩（Summary，长变短）
   - 手段三：关键信息提取（Entity/Memory Bank，结构化抽实体）
   - 三层结构：工作记忆（当前用）、episodic情景记忆（个人经历）、semantic语义记忆（通用事实）
5. 若只记得"忆库存"：
   - 忆 = 长期记忆问题
   - 库 = 向量数据库
   - 存 = 存对话历史
   - 然后因果推导：存多了会爆 → 要压缩（Summary）→ 压缩完还要结构化（Entity提取）→ 最后分层管理（工作/episodic/semantic）
6. 数字保险：三种核心技术（向量存储、总结压缩、关键提取）+ 一种组织方式（分层）= 完整答案。

## 面试话术

"让Agent具备长期记忆，核心思路是把超出上下文窗口的历史信息持久化到外部存储。通常的做法有几层：首先用向量数据库存储对话历史的embedding，需要时做语义检索；其次对超长对话做Summary压缩，避免信息爆炸；另外还可以做关键信息提取，把实体和重要事实抽到Memory Bank里。更精细一点会参考人类记忆做分层，比如工作记忆、情景记忆和语义记忆分开管理。"

"本质上就是'存得下、找得着、压得短、分得清'，向量库解决存和找，Summary解决长度问题，Entity提取和分层解决结构化问题。"

## 追问预演

**Q1**：向量数据库具体怎么存对话历史？
**A**：把每轮对话或每段文本用embedding模型转成向量，存入FAISS/Milvus/Chroma，检索时把当前问题也转成向量，做相似度匹配召回最相关的历史片段。

**Q2**：Summary压缩是怎么做的？
**A**：可以定期让LLM把最近N轮对话总结成一段要点，也可以增量更新，比如每次新来一轮就融合进已有摘要，替换原始长文本。

**Q3**：Memory Bank和向量库有什么区别？
**A**：向量库存的是原始或半原始文本片段，按语义检索；Memory Bank是结构化存储，比如键值对、实体关系图，更适合精确查询和推理。

**Q4**：分层记忆的三层各指什么？
**A**：工作记忆是当前任务相关的短期信息；episodic记忆是用户的个人经历和事件；semantic记忆是通用事实和世界知识。三层可以分别用不同存储和召回策略。
