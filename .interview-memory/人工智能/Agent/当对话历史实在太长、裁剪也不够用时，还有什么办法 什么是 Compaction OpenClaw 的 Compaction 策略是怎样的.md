# 原始知识

- Compaction是当对话历史**超过阈值时的深度压缩策略**
- 策略一：**摘要压缩**，将多轮对话压缩为一段摘要，**保留关键信息**
- 策略二：**关键信息提取**，保留**决策点和结论**，丢弃中间推理过程
- 策略三：**归档存储**，将完整历史存入**长期记忆**，上下文只保留摘要和最近几轮
- OpenClaw做法：**自动触发Compaction**
- **多级压缩**
- **用户可配置压缩策略**
- **支持自定义压缩Prompt**

# 单字锚点 + 防崩推导

## 锚点字：压

- **扩展口诀**：压（压缩）摘（摘要）存（存储）三Compaction
- **为什么选它**："压"是Compaction的本质动作，力度比"裁"更重，区分明显
- **推导链**：
  1. 看到"压" → 想到**压缩/Compaction**
  2. 什么时候压？**裁剪也不够用时**，对话历史**超过阈值**
  3. "摘" → **摘要压缩**：多轮压成一段，保留关键信息
  4. "存" → **归档存储**：完整历史存长期记忆，上下文留摘要+最近几轮
  5. 还少一个？→ **关键信息提取**：保留**决策点和结论**，扔中间推理
  6. OpenClaw特色：**自动触发** + **多级压缩** + **可配置策略** + **自定义压缩Prompt**

# 一字一画面

- **压** —— 像[液压机压扁汽车]：黄色工业液压机缓缓下降，废旧轿车"嘎吱"被压成一块扁平铁饼，体积骤减九成（本质：Compaction是暴力压缩，大幅减少Token占用）
- **摘** —— 像[编辑改稿红笔划]：A4稿纸上红笔大段大段划线，旁边批注"保留结论，删除过程"，最后只剩半页精华（本质：摘要压缩保留关键信息）
- **存** —— 像[档案室铁柜归档]：棕色牛皮纸档案袋贴好标签推进铁柜抽屉，柜门"咔哒"上锁，桌上只留一张索引卡（本质：完整历史归档长期记忆，上下文只留索引）

# 防混补丁

- 容易和 **滑动窗口裁剪** 搞混？一句补丁：**裁剪是扔旧消息，Compaction是压旧消息，裁是删、压是变，压完还能读摘要**

# 追问预演

| 追问 | 简要答案 |
|------|----------|
| Compaction的阈值通常设多少？ | 看模型Context Window大小，通常是窗口的70%-80%，预留空间给当前轮次 |
| 多级压缩是什么意思？ | 先轻压成摘要，还不够再重压成关键点列表，极端情况压成一句话决策链 |
| 自定义压缩Prompt有什么用？ | 让业务场景决定什么信息更重要，比如客服场景保留用户诉求和解决方案，丢弃寒暄 |
