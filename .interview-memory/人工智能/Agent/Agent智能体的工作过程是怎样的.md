## 速记卡片

环 | 环感知，思行反 | 感知→思考→行动→反馈→循环

---

## 原始知识

- **感知**：接收用户输入与环境信息
- **思考**：LLM推理与决策（下一步做什么）
- **行动**：调用工具或执行操作
- **反馈**：观察执行结果
- **循环**：直到任务完成（ReAct模式）

## 单字锚点 + 防崩推导

**锚点字**：环

**扩展口诀**：环感知，思行反

**为什么选它**：工作过程是一个"循环"，"环"字直接点出流程型题目的核心特征——闭环迭代。

**推导链**：
1. 环 → 循环、闭环 → 过程不是一次性的，是转圈的
2. 感 → 感知（接收输入/环境信息）—— 循环的起点，先看再动
3. 知 → 思考（LLM推理决策）—— "感"后自然要"知"，谐音"思"→思考
   - 口诀里是"思"，感→思，感知之后是思考
4. 行 → 行动（调用工具/执行）—— 想完了就干
5. 反 → 反馈（观察结果）—— 干完了要看效果
6. 环闭合：反馈后又回到感知，判断任务是否完成，没完继续下一轮
7. 若只记得"环"：
   - 环 = 循环 = ReAct = Reasoning + Acting
   - ReAct本身就是循环：推理→行动→观察→再推理...
   - 所以只要想起ReAct，四步自然出来：观察（感知）→推理（思考）→行动→观察（反馈）
8. 顺序保险：感→思→行→反，像人做事：先看→再想→再做→再看。

## 面试话术

"Agent的工作过程可以概括为一个感知-思考-行动-反馈的循环。首先感知用户输入和环境信息，然后LLM进行推理决策，接着调用工具执行行动，最后观察执行结果作为新的反馈。如果任务没完成，就进入下一轮循环，这就是经典的ReAct模式。"

"说白了就像人做事：先看情况，再想办法，然后去执行，执行完看看效果，不行再来一遍。Agent靠这个闭环才能从简单的'一问一答'进化到'自主完成复杂任务'。"

## 追问预演

**Q1**：ReAct模式具体是什么？
**A**：ReAct = Reasoning + Acting，让LLM交替进行推理轨迹和行动执行，通过观察环境反馈来动态调整下一步策略，比单步推理更 robust。

**Q2**：如果行动失败了怎么办？
**A**：失败结果作为负面反馈进入下一轮循环，LLM会重新推理，尝试别的工具或调整参数，直到成功或达到重试上限。

**Q3**：循环什么时候终止？
**A**：通常有几种条件：任务目标达成、达到最大迭代次数、遇到不可恢复错误、或者用户主动中断。

**Q4**：感知模块只接收文本输入吗？
**A**：不一定，多模态Agent还可以接收图片、音频、传感器数据等，感知层越丰富，Agent能处理的任务越广。
