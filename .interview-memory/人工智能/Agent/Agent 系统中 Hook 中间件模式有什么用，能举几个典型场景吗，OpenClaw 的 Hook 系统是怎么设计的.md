## 速记卡片

钩 | 钩前后，五场景 | 生命周期插逻辑，不改核心代码

## 原始知识

- 关键**生命周期**插入自定义逻辑
- **不修改核心**代码
- **插件化**扩展
- **请求预处理**
- **响应后处理**
- 工具调用**前后拦截**
- **错误处理**
- **日志记录**
- **权限校验**
- 定义**Hook接口**
- 支持**Before/After/Error**三个切入点
- 可**链式组合**
- **优先级排序**
- 支持**同步和异步**Hook

## 顺口溜

钩前钩后钩出错，链上排序不闯祸。

## 单字锚点 + 防崩推导

- **锚点字**：钩
- **扩展口诀**：钩前后，五场景
- **为什么选它**：题目核心是"Hook中间件"，"钩"字就是题眼，钩住生命周期，插入逻辑
- **推导链**：
  1. 钩 → Hook → 钩子，钩住关键节点
  2. 前 → Before → 请求预处理、工具调用前拦截、权限校验
  3. 后 → After → 响应后处理、日志记录
  4. 出错 → Error → 错误处理
  5. 五场景 → 预处理、后处理、拦截、错误、日志（权限可归到预处理）
  6. 链式组合 + 优先级 + 同步异步 → 工程实现细节

## 一字一画面

- **钩** —— 像钓鱼钩：鱼咬饵的瞬间钩子挂住鱼嘴，不需要改变水流方向（本质：在关键时机捕获事件，不改动主干流程）
- **前** —— 像机场值机柜台：旅客进安检前先值机、称行李、贴标签，没票的不让进（本质：准入控制，前置过滤）
- **后** —— 像餐厅结账小票：吃完饭后打印消费明细，记录吃了什么、花了多少（本质：事后记录，留痕审计）

## 面试话术

1. "Hook最大的价值是在Agent关键生命周期里插自定义逻辑，但不用改核心代码，比如我想在每次调用工具前加权限校验，直接注册一个Before Hook就行。"
2. "OpenClaw把Hook分成Before、After、Error三个切入点，支持链式组合和优先级排序，同步异步都能跑，典型的场景有请求预处理、响应后处理、工具拦截、错误兜底和日志审计。"

## 防混补丁

- **容易和什么搞混**：容易和AOP面向切面搞混，以为是编译期织入。
- **补丁口诀**：Hook是运行时插桩，不是编译期代理；注册即生效，卸载即消失。

## 追问预演

1. **Q：Hook和直接改源码有什么区别？**
   A：Hook是扩展点，升级核心不影响Hook；改源码是侵入式，升级要合并代码。
2. **Q：Hook链如果有一个报错怎么办？**
   A：OpenClaw支持配置失败策略，可以中断后续Hook，也可以继续执行并记录错误。
3. **Q：异步Hook怎么保证顺序？**
   A：优先级排序对异步Hook同样生效，同优先级按注册顺序执行，不同优先级等前一个优先级全部完成再下一个。
