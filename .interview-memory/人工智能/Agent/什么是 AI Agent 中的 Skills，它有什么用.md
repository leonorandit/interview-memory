## 速记卡片

| 锚点字 | 口诀 | 核心差异 |
|--------|------|----------|
| 技 | 技封装，复调用 | Skills=可复用的能力单元 |

## 原始知识

- **封装**特定功能的工具集合
- 如代码编写Skill包含**文件读写**
- 模块化的Agent能力
- 便于**复用和组合**
- **降低开发复杂度**
- 支持Skill市场共享
- 类似**App Store**

## 顺口溜

技封装，复调用，
模块化，降复杂。

## 单字锚点 + 防崩推导

**锚点字：技**

- **扩展口诀**：技封装，复调用
- **为什么选它**："技"=技能，直接对应Skills的中文含义，一想到"技"就想到Agent的"技能包"。
- **推导链**：
  1. 技 → 技能 → Skills
  2. Skills是什么 → 封装好的能力单元
  3. 封装了什么 → 工具集合（如代码编写Skill=文件读写+代码执行+调试）
  4. 有什么用 → 模块化、复用、组合、降低复杂度
  5. 进阶点 → 可以像App Store一样共享

## 一字一画面

**技** —— 像**多功能瑞士军刀**：红色刀身啪地弹开，螺丝刀、剪刀、开瓶器各就各位，按需取用（本质：Skills是把零散工具封装成标准化能力包）。

## 面试话术

"Skills是Agent可复用的能力单元，你可以把它理解成Agent的瑞士军刀——把特定功能的工具集合封装在一起，比如一个代码编写Skill就包含了文件读写、代码执行和调试的能力。这样做最大的好处是模块化和复用，不同Agent可以像从App Store下载App一样去组合已有的Skill，大幅降低开发复杂度。"

## 防混补丁

容易和**MCP**搞混？记住：**技是模块，协是协议**——Skills是"能力包"，MCP是"通信规则"。

## 追问预演

| 追问 | 简要答案 |
|------|----------|
| Skills和Tool有什么区别？ | Tool是单个工具，Skill是工具的封装集合，粒度更大。 |
| 如何设计一个好的Skill？ | 单一职责、接口标准化、输入输出契约清晰、独立可测试。 |
| Skill之间可以互相调用吗？ | 可以，通过依赖管理实现Skill的组合和复用。 |
