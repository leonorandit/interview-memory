## 原始知识

- **ReAct=Reasoning + Acting**：推理与行动结合
- **循环步骤**：Thought思考当前状态/Action选择行动/Observation观察结果
- **与CoT区别**：ReAct有行动反馈/CoT只有推理
- **构建方法**：定义Prompt模板包含Thought/Action/Observation/定义可用工具集/LLM按格式输出/解析执行
- **优势**：减少幻觉/可解释性强/能处理复杂多步任务

## 顺口溜

反观行，三循环，有反馈，少幻觉。

## 单字锚点 + 防崩推导

- **锚点字**：反
- **扩展口诀**：反观行，三循环
- **为什么选它**："反"是ReAct的首字谐音（Reasoning+Acting=反），一看到"反"就想起推理+行动
- **推导链**：
  1. 反 → ReAct（Reasoning + Acting）
  2. 反的第一步 → 思（Thought，思考当前状态）
  3. 思完要动 → 观（Action，选择行动/调用工具）
  4. 动完要看 → 行（Observation，观察执行结果）
  5. 三步成一环 → 三循环（Thought-Action-Observation）
  6. 循环的优势 → 减少幻觉/可解释性强/能处理复杂多步任务

## 一字一画面

- **反** —— 像大脑和双手搭档：大脑想一步，双手做一步，做完大脑再评估。（本质：推理与行动的交替结合）
- **思** —— 像棋手沉思：盯着棋盘默念"如果走马，对方会如何应对"。（本质：Thought步骤对当前状态的推理）
- **观** —— 像按电梯按钮：想好去几楼后，手指按下对应数字。（本质：Action步骤选择具体行动）
- **行** —— 像电梯到达后的铃声：叮的一声，告诉你现在到了哪一层。（本质：Observation步骤获取环境反馈）
- **循** —— 像洗衣机的转动：浸水-洗涤-漂洗反复循环，直到衣服干净。（本质：多轮迭代逼近目标）

## 面试话术

1. "ReAct的核心是'反观行'三循环：Thought思考当前状态，Action选择行动，Observation观察结果。它和CoT最大的区别是ReAct有行动反馈，能根据环境修正推理。"
2. "构建ReAct Agent需要定义包含Thought/Action/Observation的Prompt模板，明确可用工具集，让LLM按固定格式输出，框架解析后执行并返回Observation，形成闭环。"

## 防混补丁

- **容易和什么搞混**：容易把ReAct和CoT搞混，以为两者都是纯推理。
- **补丁口诀**：CoT只动嘴，ReAct又动嘴又动手，动手后有反馈。

## 追问预演

1. **Q：ReAct和CoT到底有什么区别？** A：CoT只有推理链，没有与环境交互；ReAct每推理一步就行动一步，行动结果反馈给下一步推理，能修正错误。
2. **Q：ReAct的Prompt模板怎么设计？** A：要求LLM按"Thought: ... Action: ... Observation: ..."的格式输出，框架正则解析Action并执行。
3. **Q：ReAct适合什么场景？不适合什么？** A：适合需要多步工具调用的复杂任务；不适合单轮问答或延迟敏感场景。
