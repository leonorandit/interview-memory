# 什么是 LangChain Agent

## 速记卡片
`代 | 代决工，循环停 | Agent代理`

## 原始知识
- LangChain Agent：能自主决策、循环执行任务的代理
- 工作原理：
  1. **代**（代理）- 接收用户任务
  2. **决**（决策）- LLM决定下一步行动（Thought）
  3. **工**（工具调用）- 执行行动（Action），调用工具
  4. **循环**（循环）- 观察结果（Observation），决定继续还是结束
  5. **停**（停止）- 任务完成，返回最终答案
- 核心模式：ReAct（推理+行动循环）
- **易混点**：Agent vs Chain
- **追问点**：Agent怎么选择工具？怎么防止死循环？

## 顺口溜
**代决工，循环停**

## 单字锚点 + 防崩推导
- **锚点字**：**代**
- **扩展口诀**：**代决工，循环停**
- **为什么选它**：Agent就是"代理"，"代"是最直接的锚点。
- **防崩推导**：
  - 代（Agent）→ 自主代理
  - 决（决策）→ LLM思考决定下一步
  - 工（工具）→ 调用工具执行
  - 循环 → 观察结果、继续决策
  - 停 → 任务完成停止

## 一字一画面
- **代** —— 像**私人助理**：你交代任务，他去办。（本质：代理）
- **决** —— 像**军师出谋划策**：先想清楚怎么做。（本质：LLM决策）
- **工** —— 像**跑腿办事**：按军师说的去执行。（本质：工具调用）
- **循环** —— 像**反馈调整**：办完一件看下结果，再决定下一件。（本质：ReAct循环）
- **停** —— 像**任务完成汇报**：事情办完了，回来报告。（本质：返回结果）

## 面试话术
- 问什么是LangChain Agent → "LangChain Agent是自主决策的代理，口诀是**代决工，循环停**。**代**——接收任务；**决**——LLM思考决策（Thought）；**工**——调用工具执行（Action）；**循环**——观察结果（Observation），决定继续还是结束；**停**——任务完成返回答案。核心是ReAct模式：推理和行动交替循环，直到问题解决。"

## 防混补丁
- 容易和"Chain"搞混。Chain是固定流程，Agent是动态决策。补丁：**Chain是流水线，Agent是调度员**。"
