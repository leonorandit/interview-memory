## 原始知识

- **Agent定义**：具备自主规划、工具调用、记忆和反馈循环的AI系统
- **单次问答 vs 多轮循环**：API一问一答，Agent反复迭代
- **无状态 vs 有记忆**：API无记忆，Agent记历史
- **纯文本 vs 可操作工具**：API只生成文本，Agent可调外部工具
- **被动回答 vs 主动规划**：API等提问，Agent自主分解目标
- **Planning规划**：拆解复杂任务
- **Tool Use工具使用**：调用函数操作外部
- **Memory记忆**：保存上下文状态
- **Reflection反思**：自我纠错改进

## 顺口溜

环工反，四区别：
Agent能循环，工具能调用，反思能纠错，记忆忘不了。

## 单字锚点 + 防崩推导

**锚点字**：环

**扩展口诀**：环工反，四区别

**为什么选它**：Agent最本质的特征就是"循环"——感知-决策-行动的闭环，这是API调用完全没有的。

**推导链**：
1. 环 → 什么环？反馈循环（感知→决策→行动→观察→再决策）
2. 环 → 循环里有什么？Planning（规划）+ Tool Use（工具使用）
3. 工 → 工具怎么用？根据用户意图自主判断调用
4. 反 → 反思什么？Reflection自我纠错，评估执行结果
5. 四区别 → 单次vs循环、无状态vs记忆、纯文本vs工具、被动vs主动

## 一字一画面

环 —— 像**跑步机履带**：黑色履带不断循环转动，人站在上面持续前进（本质：闭环迭代，一轮结束后自动进入下一轮）

工 —— 像**瑞士军刀弹出**：红色刀身咔哒一声，不同刀片按需弹出（本质：工具按需调用，不是固定输出）

反 —— 像**后视镜倒车**：司机盯着后视镜调整方向，错了就修正（本质：Reflection观察结果并自我纠错）

## 面试话术

"Agent和直接调API最大的区别是一个字——'环'。调API就像问一句答一句，无状态、纯文本；Agent是一个有反馈循环的系统，它会自主规划、调用工具、记住上下文、还能反思纠错。简单说，API是'问答机'，Agent是'能干活的小助手'。"

## 防混补丁

容易和**RAG系统**搞混？补丁口诀：RAG是查资料再回答，Agent是**动手操作**再反馈。

## 追问预演

**Q：Agent的Planning具体怎么做？**
A：把用户目标拆解成子任务列表，按依赖关系排序执行，类似ReAct或CoT模式。

**Q：Tool Use是谁来执行？**
A：LLM生成函数调用JSON，由系统框架解析并执行真实函数，结果再回传给LLM。

**Q：Reflection怎么实现？**
A：执行完一步后让LLM评估结果是否满足预期，不满足则重新规划。
