## 速记卡片

| 锚点字 | 口诀 | 核心差异 |
|--------|------|----------|
| 角 | 角规工示，四职责 | 角色规范工具示例 |

## 原始知识

- **设定角色身份**：Agent是谁、风格语气
- **定义行为规范**：能做什么、不能做什么
- **描述可用工具**：工具列表及调用方式
- **Few-shot示例**：给样例教输出格式
- **设定输出格式**：JSON/Markdown等格式要求
- **分层Prompt**：核心指令放System，动态内容放User
- **Prompt压缩技术**：删除冗余词、缩写
- **关键信息前置**：LLM对开头更敏感
- **定期清理冗余**：去掉过期历史
- **外部记忆库**：不塞Prompt，按需检索

## 顺口溜

角规工示，四职责：
定角色，立规矩，教工具，给示例。

## 单字锚点 + 防崩推导

**锚点字**：角

**扩展口诀**：角规工示，四职责

**为什么选它**：System Prompt首先是"角色设定"，就像演员拿到剧本先看自己演谁。

**推导链**：
1. 角 → 角色（设定Agent身份、语气、风格）
2. 规 → 规范（行为约束，能做什么不能做什么）
3. 工 → 工具（描述可用工具及调用方式）
4. 示 → 示例（Few-shot示例教输出格式）
5. 四职责 → 角色、规范、工具、示例四大块
6. 太长怎么办？分层 + 压缩 + 前置 + 外部记忆

## 一字一画面

角 —— 像**戏服衣架**：一件绣着名字的戏服挂在首位，演员进来先穿上它（本质：角色身份是System Prompt的根基）

规 —— 像**交通红绿灯**：红黄绿三色灯立在路口，车辆见灯行止（本质：行为规范划定Agent的允许/禁止区间）

工 —— 像**工具箱内衬**：红色内衬上印着每把扳手的轮廓，对号入座（本质：工具描述让LLM知道有什么可用）

示 —— 像**烘焙食谱步骤图**：彩色照片一步一步展示成品样子（本质：Few-shot示例降低LLM理解成本）

## 面试话术

"System Prompt主要承担四个职责，我记成'角规工示'。角是角色身份，规是行为规范，工是工具描述，示是Few-shot示例。如果System Prompt太长，我的处理策略是分层——核心指令死守System，动态示例和上下文往User消息里放；再加上Prompt压缩、关键信息前置，实在不行就上外部记忆库，按需检索而不是全塞进去。"

## 防混补丁

容易和**User Prompt**搞混？补丁口诀：System是底层人设，User是具体任务，角色在System，问题在User。

## 追问预演

**Q：关键信息为什么要前置？**
A：LLM对Prompt开头注意力更强，中间容易丢失，重要的放前面。

**Q：外部记忆库怎么按需检索？**
A：用户提问Embedding后去向量库检索相关知识片段，拼接进User Prompt。

**Q：Few-shot示例多少条合适？**
A：一般1-3条，太多占Token且可能过拟合样例风格。
