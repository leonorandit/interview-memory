# 单字锚点 + 防崩推导

## 锚点字：核

- **扩展口诀**：核（核心）运（Runner运行）记（Memory记忆）工（Tool工具）四组件
- **为什么选它**："核"直接对应"核心组件"，是题目自带的题眼
- **推导链**：
  1. 看到"核" → 想到**核心组件**
  2. "运" → **Agent Runner**（运行/执行引擎），它是**驱动者**
  3. "记" → **Memory Manager**（记忆管理），管理**短期+长期记忆**
  4. "工" → **Tool Manager**（工具管理），负责**注册和调用**
  5. 还少一个？四组件！→ **Context Manager**（上下文管理），维护**对话窗口**
  6. 关系：Runner是**大脑+指挥**，其他三个是**手脚+记忆**，四者**协同**

# 一字一画面

- **核** —— 像[核桃剖开四瓣]：棕色核桃被敲开，露出四瓣金黄色果仁紧密排列（本质：核心组件是一个整体，缺一不可）
- **运** —— 像[赛车引擎轰鸣]：红色跑车引擎盖下V8发动机转动，带动四个轮子飞奔（本质：Runner驱动整体流程）
- **记** —— 像[便签墙贴满便利贴]：黄色蓝色便利贴层层叠加，新贴盖住旧的但边缘露出关键词（本质：记忆分短期活跃和长期存档）
- **工** —— 像[瑞士军刀弹开工具]：红色军刀"咔咔"弹出螺丝刀剪刀开瓶器，各就各位（本质：工具管理器按需调度）

# 面试话术

"OpenClaw的核心组件有四个：Agent Runner执行引擎、Context Manager上下文管理器、Memory Manager记忆管理器，还有Tool Manager工具管理器。Runner是总指挥，负责驱动整个流程；Context Manager维护对话窗口；Memory Manager管理短期和长期记忆；Tool Manager负责工具的注册和调用。它们四个协同工作，完成Agent的完整任务链路。"

# 防混补丁

- 容易和 **微服务架构** 搞混？一句补丁：**四组件是单Agent内部分工，不是独立服务，共进程共生死**

# 追问预演

| 追问 | 简要答案 |
|------|----------|
| Context Manager和Memory Manager有什么区别？ | Context管当前对话窗口里的原始消息，Memory管提取后的短期/长期记忆，Context是RAW，Memory是COOKED |
| Tool Manager是怎么注册工具的？ | 通常通过装饰器或配置文件注册，Agent Runner在执行阶段通过Tool Manager查找并调用 |
| 如果Runner挂了，其他组件怎么办？ | 四者共生命周期，Runner是入口，Runner停止则整个Agent实例终止 |
