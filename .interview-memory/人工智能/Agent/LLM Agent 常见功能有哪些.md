## 原始知识

- **自主问答**：独立理解问题并给出答案
- **代码生成/执行**：写代码、跑代码、修bug
- **多工具调度**：同时协调多个API/服务
- **信息检索与摘要**：搜资料、读长文、出摘要
- **多轮对话与记忆**：跨轮次保持上下文与偏好
- **文件处理**：读文档、写报告、解析表格

## 单字锚点 + 防崩推导

**锚点字**：能

**扩展口诀**：能问答，三功能

**为什么选它**：问题问"功能有哪些"，"能"字直接对应"能力/功能"，看到"能"就想起"Agent能干什么"。

**推导链**：
1. 能 → 能力 → Agent的功能清单
2. 问答 → 第一功能：自主问答（最基础，聊天对话）
3. 三功能 → 实际六大功能，但口诀给"三"，怎么拆？
   - 拆法一：问答类（自主问答 + 多轮对话与记忆）、生成类（代码生成 + 文件处理）、外部类（多工具调度 + 信息检索与摘要）
   - 拆法二：记住"三"是概数，代表"多"，面试时展开六个
   - 最稳路径：能 → 问答（1自主问答 2多轮对话）→ 生成（3代码 4文件）→ 外部（5调度 6检索），六脉神剑，口诀"三功能"提示分三大类
4. 若只记得"能问答"三个字：
   - 能 = 能力
   - 问 = 自主问答、多轮对话
   - 答 = 给出答案的方式 → 代码、文件、检索摘要都是"答"的不同形态
   - 再补：调度 = 多工具调度（答得好的前提是工具用得好）
5. 数字保险：记住6个功能，可以数手指——两手各三根。

## 面试话术

"LLM Agent的常见功能可以分这几块：最基础的是自主问答和多轮对话；进阶一点有代码生成与执行、文件处理；再往上就是多工具调度和信息检索摘要。说白了，Agent不只是聊天，还能写代码、读文档、调API、搜资料，把LLM从'动嘴'变成'动手'。"

"如果让我总结，Agent的核心能力是'问—搜—写—调—记—管'，分别对应问答、检索、代码/文件生成、工具调度、记忆、文件处理。"

## 防混补丁

容易和"Agent架构组成"搞混？

**补丁口诀**：架构是里子四根柱，功能是面子六扇窗。

（架构回答"由什么模块构成"，功能回答"对外能做什么"——别把Planning/Action架构词当成功能来答。）

## 追问预演

**Q1**：多工具调度具体怎么实现？
**A**：通过Function Calling让LLM判断需要调哪个工具，生成参数，并行或串行执行，再把结果汇总返回给用户。

**Q2**：信息检索和RAG有什么区别？
**A**：RAG是检索增强生成的一种实现方式，信息检索是Agent的功能之一，Agent做检索可以用RAG，也可以直接调用搜索引擎API。

**Q3**：代码执行功能安全吗？
**A**：一般放在沙箱环境里执行，比如Docker容器，限制网络和文件访问，防止恶意代码。

**Q4**：文件处理支持哪些格式？
**A**：常见PDF、Word、Excel、TXT、Markdown，解析后提取文本或结构化数据，再交给LLM处理。
