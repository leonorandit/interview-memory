## 原始知识

- **死循环三原因**：工具调用无结果、推理陷入循环、目标无法达成一直尝试
- **最大轮次限制**：设上限防无限循环
- **设置超时时间**：时间到强制中断
- **人类介入机制**：卡死时人工接管
- **终止条件**：Agent目标达成立即停
- **历史去重**：避免重复调用相同工具
- **监控告警**：检测循环模式自动中断

## 顺口溜

限深超，三解法：
限轮次，深去重，超时报警人工控。

## 单字锚点 + 防崩推导

**锚点字**：限

**扩展口诀**：限深超，三解法

**为什么选它**：死循环本质是"无限制"，"限"字直击问题核心——要给Agent套上缰绳。

**推导链**：
1. 限 → 限制什么？限制轮次（最大轮次限制）
2. 限 → 限制什么？限制时间（设置超时时间）
3. 深 → 深入检查（历史去重，避免重复调用）
4. 超 → 超过阈值怎么办？（监控告警 + 人工介入）
5. 三解法 → 轮次限制、超时机制、人工介入三大兜底

## 一字一画面

限 —— 像**红色警戒线**：一条鲜红的激光线在走廊尽头闪烁（本质：划定边界，越线即停）

深 —— 像**潜水艇声呐**：屏幕上一圈圈波纹扫描，发现重复目标立刻标红（本质：去重检测，识别重复模式）

超 —— 像**厨房计时器**：黄色圆盘滴答作响，时间一到铃声大作（本质：时间盒，强制中断）

## 面试话术

"死循环确实遇到过，核心思路就一个字——'限'。我通常会三层防护：第一层设最大轮次，比如最多调10轮工具；第二层设超时，单步超过30秒就熔断；第三层做历史去重，避免重复调同一个接口。最后再加上监控告警，真卡住就人工介入。"

## 防混补丁

容易和**递归栈溢出**搞混？补丁口诀：递归是代码调用自己，Agent循环是决策反复绕圈。

## 追问预演

**Q：如果工具调用确实需要很多轮怎么办？**
A：可以分层，复杂任务先拆解子目标，每个子目标单独设轮次限制。

**Q：历史去重具体怎么做？**
A：记录已调用的工具名+参数签名，MD5去重或语义相似度判断。

**Q：监控告警怎么检测循环模式？**
A：看工具调用序列是否进入周期性重复，或连续多轮返回相同结果。
