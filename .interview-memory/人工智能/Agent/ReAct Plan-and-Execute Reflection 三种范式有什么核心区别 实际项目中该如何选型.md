# ReAct、Plan-and-Execute、Reflection 三种范式有什么核心区别？实际项目中该如何选型？

## 速记卡片

## 原始知识

### 三种范式的核心区别

| 维度 | ReAct | Plan-and-Execute | Reflection |
|------|-------|------------------|------------|
| **核心思想** | 边想边做，推理与行动交替 | 先想后做，计划再执行 | 做完再想，自我评估修正 |
| **执行节奏** | 每一步都重新思考 | 一次性规划，分步执行 | 执行完后统一反思 |
| **灵活性** | 高，随时调整策略 | 中，计划内执行 | 高，通过反思迭代改进 |
| **适用任务** | 动态环境、需实时反馈 | 多步骤复杂任务 | 质量要求高、需迭代优化 |
| **代表框架** | LangChain ReAct、AutoGPT | BabyAGI、Plan-and-Solve | Reflexion、Self-Refine |

### 实际项目选型指南

**选 ReAct 当**：
- 任务步骤不确定，需要实时观察环境反馈
- 需要与外部工具频繁交互（搜索、API、数据库）
- 环境动态变化，计划赶不上变化
- 例子：智能客服、实时数据分析、交互式编程助手

**选 Plan-and-Execute 当**：
- 任务步骤明确，可以一次性规划清楚
- 减少 LLM 调用次数，降低成本
- 执行过程中不需要频繁改变策略
- 例子：报表生成、数据清洗流水线、固定流程自动化

**选 Reflection 当**：
- 对输出质量要求高，需要迭代优化
- 有明确的评估标准可以自我打分
- 执行成本高，一次做对很重要
- 例子：代码生成、文章写作、复杂报告生成

### 组合使用
实际项目中常常是**组合使用**：
```
Plan-and-Execute（先定大计划）
    ↓ 每一步用
ReAct（边做边调整）
    ↓ 执行完用
Reflection（检查质量，不满意重来）
```

**选** —— 像 **三把钥匙**：[三把不同颜色] 红色钥匙（ReAct）开动态门，蓝色钥匙（Plan）开固定门，绿色钥匙（Reflection）开质量门。

## 面试话术
ReAct、Plan-and-Execute、Reflection 的核心区别在于**思考与行动的时机**。ReAct 是边想边做，适合动态环境；Plan-and-Execute 是先想后做，适合步骤明确的任务；Reflection 是做完再想，适合需要迭代提升质量的场景。实际选型看三个因素：**任务确定性**（确定用 Plan，不确定用 ReAct）、**环境动态性**（动态用 ReAct）、**质量要求**（高要求加 Reflection）。很多时候是组合使用：先用 Plan 定大框架，每步用 ReAct 灵活执行，最后用 Reflection 检查质量。

## 防混补丁
**别混**：不要把三种范式当成互斥的，它们解决的是不同维度的问题。口诀：**大框用计，灵活处放反，收工前反思**。
