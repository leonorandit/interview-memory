# 什么是 Function Calling？原理是什么？

## 速记卡片
`外 | 识调参，外执行 | 模型不执行`

## 原始知识
- **定义**：大模型识别需要外部工具，生成结构化调用请求（函数名+参数）的能力
- **核心流程**：用户提问 + 工具列表 → 模型判断是否需要工具 → 生成 JSON 调用 → 外部系统执行 → 结果回传 → 模型生成最终回答
- **关键认知**：模型**只生成调用请求，不真正执行函数**
- **常见叫法**：OpenAI 叫 Function Calling，Google/Anthropic 叫 Tool Calling
- **追问点**：模型怎么知道有哪些工具？参数生成错了怎么办？

## 顺口溜
**识调参，外执行**

## 单字锚点 + 防崩推导
- **锚点字**：**外**
- **扩展口诀**：**外执行，模生参**
- **为什么选它**：最大误区是以为模型自己调 API。记住"外"，就记住执行一定在外部。
- **防崩推导**：
  - 外 → 外部执行 → 模型本身不能联网、不能查库
  - 执行 → 外部代码/系统跑函数 → 拿到结果
  - 模 → 模型只负责判断和生成
  - 生 → 生成结构化参数（JSON）
  - 参 → 参数按 Schema 填写
  - 识 → 识别用户意图需要工具

## 一字一画面
- **识** —— 像**医生问诊**：病人说头疼，医生判断需要做 CT。（本质：意图识别，判断是否需要工具）
- **调** —— 像**医生开检查单**：写上"头部 CT"。（本质：选择函数/工具）
- **参** —— 像**检查单填详细信息**：部位、是否增强。（本质：生成结构化参数 JSON）
- **外** —— 像**护士带病人去放射科**：医生自己不去拍片。（本质：模型不执行，只发请求）
- **执行** —— 像**CT 机拍完出片子**。（本质：外部系统运行函数）
- **回** —— 像**护士把片子送回诊室**。（本质：结果回传给模型）
- **答** —— 像**医生看着片子下诊断**："你是偏头痛"。（本质：模型基于工具结果生成回答）

## 面试话术
- **问概念** → "Function Calling 说白了就是**识调参，外执行**。用户提问时，我们把可用工具的 Schema 告诉大模型。模型判断需要工具后，生成一个 JSON，包含要调哪个函数、参数是什么。外部代码拿到这个 JSON 去真正执行，再把结果喂回模型，最后模型生成回答。核心记住：模型是开方子的医生，不是拍 CT 的机器。"
- **问原理** → "原理上分为六步：1. 注册工具（把函数签名和描述传给模型）；2. 用户提问；3. 模型判断并生成调用 JSON；4. 外部解析 JSON 并执行函数；5. 结果以消息角色 `tool` 回传；6. 模型基于结果生成最终回答。"

## 防混补丁
- **容易和"模型直接执行代码"搞混**。模型只生成调用请求，不执行代码。
- **补丁**：**模型是开方子的医生，不是拍 CT 的机器**。
- **容易和 MCP 搞混**。Function Calling 是能力，MCP 是标准化协议。
- **补丁**：**Function Calling 是会开方子的能力，MCP 是医院统一处方单格式**。

## 追问预演
| 追问 | 简要答案 |
|------|----------|
| 模型怎么知道有哪些工具？ | 通过 System Prompt 或 API 的 `tools` 字段传入 JSON Schema |
| 参数生成错了怎么办？ | 代码层校验必填项和类型；回传错误让模型重试；加示例 Few-shot |
| Function Calling vs Tool Calling？ | 本质一样，OpenAI 叫 Function Calling，其他厂商多叫 Tool Calling |
| 和 Agent 什么关系？ | Function Calling 是 Agent 调用工具的核心机制，Agent 在此基础上加规划、记忆、循环 |
| 不支持 Function Calling 的模型怎么实现？ | Prompt 工程强制输出 JSON 格式，再用正则/JSON 解析提取 |
