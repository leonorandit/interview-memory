## 速记卡片

核 | 核记工，三优先 | Runner先跑，Context续命，Tool赋能

## 原始知识

- **Agent Runner**执行引擎
- 负责核心**推理循环**
- 最先做因为是**一切的基础**
- **Context Manager**上下文管理
- 控制**Token**和记忆
- 保证Agent能**持续工作**
- **Tool Manager**工具管理
- 让Agent具备**外部能力**
- 是Agent区别于**Chat的核心**
- **MVP最小可用**
- 核心循环**跑通才能迭代**
- 上下文管理是**工程核心约束**
- 工具调用是**Agent价值所在**

## 顺口溜

核跑圈，记续命，工干活，三足鼎。

## 单字锚点 + 防崩推导

- **锚点字**：核
- **扩展口诀**：核记工，三优先
- **为什么选它**：题目核心是"从零搭建，先做哪三个"，"核"字点出核心模块，三模块中最基础的是Runner
- **推导链**：
  1. 核 → 核心 → Agent Runner执行引擎
  2. 记 → 记忆/上下文 → Context Manager
  3. 工 → 工具 → Tool Manager
  4. 三优先 → Runner最先、Context其次、Tool第三
  5. Runner为什么先 → 推理循环是一切基础，没它Agent不会动
  6. Context为什么次 → Token和记忆是工程约束，跑久了必崩
  7. Tool为什么三 → 外部能力是Agent区别于ChatGPT的关键

## 一字一画面

- **核** —— 像汽车发动机：没有发动机，再好的内饰和导航都是摆设，一拧钥匙先听引擎响（本质：执行引擎是动力源，决定Agent能不能运转）
- **记** —— 像手机电量管理：屏幕再亮、APP再多，没电就全停，电量显示20%时自动降频保命（本质：上下文管理是续航系统，决定Agent能跑多久）
- **工** —— 像瑞士军刀：没有刀的手只是手，有了刀才能削苹果、开酒瓶，能力边界被工具定义（本质：工具扩展是Agent超越对话的核心竞争力）

## 面试话术

1. "我会先做Agent Runner，因为推理循环是Agent的心脏，没有它后面什么都跑不起来，MVP阶段先把核心循环打通最重要。"
2. "第二个做Context Manager，因为Token限制和记忆管理是工程上的硬约束，不做这层Agent聊几轮就崩了，根本没法持续工作。"
3. "第三个做Tool Manager，因为能调用外部工具才是Agent区别于普通Chat的核心价值所在，但前提是前两个已经跑稳了。"

## 防混补丁

- **容易和什么搞混**：容易和基础设施搞混，以为先做日志、监控、配置中心。
- **补丁口诀**：不是先做基建，是先做业务核心；Runner跑通了，监控才有意义。

## 追问预演

1. **Q：为什么不先做Tool Manager？**
   A：工具再好，Agent不会调用、调完不会续上下文，还是跑不起来；Runner和Context是1，Tool是后面的0。
2. **Q：Context Manager和Runner能合并吗？**
   A：短期可以，但很快会拆；Runner负责"怎么思考"，Context负责"记住什么"，关注点不同。
3. **Q：MVP阶段Tool Manager要支持多少个工具？**
   A：一到两个就够了，比如搜索和代码执行，验证"能调用-能返回-能继续"的闭环即可。
