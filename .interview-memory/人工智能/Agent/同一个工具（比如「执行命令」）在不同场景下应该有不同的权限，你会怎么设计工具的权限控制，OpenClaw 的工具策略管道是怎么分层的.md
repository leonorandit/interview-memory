## 速记卡片

权 | 权角审执，四分层 | 管道式处理，每层可插拔

## 原始知识

- **角色**权限
- **场景**权限
- **用户**权限
- **动态**权限
- **Policy**层定义规则
- **Filter**层过滤条件
- **Approval**层审批机制
- **Execution**层执行控制
- **管道式处理**
- 每层**可插拔**

## 顺口溜

权定角，审过滤，执行前，四层护。

## 单字锚点 + 防崩推导

- **锚点字**：权
- **扩展口诀**：权角审执，四分层
- **为什么选它**：题目核心是"权限控制"，"权"字就是题眼，权限分层，层层把关
- **推导链**：
  1. 权 → 权限 → 工具权限控制
  2. 角 → 角色 → Policy层定义角色规则
  3. 审 → 审批 → Approval层人工或自动审批
  4. 执 → 执行 → Execution层最终执行控制
  5. 四分层 → Policy、Filter、Approval、Execution
  6. 可插拔 → 每层都是独立插件，按需组合

## 一字一画面

- **权** —— 像古代调兵虎符：右半边在将军手里，左半边在皇帝手里，两半合拢才能出兵（本质：权限是凭证，缺一不可）
- **角** —— 像剧场角色牌：演员进场先领角色牌，牌上写着能进哪个房间、不能进哪个房间（本质：角色预定义行为能力边界）
- **审** —— 像机场安检仪：行李先过X光机，可疑物品被拦下人工复检，没问题才放行（本质：中间拦截，异常阻断）
- **执** —— 像电梯刷卡层：刷卡后只能点亮有权限的楼层，没权限的楼层按钮不亮（本质：最终执行点硬拦截）

## 面试话术

1. "我会把权限设计成管道模式，从Policy定义规则开始，Filter做条件过滤，Approval做敏感操作审批，Execution做最终执行拦截，每层都可插拔。"
2. "比如执行命令这个工具，Policy层规定只有运维角色能调，Filter层检查当前场景是不是生产环境，Approval层要求高危命令二次确认，Execution层才真的下发命令。"

## 防混补丁

- **容易和什么搞混**：容易和RBAC角色权限搞混，以为只是角色判断。
- **补丁口诀**：不是单点判断，是管道递进；每层都过，层层留痕。

## 追问预演

1. **Q：Filter层和Policy层有什么区别？**
   A：Policy是规则定义（什么角色能做什么），Filter是运行时条件判断（当前环境是否允许），Policy回答"能不能"，Filter回答"现在行不行"。
2. **Q：Approval层怎么实现自动审批？**
   A：可以配置规则引擎，比如只读命令自动过，写命令转人工，或者低危操作自动审批、高危操作阻断。
3. **Q：动态权限是什么意思？**
   A：不是静态角色绑定，而是根据实时上下文动态计算，比如同一用户白天能执行A命令，晚上只能执行B命令。
