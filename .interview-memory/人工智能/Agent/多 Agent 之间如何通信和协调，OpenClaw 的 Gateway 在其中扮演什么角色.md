## 单字锚点 + 防崩推导

**锚点字：总**

- **扩展口诀**：总转协，三角色
- **为什么选它**："总"=总线，多Agent通信的核心方式是消息总线，而Gateway就是总线的枢纽。
- **推导链**：
  1. 总 → 总线 → 消息总线
  2. 多Agent通信方式 → 消息总线、直接RPC、共享状态、事件驱动
  3. OpenClaw Gateway角色 → 消息路由转发（路）
  4. 注册发现 → Agent上线注册、下线注销（注）
  5. 管控 → 负载均衡、权限控制、日志审计（权/审）
  6. 统一入口 → 所有Agent通信走Gateway
  7. 通信模式 → 广播和点对点
  8. 三角色 → 路由转发者、注册中心、管控中枢

## 一字一画面

**总** —— 像**机场中央调度塔**：玻璃幕墙环绕的控制室里，绿色雷达屏幕旋转，调度员头戴耳机手持话筒，指挥不同跑道的飞机起飞降落，广播和专线通话同时进行中（本质：Gateway是通信中枢，负责谁找谁、怎么走、安不安全）。

## 面试话术

"多Agent之间的通信方式主要有四种：消息总线、直接RPC调用、共享状态和事件驱动。在OpenClaw里，Gateway扮演着三个核心角色：首先是消息路由转发者，负责把消息送到正确的Agent；其次是注册发现中心，Agent上线和下线都在Gateway登记；最后是管控中枢，负责负载均衡、权限控制和日志审计。它是所有Agent通信的统一入口，既支持广播也支持点对点通信。"

## 防混补丁

容易和**API Gateway**搞混？记住：**总管Agent，API管用户**——OpenClaw Gateway是Agent间通信的中枢，而传统API Gateway面向的是外部客户端请求。

## 追问预演

| 追问 | 简要答案 |
|------|----------|
| Gateway单点故障怎么办？ | 支持多节点集群部署，采用一致性哈希做状态分片。 |
| 广播通信会不会消息风暴？ | 有TTL限制和消息去重，超范围或重复消息自动丢弃。 |
| Agent间通信加密吗？ | Gateway层支持TLS加密，敏感消息额外做端到端加密。 |
