## 原始知识

- 任务复杂需**分工**
- 不同领域**专家Agent**
- **并行处理**提高效率
- **主从架构**解耦
- Parent Agent可**spawn Subagent**
- Subagent**继承父Agent配置**
- **独立Context和记忆**
- 通过**消息总线**通信
- 支持**层级嵌套**
- 结果**回传父Agent**

## 单字锚点 + 防崩推导

**锚点字：协**

- **扩展口诀**：协分专，三场景
- **为什么选它**："协"=协作，题目核心就是"多Agent协作"，与题2的"协"不同场景（题2是协议），但同为协——这里协=协作。
- **推导链**：
  1. 协 → 协作 → 多Agent协作
  2. 为什么协作 → 任务复杂需分工（分）
  3. 协作类型 → 不同领域专家Agent（专）
  4. 协作好处 → 并行处理提高效率
  5. 架构 → 主从架构解耦
  6. OpenClaw支持 → Parent spawn Subagent
  7. Subagent特性 → 继承配置、独立Context和记忆
  8. 通信 → 消息总线
  9. 结构 → 层级嵌套，结果回传父Agent

## 一字一画面

**协** —— 像**交响乐队指挥台**：指挥家挥动银色指挥棒，弦乐、管乐、打击乐各组乐手分散就座，乐谱独立但节奏统一，最终汇聚成完整乐章（本质：多Agent协作是分工有序、各自独立又统一协调的系统性工作）。

## 面试话术

"需要多Agent协作的场景主要有三个：第一，任务本身很复杂，必须拆分给不同角色分工完成；第二，涉及多个专业领域，需要各自的专家Agent来处理；第三，为了并行处理提高效率，同时降低单个Agent的复杂度。在OpenClaw里，Parent Agent可以通过spawn创建Subagent，Subagent会继承父Agent的配置但拥有独立的Context和记忆，它们之间通过消息总线通信，支持层级嵌套，最后把结果回传给父Agent。"

## 防混补丁

容易和**微服务拆分**搞混？记住：**协有记忆，服无状态**——Subagent有独立Context和记忆，而微服务通常是无状态的。

## 追问预演

| 追问 | 简要答案 |
|------|----------|
| Subagent和Parent的Context怎么隔离？ | Subagent有独立Context，但可通过消息总线选择性同步关键信息。 |
| 嵌套层级有限制吗？ | 有限制，防止无限递归和资源耗尽，通常有最大深度配置。 |
| 多Agent协作怎么避免重复工作？ | 通过任务分配协议+共享状态+结果聚合去重。 |
