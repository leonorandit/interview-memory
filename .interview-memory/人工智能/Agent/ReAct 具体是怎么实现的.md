# ReAct 是啥？具体是怎么实现的？

## 场景触发词
ReAct 是什么、ReAct 怎么实现、ReAct 原理、Agent 推理+行动、ReAct 代码实现

## 速记卡片
`循 | 循观思行，四步环 | ReAct 实现`

## 原始知识

**ReAct = Reasoning（推理）+ Acting（行动）**

核心思想：让 LLM 在**推理**和**行动**之间循环，通过与外部环境交互来完成任务。

**具体实现——四步循环**：

```
观察（Observation）
    ↓
思考（Thought）
    ↓
行动（Action）
    ↓
观察（Observation）
    ↓
...（循环直到任务完成）
```

1. **观察（Observation）**
   - 接收当前环境信息：用户输入、工具返回结果、系统状态
   - 作为下一步推理的输入

2. **思考（Thought）**
   - LLM 生成内部推理文本，分析当前状态
   - 决定下一步需要做什么
   - 输出格式：`Thought: 我需要先搜索相关信息...`

3. **行动（Action）**
   - 根据思考结果，调用具体工具或给出最终答案
   - 输出格式：`Action: search("关键词")` 或 `Action: finish("最终答案")`

4. **观察（Observation）**
   - 获取行动执行后的结果
   - 反馈给下一轮思考

**关键实现细节**：
- **Prompt 模板**：定义 Thought/Action/Observation 的输出格式，用 few-shot 示例教会模型
- **工具解析**：从 Action 输出中提取工具名和参数，执行后把结果包装成 Observation
- **停止条件**：当 Action 为 `finish` 或达到最大循环次数时停止
- **错误处理**：工具调用失败时，把错误信息作为 Observation 反馈，让模型重新思考

## 单字锚点 + 防崩推导
- **锚点字**：循
- **扩展口诀**：循观思行
- **为什么选它**：“循”= 循环，ReAct 的核心就是**循环**迭代
- **防崩推导**：
  1. 循 → 循环
  2. 观 → **观**察环境，获取当前信息
  3. 思 → **思**考分析，决定下一步
  4. 行 → **行**动执行，调用工具或输出结果
  5. 四步环 → 观察→思考→行动→观察，循环往复

## 一字一画面
**循** —— 像 **跑步机**：[黑色传送带] 人不断循环跑动，但每一步都在前进，直到跑到目标距离。
**观** —— 像 **侦察兵望远镜**：[绿色镜头] 先观察敌情，把看到的情报带回来。
**思** —— 像 **将军在帐篷里看地图**：[昏黄灯光] 根据侦察兵的情报，分析局势，决定下一步战术。
**行** —— 像 **骑兵出击**：[马匹奔腾] 将军下令后，部队执行具体行动，打完仗再回报战果。

## 面试话术
ReAct 是 **Reasoning + Acting** 的缩写，核心就是四步循环：先**观察**环境，然后**思考**分析，接着**行动**执行（调用工具或输出答案），再把行动结果作为新的**观察**进入下一轮。实现上主要靠 Prompt 模板定义好 Thought/Action/Observation 的输出格式，用 few-shot 教会模型怎么循环，再加上一个工具解析器执行 Action 并把结果喂回去。循环直到模型输出 `finish` 或达到最大步数。

## 防混补丁
**别混**：ReAct 和 CoT 的区别在于 CoT 只思考不行动，ReAct 会调用工具改变环境。口诀：**CoT 闭门造车，ReAct 出门办事**。
