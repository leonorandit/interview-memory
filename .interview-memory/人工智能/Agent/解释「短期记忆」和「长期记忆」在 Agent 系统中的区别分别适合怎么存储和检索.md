## 速记卡片

| 锚点字 | 口诀 | 核心差异 |
|--------|------|----------|
| 短 | 短快长慢，两记忆 | 短期快取长期慢寻 |

## 原始知识

- **短期记忆内容**：当前对话上下文、对话历史、最近几轮交互
- **短期存储介质**：内存或Redis
- **短期访问特点**：快速访问、有Token上限
- **长期记忆内容**：用户画像、跨会话知识、历史任务经验
- **长期存储介质**：向量数据库或图数据库
- **长期检索方式**：Embedding检索、语义搜索
- **核心区别**：时效性、容量、存储介质、检索方式
- **OpenClaw做法**：Message History短期、Vector Store长期、Entity Store实体关系

## 顺口溜

短快长慢，两记忆：
短期像内存快如风，长期像档案慢如钟。

## 单字锚点 + 防崩推导

**锚点字**：短

**扩展口诀**：短快长慢，两记忆

**为什么选它**：用"短"对"长"，天然对比关系，一眼看出两种记忆的差异维度。

**推导链**：
1. 短 → 短期记忆（当前会话、最近几轮）
2. 短 → 短期特点？快（内存/Redis、毫秒级访问）
3. 长 → 长期记忆（跨会话、用户画像、历史经验）
4. 长 → 长期特点？慢（向量数据库、需要Embedding检索）
5. 两记忆 → 四种区别维度：时效性、容量、存储介质、检索方式
6. OpenClaw：Message History（短期）+ Vector Store（长期）+ Entity Store（实体关系）

## 一字一画面

短 —— 像**桌面便签纸**：黄色小纸片贴在显示器边框，随手一瞥就能看到（本质：触手可及，但容量极小，贴多了会掉）

快 —— 像**闪存U盘插拔**：银色U盘咔哒插入USB口，文件夹瞬间弹出（本质：Redis/内存读写，毫秒级响应）

长 —— 像**地下档案库**：金属抽屉柜排列到走廊尽头，档案员推着小车慢慢找（本质：数据量大，需要检索系统定位）

慢 —— 像**图书馆索引卡**：木质卡片柜，每张卡片写着位置编码，按编码去书架上取（本质：Embedding检索，先语义匹配再召回内容）

## 面试话术

"短期记忆和长期记忆我记成'短快长慢'。短期记忆是当会话的上下文，存内存或Redis，访问快但有Token上限；长期记忆是跨会话的用户画像和历史经验，存向量数据库或图数据库，通过Embedding做语义检索。OpenClaw也是这个思路，Message History管短期，Vector Store管长期，Entity Store管实体关系。"

## 防混补丁

容易和**缓存**搞混？补丁口诀：缓存是技术概念，记忆是语义概念；缓存存原始数据，记忆存**理解后的知识**。

## 追问预演

**Q：短期记忆截断了怎么办？**
A：可以用Summarization做摘要归档到长期记忆，或者触发长期记忆检索补全上下文。

**Q：图数据库适合存什么？**
A：实体关系网络，比如用户偏好图谱、知识图谱，适合关系型推理。

**Q：长期记忆怎么更新？**
A：会话结束后提取关键信息，Embedding入库，定期去重和衰减旧记忆。

## 关联知识点

- 向量数据库与Embedding检索（长期记忆的基础设施）
- Context Window（短期记忆的硬上限来源）
- Redis缓存（短期记忆的常用存储）
