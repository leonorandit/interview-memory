# 用的什么向量数据库Qdrant和Milvus有什么区别

## 速记卡片
`选 | 选Q轻，选M重 | 向量数据库选型对比`

## 原始知识
- **Qdrant**：Rust 编写，单机性能极强，轻量易部署，过滤能力强，API 友好（REST/gRPC）
- **Milvus**：Go/C++ 编写，云原生分布式架构，支持超大规模，功能全但部署复杂
- **Qdrant 适用**：中小规模（千万级以内）、快速上线、强过滤需求、资源有限团队
- **Milvus 适用**：超大规模（十亿级）、分布式扩展、多租户、企业级运维、GPU 加速
- **易混点**：Milvus 性能一定比 Qdrant 好
- **追问点**：其他向量数据库（Pinecone/Weaviate/pgvector/Chroma）、HNSW 索引、距离度量

**选Q轻，选M重**

## 单字锚点 + 防崩推导
- **锚点字**：**选**
- **扩展口诀**：**选Q轻，选M重**
- **为什么选它**：向量数据库选型是核心决策，"选"字直接对应面试场景
- **防崩推导**：
  - 选 → 向量数据库选型
  - Q → Qdrant → Rust 写 → 单机性能强

  - M → Milvus → Go/C++ → 分布式
  - 重 → 重量级 → 云原生、超大规模、部署复杂、功能全

## 一字一画面
- **选** —— 像**买车**：先看用途再选车型。（本质：按场景选型）
- **Q** —— 像**跑车**：Rust 引擎，起步快、操控灵。（本质：单机性能强）
- **轻** —— 像**电瓶车**：充电就能跑，不需要建加油站。（本质：轻量部署）
- **M** —— 像**重型卡车**：能拉几十吨货，但需要专业司机和停车场。（本质：分布式大规模）
- **重** —— 像**物流枢纽**：分拣中心、调度系统、多仓库联动。（本质：云原生企业级）

## 面试话术
- "我们选型时对比了 Qdrant 和 Milvus，核心差异是**选Q轻，选M重**。Qdrant 用 Rust 写，单机性能极强，部署轻量，过滤能力强，适合中小规模、追求快速上线的场景。Milvus 是云原生分布式架构，支持超大规模向量存储，功能全但部署复杂，适合企业级大数据量场景。最终我们根据数据量和团队运维能力做了选择。"

## 防混补丁
- 容易以为 Milvus 性能一定比 Qdrant 好。实际上在中小规模下 Qdrant 单机性能可能更优。补丁：**小规模Qdrant快，大规模Milvus稳**。

## 追问预演
- **Q: 什么场景选 Qdrant？**
  - A: 数据量千万级以内、追求快速部署、需要强过滤能力、团队资源有限、中小团队首选。
- **Q: 什么场景选 Milvus？**
  - A: 十亿级向量、需要分布式扩展、多租户隔离、需要 GPU 加速、企业级运维监控。
- **Q: 还有哪些向量数据库可选？**
  - A: Pinecone（全托管 SaaS）、Weaviate（GraphQL 接口）、pgvector（Postgres 插件，适合已有 PG 架构）、Chroma（轻量本地，适合原型）。
