# Function Calling Skill MCP 的区别

## 速记卡片
`调 | 调封规，三层分 | 能力vs封装vs协议`

## 原始知识
- **Function Calling**：**模型能力层**。模型根据 prompt 中的工具描述，自主决策"调不调、调哪个、填什么参数"，输出标准调用格式（JSON）。
- **Skill**：**应用封装层**。面向业务语义的函数集合，一个 Skill 可包含多个 Function，强调复用性和业务抽象。
- **MCP**：**协议标准层**。定义模型与外部工具/数据源交互的统一接口规范（菜单格式、通信方式、安全机制）。
- **关系**：MCP 定规矩 → Skill 按规矩封装业务 → Function Calling 是模型使用 Skill 的能力。
- **易混点**：Skill 不是单个 Function，而是更高层的业务抽象。
- **追问点**：为什么需要 MCP？没有 MCP 直接用 Function Calling 行不行？

## 单字锚点 + 防崩推导
- **锚点字**：**调**
- **扩展口诀**：**调封规**
- **为什么选它**：三者里最直观、最靠近用户感知的是"模型调用了什么"。从"调"字出发，向上追问是谁在调（Function Calling）、调的是什么（Skill）、按什么规矩调（MCP）。
- **防崩推导**：
  - 调 → 模型主动决策调用 → **Function Calling**（能力层）
  - 封 → 零散函数被包成业务单元 → **Skill**（封装层，一个Skill可含多Function）
  - 规 → 统一的接口标准和通信规范 → **MCP**（协议层）
  - 三层关系：MCP 定标准 → Skill 按标准实现 → Function Calling 消费 Skill

## 一字一画面
- **调** —— 像**服务员填单**：顾客说"我想吃红烧肉"，服务员看菜单后决定点什么、写单子给后厨。（Function Calling：模型主动判断、生成调用请求）
- **封** —— 像**预制菜包**：不是散装卖猪肉、酱油、冰糖，而是打包成"红烧肉料包"，一键下锅就是一道菜。（Skill：业务封装，对外暴露"查天气"而不是零散经纬度接口）
- **规** —— 像**餐厅加盟手册**：规定菜单必须有哪些字段、下单用什么格式、后厨怎么对接供应链。（MCP：标准化协议，让所有餐厅和供应商说同一种语言）
- **三层分** —— 像**剧场分工**：演员上台表演（调/Function Calling），剧组把道具服装打包成"场景包"（封/Skill），剧场守则规定谁什么时候上场、怎么递道具（规/MCP）。

## 面试话术
- 问区别 → "三者是**不同层次**的概念，口诀是**调封规**。Function Calling 是**模型能力**，模型判断什么时候调工具、填什么参数；Skill 是**应用层封装**，一个 Skill 可包含多个函数，面向业务语义；MCP 是**协议标准**，定义工具和模型交互的统一接口。打个比方：Function Calling 是服务员点菜的能力，Skill 是预制菜包，MCP 是餐厅加盟手册。"
- 问关系 → "MCP 定规矩，Skill 按规矩封装业务能力，Function Calling 是模型使用这些 Skill 的能力。没有 MCP，每接一个新工具都要写适配代码；有了 MCP，工具和模型说同一种语言。"

## 防混补丁
- 容易把 Skill 当成单个 Function。补丁：**Skill 是菜包，Function 是食材**。
- 容易把 MCP 当成一种模型能力。补丁：**MCP 是菜单格式标准，不是点菜的人**。

## 追问预演
1. **为什么需要 MCP？没有它行不行？**
   → 没有 MCP 时每接入一个新工具都要写特定适配代码，耦合严重。MCP 统一了工具描述、调用格式、通信机制，让工具提供方和模型消费方解耦。
2. **一个 Skill 里可以有多个 Function 吗？**
   → 可以。Skill 是更高层的业务抽象，例如"天气 Skill"可包含查当前天气、查未来七天、查空气质量等多个 Function。
3. **Function Calling 只能配合 MCP 使用吗？**
   → 不是。Function Calling 是通用模型能力，OpenAI、Claude 等原生都支持。MCP 只是让工具描述和调用过程标准化，降低集成成本。
4. **实际工程中三者怎么配合？**
   → 工具开发者按 MCP 协议暴露能力（Server）→ 平台将能力封装为 Skill 注册到 Agent → Agent 通过 Function Calling 调用 Skill → 执行结果返回给模型生成回答。
