# 什么是 MCP 协议

## 速记卡片
`标 | 模上文，工标调 | MCP协议`

## 原始知识
- MCP（Model Context Protocol）：Anthropic推出的标准化工具调用协议
- 核心定位：
  - **模**（Model）- 面向大模型
  - **上文**（Context）- 标准化上下文传递
  - **工**（工具）- 标准化工具描述和调用
  - **标**（标准）- 统一协议标准
  - **调**（调用）- 模型调用外部工具
- 作用：解决每个工具各自一套API的问题，统一工具接入标准
- **易混点**：MCP vs Function Calling
- **追问点**：为什么需要MCP？和A2A什么关系？

## 顺口溜
**模上文，工标调**

## 单字锚点 + 防崩推导
- **锚点字**：**标**
- **扩展口诀**：**模上文，工标调**
- **为什么选它**：MCP的核心价值是"标准化"，"标"是灵魂。
- **防崩推导**：
  - 标（标准）→ 标准化工具调用协议
  - 模（Model）→ 面向大模型
  - 上文（Context）→ 上下文传递
  - 工（工具）→ 工具描述和调用
  - 调（调用）→ 模型调用外部工具

## 一字一画面
- **标** —— 像**万能转接头**：不管什么插头，插上就能用。（本质：标准化协议）
- **模** —— 像**智能音箱**：需要接各种家电。（本质：面向大模型）
- **工** —— 像**工具箱**：锤子、螺丝刀统一规格。（本质：工具标准化）
- **调** —— 像**遥控器**：统一控制所有设备。（本质：统一调用）

## 面试话术
- 问什么是MCP → "MCP是Model Context Protocol，Anthropic推出的标准化工具调用协议，口诀是**模上文，工标调**。**模**——面向大模型；**上文**——标准化上下文传递；**工**——标准化工具描述；**标**——统一协议标准；**调**——模型统一调用外部工具。以前每个工具（GitHub、Slack、数据库）各自一套API，MCP就像万能转接头，让模型用统一方式调用任何工具。"

## 防混补丁
- 容易和"Function Calling"搞混。Function Calling是模型能力，MCP是在此之上的协议标准。补丁：**Function Calling是会说话，MCP是定规矩**。"
