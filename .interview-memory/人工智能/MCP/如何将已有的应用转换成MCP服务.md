# 如何将已有的应用转换成 MCP 服务

## 速记卡片
`转 | 包暴露，MCP化 | 应用改造`

## 原始知识
- 转换步骤：
  1. **包**（封装）- 将现有API/功能封装成MCP Server
  2. **暴露**（暴露）- 用MCP协议暴露工具接口（Tool Schema）
  3. **MCP化**（MCP化）- 实现MCP协议生命周期（初始化、调用、关闭）
- 关键：定义Tool Schema（名称、描述、参数），处理Call请求
- SDK：@modelcontextprotocol/sdk（Node.js）、mcp（Python）
- **易混点**：改造 vs 重写
- **追问点**：现有REST API怎么转？需要改业务逻辑吗？

## 顺口溜
**包暴露，MCP化**

## 单字锚点 + 防崩推导
- **锚点字**：**转**
- **扩展口诀**：**包暴露，MCP化**
- **为什么选它**：面试题核心是"转换"，"转"是直接钩子。
- **防崩推导**：
  - 转（转换）→ 已有应用转MCP服务
  - 包（封装）→ 封装成MCP Server
  - 暴露 → 暴露工具接口
  - MCP化 → 实现MCP协议

## 一字一画面
- **包** —— 像**包礼物**：把东西装进统一包装。（本质：封装成MCP Server）
- **暴露** —— 像**贴标签**：告诉别人这里面有什么。（本质：Tool Schema描述）
- **MCP化** —— 像**翻译**：把方言翻译成普通话。（本质：协议适配）

## 面试话术
- 问怎么转换 → "已有应用转MCP服务口诀是**包暴露，MCP化**。**包**——用MCP SDK封装现有API为MCP Server；**暴露**——定义Tool Schema（工具名、描述、参数），让模型知道能调什么；**MCP化**——实现MCP协议生命周期：初始化、处理Call请求、返回结果。不需要改业务逻辑，只需加一层MCP协议适配。比如原有REST API查询用户，包装成MCP Tool 'query_user'，参数是user_id。"

## 防混补丁
- 容易觉得"要重写应用"。其实只需加适配层。补丁：**转MCP是加外套，不是换内脏**。"
