# MCP 的工作流程是什么

## 速记卡片
`程 | 发连调，返结果 | 工作流程`

## 原始知识
- MCP工作流程：
  1. **发**（发现）- Host发现可用的MCP Server
  2. **连**（连接）- Client与Server建立连接（stdio或SSE）
  3. **调**（调用）- 模型决定调用工具，Client向Server发送请求
  4. **返**（返回）- Server执行工具，返回结果给Client
  5. **结果**（结果）- 模型接收结果，生成最终回答
- **易混点**：MCP流程 vs 普通API调用
- **追问点**：工具列表怎么获取？错误怎么处理？

## 顺口溜
**发连调，返结果**

## 单字锚点 + 防崩推导
- **锚点字**：**程**
- **扩展口诀**：**发连调，返结果**
- **为什么选它**：面试题问"流程"，"程"是核心。
- **防崩推导**：
  - 程（流程）→ MCP工作流程
  - 发（发现）→ 发现Server
  - 连（连接）→ 建立连接
  - 调（调用）→ 调用工具
  - 返（返回）→ 返回结果

## 一字一画面
- **发** —— 像**找供应商**：先找到有哪些供应商。（本质：发现Server）
- **连** —— 像**签合同**：和供应商建立合作关系。（本质：建立连接）
- **调** —— 像**下订单**：告诉供应商要什么。（本质：工具调用）
- **返** —— 像**收货**：供应商把货送回来。（本质：返回结果）

## 面试话术
- 问工作流程 → "MCP工作流程口诀是**发连调，返结果**。**发**——Host发现可用的MCP Server，获取工具列表；**连**——Client与Server建立连接（stdio或SSE）；**调**——模型决定调用工具，Client向Server发送调用请求；**返**——Server执行工具，返回结果；**结果**——模型接收工具结果，生成最终回答。整个过程是标准化的，任何符合MCP协议的Server都能被任何MCP Client调用。"

## 防混补丁
- 容易和普通API调用搞混。MCP是标准化的发现+调用流程，API是固定端点。补丁：**API是打固定电话，MCP是万能通讯录**。"
