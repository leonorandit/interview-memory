# MCP 与 Function Calling 的区别

## 速记卡片
`能 | 标协分，规对调 | 区别对比`

## 原始知识
- Function Calling：
  - **调**（调用）- 模型层面的能力，输出函数调用参数
  - 是LLM的"能力"，不是标准
- MCP：
  - **标**（标准）- 协议标准，统一工具接入方式
  - **协**（协议）- 通信协议层
  - **规**（规范）- 规范工具描述、调用、返回格式
- 关系：MCP基于Function Calling能力之上，增加了标准化协议
- **易混点**：两者是不是替代关系
- **追问点**：有了MCP还需要Function Calling吗？

## 顺口溜
**标协分，规对调**

## 单字锚点 + 防崩推导
- **锚点字**：**能**
- **扩展口诀**：**标协分，规对调**
- **为什么选它**：Function Calling是模型"能力"，MCP是"协议"，"能"区分层次。
- **防崩推导**：
  - 能（能力）→ Function Calling是能力，MCP是协议
  - 标（标准）→ MCP是标准协议
  - 协（协议）→ 通信协议
  - 分（分层）→ 两者分层不同
  - 规（规范）→ MCP规范工具格式
  - 调（调用）→ Function Calling是调用能力

## 一字一画面
- **标** —— 像**通用插座**：什么电器都能插。（本质：MCP标准化）
- **调** —— 像**人会用手**：是本能能力。（本质：Function Calling能力）
- **规** —— 像**使用说明书**：规定了怎么用。（本质：MCP规范）
- **分** —— 像**分层建筑**：能力在底层，协议在上层。（本质：分层关系）

## 面试话术
- 问区别 → "MCP和Function Calling的区别口诀是**标协分，规对调**。**标**——MCP是标准化协议；**协**——通信协议层；**分**——两者分层不同；**规**——MCP规范工具描述、调用、返回格式；**调**——Function Calling是模型层面的调用能力。关系是：Function Calling是'会说话'，MCP是'定外交辞令'。有了MCP还需要Function Calling，因为MCP基于Function Calling能力实现协议层。"

## 防混补丁
- 容易觉得"MCP替代Function Calling"。其实MCP依赖Function Calling。补丁：**Function Calling是腿，MCP是跑鞋**。"
