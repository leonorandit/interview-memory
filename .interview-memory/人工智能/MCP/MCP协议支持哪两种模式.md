# MCP 协议支持哪两种模式

## 速记卡片
`模 | 标服事，本远两 | 两种模式`

## 原始知识
- MCP两种传输模式：
  1. **标**（stdio）- 标准输入输出，本地进程间通信
     - **本**（本地）- Server和Host在同一台机器
     - 优点：简单、安全、无网络开销
  2. **服**（SSE）- Server-Sent Events，基于HTTP的服务端推送
     - **事**（事件流）- 服务器向客户端推送事件
     - **远**（远程）- 支持远程Server连接
     - 优点：支持分布式、跨网络
- **易混点**：stdio vs SSE怎么选
- **追问点**：SSE和WebSocket区别？安全性怎么保证？

## 顺口溜
**标服事，本远两**

## 单字锚点 + 防崩推导
- **锚点字**：**模**
- **扩展口诀**：**标服事，本远两**
- **为什么选它**：面试题问"哪两种模式"，"模"是记忆钩子。
- **防崩推导**：
  - 模（模式）→ 两种传输模式
  - 标（stdio）→ 标准输入输出
  - 服（SSE）→ Server-Sent Events
  - 事（事件）→ 服务端推送事件
  - 本（本地）→ stdio本地通信
  - 远（远程）→ SSE远程通信
  - 两 → 两种模式

## 一字一画面
- **标** —— 像**对讲机**：短距离，直接通话。（本质：stdio本地通信）
- **服** —— 像**电话**：远距离，通过网络连接。（本质：SSE远程通信）
- **本** —— 像**办公室对话**：同事之间当面说。（本质：本地模式）
- **远** —— 像**视频会议**：和外地同事远程沟通。（本质：远程模式）

## 面试话术
- 问两种模式 → "MCP支持两种传输模式，口诀是**标服事，本远两**。**标**——stdio标准输入输出，本地进程间通信；**服**——SSE Server-Sent Events，基于HTTP的服务端推送；**事**——事件流推送；**本**——stdio适合本地Server；**远**——SSE支持远程分布式Server。stdio简单安全，适合本地工具；SSE灵活可扩展，适合云服务场景。"

## 防混补丁
- 容易和"WebSocket"搞混。SSE是单向推送（Server→Client），WebSocket是双向。补丁：**SSE是广播，WebSocket是对讲机**。"
