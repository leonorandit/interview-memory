# 什么是 MCP（模型上下文协议）

## 速记卡片
`接 | 接三原，本地安 | 协议标准化`

## 原始知识
- **定义**：Anthropic 提出的开放协议，标准化 AI 模型与外部数据源、工具的集成
- **架构三层**：Host（宿主应用）→ Client（客户端）→ Server（服务器）
- **三大原语**：Resources（资源，只读数据）、Prompts（提示，预设模板）、Tools（工具，可执行函数）
- **传输方式**：stdio（本地进程通信）、SSE（Server-Sent Events，远程）
- **工作流程**：发现 → 连接 → 能力协商 → 调用
- **安全模型**：本地优先、权限控制、用户确认、数据不出本地
- **核心痛点**：解决 N×M 适配问题（N 个模型 × M 个工具）
- **追问点**：与 Function Calling 的本质区别
