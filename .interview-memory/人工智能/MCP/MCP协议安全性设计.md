# MCP 协议安全性设计

## 速记卡片
`安 | 认授加，沙审四层 | 安全设计`

## 原始知识
- MCP安全性四层设计：
  1. **认**（认证）- Server身份认证，防止伪造
  2. **授**（授权）- 权限控制，细粒度访问控制
  3. **加**（加密）- 传输层TLS加密，防止窃听
  4. **沙**（沙箱）- Server运行在沙箱/隔离环境
  5. **审**（审计）- 操作日志审计，可追溯
- **易混点**：MCP安全 vs 应用安全
- **追问点**：stdio模式怎么保证安全？SSE怎么认证？

## 顺口溜
**认授加，沙审四层**

## 单字锚点 + 防崩推导
- **锚点字**：**安**
- **扩展口诀**：**认授加，沙审四层**
- **为什么选它**：面试题核心是"安全性"，"安"是直接钩子。
- **防崩推导**：
  - 安（安全）→ 安全性设计
  - 认（认证）→ 身份认证
  - 授（授权）→ 权限控制
  - 加（加密）→ 传输加密
  - 沙（沙箱）→ 隔离运行
  - 审（审计）→ 操作审计
  - 四层 → 四层安全设计

## 一字一画面
- **认** —— 像**查身份证**：确认你是谁。（本质：身份认证）
- **授** —— 像**门禁卡**：有卡才能进相应区域。（本质：权限控制）
- **加** —— 像**密封信封**：别人看不到里面内容。（本质：传输加密）
- **沙** —— 像**防弹玻璃**：即使炸了也不影响外面。（本质：沙箱隔离）
- **审** —— 像**监控录像**：事后能查谁干了什么。（本质：操作审计）

## 面试话术
- 问安全性 → "MCP安全性设计口诀是**认授加，沙审四层**。**认**——Server身份认证，防止伪造；**授**——权限控制，细粒度访问；**加**——TLS传输加密；**沙**——Server运行在沙箱隔离环境；**审**——操作日志审计。stdio模式靠进程隔离保证安全，SSE模式靠TLS+Token认证。安全是MCP设计的核心考量之一。"

## 防混补丁
- 容易觉得"stdio不安全"。其实stdio的进程隔离很安全。补丁：**stdio是自家院子，SSE是小区大门，都有锁**。"
