# 什么是 AI 编程中的自动修复循环（Auto-fix Loop）它的工作流程和退出策略怎么设计

## 速记卡片
`环 | 报错修测环，三退防死转 | 闭环自修复`

## 原始知识
- 自动修复循环：AI生成代码→编译/测试→报错→AI修复→再测试的闭环
- 工作流程：生成 → 检测 → 诊断 → 修复 → 验证
- 退出策略：**次数上限**（如3次）、**成功通过**、**错误固化**（同一错误重复出现）
- **加粗易混点**：不是无限循环，必须设退出防死转
- **加粗易混点**：修复失败时要回退到上版本，不能带病推进

## 顺口溜
**报错修测环，三退防死转**

## 单字锚点 + 防崩推导
- **锚点字**：**环**
- **扩展口诀**：**环报修测退**
- **为什么选它**：Auto-fix的核心就是"循环"，听到Auto-fix就想到环。
- **防崩推导**：
  - 环 → 闭环流程 → 报错才能进环
  - 环 → 修 → AI根据错误信息改代码
  - 环 → 测 → 改完必须再验证
  - 环 → 退 → 次数上限/错误固化/成功通过，三种退出

## 一字一画面
- **环** —— 像**跑步机上的仓鼠轮**：AI在圈里跑，报错就推动轮子转一圈，修完跑回来再测试。（本质：反馈驱动闭环）
- **报** —— 像**红灯亮起**：编译失败或测试不通过，红灯闪，触发修复动作。（本质：错误信号是入口）
- **修** —— 像**医生看化验单**：AI拿着报错堆栈诊断病因，对症下药改代码。（本质：错误上下文指导修复）
- **测** —— 像**安检门**：修完必须再过一遍安检，绿牌才放行。（本质：验证是质量闸门）
- **退** —— 像**电梯限重三次**：超重三次就停运报警，不再硬上。（本质：退出策略防资源耗尽）

## 面试话术
- "Auto-fix Loop就是**报错修测环**。AI生成代码后走编译和测试，报错就进环修复，修完再测。退出策略设三层保险：成功退出、次数上限退出、还有错误固化退出——同一错误重复出现说明修不了，及时止损。"

## 防混补丁
- 容易和"Agent自主循环"搞混。Auto-fix是**被动触发**（报错才转），Agent是**主动规划**（自己定目标）。补丁：**环转靠报错，Agent自己跑**。

## 追问预演
- Q：如果AI越修越错怎么办？ → A：设错误固化检测，连续两次同一错误就人工介入。
- Q：修复上下文怎么给AI？ → A：给报错堆栈+失败测试用例+原始需求，三合一Prompt。
- Q：和CI/CD怎么结合？ → A：嵌在Pipeline里，作为Code Generation Stage的后置校验环节。
