# 如何评估大模型的能力主流的 Benchmark 有哪些

## 速记卡片
`榜 | 榜评多面考，MMLU考知，GSM8K算数，HumanEval码 | 维度分解`

## 原始知识
- 评估维度：**知识**、**推理**、**代码**、**多语言**、**安全性**、**长文本**
- MMLU：多学科知识问答，测广度
- GSM8K：小学数学推理，测逻辑链
- HumanEval：函数级代码生成，测编程
- C-Eval：中文综合评估
- TruthfulQA：测幻觉和事实性
- **加粗易混点**：没有单一榜单能全面衡量，必须多榜综合
- **加粗易混点**：排行榜分数高≠实际好用，还要看人类偏好评测（如MT-Bench）

## 顺口溜
**榜评多面考，MGHCT**

## 单字锚点 + 防崩推导
- **锚点字**：**榜**
- **扩展口诀**：**榜分六面考**
- **为什么选它**：评估大模型必谈Benchmark，"榜"是天然钩子。
- **防崩推导**：
  - 榜 → 考知识 → MMLU（多学科选择题）
  - 榜 → 考推理 → GSM8K（数学一步步推）
  - 榜 → 考代码 → HumanEval（写函数跑测试）
  - 榜 → 考中文 → C-Eval
  - 榜 → 考真实 → TruthfulQA（防幻觉）
  - 榜 → 考偏好 → MT-Bench（人类打分）

## 一字一画面
- **榜** —— 像**高考排行榜**：单科状元不一定总分高，看综合排名。（本质：多维评估）
- **M** —— 像**百科全书抢答赛**：历史理化生物全考，看知识覆盖面。（MMLU）
- **G** —— 像**小学生做应用题**：列式子、一步步算，不能跳步。（GSM8K）
- **H** —— 像**程序员现场刷LeetCode**：给描述写函数，跑通测试用例才算过。（HumanEval）
- **C** —— 像**语文综合卷**：古文、现代文、常识全包，考母语深度。（C-Eval）
- **T** —— 像**测谎仪**：故意问陷阱问题，看模型会不会瞎编。（TruthfulQA）

## 面试话术
- "大模型评估是**榜评多面考**，没有单一标准。知识看MMLU，推理看GSM8K，代码看HumanEval，中文看C-Eval，幻觉看TruthfulQA。实际落地还要加人类偏好评测，榜单高不代表用户体验好。"

## 防混补丁
- 容易把"Benchmark高分"当成"模型强"。补丁：**榜是考场，用是战场**，考场第一战场可能水土不服，还要看实际业务指标。

## 追问预演
- Q：怎么防止模型刷榜？ → A：用held-out测试集、动态更新题目、加人工校验。
- Q：业务里怎么评估？ → A：建业务专属评测集，加人工标注的Good/Bad Case。
- Q：长文本评估用什么？ → A：LongBench、Needle in a Haystack（大海捞针）。
