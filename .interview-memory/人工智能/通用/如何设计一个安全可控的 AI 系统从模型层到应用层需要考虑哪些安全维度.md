# 如何设计一个安全可控的 AI 系统从模型层到应用层需要考虑哪些安全维度

## 速记卡片
`盾 | 盾分三层守，模数应各不同 | 纵深防御`

## 原始知识
- 模型层：对齐训练（RLHF）、红队测试、模型水印、输出过滤
- 数据层：隐私脱敏、来源审计、偏见检测、防投毒
- 应用层：输入过滤、权限控制、审计日志、人机回环（Human-in-the-loop）
- **加粗易混点**：安全不是单点，是**纵深防御**
- **加粗易混点**：可控≠完全无风险，是**风险可接受+可审计+可回退**

## 顺口溜
**盾分三层守，模数应**

## 单字锚点 + 防崩推导
- **锚点字**：**盾**
- **扩展口诀**：**盾牌三层厚**
- **为什么选它**：安全系统的核心意象就是盾牌，最直观。
- **防崩推导**：
  - 盾 → 厚 → 三层 → 模型层、数据层、应用层
  - 模型层 → 对齐+红队+水印 → 让模型本身不想作恶
  - 数据层 → 脱敏+审计+防投毒 → 不让坏数据教坏模型
  - 应用层 → 过滤+权限+日志+人机回环 → 最后一道闸

## 一字一画面
- **盾** —— 像**中世纪三层塔盾**：最里层加固（模型对齐），中间层缓冲（数据治理），最外层格挡（应用拦截）。（本质：纵深防御体系）
- **模** —— 像**给战士洗脑+体检**：RLHF让它不想伤人，红队测试模拟攻击找弱点。（模型层安全）
- **数** —— 像**食堂验菜+去农药**：数据进锅前先检测偏见和毒性，隐私食材要脱敏。（数据层安全）
- **应** —— 像**城门守卫**：查通行证（权限）、搜身（输入过滤）、记名册（审计日志）、大事请示上司（人机回环）。（应用层安全）

## 面试话术
- "AI安全设计是**盾分三层守**。模型层做RLHF对齐和红队测试，让模型本质向善；数据层做脱敏和防投毒，堵住污染源头；应用层做输入过滤、权限控制和审计日志，设最后一道闸。关键是纵深防御，任何一层都不能单打独斗。"

## 防混补丁
- 容易把"安全"只理解为"输出过滤"。补丁：**过滤是城门，盾有三层厚**，只在应用层拦是远远不够的。

## 追问预演
- Q：红队测试怎么做？ → A：雇人专门诱导模型出有害内容，找到弱点后再对齐加固。
- Q：怎么防Prompt Injection？ → A：输入过滤+分隔符隔离+权限最小化+输出编码。
- Q：模型水印有什么用？ → A：追踪泄露源头，防内部人员盗用模型商用。
