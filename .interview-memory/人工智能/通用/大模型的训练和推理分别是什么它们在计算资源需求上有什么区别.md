# 大模型的训练和推理分别是什么它们在计算资源需求上有什么区别

## 速记卡片
`训 | 训吃推省算，训要并推要并，训耗存推耗卡 | 阶段目标不同`

## 原始知识
- 训练：用大量数据调参数，让模型学会规律（前向+反向传播）
- 推理：用已训练好的模型做预测（只有前向传播）
- 训练吃：**显存大**（存参数+梯度+优化器状态）、**计算密集**（反向传播）、**通信密集**（多卡同步）
- 推理省：**显存小**（只存参数）、**计算单向**、**可批处理**
- **加粗易混点**：训练显存≈推理显存×3~4倍（因梯度和优化器状态）
- **加粗易混点**：推理卡可以量化压缩（INT8/INT4），训练一般不行

## 顺口溜
**训吃推省算，训三推一**

## 单字锚点 + 防崩推导
- **锚点字**：**训**
- **扩展口诀**：**训吃显存推省卡**
- **为什么选它**：训练是"烧钱"环节，印象最深刻，锚"训"自然带出对比。
- **防崩推导**：
  - 训 → 要调参数 → 反向传播 → 存梯度+优化器状态 → 显存爆炸
  - 训 → 数据量大 → 多卡并行 → 卡间通信 bottleneck
  - 推 → 不动参数 → 只前向 → 只存模型权重 → 显存省
  - 推 → 可量化 → INT4省更多 → 吞吐高

## 一字一画面
- **训** —— 像**厨师学做菜**：大量食材反复炒、尝、调整配方，过程又费料又费时。（本质：参数学习，双向计算）
- **吃** —— 像**三只冰箱并排放**：一只放食材（参数），一只放半成品（梯度），一只放调料（优化器状态）。（本质：显存三件套）
- **推** —— 像**快餐窗口出餐**：配方已定，按单组装，速度要快。（本质：前向预测）
- **省** —— 像**把大菜谱缩印成小卡片**：INT4量化， still能读，不占地方。（本质：推理压缩）

## 面试话术
- "训练是**训**，推理是**推**。训练要前向算完再反向调参数，所以显存要吃三倍——参数、梯度、优化器状态都得存；推理只前向，只存参数，还能量化压缩。训练卡要A100/H100这种高显存高带宽，推理卡可以便宜些，甚至边缘设备跑INT4。"

## 防混补丁
- 容易以为"推理不需要GPU"。补丁：**推也要卡，只是少吃**。大模型推理没GPU照样慢，只是相对训练省得多。

## 追问预演
- Q：为什么训练不能量化？ → A：量化误差在反向传播中会累积放大，导致模型学偏。
- Q：推理怎么进一步加速？ → A：KV Cache、Continuous Batching、 speculative decoding。
- Q：训练多卡通信瓶颈怎么解？ → A：ZeRO分片、Pipeline并行、Tensor并行组合。
