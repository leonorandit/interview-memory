# 对比使用过哪些主流大模型？你们项目中最终选用了哪个模型？为什么？

## 速记卡片
`型 | 能成延稳私定，六维选 | 大模型选型`

## 原始知识
- **主流闭源模型**：GPT-4o/GPT-5、Claude 3.5/4、Gemini 2.5 Pro、Kimi K1.5、豆包 Pro、文心一言
- **主流开源模型**：Llama 4、Qwen 3、DeepSeek-V4/R1、Mistral Large、Gemma 3、GLM-4
- **选型六维度**：
  - **能**（能力）：推理、代码、多模态、长文本、中文理解
  - **成**（成本）：API 价格、私有化部署成本、运维人力
  - **延**（延迟）：首 token 延迟（TTFT）、吐字速度（TPS）
  - **稳**（稳定性）：SLA 保障、并发承载、服务可用性
  - **私**（隐私）：数据是否出域、合规要求、本地化部署能力
  - **定**（定制）：是否支持微调、LoRA、领域适配、架构修改
- **典型场景匹配**：
  - 复杂推理/代码 → Claude 4 / DeepSeek-R1
  - 通用对话/多模态 → GPT-4o / Gemini 2.5
  - 长文本/文档处理 → Kimi / Gemini 1M context
  - 中文场景/低成本 → Qwen 3 / DeepSeek / 豆包
  - 私有化/金融政务 → Qwen / Llama / GLM-4 本地部署
- **答题套路**：先分类对比（闭源 2-3 个 + 开源 2-3 个），再说选型框架（六维），最后结合具体项目场景给出选择理由和量化收益

## 一字一画面
- **型** —— 像**买车**：要动力（能力）、油耗（成本）、提速（延迟）、质量（稳定）、安全（隐私）、改装空间（定制）。（本质：六维选型框架）
- **能** —— 像**发动机马力**：DeepSeek-R1 和 Claude 推理强，GPT-4o 多模态全。（本质：模型能力）
- **成** —— 像**油价**：DeepSeek、豆包、Qwen API 便宜，Claude/GPT 贵。（本质：成本）
- **延** —— 像**百公里加速**：小模型（7B/13B）快，大模型（70B+）慢。（本质：延迟）
- **稳** —— 像**质保**：闭源有 SLA，开源自己兜底。（本质：稳定性）
- **私** —— 像**车窗贴膜**：数据不能出公司 → 必须本地部署开源。（本质：隐私合规）
- **定** —— 像**改装店**：开源能换轮胎改排气，闭源只能贴贴纸。（本质：定制化）

## 面试话术
"我实际对比过 GPT-4o、Claude 3.5、DeepSeek-V3、Qwen 2.5 等主流模型。选型框架口诀是**能成延稳私定**六个维度。**能**——看推理、代码、多模态能力；**成**——API 价格和部署成本；**延**——首 token 延迟和吐字速度；**稳**——SLA 和并发承载；**私**——数据是否出域；**定**——是否支持微调和私有化。比如我们之前的 RAG 项目，因为涉及内部文档隐私，最终选了 Qwen 2.5 私有化部署，在能力满足需求的前提下，成本和延迟都比调闭源 API 低 60%。"

## 防混补丁
- 容易只对比能力忽略成本。补丁：**面试选型题 = 能力 + 成本 + 场景匹配，只说谁强谁弱是学生思维**。

## 追问预演
- **追问1**：如果预算无限，你会选哪个？
  → 也不选最贵的，选最匹配的。复杂推理选 Claude，多模态选 GPT-4o，长文本选 Gemini。预算无限反而更要做 A/B test。
- **追问2**：怎么快速评估一个模型适不适合业务？
  → 准备 50-100 条业务真实样本做评测，看准确率和延迟，再算成本。Benchmark 只能做初筛。
- **追问3**：开源模型能力不够怎么办？
  → 先尝试 RAG/提示工程补齐，不够再考虑 SFT 或 MoE 架构，最后才上闭源 API 兜底。
