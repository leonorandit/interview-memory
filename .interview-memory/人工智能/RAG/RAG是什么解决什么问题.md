# RAG是什么解决什么问题

## 速记卡片
`外 | 外脑补四病 | 外接知识脑，补治四大病`

## 原始知识
- **RAG** = Retrieval-Augmented Generation（检索增强生成）
- **是什么**：大模型生成前，先从**外部知识库检索**相关文档，将检索结果作为**上下文补充**进 Prompt，再让模型生成答案
- **解决四大问题**：
  1. **幻觉**（Hallucination）—— 大模型凭空编造，RAG 让答案有据可依
  2. **知识过时** —— 训练数据有截止日期，RAG 引入实时/最新文档
  3. **私域知识缺失** —— 大模型没见过企业内部数据，RAG 接入私域知识库
  4. **不可追溯/黑盒** —— 不知道答案从哪来，RAG 可标注来源、提供引用
- **对比微调**：RAG 不改模型参数，知识更新只需换文档；微调是改变模型权重
- **优势**：成本低、灵活性高、知识实时、可解释性强

**外脑补四病**

## 单字锚点 + 防崩推导
- **锚点字**：**外**
- **扩展口诀**：**外查补生**
- **为什么选它**：RAG 的本质就是引入"外部知识"来弥补大模型的不足。一念"外"字，整个架构就立住了。
- **防崩推导**：
  1. **外** → 外部知识库/文档 → 不是模型参数里的知识
  2. **查** → 检索（Retrieval）→ 从向量库/搜索引擎找相关文档
  3. **补** → 补充到 Prompt 上下文 → 增强（Augmented）
  4. **生** → 大模型基于上下文生成（Generation）
  5. **四病** → 外查补 → 有依据 → **防幻觉**；外 → 实时文档 → **知识不过时**；外 → 企业内部数据 → **私域知识**；查 → 可追溯 → **可解释**
