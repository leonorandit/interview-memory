# 什么是 RAG 混合检索？如何实现向量检索和关键词检索结合？

## 速记卡片
`混 | 向量加词混双兵，RRF来收队 | 语义+精确`

## 原始知识
- **向量检索**：语义匹配，擅长同义/近义，但怕专有名词、ID、缩写
- **关键词检索（BM25/TF-IDF）**：字面匹配，擅长精确词、稀有词
- **混合**：两者并行召回，取长补短
- **RRF**：Reciprocal Rank Fusion，按排名位置融合，无需统一分数
- **加权融合**：线性加权，需要校准两种分数的尺度
- **加粗易混点**：混合检索不是简单取并集，需要融合排序

## 顺口溜
向量加词混

## 单字锚点 + 防崩推导
- **锚点字**：**混**
- **扩展口诀**：**向量加词混**
- **为什么选它**："混"直接对应"混合检索"，且谐音"混双"（混合双打），画面感强。
- **防崩推导**：
  - 混 → 混合 → 向量检索 + 关键词检索
  - 混 → 向量 → 语义匹配，懂"汽车≈轿车"
  - 混 → 词 → BM25字面匹配，懂"GPT-4"就是"GPT-4"
  - 混 → 两者并行 → 召回结果合并
  - 混 → RRF融合 → 按排名位置加权融合，不用管分数单位

## 一字一画面
- **混** —— 像**乒乓球混双比赛**：男选手（向量检索）擅长拉弧圈（语义泛化），女选手（关键词检索）擅长快攻（精确打击），两人配合才能赢。（本质：两种检索能力互补）
- **向** —— 像**用余光找人**：你不确定他叫什么，但记得他穿红衣服、戴眼镜（语义特征）。（本质：向量检索的语义匹配）
- **词** —— 像**点名册找人**：直接喊"张三"，精确命中。（本质：关键词检索的字面匹配）
- **RRF** —— 像**两个裁判打分合并**：A裁判给第一名，B裁判给第三名，合并后算综合排名，不用管A的满分是10分还是100分。（本质：Reciprocal Rank Fusion按排名融合）

## 面试话术
- "混合检索就是**向量加词混双兵，RRF来收队**。向量检索负责语义泛化，关键词检索负责精确匹配，两者并行召回后用RRF融合排序，不需要统一分数尺度。比如用户搜'GPT-4发布日期'，向量检索可能召回'ChatGPT历史'，关键词检索精确命中'GPT-4'和'发布日期'，混合后结果更全面。"
- "我们线上用的是Elasticsearch做BM25 + Milvus做向量，召回后走RRF融合，再送Reranker精排。"

## 防混补丁
- 容易把"混合检索"当成"两个结果取并集"。其实并集会引入大量噪音，需要融合排序。补丁：**混不是堆，要融合排**。

## 追问预演
- **追问1**：RRF公式是什么？→ RRF score = Σ 1/(k + rank)，k常取60，按排名倒数加权求和。
- **追问2**：为什么不用线性加权？→ 向量分数和BM25分数尺度不同，直接加权需要复杂校准，RRF只用排名更鲁棒。
- **追问3**：混合检索会增加延迟吗？→ 会，但两种检索可以并行发，延迟增加可控，收益通常大于成本。
