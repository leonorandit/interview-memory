# 如何处理 RAG 检索不到相关文档的情况？

## 速记卡片
`空 | 空时补退问，三层不卡壳 | 兜底策略`

## 原始知识
- **补**：补充通用知识/常识库，让模型直接回答
- **退**：退化为纯LLM生成（Fallback），但需提示用户未引用
- **问**：反问用户澄清需求，缩小范围
- **扩**：扩大检索范围（降阈值、扩Top-K、跨索引）
- **加粗易混点**："检索为空" ≠ "模型不会答"，关键看有没有兜底设计

## 顺口溜
空时补退问

## 单字锚点 + 防崩推导
- **锚点字**：**空**
- **扩展口诀**：**空时补退问**
- **为什么选它**："空"直接对应"检索结果为空"这个核心场景，是问题的起点。
- **防崩推导**：
  - 空 → 结果为空 → 先扩 → 降阈值/扩Top-K/换索引试试
  - 空 → 扩了还空 → 补 → 用常识库/通用知识回答
  - 空 → 补不了 → 退 → 纯LLM生成，但标明未引用
  - 空 → 连退都不行 → 问 → 反问用户澄清
  - 三层兜底，系统不卡壳

## 一字一画面
- **空** —— 像**打开冰箱发现里面什么都没有**：但厨房外还有超市（常识库）、外卖（LLM Fallback）、或者打电话问朋友（反问用户）。（本质：检索为空时的多层兜底）
- **补** —— 像**从超市买现货**：冰箱空了，直接去超市拿常识来补。（本质：通用知识库兜底）
- **退** —— 像**没菜谱凭经验炒菜**：没有食材（检索结果），厨师凭经验做，但要告诉客人"这道菜我没按菜谱做"。（本质：Fallback生成+免责声明）
- **问** —— 像**服务员回问你**："您是要番茄炒蛋还是番茄蛋汤？"缩小范围再去找。（本质：澄清反问）

## 面试话术
- "检索为空不能崩，我们的策略是**空时补退问**。第一层是扩大检索范围，降阈值、扩Top-K；第二层是补通用知识或退化为LLM直接生成，但要明确提示用户未引用文档；第三层是反问用户澄清需求。"
- "线上我们会对空检索做埋点监控，区分是真的没有知识，还是索引质量问题。"

## 防混补丁
- 容易和"检索质量差"搞混。检索为空是**结果数为0**，检索质量差是**召回了很多但不相关**。补丁：**空是零条，差是多条烂**。

## 追问预演
- **追问1**：怎么判断是"真没有知识"还是"索引没建好"？→ 看同义词/扩展查询能否召回，能则是索引问题，不能则是知识盲区。
- **追问2**：Fallback生成怎么防止幻觉？→ 加系统提示"未检索到相关资料，以下回答基于模型自身知识"，并限制回答长度。
- **追问3**：反问用户怎么设计？→ 用LLM生成澄清问题，提供2-3个选项让用户点选。
