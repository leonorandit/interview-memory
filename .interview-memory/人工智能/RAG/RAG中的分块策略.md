# RAG 中的分块策略

## 速记卡片
`策 | 固段义，递智切 | 五大策略`

## 原始知识
- 常见分块策略：
  1. **固**（固定长度）- 按固定字符/token数切分，简单但可能切断语义
  2. **段**（按段落）- 按换行/段落切分，保留自然边界
  3. **义**（按语义）- 用模型判断语义边界，在语义完整处切分
  4. **递**（递归）- 先按大分隔符切，再按小分隔符切，层级递进
  5. **智**（Agentic/智能）- 用LLM判断最佳切分点，最智能但成本高
- 对比：固定简单快但粗糙；语义/Agentic精准但慢贵
- **易混点**：递归分块 vs 固定分块
- **追问点**：什么时候用哪种？Agentic分块原理？

## 单字锚点 + 防崩推导
- **锚点字**：**策**
- **扩展口诀**：**五策略，固段义**
- **为什么选它**：面试题问的是"策略"，"策"字直接对应问题核心。
- **防崩推导**：
  - 策（策略）→ 五种分块策略
  - 固（固定长度）→ 最简单，按固定大小切
  - 段（按段落）→ 按自然段落边界切
  - 义（按语义）→ 按语义完整性切
  - 递（递归）→ 层级递进切分
  - 智（智能）→ 用LLM智能判断切分点
  - 切（切分）→ 最终目的都是把文档切好

## 一字一画面
- **固** —— 像**切蛋糕按尺寸**：每块必须一样大，不管奶油花在哪儿。（本质：固定长度分块）
- **段** —— 像**按章节切书**：一章一章切，保留自然边界。（本质：按段落分块）
- **义** —— 像**在句末切**：话没说完不切，保证语义完整。（本质：按语义边界分块）
- **递** —— 像**先切大块再切小块**：先把肉切条，再把条切丁。（本质：递归层级切分）
- **智** —— 像**大厨根据纹理切**：看肉的纹理决定怎么切最好。（本质：LLM智能判断切分点）

## 面试话术
- 问有哪些策略 → "RAG中常见的分块策略，口诀是**固段义，递智切**。**固**——固定长度切分，简单快速但可能切断语义；**段**——按段落切分，保留自然边界；**义**——按语义切分，保证语义完整；**递**——递归切分，先按大分隔符再按小分隔符；**智**——Agentic智能切分，用LLM判断最佳切分点，最精准但成本高。实际项目中，递归分块（LangChain默认）是最常用的平衡方案。"
- 问怎么选 → "追求速度用固定长度，追求质量用语义或Agentic，一般场景用递归分块。"

## 防混补丁
- 容易觉得"固定长度最好"，其实它经常切断语义。补丁：**固定是快刀，语义是手术刀**。

## 追问预演
- **问**：递归分块原理？→ 先尝试大分隔符（如段落），如果块还太大，再用小分隔符（如句子），层层递进。
- **问**：Agentic分块怎么做？→ 用LLM分析文档结构，在主题转换处切分，保证每块主题单一。
- **问**：Markdown/代码怎么分块？→ 按标题层级分（Markdown），按函数/类分（代码），保留结构信息。

## 手撕代码（递归分块伪代码）

```python
from langchain.text_splitter import RecursiveCharacterTextSplitter

# 递归分块：层级递进
text_splitter = RecursiveCharacterTextSplitter(
    chunk_size=500,
    chunk_overlap=50,
    separators=["\n\n", "\n", "。", "！", "？", " ", ""]  # 从大分隔符到小分隔符
)

chunks = text_splitter.split_text(document)

# Agentic分块：用LLM判断切分点
split_points = llm.generate(
    prompt=f"分析以下文档，找出主题转换的位置，返回切分点列表：\n{document}"
)
```
