# 如何规避RAG系统中大模型的幻觉

## 速记卡片
`严 | 严控查验 | 反幻觉四关`

## 原始知识
- **严（Prompt约束）**：严格限定模型只能基于检索内容回答，不知道就明确说不知道
- **控（参数控制）**：降低 temperature、top-p，减少生成随机性和发散
- **查（检索优化）**：混合检索 + Rerank 精排，确保喂给模型的资料准且相关
- **验（后验校验）**：要求引用溯源、Self-RAG 自检、人工/规则校验
- **易混点**：RAG 只能缓解幻觉，不能根除
- **追问点**：Self-RAG 原理、temperature 取值、Prompt 怎么写约束

## 顺口溜
**严控查验**

## 单字锚点 + 防崩推导
- **锚点字**：**严**
- **扩展口诀**：**严控查验**
- **为什么选它**：反幻觉的第一要务是"严"。安检不严，什么都能混进去。
- **防崩推导**：
  - 严 → Prompt 严格约束 → "只能基于提供的资料回答，不知道就说不知道"
  - 控 → 控制生成参数 → temperature 调低、top-p 减小，减少发散
  - 查 → 检索优化 → 混合检索、Rerank 精排，确保喂料准
  - 验 → 后验校验 → 引用来源、Self-RAG 自检、人工审核

## 一字一画面
- **严** —— 像**安检口保安严格检查**："只有名单上的人才能进！"（Prompt 严格约束，只准用检索到的知识）
- **控** —— 像**空调遥控器调低温度**：温度越低，模型越冷静，不乱发散。（降低 temperature）
- **查** —— 像**安检仪扫描行李**：先过机（向量检索），再人工开包检查（Rerank 精排）。（检索优化）
- **验** —— 像**出站验票**：必须出示车票和身份证，核对是否对应。（引用溯源、Self-RAG 校验）

## 面试话术
- 问反幻觉 → "我概括为**严控查验**四道关。**严**是 Prompt 严格约束，只能基于检索资料回答；**控**是调低 temperature 等参数，减少随机性；**查**是优化检索质量，用混合检索和 Rerank 确保召回准；**验**是后验校验，要求模型标注引用来源，甚至用 Self-RAG 让模型自检。"
