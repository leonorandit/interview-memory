# 大模型的 RAG 主要用来解决什么问题？

## 场景触发词
RAG 解决什么问题、为什么用 RAG、RAG 痛点、大模型幻觉、知识过时、私域数据、RAG 价值

## 速记卡片
`幻 | 幻旧私源省 | 五大痛点`

## 原始知识

- **幻觉（幻）**
  - LLM 会编造不存在的信息，听起来像真的但实际是瞎编
  - RAG 用检索到的真实文档约束生成，让模型"有据可依"
  - **易混点**：RAG 能大幅降低幻觉，但不能100%消除（检索不准或模型不按要求用检索内容时仍会 hallucinate）
  - **追问点**：RAG 和微调哪个解决幻觉更好？答：RAG，因为答案基于真实文档

- **知识过时（旧）**
  - 模型训练数据有截止日期，不知道最新事件、新产品、新政策
  - RAG 把最新文档实时接入，让模型"开卷考试"
  - **易混点**：不是让模型学会新知识，而是检索时动态查最新资料
  - **追问点**：知识更新频率高时，RAG 比微调成本低得多

- **私域数据（私）**
  - 模型没有企业内部的专有知识：产品手册、客户资料、内部规章、个人笔记
  - RAG 外挂私有文档库，让模型能回答"你家产品怎么用"
  - **易混点**：私域数据不需要也不应该用来训练通用模型
  - **追问点**：数据安全和权限怎么控制？需要访问控制和脱敏

- **可解释/可追溯（源）**
  - 纯 LLM 生成不知道答案从哪来，无法验证
  - RAG 能标注信息来源（"根据《XX手册》第3页"），提升可信度
  - **易混点**：可追溯性是 RAG 相比微调的显著优势
  - **追问点**：怎么实现引用标注？在 Prompt 里要求模型标注来源，或后处理匹配

- **成本低（省）**
  - 微调需要大量算力和标注数据，知识更新要重新训练
  - RAG 只需更新文档库，换换文件就能让模型掌握新知识
  - **易混点**：RAG 不是免费的，有检索和向量存储成本，但比微调便宜一个数量级
  - **追问点**：RAG 和微调怎么结合？可以 RAG 做事实检索，微调做风格和行为调整

## 顺口溜
**幻旧私源省**

## 单字锚点 + 防崩推导
- **锚点字**：幻
- **扩展口诀**：幻旧私源省
- **为什么选它**：“幻”是 RAG 最核心要解决的痛点，也是面试官最常问的问题
- **防崩推导**：
  1. 幻 → RAG 解决 LLM **幻**觉问题（瞎编）
  2. 旧 → 解决知识**旧**/过时（训练数据有截止日期）
  3. 私 → 解决**私**域数据无法访问（企业内部知识）
  4. 源 → 解决答案无**源**可查（可解释性、可追溯）
  5. 省 → 解决成本高问题（**省**钱，比微调便宜）

## 一字一画面
- **幻** —— 像 **说谎的匹诺曹**：[木偶长鼻子] 鼻子越变越长，RAG 给它戴上真理口罩，只能说实话。（解决幻觉）
- **旧** —— 像 **过期报纸**：[泛黄报纸] 上面的新闻是去年的，RAG 换上今天的实时新闻。（解决知识过时）
- **私** —— 像 **公司内部保险箱**：[银色保险柜] 里面的机密文件外人看不到，RAG 是合法钥匙。（解决私域数据）
- **源** —— 像 **论文脚注**：[书本脚注] 每句话后面都标注出自哪本书第几页。（解决可追溯性）
- **省** —— 像 **省钱优惠券**：[红色优惠券] 不用花大钱重新训练模型，换换文档就更新知识。（解决成本）

## 面试话术
- 问解决什么问题 → “大模型 RAG 主要解决五大问题，口诀是**幻旧私源省**。第一，**幻**——幻觉，LLM 会编造不存在的信息，RAG 用真实文档约束生成；第二，**旧**——知识过时，模型训练有截止日期，RAG 能接入最新数据；第三，**私**——私域数据，模型不知道企业内部的专有知识，RAG 外挂企业文档库；第四，**源**——可追溯，RAG 能标注答案来自哪份文档，提升可解释性；第五，**省**——成本低，知识更新只需换文档，不用重新训练模型。”
- 问 RAG vs 微调 → “RAG 是动态查资料，适合知识频繁更新和需要可追溯的场景；微调是把知识教进模型参数，适合改变模型行为或风格。两者可以结合：RAG 保证事实准确，微调优化回答风格。”

## 防混补丁
- 容易和“长上下文”搞混。长上下文是把整本书塞给模型读，RAG 是只给最相关的几页。补丁：**长上下文是全书硬塞，RAG 是精准投喂**。
- 容易认为 RAG 能100%解决幻觉。补丁：**RAG 降幻觉，不除根；检索不准或文档有错，仍会 hallucinate**。

## 追问预演
1. **问**：RAG 能100%解决幻觉吗？
   **答**：不能。检索不准、文档本身有错、或 LLM 没按要求使用检索内容时，仍会产生幻觉。需要配合 Rerank、事实核查等手段进一步降低。
2. **问**：什么场景不适合 RAG？
   **答**：需要深度推理而非事实检索的场景、数据量极小且固定的场景、或要求模型改变底层行为能力的场景，可能直接微调或 prompt 工程更合适。
3. **问**：RAG 和微调怎么选？
   **答**：知识频繁更新、需要可追溯、私域数据多 → 选 RAG；需要改变模型风格、固定领域知识、对延迟极敏感 → 选微调。也可以 RAG + 微调结合使用。
4. **问**：私域数据安全怎么保障？
   **答**：访问控制（谁可查什么文档）、数据脱敏（敏感信息替换）、权限隔离（不同用户看不同知识库）、审计日志（记录查询行为）。
