# 向量检索和关键词检索的区别

## 速记卡片
`义 | 义稠懂，稀字对 | 语义vs字面`

## 原始知识
- **向量检索（稠密）**：文本 → Embedding模型 → 高维向量 → ANN近似最近邻搜索
- **关键词检索（稀疏）**：文本 → 分词 → 词频统计/倒排索引 → 精确匹配（BM25、TF-IDF）
- **向量检索优势**：语义理解、跨语言、容错（错别字/同义词）
- **关键词检索优势**：精确匹配、可解释、对新词/专有名词友好、无需训练
- **易混点**：谁更准？看场景，不是绝对替代
- **追问点**：为什么RAG要用混合检索？

## 顺口溜
**义稠懂，稀字对**

## 单字锚点 + 防崩推导
- **锚点字**：**义**
- **扩展口诀**：**义稠对稀**
- **为什么选它**：两者的本质分水岭是"是否理解语义"，不是技术实现。先想"义不义"。
- **防崩推导**：
  - 义 → 语义理解 → 需要模型编码 → 输出稠密向量（每维都有值）→ 靠距离算相似 → 向量检索
  - 对 → 字面匹配 → 分词统计 → 输出稀疏向量（大部分为0）→ 精确对上了才算 → BM25/关键词检索

## 一字一画面
- **义** —— 像**知音聊天**：你说"心情低落"，他懂你"难过"，不需要你说"难过"这两个字。（语义理解）
- **稠** —— 像**浓雾中的灯塔**：每一束光都很微弱，但所有方向的光汇聚起来，就能指明方向。（稠密向量：每维都承载压缩后的语义信息）
- **懂** —— 像**翻译官同传**：中文"苹果"和英文"apple"指向同一个向量，跨语言也能懂。（跨语言、同义词归一）
- **稀** —— 像**考试答题卡**：只在有词的位置涂黑，其他地方全是空白。（稀疏向量：绝大多数维度为0）
- **字** —— 像**字典查生字**：偏旁部首必须完全对上，差一笔都查不到。（精确匹配，容错性差）
- **对** —— 像**钥匙开锁**：齿形必须严丝合缝，差一毫米就开不了。（字面一致性要求高）

## 面试话术
- 问区别 → "核心差异是**义不义和稀不稀**。向量检索是稠密向量，通过Embedding模型理解语义，像知音聊天，你说'笔记本'他也能找到'电脑'相关；关键词检索是稀疏向量，基于词频和倒排索引做字面匹配，像字典查字，必须对上关键词。RAG里常把两者混着用，精确问题用关键词，语义泛化用向量。"
- 问为什么混合 → "因为各有盲区。向量检索对专有名词、新词、ID类信息容易丢；关键词检索对同义词、跨语言、错别字无能为力。混着用才能召回全。"

## 防混补丁
- 容易和"稠密/稀疏矩阵"搞混，但这里不是数据结构考试，是检索方式。补丁：**稠稀讲的是向量的样子，不是数据库存储格式。**

## 追问预演
1. **为什么向量检索对新词/ID不友好？**
   → Embedding模型训练时没见过这个词，编码出来的向量可能偏离语义空间。
2. **BM25为什么比TF-IDF好？**
   → 引入了文档长度归一化和词频饱和函数，避免长文档天然占优。
3. **混合检索后怎么融合结果？**
