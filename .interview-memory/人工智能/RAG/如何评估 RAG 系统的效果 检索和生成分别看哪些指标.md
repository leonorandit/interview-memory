# 如何评估 RAG 系统的效果？检索和生成分别看哪些指标？

## 速记卡片
`评 | 评准全信流畅，检索生成分开看 | 检索指标+生成指标`

## 原始知识
- **检索指标**：Recall@K、Precision@K、NDCG、MRR、Hit Rate
- **生成指标**：Faithfulness（忠实度）、Answer Relevance（答案相关性）、Context Precision（上下文精准度）、Context Recall（上下文召回率）
- **端到端**：RAGAS、ARES、自定义业务指标
- **加粗易混点**：检索Recall高 ≠ 生成质量好，中间还有Rerank和Prompt组装环节

## 顺口溜
准全信流畅

## 单字锚点 + 防崩推导
- **锚点字**：**评**
- **扩展口诀**：**准全信流畅**
- **为什么选它**："评"直接对应"评估"，是RAG优化的方向盘。
- **防崩推导**：
  - 评 → 评估 → 分两块：检索+生成
  - 评 → 准 → Precision（检索准不准）
  - 评 → 全 → Recall（检索全不全）
  - 评 → 信 → Faithfulness（生成忠不忠实）
  - 评 → 流畅 → Answer Relevance（答案是否相关流畅）
  - 检索好 ≠ 生成好，要分开评

## 一字一画面
- **评** —— 像**美食评委打分**：先看成色（检索），再尝味道（生成），两项分别打分才算总分。（本质：检索和生成独立评估）
- **准** —— 像**射箭中靶心**：10支箭有几支在红圈（相关文档）里。（本质：Precision@K）
- **全** —— 像**捞鱼网有没有漏**：池塘里5条金鱼，捞上来几条。（本质：Recall@K）
- **信** —— 像**学生背书不添油加醋**：老师原文是"孔子说"，学生没说成"老子说"。（本质：Faithfulness忠实度）
- **流** —— 像**导游回答问题**：不仅答对了，还条理清晰让人听得懂。（本质：Answer Relevance流畅相关）

## 面试话术
- "RAG评估要**评准全信流畅，检索生成分开看**。检索侧看Recall@K、NDCG、MRR，衡量找不找得到、找得准不准；生成侧用RAGAS框架看Faithfulness、Answer Relevance、Context Precision，衡量答得真不真、对不对题。"
- "很多团队只看检索指标，但检索好生成不一定好，因为Rerank和Prompt组装也会出问题，所以必须端到端评估。"

## 防混补丁
- 容易把"检索Recall高"当成"系统效果好"。其实检索召回10篇但只有1篇有用，生成还是会崩。补丁：**全而不准，垃圾进垃圾出**。

## 追问预演
- **追问1**：RAGAS具体怎么用？→ 用LLM作为裁判，对Faithfulness、Answer Relevance、Context Precision每个维度打0-1分。
- **追问2**：没有标注数据怎么评？→ 用LLM生成合成问答对，或用无参考指标如BERTScore。
- **追问3**：线上怎么监控？→ 埋点看用户满意度、点赞点踩率、追问率，间接反映RAG效果。
