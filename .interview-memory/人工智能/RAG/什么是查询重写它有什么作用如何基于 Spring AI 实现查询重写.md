## 原始知识

- **改写**：将原始query转化为更适合检索的形式
- **消除歧义**：处理一词多义、指代不明
- **扩展同义词**：把口语化转成术语化
- **纠错补全**：修正错别字、补全省略词
- **分解子查询**：复杂问题拆成多个检索单元
- **提高召回率**：扩大检索覆盖范围
- **解决术语不匹配**：用户说"高烧"，文档写"高热"
- **处理口语化**：把"咋回事"转成规范医学表述
- **QueryTransformer接口**：Spring AI 重写入口
- **自定义Prompt**：用LLM驱动生成变体query
- **并行检索**：多版本query同时搜，取并集

## 单字锚点 + 防崩推导

- **锚点字**：改
- **扩展口诀**：改意同，三作用
- **为什么选它**：查询重写的核心动作就是"改"字——改写法、改表达、改形式，一字定音。
- **推导链**：
  1. 看到"改" → 想到改写query，不是改文档
  2. "改"怎么改？→ 消歧义、扩同义、纠错、拆子查询
  3. 改了有啥用？→ 三作用：提召回、对术语、通口语
  4. Spring AI 咋实现？→ QueryTransformer + LLM Prompt + 并行变体检索

## 一字一画面

- **改** —— 像**整容医生拿手术刀**：对着原始query咔嚓几下，消掉赘肉（歧义）、填上假体（同义词）、矫正歪鼻（纠错），最后变成一张标准证件照（规范query），丢进检索库比对。（本质：query标准化变形）

## 面试话术

"查询重写就是在检索前，用LLM把用户的原始query改写成更规范、更检索友好的形式。比如用户问'高烧怎么办'，重写成'高热的治疗方案'，这样就和医学文献里的术语对上了。Spring AI里可以用QueryTransformer接口，配合自定义Prompt让模型生成多个变体query，再并行检索取并集，召回率明显提升。"

## 追问预演

| 追问 | 简要答案 |
|------|----------|
| 查询重写会不会引入语义偏移？ | 会，所以要控制Prompt，让LLM保证语义等价，并对结果做校验。 |
| 并行检索多个变体query，怎么合并结果？ | 一般取并集去重，或用RRF融合排序，避免简单截断。 |
| 除了Spring AI，LangChain怎么实现？ | LangChain有MultiQueryRetriever，原理类似，都是生成多个子query再检索。 |
