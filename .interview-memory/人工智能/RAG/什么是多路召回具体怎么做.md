# 什么是多路召回？具体怎么做？

## 速记卡片
`多 | 多路并，融去重 | 召回策略`

## 原始知识
- **定义**：多种召回策略并行执行，各取所长
- **为什么不用单路**：向量语义好但字面弱，关键词精准但泛化差，规则灵活但天花板低
- **常见路子**：向量路、稀疏路（BM25/TF-IDF）、规则路、图路、协同过滤路
- **核心步骤**：**并**（多路独立召回）→ **融**（合并结果）→ **去**（去重）→ **重**（加权截断，送精排）
- **追问点**：各路权重怎么调？融合公式是什么？

## 顺口溜
**多路并，融去重**

## 单字锚点 + 防崩推导
- **锚点字**：**多**
- **扩展口诀**：**多路并，融去重**
- **为什么选它**：多路召回的灵魂就是"不只一条路"，先想"多"字，自然带出并行和融合。
- **防崩推导**：
  - 多 → 不止一种策略 → 向量 + 关键词 + 规则 + 图
  - 路 → 各跑各的，互不干扰，并行召回 Top-K
  - 并 → 同时跑，拉宽候选池，避免单路盲区
  - 融 → 把各路的 Top-K 合并到一个池子
  - 去 → 同一物品被多路召回，需要去重
  - 重 → 加权排序，截断留前 N，送给精排/Rerank

## 一字一画面
- **多** —— 像**多条管道同时抽水**：向量管抽"语义相似水"，关键词管抽"字面匹配水"，规则管抽"运营指定水"。（本质：多策略覆盖不同信息维度）
- **路** —— 像**高速公路多车道**：各行其道，一辆车在向量路跑，另一辆在关键词路跑，不堵车。（本质：独立执行，互不干扰）
- **并** —— 像**三支搜救队同时出动**：天上直升机、地面搜救犬、水里潜水员，同时覆盖海陆空。（本质：并行拉宽候选集，降低漏召）
- **融** —— 像**把各队救的人集中到广场**：先汇合，再统一处理。（本质：结果合并）
- **去** —— 像**广场入口查身份证去重**：同一个人被两队救了，只算一次。（本质：去重防重复曝光）
- **重** —— 像**急诊分诊台按病情优先级留前 50 人**：轻症的回家，重症的优先见医生。（本质：加权截断，送精排）

## 面试话术
- **问概念** → "多路召回的核心口诀是**多路并，融去重**。因为单路召回总有盲区，比如向量召回语义泛化好，但对精准关键词匹配弱；BM25 关键词召回精准，但语义扩展差。所以我们会开多条路并行召回，再把结果融合、去重、加权截断，最后送给精排或 Rerank。"
- **问具体做法** → "常见路子有向量路、BM25 稀疏路、规则路、图路；融合方式可以是加权求和、轮询（Round Robin）、或插值融合。线上一般通过 AB 实验或离线 Recall@K 来调各路权重。"

## 防混补丁
- **容易和"混合检索"搞混**。多路召回是大概念，混合检索（稠密向量+稀疏向量）只是其中一种"路"。
- **补丁**：**多路是地图，混合是其中一条高速**。

## 追问预演
| 追问 | 简要答案 |
|------|----------|
| 多路融合具体公式？ | 常见有加权求和 `score = Σ w_i * score_i`、轮询、或模型融合（如 LambdaMART） |
| 各路权重怎么调？ | 离线：看 Recall@K / MRR；线上：AB 实验调参 |
| 去重怎么做？ | 按 Item ID 去重；取最高分或指定某一路优先 |
| 多路召回性能怎么保证？ | 各路并行调用；控制每路召回量（如每路只召 Top-100）；异步 IO |
| 和两阶段检索（粗排+精排）区别？ | 多路召回是"多条粗管进水"，粗排精排是"水池里再筛选" |
