## 速记卡片

嵌 | 嵌转量，语义存 | Embedding 向量化 + 项目选型

## 原始知识

- **Embedding（嵌入/向量化）**
  - 把文本、图片、代码等离散数据映射成**高维稠密向量**（通常 256~4096 维）
  - 核心特性：**语义相近 → 向量距离近**（余弦相似度 / 欧氏距离）
  - 类比：把单词/句子放到一个多维语义空间中，意思相近的靠得近
- **工作原理**
  - Transformer 编码器（如 BERT）提取上下文语义
  - Pooling 层（Mean/CLS/Last Hidden）把 token 序列压成一个向量
  - 归一化后输出固定长度的稠密向量
- **主流 Embedding 模型**
  - **OpenAI**：text-embedding-ada-002 / text-embedding-3-small / 3-large（通用强、API贵）
  - **BAAI/BGE**：BGE-large-zh / BGE-M3（中文最优、开源、可私有化）
  - **M3E**：m3e-base / m3e-large（中文社区热门）
  - **E5**：e5-large-v2 / e5-mistral（英文最强、多语言版）
  - **GTE**：gte-large-zh（阿里，中文场景）
  - **多模态**：CLIP（图文）、Jina（长文本）
- **项目选型六维度**
  - **语**（语言）：中文场景优先 BGE / M3E / GTE，英文优先 E5 / OpenAI
  - **域**（领域）：通用选 BGE-M3，医疗/法律垂直领域需微调或选领域模型
  - **上**（上限）：向量化是 RAG 的瓶颈，QPS 要求高的选轻量模型（小模型+GPU）
  - **速**（速度）：模型参数量 vs 维度，维度低（256/512）检索更快
  - **维**（维度）：768 维通用，1024+ 精度更高但存储翻倍
  - **证**（证明）：MTEB/C-MTEB 榜单分数是初筛依据，但最终看业务评测

## 顺口溜

## 单字锚点 + 防崩推导

- **锚点字**：**嵌**
- **扩展口诀**：嵌转量，语义存
- **为什么选它**：Embedding 直译就是"嵌入"，把语义嵌进向量空间。
- **防崩推导**：
  - 嵌 → 嵌入 → 把文本嵌进高维向量空间
  - 转 → Transformer 编码器转换
  - 量 → 输出固定长度的稠密向量
  - 语义存 → 语义信息被编码保存在向量中，相似文本向量距离近

## 一字一画面

- **嵌** —— 像**把宝石镶进戒指**：文本是宝石，向量空间是戒指托座，镶好后宝石在三维空间有了固定坐标。（本质：离散文本 → 连续向量）
- **转** —— 像**榨汁机**：苹果（文本）进去，苹果汁（向量）出来，保留了苹果的味道（语义），但形态完全不同。（本质：Transformer 编码语义）
- **量** —— 像**GPS 坐标**：北京天安门、上海外滩在地球仪上有各自的经纬度坐标，两个地点越近，坐标差越小。（本质：向量距离 ≈ 语义距离）

## 面试话术

"Embedding 就是把文本转换成高维稠密向量的技术，口诀是**嵌转量，语义存**。通过 Transformer 编码器把文本映射到向量空间，语义相近的文本在向量空间里距离也相近。我们项目用的是 **BGE-large-zh**，选型主要考虑四个因素：第一**语言**，我们是中文文档库，BGE 在 C-MTEB 中文榜单上表现最好；第二**私有化**，数据不能出域，BGE 开源可本地部署；第三**速度**，用 ONNX 量化后单卡 QPS 能到 2000+，满足业务需求；第四**成本**，开源免费，相比调 OpenAI API 每个月能省几万块。上线后 RAG 的召回准确率从关键词检索的 62% 提升到了 89%。"

## 防混补丁

容易和 "One-Hot 编码" 搞混。One-Hot 是高维稀疏向量，词与词之间正交（距离都一样），没有语义信息。补丁：**One-Hot 是查字典，Embedding 是测语义距离**。

## 追问预演

- **Q：为什么不用 OpenAI 的 Embedding API？**
  → 数据隐私要求不能出域，且调用量大了成本高。BGE 本地部署可控且免费。
- **Q：Embedding 模型需要微调吗？**
  → 通用场景不需要，垂直领域（法律/医疗）建议用领域数据做对比学习微调，召回率能再提升 10-15%。
- **Q：向量维度越高越好吗？**
  → 不是。768 维是性价比甜点，1024+ 精度提升有限但存储和计算翻倍。实际选型要在业务评测集上测 Recall@K，选拐点。
- **Q：多语言场景怎么选？**
  → BGE-M3 支持 100+ 语言，或者 E5-mistral，多语言统一向量空间，不用分语言建索引。
