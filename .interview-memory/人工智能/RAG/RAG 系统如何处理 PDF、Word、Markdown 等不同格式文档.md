# RAG 系统如何处理 PDF、Word、Markdown 等不同格式文档？

## 速记卡片
`解 | 解析洗切标准化，格式统一再入库 | 多格式→纯文本`

## 原始知识
- **解**：格式解析（PDF用PyPDF/MinerU，Word用python-docx，Markdown直接读）
- **析**：结构化提取（表格、图片、标题层级）
- **洗**：去噪（页眉页脚、重复内容、乱码）
- **切**：按语义切分，保留上下文
- **标**：标准化（统一编码、时间格式、命名实体）
- **加粗易混点**：PDF解析是最难的， scanned PDF 还需要OCR

## 顺口溜
解析洗切标准化

## 单字锚点 + 防崩推导
- **锚点字**：**解**
- **扩展口诀**：**解析洗切标准化**
- **为什么选它**："解"谐音"解方程"，对应"解析文档"这个动作，是多格式处理的第一步。
- **防崩推导**：
  - 解 → 解析 → 不同格式用不同工具解包
  - 解 → 析 → 提取结构（标题、表格、图片）
  - 解 → 洗 → 去噪去重去乱码
  - 解 → 切 → 语义切分
  - 解 → 标准化 → 统一格式再入库

## 一字一画面
- **解** —— 像**拆快递**：PDF是硬壳包装（难拆），Word是拉链袋（好拆），Markdown是张纸（直接看）。（本质：不同格式解析难度不同）
- **析** —— 像**分拣员把包裹里的东西分类**：衣服放A区，电子产品放B区，易碎品贴标签。（本质：提取标题、表格、图片等结构）
- **洗** —— 像**洗菜去泥**：把页眉页脚（泥）冲掉，把烂叶子（乱码）摘掉。（本质：数据清洗）
- **切** —— 像**把洗好的菜切成段**：按语义切，不能一刀把单词切成两半。（本质：语义Chunking）
- **标** —— 像**贴统一价签**：不管哪来的货，最后都贴一样的标签。（本质：标准化入库）

## 面试话术
- "多格式处理我们总结为**解析洗切标准化**。PDF用MinerU或PyPDF解析，注意scanned PDF要OCR；Word用python-docx提取段落和表格；Markdown最轻量直接读。之后统一做数据清洗、语义切分、标准化，再写入向量库。"
- "PDF是坑最多的，尤其是扫描件和复杂排版，我们线上会对解析结果做质量校验。"

## 防混补丁
- 容易认为"Markdown最简单所以不需要处理"，其实MD里也可能有HTML标签、 frontmatter噪音。补丁：**MD虽轻，也要洗**。

## 追问预演
- **追问1**：PDF里有表格怎么处理？→ 用MinerU或Camelot提取表格，转成Markdown表格或JSON保留结构。
- **追问2**：图片里的文字怎么办？→ OCR（PaddleOCR/EasyOCR）提取后作为文本块入库。
- **追问3**：怎么保证标题层级不被破坏？→ 解析时保留标题标记（H1/H2），切分时按层级做上下文增强。
