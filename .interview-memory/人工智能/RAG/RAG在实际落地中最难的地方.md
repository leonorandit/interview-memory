# RAG在实际落地中最难的地方

## 速记卡片
`脏 | 脏切准幻评 | 落地五座山`

## 原始知识
- **脏**：数据清洗难，PDF/扫描件/图片/表格解析困难，噪声大
- **切**：文档分块（Chunking）无银弹，大小难平衡，语义易断裂
- **准**：检索精度低，召回不足或噪声多，跨文档推理难
- **幻**：幻觉问题，生成内容脱离检索资料，编造引用
- **评**：评估困难，无标准答案，离线指标与在线满意度不一致
- **追问点**：哪座山最痛？通常"脏"占80%工作量，"准"决定天花板
- **易混点**：落地难点 vs RAG主流程

## 顺口溜
**脏切准幻评**

## 单字锚点 + 防崩推导
- **锚点字**：**脏**
- **扩展口诀**：**脏切难准评**
- **为什么选它**：数据质量是RAG的地基，地基脏，上面全歪。先想"脏不脏"。
- **防崩推导**：
  - 脏 → 原始数据质量差 → PDF/图片/扫描件解析困难 → 占80%工作量
  - 脏 → 需要切 → 切大了浪费token/上下文超限，切小了语义断裂
  - 切 → 需要查 → 查不准（召回低、噪声多、语义鸿沟）→ 决定RAG天花板
  - 查 → 给模型生 → 模型乱说 → 幻觉（编造、脱离检索内容）
  - 生 → 好不好 → 评不了（无标准答案、主观性强、离线在线不一致）

## 一字一画面
- **脏** —— 像**下水道里的油污文件**：PDF扫描件歪七扭八，表格嵌在图片里，文字像油渍一样粘在一起，机器读出来全是乱码。（数据清洗与文档解析）
- **切** —— 像**切生日蛋糕**：切太大一块吃不下（上下文超限），切太小奶油和蛋糕分离（语义断裂），切歪了把樱桃切烂（破坏关键信息）。（文档分块策略）
- **准** —— 像**大海捞针**：针是"公司2024年Q3营收"，捞上来一堆"营收预测""Q2财报""行业分析"，就是没针。（检索精度与召回质量）
- **幻** —— 像**吹牛大王**：明明资料里写的是A，他偏说成B，还一本正经地引用不存在的第5页第3行。（生成幻觉与事实一致性）
- **评** —— 像**给作文打分**：没有标准答案，张三觉得满分，李四觉得跑题，到底好不好谁也说不清，用户点赞但离线指标低。（评估体系与效果衡量）

## 面试话术
- 问落地难点 → "RAG落地有五大难点，口诀是**脏切准幻评**。第一是'脏'——原始数据质量差，PDF扫描件、图片表格解析困难，数据清洗往往占80%工作量；第二是'切'——文档分块没有银弹，切大了浪费token，切小了丢失上下文；第三是'准'——检索召回精度不够，用户问题与文档语义鸿沟大，这直接决定RAG效果天花板；第四是'幻'——模型容易脱离检索内容编造答案，甚至伪造引用来源；第五是'评'——缺乏客观统一的评估标准，离线指标和在线用户满意度经常不一致。实际项目中，我觉得'脏'和'准'是最痛的。"
- 追问最痛 → "最痛的是'脏'和'准'。数据清洗是隐性成本大头，很多团队低估；检索不准的话，后面生成再好也救不回来。"

## 防混补丁
- 容易和"RAG主要流程"搞混。流程是**切存查排合生**，落地难点是**脏切准幻评**。补丁：**流程是步骤，落地是坑**。
- 容易漏掉"评估难"，只记得技术难点。补丁：**技术五座山，评是第六座**。

## 追问预演
1. **Q: 怎么解决数据脏的问题？**
   A: 多模态文档解析（OCR+版面分析）、规则清洗（去页眉页脚去重）、质量过滤（低质量/短文本丢弃）、人工标注迭代。
2. **Q: 分块策略有哪些？**
   A: 固定长度、递归字符、语义分块（Embedding相似度）、Agentic分块、按文档结构分块（标题/段落/表格）。
3. **Q: 检索不准怎么优化？**
   A: 混合检索（稀疏BM25+稠密向量）、查询重写/扩展、Rerank精排、元数据过滤、HyDE（假设文档嵌入）、多跳检索。
4. **Q: 怎么缓解幻觉？**
   A: 严格引用约束（要求模型基于检索内容回答）、RAG-specific微调、Faithfulness评估、多轮检索验证、溯源高亮。
5. **Q: 评估指标有哪些？**
   A: 检索端：Recall@K、MRR、NDCG；生成端：Faithfulness、Answer Relevance、Context Precision；人工评估：实用性、流畅度、引用准确性。
6. **Q: 成本和延迟怎么考虑？**
   A: 这是工程落地附加难点。优化方向：向量数据库选型（HNSW索引）、缓存热点查询、精排模型轻量化、检索结果截断、异步预热。
