# RAG 中的查询扩展

## 速记卡片
`扩 | 一扩多召回 | 扩撒网`

## 原始知识
- 查询扩展（Query Expansion）：将用户原始查询改写/扩展为多个相关查询再检索
- 核心方法：
  - 同义词扩展、LLM生成改写、伪相关反馈（PRF）
  - HyDE（Hypothetical Document Embeddings）：让LLM先写假设答案，再用答案Embedding去检索
- 为什么需要：
  - 用户查询太短/太模糊，与文档用词不匹配（词汇鸿沟）
  - 提高召回率（Recall），防止相关文档被漏掉
- **易混点**：查询扩展 vs Rerank、扩展 vs 重写（Rewrite）
- **追问点**：有哪些扩展方法？HyDE原理？扩展会不会引入噪声？

## 顺口溜
**一扩多召回**

## 单字锚点 + 防崩推导
- **锚点字**：**扩**
- **扩展口诀**：**一扩多召回**
- **为什么选它**：面试题的关键词就是"查询扩展"，"扩"是整个技术的核心动作。
- **防崩推导**：
  - 扩（查询扩展）→ 把一个查询变成多个相关查询
  - 多（多查询）→ 用不同角度/同义词去检索向量数据库
  - 召（召回）→ 核心目的是提高召回率
  - 不漏 → 防止因为词汇不匹配而漏掉相关文档

## 一字一画面
- **扩** —— 像**把网撒大**：原来用一根鱼竿钓鱼，现在撒一张大网。（本质：扩大检索范围，提高覆盖面）
- **多** —— 像**多问几个人**：同一个问题用不同方式问，确保没人漏听。（本质：多角度查询，消除词汇鸿沟）
- **召** —— 像**点名册**：确保所有相关的人都被点到，一个不漏。（本质：提高召回率）
- **歧** —— 像**消除误会**："苹果"可能指水果也可能指手机，扩展后明确真实意图。（本质：消除查询歧义）

## 面试话术
- 问什么是查询扩展 → "查询扩展是RAG中提高召回率的关键技术，口诀是**一扩多召回**。**扩**——把用户的原始查询扩展成多个相关查询，比如用LLM生成同义改写、用HyDE生成假设文档；**多**——用多个角度去检索向量数据库；**召**——核心目的是提高召回率，防止因为用户用词和文档用词不一致而漏掉相关内容。比如用户问'怎么修车'，文档里写的是'汽车维修方法'，直接检索可能匹配不到，扩展后就能召回。"
- 问和Rerank区别 → "查询扩展是**增加候选集**（提高召回），Rerank是**精排候选集**（提高精度）。口诀：**扩展是撒网，Rerank是挑鱼**。"

## 防混补丁
- 容易和"Rerank"搞混。查询扩展是增加候选集（提高召回），Rerank是精排候选集（提高精度）。补丁：**扩展是撒网，Rerank是挑鱼**。
- 容易和"查询重写（Query Rewrite）"搞混。重写是改成更好的一个查询；扩展是生成多个查询一起检索。补丁：**重写是换鱼竿，扩展是多根竿**。

## 追问预演
- **问**：有哪些扩展方法？→ 同义词扩展、LLM生成扩展查询、伪相关反馈（PRF，用首轮检索结果里的词扩展）、HyDE（假设文档嵌入）。
- **问**：HyDE是什么？→ Hypothetical Document Embeddings，让LLM根据查询生成一个假设答案，再用这个答案的Embedding去检索，解决查询太短的问题。
- **问**：扩展会不会引入噪声？→ 会，扩展太多不相关的查询会降低精度。通常需要和Rerank配合：扩展提高召回，Rerank精排去噪。

## 手撕代码（查询扩展伪代码）

```python
# 方法1: LLM生成扩展查询
expanded_queries = llm.generate(
    prompt=f"基于查询'{query}'生成3个语义相近的扩展查询"
)

# 方法2: HyDE（假设文档嵌入）
hypothetical_doc = llm.generate(f"请回答这个问题：{query}")
hyde_embedding = embedding_model.encode(hypothetical_doc)

# 检索：原始查询 + 扩展查询 + HyDE 联合检索
all_queries = [query] + expanded_queries + [hypothetical_doc]
all_embeddings = [embedding_model.encode(q) for q in all_queries]
results = vector_db.search(all_embeddings, top_k=10)

# 后续接 Rerank 精排
ranked = reranker.rerank(query, results, top_k=5)
```
