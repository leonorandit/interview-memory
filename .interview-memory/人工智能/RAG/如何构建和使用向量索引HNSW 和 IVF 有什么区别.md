# 如何构建和使用向量索引HNSW 和 IVF 有什么区别

## 速记卡片
`索 | 索分图聚找，HNSW建图层，IVF先分桶 | 图vs聚类`

## 原始知识
- 向量索引：把高维向量组织成数据结构，加速最近邻搜索（ANN）
- HNSW：分层可导航小世界图，每层建图，搜索时从上层粗粒度往下层细粒度跳
- IVF：倒排文件索引，先K-means聚类分桶，查询时找最近几个桶再精排
- 区别：HNSW**建图成本高、内存大、查询快**；IVF**建索引快、内存省、查询稍慢**
- **加粗易混点**：HNSW适合小中型数据集+高QPS；IVF适合大规模+内存敏感场景
- **加粗易混点**：HNSW没有训练阶段，IVF需要训练聚类中心

## 顺口溜
**索分图聚找，H快I省**

## 单字锚点 + 防崩推导
- **锚点字**：**索**
- **扩展口诀**：**索引分两家**
- **为什么选它**：问题是关于"向量索引"的，锚"索"直接对应。
- **防崩推导**：
  - 索 → 怎么加速找？ → 两种路线
  - 索 → 图路线 → HNSW → 多层图 → 上层跳下层 → 查询快、内存大
  - 索 → 聚类路线 → IVF → K-means分桶 → 先找桶再精排 → 省内存、建快
  - 索 → 怎么选？ → 小规模高QPS选H，大规模省内存选I

## 一字一画面
- **索** —— 像**找朋友游戏**：一堆人里最快找到最像的人，需要提前排好队形。（本质：ANN加速）
- **图** —— 像**地铁换乘图**：外层是城际铁路（ coarse），内层是市内地铁（fine），层层换乘客到终点。（HNSW分层图搜索）
- **聚** —— 像**商场楼层导览**：先按品类分楼层（聚类桶），再找具体店铺（桶内精排）。（IVF倒排分桶）
- **快** —— 像**高速ETC**：不停车通过，但建收费站贵。（HNSW查询快、内存贵）
- **省** —— 像**公交+步行**：便宜但多转几趟。（IVF省内存、查询多一步）

## 面试话术
- "向量索引解决高维最近邻搜索问题，主流有两种：**索分图聚找**。HNSW是建多层图，查询像地铁换乘，一层层往下跳，查询快但内存吃得多；IVF是先K-means聚类分桶，查询先找最近几个桶再精排，省内存但慢一拍。小数据高并发选HNSW，大数据省成本选IVF，也可以两者组合成IVF_PQfs+HNSW。"

## 防混补丁
- 容易把"向量索引"和"传统B+树索引"混为一谈。补丁：**索是高维近邻，树是精确范围**，向量索引是近似搜索（ANN），允许微小精度换速度。

## 追问预演
- Q：HNSW的M和ef参数什么意思？ → A：M是每层最大连接数，ef是搜索时考察的候选集大小，越大越准越慢。
- Q：IVF的nlist和nprobe怎么调？ → A：nlist是桶数，nprobe是查询时查几个桶，桶多建索引慢查准，探多多查准查慢。
- Q：还有什么其他索引？ → A：PQ（乘积量化省内存）、DiskANN（磁盘版HNSW省内存）。
