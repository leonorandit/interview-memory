# RAG 系统如何标注信息来源和提供引用

## 速记卡片
`引 | 引随查带标，块里埋ID，生时挂尾注 | 溯源可信`

## 原始知识
- 来源标注流程：文档预处理埋ID → 检索带回元数据 → 生成时拼接引用 → 输出附尾注
- 技术实现：chunk级溯源（每个chunk带doc_id+page+url）、生成Prompt加指令"请标注来源"
- 引用格式：[^1^] 或 [来源: 文档名]
- **加粗易混点**：引用不能是幻觉编的，必须严格来自检索结果
- **加粗易混点**：多chunk合并时要处理重复来源，避免尾注爆炸

## 顺口溜
**引随查带标，块埋生挂注**

## 单字锚点 + 防崩推导
- **锚点字**：**引**
- **扩展口诀**：**引从块中来**
- **为什么选它**：整个问题是关于"引用"的，锚"引"最精准。
- **防崩推导**：
  - 引 → 要溯源 → 块里先埋ID（预处理）
  - 引 → 要准确 → 检索带回什么就引什么
  - 引 → 要规范 → 生成时加引用指令
  - 引 → 要可读 → 输出末尾列尾注

## 一字一画面
- **引** —— 像**论文参考文献**：每句话后面括号角标，末尾统一列出处。（本质：学术引用范式）
- **埋** —— 像**给每个乐高积木印编号**：组装前印好，拆拼都能溯源。（本质：chunk级元数据）
- **带** —— 像**快递单随货走**：检索召回的chunk必须贴好来源标签。（本质：检索结果携带上下文）
- **挂** —— 像**商品吊牌**：生成完在回答末尾挂一排吊牌，点哪看哪。（本质：输出层引用渲染）

## 面试话术
- "RAG引用是**引随查带标**。预处理时给每个chunk埋doc_id和页码；检索召回时把元数据一起带进上下文；生成Prompt里加一句'请标注信息来源'；最后输出[^1^][^2^]这种尾注，用户可点击溯源。关键是引用必须严格来自检索结果，不能是模型自己编的。"

## 防混补丁
- 容易和"模型自带知识"搞混。补丁：**引只能查，不能编**，模型自己知道的不算引用，必须来自检索库。

## 追问预演
- Q：多chunk同一来源怎么展示？ → A：去重合并，同一文档只列一次，标多个页码范围。
- Q：怎么防引用幻觉？ → A：后处理校验，检查引用的ID是否在检索结果列表中。
- Q：用户点击引用后跳到哪？ → A：跳原文档对应chunk或高亮段落，需前端做锚点定位。
