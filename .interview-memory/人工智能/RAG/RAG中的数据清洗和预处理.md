# RAG 中的数据清洗和预处理

## 速记卡片
`洗 | 洗整去重标 | 先洗整再去重后贴标`

## 原始知识
- 数据清洗是RAG流程中"切分"之前的关键步骤，直接影响检索精度
- 五步流程：
  1. **洗**（清洗去噪）- 去除HTML标签、广告、页眉页脚、特殊符号、乱码
  2. **整**（规范化+结构化）- 统一编码、统一格式，识别标题、段落、表格、代码块
  3. **去**（去重）- 去掉重复或近似重复的文档/段落
  4. **重**（质量过滤）- 筛选太短/太长、无意义、低质量的片段
  5. **标**（元数据标注）- 添加来源、类别、时间等标签，用于检索过滤
- **易混点**：数据清洗 vs 数据增强
- **追问点**：怎么去重？怎么处理PDF/图片？元数据有什么用？

## 单字锚点 + 防崩推导
- **锚点字**：**洗**
- **扩展口诀**：**先洗整，再去重，后贴标**
- **为什么选它**：面试题的关键词就是"数据清洗和预处理"，"洗"是整个流程的起点，也是面试官最关心的环节。
- **防崩推导**：
  - 洗（清洗去噪）→ 先去掉HTML标签、广告、噪声等无关内容
  - 整（规范化+结构化）→ 统一格式，识别标题段落表格等结构
  - 去（去重）→ 去掉重复或近似重复的文档
  - 重（质量过滤）→ 筛选掉太短/太长/低质量的片段
  - 标（元数据标注）→ 添加来源、类别、时间等标签，方便后续过滤检索

## 一字一画面
- **洗** —— 像**洗菜**：先把菜叶上的泥土、虫子、烂叶洗掉。（本质：清洗去噪，去掉无关内容）
- **整** —— 像**切菜前理整齐**：把菜叶理顺，按茎和叶分类，知道哪里该切。（本质：规范化+结构化，识别文档结构）
- **去** —— 像**挑出重复的菜叶**：两棵菜长得一样，只留一棵。（本质：去重，去掉近似重复内容）
- **重** —— 像**筛选优质食材**：太小的不要，烂掉的扔掉，只留好的。（本质：质量过滤，筛选高质量片段）
- **标** —— 像**给保鲜盒贴标签**：写上"青菜-冷藏-3天"，方便以后找。（本质：元数据标注，方便检索时过滤）

## 面试话术
- 问数据清洗怎么做 → "RAG的数据清洗和预处理，口诀是**洗整去重标**。**洗**——清洗去噪，去掉HTML标签、广告、页眉页脚等无关内容；**整**——规范化+结构化，统一编码格式，识别标题、段落、表格、代码块等结构；**去**——去重，去掉重复或近似重复的文档/段落；**重**——质量过滤，筛选掉太短/太长、无意义、低质量的片段；**标**——元数据标注，添加来源、类别、时间等标签，方便后续按条件过滤检索。清洗做得越好，检索精度越高。"
- 问为什么要做清洗 → "脏数据会导致检索噪声大、Embedding质量差。比如HTML标签会被编码进向量，广告内容会污染检索结果，重复文档会浪费存储和计算资源。"

## 防混补丁
- 容易和"数据增强"搞混。数据清洗是去掉坏的，数据增强是增加好的（如生成更多变体）。补丁：**清洗是减法，增强是加法**。

## 追问预演
- **问**：怎么去重？→ 可以用SimHash、MinHash、文本相似度（如余弦相似度）等方法检测近似重复。
- **问**：怎么处理PDF？→ 先用PDF解析器提取文本，注意处理表格、图片、多栏布局。复杂PDF可以用OCR或专用解析库（如PyPDF、pdfplumber）。
- **问**：元数据有什么用？→ 元数据可以用于检索时过滤（如只看最近一年的文档），也可以用于Rerank时加权，还能在生成回答时标注引用来源。

## 手撕代码（数据清洗伪代码）

```python
# 1. 洗（清洗去噪）
text = remove_html_tags(raw_text)
text = remove_headers_footers(text)
text = remove_ads(text)
text = normalize_encoding(text)

# 2. 整（规范化+结构化）
text = normalize_whitespace(text)
chunks = extract_structure(text)  # 识别标题、段落、表格、代码块

# 3. 去（去重）
unique_chunks = deduplicate(chunks, threshold=0.9)  # SimHash/MinHash

# 4. 重（质量过滤）
quality_chunks = filter_quality(
    unique_chunks, 
    min_len=50,      # 太短无意义
    max_len=1000,    # 太长检索不精准
    min_chinese_ratio=0.5  # 乱码过滤
)

# 5. 标（元数据标注）
for chunk in quality_chunks:
    chunk.metadata = {
        "source": doc_source,
        "category": doc_category,
        "created_at": doc_date,
        "doc_id": doc_id
    }
```
