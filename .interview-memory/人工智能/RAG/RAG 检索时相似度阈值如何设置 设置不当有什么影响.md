# RAG 检索时相似度阈值如何设置？设置不当有什么影响？

## 速记卡片
`阈 | 阈值高低定边界，太松太紧都危险 | 精度vs召回`

## 原始知识
- **高阈值**（如0.8+）：严格过滤，精度高但可能漏召回
- **低阈值**（如0.3-）：召回多但引入噪音，幻觉风险大
- **动态阈值**：根据查询难度、领域自适应调整
- **RRF/Rerank后截断**：不用单一阈值，用排序位次
- **加粗易混点**：余弦相似度和欧氏距离方向相反，阈值定义不同

## 顺口溜
高低定边界

## 单字锚点 + 防崩推导
- **锚点字**：**阈**
- **扩展口诀**：**高低定边界**
- **为什么选它**："阈"是门槛的意思，直接对应相似度阈值这个核心概念，信息密度极高。
- **防崩推导**：
  - 阈 → 门槛 → 高门槛（高阈值）→ 严格筛选 → 精度高，召回低
  - 阈 → 低门槛（低阈值）→ 宽松通过 → 召回高，噪音多
  - 阈 → 太松 → 垃圾进Prompt → LLM幻觉
  - 阈 → 太紧 → 该有的知识进不来 → 答不上
  - 阈 → 动态调 → 根据场景自适应

## 一字一画面
- **阈** —— 像** nightclub 门口的保安**：阈值高就是帅哥美女才能进（精度高但人少）；阈值低就是谁都能进（人多但鱼龙混杂）。（本质：相似度门槛控制质量与数量）
- **高** —— 像**高考分数线650**：只收尖子生，但可能漏掉偏科天才。（本质：高阈值精度高召回低）
- **低** —— 像**社区大学零门槛**：什么人都有，课堂里可能混进卖保险的。（本质：低阈值召回高噪音大）
- **动** —— 像**弹性检票口**：高峰期放宽，淡季收紧，根据人流量自动调节。（本质：动态阈值自适应）

## 面试话术
- "阈值设置是**高低定边界**的艺术。太高容易漏召回，该答的答不上；太低噪音混进Prompt，LLM容易产生幻觉。我们线上会根据业务场景动态调整，比如医疗场景阈值设高，客服场景相对放宽，同时结合Rerank做二次筛选。"
- "实践中我更倾向于先低阈值多召回，再用Rerank精排截断，比死磕一个阈值更稳。"

## 防混补丁
- 容易把余弦相似度和欧氏距离的阈值搞混。余弦 similarity 越高越相似（0~1），欧氏 distance 越低越相似。补丁：**余弦高分近，欧氏低分近**。

## 追问预演
- **追问1**：怎么找最优阈值？→ 在验证集上扫不同阈值，看Precision-Recall曲线拐点。
- **追问2**：不同领域阈值一样吗？→ 不一样，医疗/法律要求高阈值，通用问答可以低一些。
- **追问3**：不用阈值怎么截断？→ 固定Top-K，或按Rerank分数的gap截断（比如第一名和第二名差距很大就只看第一）。
