# 怎么量化你的 RAG 效果？

## 速记卡片
`量 | 量准召，量相忠，量业人 | 三维量化`

## 原始知识
- 检索层：Precision、Recall、MRR、NDCG、Hit@K、MAP
- 生成层：Faithfulness（忠实度/事实一致性）、Answer Relevance（回答相关性）、Context Relevance（上下文相关性）
- 端到端框架：RAGAS、ARES、TruLens、CRAG
- 业务/人工层：用户满意度、采纳率、转化率、人工标注、A/B Test、Bad Case 分析
- **易混点**：传统 NLP 指标 BLEU/ROUGE 在 RAG 中不适用
- **追问点**：RAGAS 原理、忠实度计算、无标注数据怎么办

## 顺口溜
**量准召，量相忠，量业人**

## 单字锚点 + 防崩推导
- **锚点字**：**量**
- **扩展口诀**：**量准召，量相忠，量业人**
- **为什么选它**：题目问"怎么量化"，锚点字直接扣题。
- **防崩推导**：
  - 量 → 量化 → 分三层：检索、生成、业务
  - 准 → 检索精确率/准确率 (Precision) + MRR、NDCG、Hit@K
  - 召 → 检索召回率 (Recall)，有没有漏
  - 相 → 生成回答与用户问题的相关性 (Answer Relevance)
  - 忠 → 生成回答与检索文档的事实一致性 (Faithfulness，防幻觉)
  - 业 → 业务指标：转化率、采纳率、满意度、停留时长
  - 人 → 人工评估：专家打分、Gold Dataset、Bad Case 复盘

## 一字一画面
- **量** —— 像**拿尺子量布**：三个师傅分别量不同部位，最后拼成完整尺寸。（本质：分维度量化）
- **准** —— 像**射箭命中率**：10支箭中靶心7支，3支脱靶。（本质：Precision，准不准）
- **召** —— 像**集合点名**：应到10人，实到8人，2人缺席。（本质：Recall，全不全）
- **相** —— 像**相亲匹配度**：对方条件和你要求的相关程度。（本质：Answer Relevance）
- **忠** —— 像**忠臣不欺君**：说的话必须有史料依据，不能编造。（本质：Faithfulness，防幻觉）
- **业** —— 像**业绩看板**：销售额、转化率、客诉率。（本质：业务指标）
- **人** —— 像**专家评审团**：专家打分，一票否决。（本质：人工评估是黄金标准）

## 面试话术
- 问怎么量化 → "我分三个层面来量化，口诀是**量准召，量相忠，量业人**。检索层看精确率、召回率，配合 MRR、NDCG；生成层核心看回答相关性和忠实度，防止幻觉；最后一定要回归业务指标和人工评估，比如用户采纳率、满意度，配合 A/B Test。"
- 问框架 → "工程上常用 RAGAS 做自动化评估，它用 LLM-as-a-Judge 来打分；再配合人工抽检和 A/B Test 交叉验证。"

## 防混补丁
- 容易和传统 NLP 指标（BLEU/ROUGE）搞混。RAG 更关注**事实一致性**而非文本 n-gram 重叠。补丁：**RAG看事实，NLP看字句**。

## 追问预演
1. **RAGAS 是什么？怎么工作的？**
   → 开源 RAG 评估框架，核心用 LLM 做裁判。主要指标：Faithfulness、Answer Relevance、Context Precision、Context Recall。不需要人工标注，通过 prompt 让 LLM 判断答案是否忠实于上下文。
2. **忠实度（Faithfulness）具体怎么算？**
   → 抽取答案中的每个事实陈述，逐条判断是否在检索到的 Context 中有支持依据。支持率 = 支持陈述数 / 总陈述数。
3. **没有标注数据怎么评？**
   → 三条路：① 用 RAGAS/ARES 等无参考评估框架；② 用 LLM 生成合成问答对作为伪标注；③ 先上线跑起来收集用户反馈。
4. **A/B Test 怎么做？**
   → 分流用户流量，控制变量（如只换检索策略或只换 prompt），对比核心业务指标（采纳率、转化率、满意度）。
5. **检索和生成哪个更重要？**
   → 检索是天花板，生成是在天花板上画画。检索不准，生成再好也是胡说。所以优先保证检索准召。
