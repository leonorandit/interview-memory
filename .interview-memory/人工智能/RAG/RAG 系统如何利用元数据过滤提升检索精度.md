# RAG 系统如何利用元数据过滤提升检索精度

## 速记卡片
`滤 | 滤前置筛精，元数据当筛子，语义当铲子 | 先粗后精`

## 原始知识
- 元数据过滤：在向量检索前/后，用标签（时间、作者、分类、权限）先缩小候选集
- 实现方式：预过滤（Pre-filtering，先元数据筛再向量搜）+ 后过滤（Post-filtering，先搜再筛）
- 常用元数据：时间范围、文档类型、用户权限、产品分类、版本号
- **加粗易混点**：预过滤速度快但可能漏相关文档；后过滤精度高但计算量大
- **加粗易混点**：不是所有向量数据库都支持预过滤，选型要注意

## 顺口溜
**滤前置筛精，元筛语义铲**

## 单字锚点 + 防崩推导
- **锚点字**：**滤**
- **扩展口诀**：**滤要先筛后挖**
- **为什么选它**：问题核心是"过滤提升精度"，锚"滤"直接命中动作。
- **防崩推导**：
  - 滤 → 先筛 → 元数据当筛子（Pre-filtering）
  - 滤 → 后挖 → 语义相似度当铲子（向量检索）
  - 滤 → 筛子目数 → 时间、作者、分类、权限
  - 滤 → 漏网处理 → 预过滤可能漏，后过滤可能慢，需权衡

## 一字一画面
- **滤** —— 像**淘金先筛石子**：大筛子晃掉明显不相关的，剩下金沙再用细筛。（本质：分层过滤）
- **筛** —— 像**实验室选培养皿**：先选日期新鲜的、作者权威的，缩小范围再细看内容。（本质：元数据预过滤）
- **铲** —— 像**考古队精准挖掘**：范围定好后，用语义铲子一寸寸挖最相关的片段。（本质：向量精检索）
- **精** —— 像**钻石分级**：先筛大小（元数据），再看切工（语义），最终只剩VVS。（本质：精度提升）

## 面试话术
- "RAG提升检索精度靠**滤前置筛精**。把元数据当筛子——时间、分类、权限这些标签先预过滤，把明显不相关的筛掉；再用语义相似度当铲子，在缩小后的候选集里精准挖掘。预过滤快但可能漏，后过滤准但慢，通常用预过滤+向量检索组合，具体看数据库支持。"

## 防混补丁
- 容易和"Rerank"搞混。元数据过滤是**粗筛**，Rerank是**精排**。补丁：**滤是筛石子，排是挑金子**，先滤后排两步走。

## 追问预演
- Q：预过滤和后过滤怎么选？ → A：数据量大选预过滤减少向量计算；精度要求高可后过滤，或两者结合。
- Q：元数据存在哪？ → A：和向量一起存向量数据库的payload里，如Pinecone metadata、Milvus scalar field。
- Q：复合过滤条件怎么写？ → A：用AND/OR表达式，如 category='财务' AND year>=2023。
