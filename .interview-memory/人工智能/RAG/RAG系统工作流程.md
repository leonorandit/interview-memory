# RAG系统工作流程

## 速记卡片
`查 | 一查二排三合四生 | 流程四步`

## 原始知识
- **查（检索）**：用户Query向量化，从知识库向量检索Top-K召回
- **排（精排）**：Rerank模型对召回文档重新排序，筛选最相关片段
- **合（合并）**：将检索结果与用户Query拼接成Prompt上下文
- **生（生成）**：LLM基于上下文生成最终回答
- **易混点**：RAG vs 微调
- **追问点**：混合检索、上下文过长、Rerank必要性

## 顺口溜
**一查二排三合四生**

## 单字锚点 + 防崩推导
- **锚点字**：**查**
- **扩展口诀**：**查排合生**
- **为什么选它**：RAG的灵魂是"查"。没有检索，就只是普通大模型对话。
- **防崩推导**：
  - 查 → 用户提问 → 向量化 → 从知识库召回相关文档（Top-K）
  - 排 → 召回结果多且杂 → Rerank精排 → 选出最相关的N条
  - 合 → 把精选文档 + 用户问题 → 拼接成完整Prompt上下文
  - 生 → LLM看着上下文 → 生成有依据的回答

## 一字一画面
- **查** —— 像**图书馆找书**：你说关键词，管理员从书架上抱回一摞相关书籍。（向量检索召回Top-K）
- **排** —— 像**管理员按相关度重新整理**：把最贴切的书放最上面，无关的剔除。（Rerank精排）
- **合** —— 像**把书页剪贴到作业本上**：把相关内容贴在问题后面，做成参考资料。（上下文合并）
- **生** —— 像**学霸看着资料写作业**：看着剪贴好的材料，写出完整准确的答案。（LLM生成）

## 面试话术
- 问流程 → "RAG的核心流程我概括为**查排合生**四步。用户提问后，系统先做向量检索召回相关文档；再用Rerank精排筛选最相关片段；然后把检索结果和原问题合并成Prompt上下文；最后交给大模型生成回答。这样既能利用外部知识，又能发挥大模型的生成能力。"
- 问优势 → "相比微调，RAG是**开卷考试**，知识更新只需要改知识库，不需要重新训练模型，而且可解释性强，能直接看到检索到了哪些文档。"
