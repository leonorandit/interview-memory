# RAG 和模型微调 Fine-tuning 有什么区别？如何选择？

## 速记卡片
`选 | 外脑换脑要分明，动态知识选RAG | 外挂vs内化`

## 原始知识
- **RAG**：外挂知识库，检索+生成，知识动态更新，不改动模型参数
- **Fine-tuning**：内化知识到模型参数，需要训练数据，知识静态（截止训练时）
- **选RAG**：知识频繁更新、需要可解释引用、数据量小
- **选微调**：需要改变模型风格/格式、领域深度推理、大量标注数据
- **加粗易混点**：RAG解决"不知道"，微调解决"不会做"

## 顺口溜
外脑换脑要分明

## 单字锚点 + 防崩推导
- **锚点字**：**选**
- **扩展口诀**：**外脑换脑要分明**
- **为什么选它**："选"直接对应题目"如何选择"，是决策的核心动作。
- **防崩推导**：
  - 选 → 选择题 → 选外脑还是换脑子
  - 选 → 外脑（RAG）→ 知识放外面，随时换书
  - 选 → 换脑（Fine-tuning）→ 知识刻进脑子，反应快但难改
  - 动态知识/要引用 → 选外脑（RAG）
  - 风格固定/深度推理 → 换脑（微调）

## 一字一画面
- **选** —— 像**考试带不带参考书**：开卷考（RAG）能查资料，闭卷考（微调）凭记忆答题。（本质：外挂知识 vs 内化知识）
- **外** —— 像**背书包去考试**：书包里装最新教材，考完还能换下一科的书。（本质：RAG知识动态更新）
- **换** —— 像**给大脑换芯片**：换完反应变快，但再想换就得重新做手术。（本质：微调改变模型参数，更新成本高）
- **分** —— 像**两条岔路**：左边牌写"知识常变选左边"，右边牌写"风格固定选右边"。（本质：选择策略）

## 面试话术
- "核心区别是**外脑换脑要分明**。RAG是给模型外挂一个动态知识库，适合知识频繁更新、需要给出引用来源的场景；Fine-tuning是把知识刻进模型参数，适合需要改变模型输出风格或做深度领域推理的场景。"
- "实际项目中我们常两者结合：RAG解决实时知识，微调解决特定任务格式。"

## 防混补丁
- 容易和"Prompt Engineering"搞混。RAG是加外部知识，Prompt是改输入指令，微调是改模型参数。补丁：**RAG加书包，Prompt改题目，微调换脑子**。

## 追问预演
- **追问1**：什么情况下两者一起用？→ 领域模型微调后+RAG，既懂领域语言又能查最新资料。
- **追问2**：微调后模型幻觉少了，为什么还需要RAG？→ 微调知识截止于训练数据，新知识还是要靠RAG补。
- **追问3**：RAG成本低，为什么不所有场景都用RAG？→ RAG不改变模型能力，复杂推理任务微调效果更好。
