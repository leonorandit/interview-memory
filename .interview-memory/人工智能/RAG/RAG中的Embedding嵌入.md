# RAG 中的 Embedding 嵌入

## 速记卡片
`向 | 文转向，语义量 | 文本向量化`

## 原始知识
- Embedding（嵌入）：把文本转换成高维向量（稠密向量），捕捉语义信息
- 为什么需要：
  - 计算机不懂文字，只懂数字
  - 向量可以计算语义相似度（余弦相似度、点积）
  - 语义相近的文本，向量距离近
- 特点：稠密、低维（相对one-hot）、语义连续
- **易混点**：Embedding vs One-Hot、Embedding vs 关键词匹配
- **追问点**：Embedding怎么训练？为什么能捕捉语义？

## 顺口溜
**文转向，语义量**

## 单字锚点 + 防崩推导
- **锚点字**：**向**
- **扩展口诀**：**文变向，语义量**
- **为什么选它**：Embedding的核心产出就是"向量"，"向"是整个技术的基石。
- **防崩推导**：
  - 向（向量）→ 文本被转换成高维向量
  - 文（文本）→ 原始输入是文字
  - 转（转换）→ 通过Embedding模型转换
  - 语（语义）→ 向量捕捉了语义信息
  - 义（意义）→ 语义相近的文本向量距离近
  - 量（量化/计算）→ 可以用数学方法计算相似度

## 一字一画面
- **向** —— 像**地图上的坐标**：每个文本在语义空间都有一个坐标。（本质：高维向量表示）
- **文** —— 像**一句话**："我爱北京"。（本质：原始文本）
- **转** —— 像**翻译机**：把中文翻译成机器能懂的数字。（本质：Embedding模型编码）
- **语** —— 像**聚类**："国王"和"女王"坐得近，"苹果"和"香蕉"坐得近。（本质：语义相近向量距离近）
- **量** —— 像**量距离**：用尺子量两个坐标点之间的距离。（本质：余弦相似度计算）

## 面试话术
- 问什么是Embedding → "Embedding是把文本转换成高维向量的技术，口诀是**文转向，语义量**。**文**——原始输入是文字；**转**——通过Embedding模型转换成向量；**向**——每个文本在语义空间有一个坐标；**语义**——向量捕捉了语义信息，语义相近的文本向量距离近；**量**——可以用余弦相似度等数学方法计算文本之间的相似度。这样计算机就能'理解'文本语义，进行检索和匹配。"
- 问和关键词匹配区别 → "关键词匹配只看字面相同，比如'电脑'和'计算机'算不同；Embedding看语义，'电脑'和'计算机'向量距离很近。补丁：**关键词是查字典，Embedding是聊得来**。"

## 防混补丁
- 容易和"One-Hot编码"搞混。One-Hot是高维稀疏、无语义；Embedding是低维稠密、有语义。补丁：**One-Hot是点名册，Embedding是座位图**。

## 追问预演
- **问**：Embedding怎么训练？→ 用大规模语料通过自监督学习（如Skip-gram、Masked Language Model）训练，让上下文相似的词向量相近。
- **问**：为什么能捕捉语义？→ 训练时模型学会了"经常一起出现的词语义相近"，所以向量空间自然地保留了语义关系。
- **问**：向量维度一般是多少？→ 常见768、1024、1536维，维度越高表达能力越强，但存储和计算成本也越高。

## 手撕代码（Embedding伪代码）

```python
from sentence_transformers import SentenceTransformer

# 加载Embedding模型
model = SentenceTransformer('BAAI/bge-large-zh')

# 文本转向量
texts = ["我爱北京", "我喜欢北京", "今天天气很好"]
embeddings = model.encode(texts)

# 计算相似度
from sklearn.metrics.pairwise import cosine_similarity
sim_matrix = cosine_similarity(embeddings)
# sim_matrix[0][1] ≈ 0.95 （"我爱北京"和"我喜欢北京"很相似）
# sim_matrix[0][2] ≈ 0.30 （"我爱北京"和"今天天气很好"不相似）
```
