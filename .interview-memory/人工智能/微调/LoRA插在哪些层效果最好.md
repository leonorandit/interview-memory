# LoRA 插在哪些层效果最好？

## 速记卡片
`Q | 先QV，后Att全，FFN看资源 | 层选优先级`

## 原始知识
- **最优**：q_proj + v_proj（性价比之王，参数少、效果稳）
- **次优**：q_proj + k_proj + v_proj + o_proj（整个 Attention，效果更全面）
- **可选**：加上 FFN 层（up_proj / down_proj / gate_proj），收益有但递减
- **原因**：Q 管查询模式、V 管输出内容，对下游任务最敏感；K 只是索引、O 只是聚合
- **权衡**：FFN 占模型 2/3 参数，插 LoRA 性价比远低于 Attention
- **易混点**：不是冻住哪层，而是**旁路加秩**
- **追问点**：为什么 QV 比 KO 更重要？秩 r 怎么选？

## 顺口溜
**先 QV，后 Att 全，FFN 看资源**

## 单字锚点 + 防崩推导
- **锚点字**：**Q**
- **扩展口诀**：**Q 发问，V 回应**
- **为什么选它**：Q 是 Query，注意力机制的起点，先想起"问问题"就能串起整个链路。
- **防崩推导**：
  - Q → 提问 → 查询模式 → q_proj 最敏感
  - Q + V → 问完要答 → Value 输出内容 → v_proj 第二敏感
  - 四件 → q + k + v + o → Attention 全覆盖 → 效果更稳但参数翻倍
  - FFN → 前馈网络 → 占 2/3 参数 → 收益递减 → **看资源**

## 一字一画面
- **Q** —— 像**球场上举手发问**：球员先举手问球在哪（Query），决定后续所有动作。（q_proj 最敏感）
- **V** —— 像**球网接Value回传**：问完必须接到回传球才能射门。（v_proj 输出内容）
- **Att 全** —— 像**四人联防**：q/k/v/o 四个人把对方前锋围死（整个 Attention），防守更稳但人多耗体力。
- **FFN** —— 像**替补席上堆满人**：多上几个替补有点用，但盒饭预算（参数量）暴涨，性价比低。

## 面试话术
- 问层选择 → "LoRA 插层有口诀：**先 QV，后 Att 全，FFN 看资源**。实践上，只插 Attention 的 q_proj 和 v_proj 性价比最高；如果要更稳，把 q、k、v、o 四个矩阵全插上；FFN 层收益有但递减，而且参数量翻倍，看资源决定是否加。"
- 问原因 → "因为 Q 管查询模式、V 管输出内容，对任务适配最敏感；K 只是索引、O 只是聚合，敏感度低。FFN 占模型 2/3 参数，LoRA 加在 FFN 上性价比远不如 Attention。"

## 防混补丁
- 容易和"全量微调冻住前几层"搞混，也容易和"AdaLoRA 自动选层"搞混。
- 补丁：**LoRA 是旁路加秩，不改原权重；冻层是全量微调的事，AdaLoRA 是它弟弟自动算账。**

## 追问预演
1. **为什么 QV 效果最好，而不是 KO？**
   → Q 控制查询模式（问什么），V 控制输出内容（答什么），两者对任务差异最敏感；K 只是键值索引，O 只是线性聚合，微调收益低。

2. **秩 r 一般选多大？**
   → 小模型/简单任务 8–16，大模型/复杂任务 32–64；r 过大容易过拟合，过小表达能力不足。

3. **把所有线性层都插上 LoRA 会怎样？**
   → 效果通常最好，但参数量接近全量微调的 10%–20%，失去 PEFT"高效"的意义，需要权衡。

4. **DoRA 和 LoRA 在层选择上有区别吗？**
   → DoRA（Weight-Decomposed LoRA）将权重分解为幅度和方向，通常同样优先 Attention 层，但对方向矩阵的微调更精细。
