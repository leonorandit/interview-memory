# PEFT 和全量微调的区别？

## 速记卡片
`量 | 全调参，PEFT省 | 参数量对比`

## 原始知识
- **参数量**：全量调全部；PEFT调<1%
- **计算资源**：全量需多卡A100；PEFT单卡可行
- **显存占用**：全量显存爆炸；PEFT显著降低
- **效果**：全量上限高；PEFT接近90-95%
- **适用场景**：全量数据足+分布差异大；PEFT资源受限+多任务
- **存储成本**：全量每任务存一份大模型；PEFT只存小适配文件
- **易混点**：不是效果绝对优劣，是资源与效果的权衡

## 顺口溜
**全调参，PEFT省** —— *全掉参，泼福特省*

## 单字锚点 + 防崩推导
- **锚点字**：**量**
- **扩展口诀**：**量比质，看场景**
- **为什么选它**：一切差异的根源是"量"——参数量、数据量、资源量、存储量。
- **防崩推导**：
  - 量 → 参数量 → 全量调全部参数；PEFT调少量参数 → 训练成本差几十倍
  - 量 → 计算量 → 全量反向传播更新全部权重 → 显存和算力需求巨大
  - 量 → 存储量 → 全量每个任务一份完整模型；PEFT一份基座+多个小适配器
  - 质 → 效果质量 → 全量上限更高；PEFT快速够用
  - 场景 → 资源少选PEFT，追求极限选全量

## 一字一画面
- **量** —— 像**菜市场称重**：全量是一整车白菜过秤，PEFT是只称一颗白菜。（本质：参数量差异决定一切成本差异）
- **全** —— 像**全屋翻新**：砸墙、改水电、换地板，动静大、花钱多。（本质：所有参数都参与更新）
- **省** —— 像**只换窗帘和沙发套**：房间格局不变，换个风格就能住。（本质：基座不动，只改少量适配参数）
- **质** —— 像**精装房vs软装房**：精装房（全量）从里到外定制；软装房（PEFT）拎包入住，效果也不差。（本质：效果差距不大，但成本差很大）

## 面试话术
- 问区别 → "核心区别在**量**字上：全量微调是把模型所有参数都打开训练，参数量大、显存占用高、每任务都要存完整模型；PEFT是冻结大部分参数，只微调少量参数或插入适配模块，**省卡、省时间、省存储**。"
- 问选择 → "实际选型看资源：如果预算充足、数据量大、任务和预训练分布差异大，全量微调上限更高；如果是中小团队、多任务切换、快速验证，PEFT是更务实的选择，效果通常能达到全量的90%以上。"

## 防混补丁
- 容易认为"全量一定比PEFT好"。补丁：**全量上限高，PEFT性价比高，没有绝对好坏，只有场景匹配**。

## 追问预演
1. **Q：PEFT能不能达到全量微调的效果？**
   A：大部分常规任务可以接近，但极端分布偏移或需要模型能力大幅重构时，全量上限更高。
2. **Q：显存节省大概多少？**
   A：LoRA通常可节省30-50%甚至更多，具体取决于秩和模型大小；Adapter也类似。
3. **Q：如果我有8张A100，是不是一定选全量？**
   A：不一定。如果要做10个任务的快速迭代，PEFT可以在单卡上并行跑多个实验，整体迭代效率反而更高。
