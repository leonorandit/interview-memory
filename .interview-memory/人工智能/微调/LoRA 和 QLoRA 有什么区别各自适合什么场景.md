## 速记卡片
`量 | 量化降存，显存减半 | QLoRA=4bit+LoRA`

## 原始知识
- **LoRA**：16/32bit 基座 + 低秩适配
- **QLoRA**：4bit 量化基座 + LoRA
- **QLoRA 用 NF4/FP4 量化**
- **分页优化器 (Paged Optimizer)** 防 OOM
- **双量化**：量化常数也量化
- **LoRA 适合**：显存充裕、追求速度
- **QLoRA 适合**：显存紧张、大模型微调

## 顺口溜
量化降存，显存减半

## 单字锚点 + 防崩推导
- **锚点字**：量
- **扩展口诀**：量化降存，显存减半，NF4 精度，分页防崩
- **为什么选它**："量"=量化，QLoRA 和 LoRA 的最大区别就是"量化"，4bit 存储是核心差异点
- **防崩推导**：量 → 量化 → 4bit → 显存降 70%+ → 但精度损失 → 用 NF4（正态分布友好）→ 双量化进一步省 → 分页优化器防 OOM → 适合大模型单卡微调

## 一字一画面
**量** —— 像**电子秤压缩**：蓝色电子秤（量）上放着 16 个金色砝码（16bit），QLoRA 用魔法手一压，变成 4 个银色小砝码（4bit），重量（显存）骤减，但秤的精度（NF4）保持不乱，旁边有绿色分页纸（Paged Optimizer）托底——本质：量化压缩基座，精度可控

## 面试话术
LoRA 和 QLoRA 的区别就在于基座模型的精度。LoRA 用 fp16 或 bf16 加载基座模型，QLoRA 用 4bit 量化加载，显存占用直接降几倍。QLoRA 用了 NF4 量化和分页优化器，训练稳定性还不错。如果你显存够，LoRA 训练更快；如果显存不够想训大模型，QLoRA 是救星。

## 防混补丁
- QLoRA 训练时 4bit，推理建议反量化到 16bit
- LoRA 权重本身还是 fp16/bf16
- 双量化是量化 scale 值，不是权重再量化
- 分页优化器把 optimizer 状态放 CPU，需要时换入

## 追问预演
Q: NF4 和 FP4 有什么区别？
A: NF4 假设权重服从正态分布，做分位数量化，精度通常比均匀量化的 FP4 更好。

Q: QLoRA 训练会比 LoRA 慢多少？
A: 通常慢 20%~50%，因为 4bit 计算需要反量化，但显存省很多，能在更小的卡上跑。
