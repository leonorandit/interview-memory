# 大模型微调的方案有哪些？

## 速记卡片
`五 | 全低指强继 | 五类方案`

## 原始知识
- **全**（Full Fine-tuning）：全量微调，更新所有参数，效果天花板但成本最高，易灾难性遗忘
- **低**（PEFT / LoRA）：参数高效微调，以 LoRA 为代表，冻结原权重只训低秩矩阵，性价比高
- **指**（SFT / Instruction Tuning）：指令微调，在有标注的指令-回答数据上监督学习，让模型听懂人话
- **强**（RLHF / DPO）：强化学习对齐，RLHF（PPO）或 DPO，基于人类偏好优化模型输出
- **继**（Continual Pre-training）：继续预训练，在领域无标注数据上继续训练，深耕垂直知识
- 常见组合：**预训练 → SFT → RLHF/DPO** 三阶段流水线
- **易混点**：预训练 vs 微调、SFT vs RLHF、LoRA vs 全量
- **追问点**：什么时候选哪种方案、RLHF 三阶段、DPO 优势、灾难性遗忘

## 顺口溜
**全低指强继**

## 单字锚点 + 防崩推导
- **锚点字**：**五**
- **扩展口诀**：**五方案，全低指强继**
- **为什么选它**：题目问"有哪些方案"，"五"直接给出数量锚点，防止漏项。
- **防崩推导**：
  - 五 → 五类主流方案
  - 全 → 全量微调（Full Fine-tuning），全部参数更新
  - 低 → PEFT / LoRA 低秩适配，只训少量参数
  - 指 → 指令微调 SFT，监督学习让模型遵循指令
  - 强 → RLHF / DPO，强化学习对齐人类偏好
  - 继 → 继续预训练，在领域数据上再训练

## 一字一画面
- **五** —— 像**五指山**：五根手指压住五行方案，缺一不可。（本质：五类方案）
- **全** —— 像**全车改装**：所有零件都换，效果最强但最贵。（本质：全量微调）
- **低** —— 像**贴膜改色**：只贴一层膜，车还是那辆车，便宜又好看。（本质：LoRA/PEFT）
- **指** —— 像**指挥官下命令**：士兵只听指令行动，指哪打哪。（本质：指令微调 SFT）
- **强** —— 像**强化训练营**：根据学员表现奖优罚劣，塑造最佳行为。（本质：RLHF/DPO 偏好对齐）
- **继** —— 像**继续教育**：大学毕业后再读研究生，深耕专业领域。（本质：继续预训练）

## 面试话术
- 问方案 → "大模型微调方案我总结为五类，口诀是**全低指强继**。**全**——全量微调，更新所有参数，效果天花板但成本最高；**低**——PEFT 以 LoRA 为代表，只训少量参数，性价比最高；**指**——指令微调 SFT，让模型听懂人话；**强**——RLHF 或 DPO，对齐人类偏好；**继**——继续预训练，在领域数据上深耕。实际工程里常用三阶段流水线：先继续预训练注入领域知识，再 SFT 学指令，最后 DPO 对齐偏好。"
- 问怎么选 → "资源充足、数据量大、追求极致效果选全量；资源有限、快速迭代选 LoRA；有高质量指令数据先做 SFT；需要对齐人类价值观再上 RLHF/DPO；垂直领域知识缺失则先继续预训练。"

## 防混补丁
- 容易把"预训练"和"微调"搞混。预训练是学通用知识（无标注），微调是学特定任务（有标注）。补丁：**先预训练学说话，后微调学听话**。
- 容易把 SFT 和 RLHF 搞混。SFT 是教模型"怎么做"，RLHF 是教模型"什么更好"。补丁：**SFT教做法，RLHF教审美**。
- 容易觉得"LoRA 不如 Full"。其实 LoRA 在很多场景能达到 95%+ 效果。补丁：**LoRA是性价比之王**。

## 追问预演
1. **全量微调和 LoRA 怎么选？**
   → 数据量大、算力足、任务复杂、追求极致效果选全量；资源有限、快速迭代、参数敏感选 LoRA。7B 模型全量需 80G+ 显存，LoRA 只需 16G 左右。
2. **RLHF 的三个阶段是什么？**
   → ① SFT 有监督微调 → ② Reward Model 训练奖励模型 → ③ PPO 用强化学习优化策略。DPO 则跳过 RM 和 PPO，直接用偏好数据优化。
3. **DPO 相比 RLHF-PPO 的优势？**
   → DPO（直接偏好优化）不需要训练单独的 Reward Model，也不需要强化学习，训练更稳定、实现更简单、显存占用更低。
4. **继续预训练和微调的区别？**
   → 继续预训练是在无标注领域数据上继续自监督训练（Next Token Prediction），学习领域知识分布；微调是在有标注数据上监督学习，学习输入到输出的映射。
5. **什么场景下需要微调而不是 Prompt Engineering？**
   → 需要稳定输出特定格式、涉及私有数据且不能放 prompt、任务复杂超出上下文容量、对延迟敏感需避免长 prompt、需要模型掌握领域术语和知识。
6. **灾难性遗忘怎么解决？**
   → ① 冻结底层只调顶层；② 使用 LoRA/Adapter 等 PEFT 方法减少参数变动；③ 混合原始数据进行回放（Replay）；④ 使用 EWC 等正则化方法约束重要参数。
