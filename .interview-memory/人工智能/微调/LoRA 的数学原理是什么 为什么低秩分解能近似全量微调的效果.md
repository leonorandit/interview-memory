## 速记卡片

| 锚点字 | 口诀 | 核心差异 |
|--------|------|----------|
| 近 | 近冻低，三原理 | 近似全量、冻结原权、低秩微调 |

## 原始知识

- 预训练权重 **W₀ ∈ R^{d×k}**
- 微调后 **W = W₀ + ΔW**
- LoRA **假设 ΔW 是低秩矩阵**
- 公式：**ΔW = BA**
- **B ∈ R^{d×r}**，**A ∈ R^{r×k}**
- **r << min(d, k)**
- 预训练模型已学到**丰富特征表示**
- 微调只需在**特定方向**做微小调整
- 这些调整集中在**低维子空间**
- 低秩矩阵足以捕捉**任务相关变化**
- 大量实验证明 **r=8 或 16** 效果接近全量微调

## 顺口溜

原权冻结不动，低秩乘积来补，子空间微调，近似全量路。

## 单字锚点 + 防崩推导

- **锚点字**：近
- **扩展口诀**：近冻低，三原理
- **为什么选它**："近"字直接回答面试题的核心——为什么低秩能"近似"全量微调效果，是因果链的终点也是记忆抓手。
- **推导链**：
  1. 近 → 近似全量微调效果
  2. 冻 → 冻结原权重 W₀，保留预训练知识
  3. 低 → 用低秩矩阵 BA 替代 ΔW
  4. 为什么能近？→ 预训练模型已学通用表示，微调只需在低维子空间做小调整
  5. r=8/16 实验证明近似度极高

## 一字一画面

**近** —— 像一架**黄色遥控飞机绕着灰色摩天大楼微调航向**：大楼主体（W₀）纹丝不动，飞机只在几条狭窄的航线（低维子空间）上轻微修正角度，最终目的地和拆了重建（全量微调）几乎一样（本质：大结构已定，小修小补就能达到相近效果）。

## 面试话术

"LoRA 的数学原理是把权重的更新 ΔW 假设为低秩矩阵，分解成两个小矩阵 BA 的乘积，其中 r 远小于原始维度。因为预训练模型已经学到了非常丰富的通用特征表示，下游微调往往只需要在少数几个关键方向上做微小调整，这些调整恰好集中在一个低维子空间里，所以低秩矩阵就能很好地近似全量微调的效果。"

## 防混补丁

- **容易和什么搞混**：认为低秩分解会丢失信息（实际不是压缩 W 本身，而是压缩 ΔW；W₀ 完整保留，只限制"变化量"的规模）。
- **补丁口诀**：原权不动完整存，只压变化不压身；低秩管的是增量，不是全身打绷带。

## 追问预演

| 追问 | 简要答案 |
|------|----------|
| 低秩假设的理论依据是什么？ | 预训练模型过参数化，实际有效自由度低；微调所需的变化矩阵本征值衰减快，主要信息集中在前几个奇异值。 |
| r 太大或太小会怎样？ | r 太小欠拟合，任务相关方向不够用；r 太大冗余，参数量上升但收益递减。 |
| LoRA 能逼近任意 ΔW 吗？ | 如果 ΔW 恰好是高秩的，LoRA 会有偏差；但实践和理论都表明预训练模型的 ΔW 通常是低秩的。 |
| 和 SVD 有什么关系？ | LoRA 可看作对 ΔW 做隐式低秩近似，但不用显式计算 SVD，而是端到端学习 BA。 |

## 关联知识点

- 预训练-微调范式
- 隐式正则化
- 本征值分布与有效秩
