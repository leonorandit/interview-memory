## 速记卡片

| 锚点字 | 口诀 | 核心差异 |
|--------|------|----------|
| 秩 | 秩冻适，三原理 | 冻权重、低秩分解、适配任务 |

## 原始知识

- **LoRA** = Low-Rank Adaptation（低秩适配）
- **PEFT** 参数高效微调方法之一
- **冻结**预训练权重 W
- 只训练**低秩分解**矩阵 A 和 B
- 公式：**W' = W + BA**
- A 是 d×r 矩阵，B 是 r×d 矩阵
- **r 远小于 d**
- 参数量从 d×d 降到 **2×d×r**
- r=8 时参数量**减少 99%** 以上
- 同时**保持模型容量**

## 顺口溜

冻结原矩阵，低秩来适配，BA相乘加，参数省百倍。

## 单字锚点 + 防崩推导

- **锚点字**：秩
- **扩展口诀**：秩冻适，三原理
- **为什么选它**："秩"是 LoRA 最核心的数学概念，低秩分解是整个方法的灵魂，一念"秩"字就能唤起整套机制。
- **推导链**：
  1. 秩 → 低秩分解 → 把大矩阵拆成两个小矩阵相乘
  2. 冻 → 冻结预训练权重 W，不改原始知识
  3. 适 → 只训练 BA 去适配下游任务
  4. 省参数 → 原来是 d×d，现在是 2×d×r，r=8 省 99%

## 一字一画面

**秩** —— 像一条**瘦长的红色水管**穿过巨大的灰色仓库：两个细长水管（A和B）对接，就能替代整座仓库的管道系统（本质：用低维结构表达高维变化）。

## 面试话术

"LoRA 的核心思路是冻结预训练权重，只引入两个低秩矩阵 A 和 B 做微调。因为微调时模型不需要在所有方向上都改变，只需要在少数关键方向上调整，所以低秩分解就能近似全量微调的效果，参数量还能减少 99% 以上。"

## 防混补丁

- **容易和什么搞混**：Adapter（Adapter 是在网络层之间插入小模块，LoRA 是直接修改权重矩阵；Adapter 增加网络深度，LoRA 不改变网络结构）。
- **补丁口诀**：Adapter插模块，LoRA改权重；冻结同相似，原理两扇门。

## 追问预演

| 追问 | 简要答案 |
|------|----------|
| LoRA 和 Adapter 有什么区别？ | LoRA 是直接对权重矩阵做低秩分解，Adapter 是在层间插入瓶颈结构；LoRA 推理时可以合并回原始权重，无额外 latency。 |
| r 一般取多少？ | 通常 8 或 16，任务复杂或矩阵维度大时可取 32 或 64。 |
| LoRA 适用于哪些层？ | 一般只对 Attention 的 Q、V 投影矩阵加 LoRA，也可对全部线性层加。 |
| 为什么冻结 W 只训 BA 不会欠拟合？ | 预训练模型已学到通用表示，微调只需在低维子空间做任务相关调整，低秩矩阵足够捕捉这些变化。 |
