# 微调有哪些方案 你做过吗

## 速记卡片
`全 | 全Lo适，省冻调 | 全量vs参数高效`

## 原始知识
- **全量微调**：所有参数更新，效果最强，需要大量数据和算力（7B 模型约需 160GB 显存）
- **LoRA**：低秩适配，主流 PEFT 方案。冻结预训练权重，只训练低秩矩阵 ΔW=BA，参数量从 d×d 降到 r×(d+d)
- **QLoRA**：LoRA + 4-bit 量化（NF4）+ 分页优化器，16GB 显存可跑 70B 模型 LoRA 微调
- **Adapter Tuning**：在 Transformer 层间插入小型适配层，只训练适配层参数
- **Prefix Tuning**：冻结模型，只训练输入前缀嵌入（prefix embeddings）
- **Prompt Tuning / P-tuning**：只优化软提示（soft prompt）嵌入
- **IA3**：学习缩放向量，对 K/V/FFN 进行逐元素缩放
- **共同思路**：冻结大部分，只调一小部分 → 省参数、省显存、省时间
- **实战经验**：
  - 数据质量 > 数量，先清洗去重
  - 学习率 1e-4 ~ 1e-5，warmup 必备
  - epochs 别太多，防过拟合
  - 混合通用数据防灾难性遗忘
  - 评估：loss 曲线 + 自动指标 + 人工抽样
- **易混点**：LoRA 不是量化，QLoRA 才是量化+LoRA
- **追问点**：LoRA 为什么有效？怎么防灾难性遗忘？

## 顺口溜
**全Lo适，省冻调**

## 单字锚点 + 防崩推导
- **锚点字**：**全**
- **扩展口诀**：**全Lo适**
- **为什么选它**：微调方案从"全不全"开始分岔——全量最强但最贵，PEFT 省资源是主流。先想"全不全"。
- **防崩推导**：
  - 全 → 全量微调（数据足、算力够时效果最好）
  - Lo → LoRA（低秩适配，只训 BA，主流首选）
  - 适 → Adapter Tuning（插入适配层）
  - 省 → PEFT 整体优势（省参数、省显存、省时间）
  - 冻 → 冻结底层参数（防灾难性遗忘）
  - 调 → Prefix / Prompt / P-tuning（只调输入端前缀）

## 一字一画面
- **全** —— 像**全屋装修**：所有参数都动，效果最好但最贵。（全量微调：所有参数更新，效果最强，7B 需约 160GB 显存）
- **Lo** —— 像**贴壁纸**：不改房子结构，只贴一层装饰，便宜又出效果。（LoRA：低秩矩阵注入，只训练 BA，r 很小如 8/16/64）
- **适** —— 像**插线板**：在原有插座上插一个适配器，不用改墙内线路。（Adapter Tuning：层间插入小型适配层）
- **省** —— 像**节能灯**：亮度够但耗电少。（PEFT 共同优势：省参数、省显存、省时间）
- **冻** —— 像**冷冻库**：底层参数冻住不动，只调顶层。（冻结层：保护预训练知识，防止灾难性遗忘）
- **调** —— 像**调频收音机**：不改装机器，只调调频按钮。（Prefix/Prompt Tuning：只调输入端前缀嵌入）

## 面试话术
- 问方案 → "微调方案口诀是**全Lo适，省冻调**。全量微调效果最强但最贵；LoRA 是主流 PEFT 方案，低秩适配省参数；Adapter Tuning 插入适配层；Prefix/Prompt Tuning 只调输入前缀。核心思路是**冻结大部分，只调一小部分**，既保留预训练能力又适应下游任务。"
- 问经验 → "实际项目中，数据质量比数量更重要，先清洗去重再调。7B 模型全量需约 160GB 显存，LoRA 用 16GB 就能跑，QLoRA 甚至能在 16GB 跑 70B。学习率通常 1e-4 到 1e-5，warmup 必备，epochs 别太多防过拟合。评估除了看 loss，一定要做人工抽样和自动化指标对比。"

## 防混补丁
- 容易把 LoRA 当成量化方法。补丁：**LoRA 是低秩适配，QLoRA 才是量化+LoRA**。
- 容易以为 Prefix Tuning 和 Prompt Tuning 一样。补丁：**Prefix 调前缀嵌入+MLP，Prompt 只调软提示**，前者更复杂效果更稳。

## 追问预演
1. **LoRA 为什么有效？**
   → 预训练权重 W 有冗余，下游任务只需低秩变化 ΔW = BA。A 随机初始化，B 零初始化，训练时只更新 BA，参数量从 d×d 降到 r×(d+d)，r 很小（如 8、16、64）。本质是在低维子空间里学习适配。
2. **QLoRA 和 LoRA 区别？**
   → QLoRA 在 LoRA 基础上，把基模型量化为 4-bit（NF4），用双量化进一步压缩，分页优化器管理显存。同样 16GB 显存可以跑 70B 模型 LoRA 微调。
3. **怎么防止灾难性遗忘？**
   → 冻结底层参数；混合预训练数据一起训练；使用 ORPO（把 SFT 和 DPO 合二为一）；设置较小的学习率。
4. **实际微调数据怎么准备？**
   → 清洗（去重、去低质量）、格式化（统一 instruction-response 格式）、配比（领域数据 : 通用数据 ≈ 1:1 到 1:3）、构造多样性（改写、扩展）。
5. **LoRA 的 r 和 alpha 怎么选？**
   → r 是低秩维度，常见 8/16/32/64，越大表达能力越强但参数越多。alpha 是缩放因子，通常 alpha = 2r。任务越复杂 r 越大。
