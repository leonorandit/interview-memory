## 原始知识

- **指令设计**：多样化指令模板，覆盖不同意图
- **数据采集**：人工标注 + 模型生成 + 人工审核
- **质量清洗**：去重、过滤低质量、格式统一
- **数据配比**：不同任务按比例混合
- **数据量与质量权衡**：质大于量，10K 好数据 > 1M 差数据
- **指令多样性**：同一任务多种问法，防过拟合
- **输入输出格式**：统一模板，方便模型学习
- **安全过滤**：去毒、去隐私、去偏见

## 单字锚点 + 防崩推导

**锚点字：构**

- 扩展口诀：**构**建数据，**构**的是指令微调的弹药库
- 为什么选它：题目核心就是"如何构建"，"构"字直接扣题
- 推导链：
  1. 构 → 构什么？→ 指令微调数据集（input + instruction + output）
  2. 构 → 第一步？→ 指令设计（多样化模板，覆盖全场景）
  3. 构 → 第二步？→ 数据采集（人工标注 + 模型生成 + 审核）
  4. 构 → 第三步？→ 质量清洗（去重、过滤、格式统一）
  5. 构 → 第四步？→ 数据配比（任务均衡，质>量）

## 一字一画面

**构** —— 像厨师备菜，先列菜单（指令设计），再买菜（数据采集），洗菜切菜（质量清洗），最后按菜单比例摆盘（数据配比），一道菜才能上桌。（本质：构建数据集是系统工程，每一步都影响最终口感。）

**指** —— 像交通指示牌，同一个路口，有人看到"直行"、有人看到"往前开"、有人看到"Go Straight"，意思一样但说法多样。（本质：指令多样化防止模型只认固定句式。）

**清** —— 像淘金，河床里全是沙子和金粒，用筛子筛掉沙子，留下的才是金子。（本质：清洗就是去重、过滤垃圾、纠正错误。）

**配** —— 像调鸡尾酒，橙汁太多就是橙汁，伏特加太多就上头，比例对了才是 Mojito。（本质：不同任务数据要均衡，防止模型偏科。）

## 面试话术

"构建 SFT 数据我分四步走：第一是指令设计，同一个任务要有多种问法，防止模型过拟合固定模板；第二是数据采集，人工标注质量高但慢，可以用模型生成加人工审核来扩量；第三是质量清洗，去重、过滤低质量、统一格式，这一步特别关键；第四是数据配比，不同任务要均衡，宁要一万条好数据不要一百万条脏数据。"

## 防混补丁

容易和"预训练数据构建"搞混？一句补丁：

> **预训练是喂 raw 语料让模型学语言，SFT 是喂指令对让模型学听话。**

（预训练：无监督，next token prediction；SFT：有监督，instruction → response。）

## 追问预演

1. **Q：模型生成数据怎么保证质量？**
   A：用强模型（如 GPT-4）生成，再经过人工或规则审核，也可以用多模型投票、self-consistency 来过滤低质量样本。

2. **Q：数据配比具体怎么调？**
   A：通常按任务重要性或数据量来加权，欠采样多的任务、过采样少的任务，也可以用课程学习逐步增加难度。

3. **Q：格式统一有多重要？**
   A：非常重要，模型对格式敏感，ChatML、Alpaca、ShareGPT 各种格式差别很大，训练时必须统一，否则模型学不到规律。
