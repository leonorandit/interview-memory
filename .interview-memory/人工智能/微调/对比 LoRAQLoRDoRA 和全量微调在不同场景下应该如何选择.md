## 速记卡片
`选 | 显小选Q，精准选D | 四者按显存和精度取舍`

## 原始知识
- **LoRA**：低秩适配，省显存，推理可合并
- **QLoRA**：4bit量化+LoRA，显存最小
- **DoRA**：权重分解 LoRA，精度更高
- **全量微调**：更新全部参数，效果上限最高
- **显存充裕+追求极致效果 → 全量**
- **显存紧张+快速实验 → QLoRA**
- **要精度但不想全量 → DoRA**
- **平衡之选 → LoRA**

## 顺口溜
显小选Q，精准选D

## 单字锚点 + 防崩推导
- **锚点字**：选
- **扩展口诀**：显小选Q，精准选D，平衡选L，土豪选全
- **为什么选它**："选"=选择，题目核心就是"在不同场景下如何选择"，直接对应考点
- **防崩推导**：选 → 选择题 → 看条件（显存/精度/速度）→ 显存小 → QLoRA（4bit+LoRA）→ 要精度 → DoRA（分解权重）→ 要平衡 → LoRA → 资源无限 → 全量

## 一字一画面
**选** —— 像**四岔路口**：黄色路牌上写着"选"，四条路分别通向——LoRA（绿色小路，省劲）、QLoRA（蓝色隧道，最窄但能通过）、DoRA（红色大桥，最稳）、全量（金色大道，最宽但费油）——本质：根据资源和目标选路

## 面试话术
如果显存特别紧张，比如单卡 24G 想训 70B 模型，首选 QLoRA。如果追求最好效果且显存管够，那就全量微调。LoRA 是性价比最高的中间方案，DoRA 比 LoRA 精度更高一点，但训练慢一些，适合对精度敏感又不想全量的场景。

## 防混补丁
- QLoRA 推理时可以用 16bit，训练用 4bit
- DoRA 不是替代 LoRA，是改进版
- 全量微调不一定比 LoRA 好，数据少时 LoRA 更稳
- 合并权重：LoRA/DoRA 可以合，QLoRA 合后需反量化

## 追问预演
Q: QLoRA 为什么能省这么多显存？
A: 基座模型用 4bit 量化存储，只训 LoRA 的少量参数，Optimizer 状态也大幅减少。

Q: DoRA 比 LoRA 强在哪？
A: DoRA 把权重分解成幅度和方向，只微调方向部分，数学上更稳定，精度通常更好。
