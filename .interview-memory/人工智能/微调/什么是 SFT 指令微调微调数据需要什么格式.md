## 速记卡片
`指 | 指令成对，问答分明 | SFT=监督+指令格式`

## 原始知识
- SFT=Supervised Fine-Tuning
- 用指令-回答对训练模型
- 数据格式：instruction + input + output
- **只训练回答部分**
- ** instruction 和 input 用 mask 遮住**
- 损失函数：回答部分的交叉熵
- 数据质量 > 数据数量

## 顺口溜
指哪打哪，问答成对

## 单字锚点 + 防崩推导
- **锚点字**：指
- **扩展口诀**：指令成对，问答分明，mask遮头，只算尾 loss
- **为什么选它**："指"=指令(Instruction)，一听就联想到 SFT 的核心是"指令跟随"
- **防崩推导**：指 → 指令 → 必须有问有答 → 数据成对出现 → input/output 格式 → 只对 output 算 loss

## 一字一画面
**指** —— 像**指挥棒**：指挥家手持银色指挥棒（指）向乐团，左边是乐谱（指令），右边是演奏（回答），棒尖只点在演奏区发光，乐谱区灰暗——本质：只对回答部分计算损失

## 面试话术
SFT 就是监督微调，核心是让模型学会"听懂指令并给出正确回答"。数据格式一般是 instruction-input-output 三元组，训练时把 instruction 和 input 部分 mask 掉，只让模型对 output 部分做 next-token prediction。

## 防混补丁
- SFT vs 预训练：预训练是无监督的，SFT 是有监督的；预训练学知识，SFT 学格式
- SFT vs RLHF：SFT 是"教答案"，RLHF 是"教偏好"
- mask 容易忘：记住"问不算，答才算"

## 追问预演
Q: 为什么只对 output 算 loss？
A: 因为 input 部分模型已经会了，我们不想改变模型对指令的理解，只想优化它生成回答的能力。

Q: SFT 数据量一般多大？
A: 通常几万到几十万条高质量数据就够了，质量远比数量重要。
