# 在进行 Fine-Tuning 时，如何选择适合的预训练模型？

## 速记卡片
`匹 | 任规数，三匹配 | 匹配任务`

## 原始知识
- **任务匹配**：NLU选BERT系列，NLG选GPT系列，多模态选CLIP/BLIP
- **规模匹配**：小数据配小模型，大数据配大模型，避免过拟合/欠拟合
- **语言/领域匹配**：中文选ChatGLM/Qwen，法律选Lawyer-LLaMA，医疗选Huatuo
- **开源生态**：社区活跃、文档完善、工具链成熟（Transformers、vLLM等）
- **许可协议**：商用需关注Apache/MIT vs 学术限制协议
- **易混点**：不是越大越好，匹配度比规模更重要

## 顺口溜
**任规数，三匹配** —— *任龟数，三匹配*

## 单字锚点 + 防崩推导
- **锚点字**：**匹**
- **扩展口诀**：**匹配好，事半功倍**
- **为什么选它**：选预训练模型的本质就是"匹配"——任务、规模、领域三维匹配。
- **防崩推导**：
  - 匹 → 任务匹配 → NLU/N LG/多模态 → 选对模型家族
  - 匹 → 规模匹配 → 模型大小 vs 数据量 → 小数据配大模型会过拟合
  - 匹 → 领域匹配 → 中文/法律/医疗 → 选对语料预训练的模型
  - 规 → 开源生态和规范 → 文档、工具链、社区支持
  - 数 → 数据规模 → 决定选多大模型

## 一字一画面
- **匹** —— 像**给马配马鞍**：赛马配轻鞍，驮货配重鞍，不匹配人马俱疲。（本质：模型和任务必须匹配）
- **任** —— 像**选武器出征**：攻城选大炮，刺选配匕首。（本质：NLU选BERT，NLG选GPT，任务决定选型）
- **规** —— 像**看地图导航**：选有清晰路标（文档）和加油站（社区）的路线。（本质：开源生态和工具链支持）
- **数** —— 像**量体裁衣**：矮子穿大衣（小数据大模型）拖地，胖子穿童装（大数据小模型）绷裂。（本质：模型规模与数据量匹配）

## 面试话术
- 问选型 → "选预训练模型我总结为三个字：**匹、规、数**。先**匹配任务**，NLU选BERT系，生成选GPT系；再**匹配规模**，数据量小别硬上大模型，容易过拟合；最后**匹配领域**，中文任务选中文预训练的模型，垂直领域优先选领域预训练模型。"
- 问补充 → "另外还要看开源生态和许可协议，社区活跃的模型坑少工具多，商用一定要避开学术限制协议，否则上线就有法律风险。"

## 防混补丁
- 容易"唯参数论"，认为参数量越大越好。补丁：**大数据配大模型，小数据配小模型，不匹配就是浪费**。

## 追问预演
1. **Q：只有1万条数据，选多大的模型？**
   A：优先选1B-7B的小模型或中等模型，配合PEFT方法，降低过拟合风险。
2. **Q：垂直领域没有专用预训练模型怎么办？**
   A：先用通用模型做继续预训练（Continual Pre-training），再做微调。
3. **Q：怎么评估预训练模型和任务的匹配度？**
   A：快速跑零样本或少样本评测，看基线水平；同时对比同规模不同模型的领域语料覆盖度。
