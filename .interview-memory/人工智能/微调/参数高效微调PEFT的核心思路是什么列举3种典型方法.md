# 参数高效微调（PEFT）的核心思路是什么？列举 3 种典型方法

## 速记卡片
`冻 | 冻大调小，插适配 | 冻结预训练参数`

## 原始知识
- **核心思路**：冻结预训练参数，只微调少量参数/插入适配模块
- **冻结大模型**： backbone 权重不动，保留预训练知识
- **微调小模块**：只训练新增或选定的少量参数
- **方法一：LoRA** —— 低秩适配，冻结W，学ΔW = BA
- **方法二：Adapter** —— 在层间插入小型全连接瓶颈模块
- **方法三：Prefix/Prompt Tuning** —— 在输入或隐藏层加可训练前缀/提示
- **易混点**：Prefix Tuning 和 Prompt Tuning 区别在层级别

## 顺口溜
**冻大调小，插适配** —— *冻大调小，差适配*

## 单字锚点 + 防崩推导
- **锚点字**：**冻**
- **扩展口诀**：**冻大调小，三方法**
- **为什么选它**：PEFT的操作灵魂是"冻"——把大模型冻住，别让它乱动。
- **防崩推导**：
  - 冻 → 冻结大部分参数 → backbone不动 → 防灾难性遗忘
  - 冻 → 调小 → 只调少量参数/小模块 → LoRA、Adapter、Prefix Tuning
  - 插 → 插入适配模块 → Adapter在层间插瓶颈层；LoRA在旁路插低秩矩阵
  - 适配 → 适配下游任务 → 各方法都是给大模型"戴不同款式的眼镜"

## 一字一画面
- **冻** —— 像**把大模型冻成蓝色冰块**：外表冰冷坚硬（参数不动），内部知识完好保存。（本质：冻结预训练权重）
- **大** —— 像**一座巨型冰山**：体量巨大，搬不动也融不起。（本质：预训练模型参数量巨大）
- **小** —— 像**冰块上挂的小暖炉**：只加热几个小点，就能让整体改变用途。（本质：只微调少量参数）
- **插** —— 像**在耳机线中间加转接头**：不改变手机和耳机本身，插入一个小模块实现适配。（本质：Adapter层间插入）
- **适配** —— 像**给冰山装上不同颜色的镜片**：红色镜片看过去是火焰世界，蓝色镜片是海洋世界，冰山本身没变。（本质：不同下游任务用不同适配参数）

## 面试话术
- 问核心思路 → "PEFT的核心思路可以概括为五个字：**冻大调小，插适配**。就是把预训练大模型的参数冻住，不动它，然后在旁边插一些小型适配模块或者只微调少量参数，让大模型快速适配下游任务。"
- 问三种方法 → "典型三种方法：一是**LoRA**，在原始权重旁加低秩矩阵做近似更新；二是**Adapter**，在Transformer层之间插入小型瓶颈网络；三是**Prefix Tuning或Prompt Tuning**，在输入或隐藏层前面加可训练的前缀向量。"

## 防混补丁
- 容易把"冻结"理解成"不用预训练模型"。补丁：**冻是保鲜不是扔掉**，冻住才能保留预训练知识。

## 追问预演
1. **Q：LoRA为什么用低秩分解？**
   A：实际微调中权重的有效更新往往是低秩的，用两个小矩阵BA近似ΔW，参数量从d×d降到d×r + r×d，r远小于d。
2. **Q：Adapter和LoRA哪个更省参？**
   A：LoRA通常更省，Adapter有额外前向计算开销；但Adapter不修改原结构，模块化更好。
3. **Q：Prefix Tuning和Prompt Tuning区别？**
   A：Prompt Tuning只在输入嵌入层加软提示；Prefix Tuning在每一层的key/value前都加可训练前缀，表达能力更强。
