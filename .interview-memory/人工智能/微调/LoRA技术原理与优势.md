# LoRA 技术原理与优势

## 速记卡片
`冻 | 冻秩快省插，原底不冻坏 | 冻结+低秩`

## 原始知识
- **冻**：冻结预训练权重，只训新增低秩矩阵 A、B
- **秩**：低秩分解（r≪d），用极少参数逼近权重变化 ΔW = BA
- **快**：参数量少 → 计算量小 → 训练迭代快
- **省**：梯度/优化器状态只存小矩阵 → 显存大幅降低
- **插**：适配器可分离/可合并 → 多任务灵活切换
- **不遗忘**：原权重不变 → 预训练知识完整保留
- **兼容强**：可与 4-bit 量化结合（QLoRA），进一步省显存

## 顺口溜
**冻秩快省插** —— *冬天织块省布插*

## 单字锚点 + 防崩推导
- **锚点字**：**冻**
- **扩展口诀**：**冻住原，插新枝**
- **为什么选它**：LoRA 的灵魂操作就是"冻结预训练权重"，一切优势都从这里长出来。
- **防崩推导**：
  - 冻 → 原权重不变 → **预训练知识不丢**（防灾难性遗忘）
  - 冻 → 只训新增 BA 小矩阵 → **参数量极少**（低秩）
  - 冻 → 梯度只回传小矩阵 → **显存省**
  - 冻 → 前向/反向计算少 → **速度快**
  - 冻 → 新权重可分离 → **插拔切换**（多任务部署灵活）

## 一字一画面
- **冻** —— 像**冰箱冻住蛋糕底**：底层蛋糕（预训练权重）冻住不动，上面挤奶油花（低秩矩阵）做装饰。（本质：不破坏底座，只改表层）
- **秩** —— 像**两根细铁丝捏造型**：原本要重捏整个泥人（全量微调），现在只弯两根细铁丝（低秩矩阵 r×d）插在泥人上调整姿态。（本质：低秩近似，极少自由度捕捉主要变化）
- **快** —— 像**只换手机壳不换手机**：不用导数据、不重启，秒换即用。（本质：参数量小，迭代加速）
- **省** —— 像**开小灶不用整厨房**：只用一个电煮锅，省电省地方。（本质：显存占用低，只存小矩阵的梯度）
- **插** —— 像**插座换不同插头**：一个基座插不同 LoRA 插头，随时拔插切换任务。（本质：适配器机制，部署可合并可分离）

## 面试话术
"LoRA 的核心思路是**冻结预训练权重，注入低秩矩阵**。除了显著减少参数量，我总结为五个字：**冻秩快省插**。冻结原权重保证了预训练知识不丢失，避免灾难性遗忘；低秩分解让训练速度和显存占用都大幅下降；最关键的是 LoRA 像插头一样**可插拔**，一个基座模型能灵活适配多个下游任务，部署时还能选择合并权重或保持分离，兼容性非常强。"

## 防混补丁
- 容易和 **Adapter** 搞混。Adapter 是在层间插入小 FFN 模块，LoRA 是旁路低秩矩阵。
- 补丁：**Adapter 插层中，LoRA 走旁路**。

## 追问预演
1. **为什么低秩就能拟合权重更新？**
   → 实验表明预训练模型的权重更新具有低本征秩，r=4~64 即可捕捉有效变化。
2. **LoRA 矩阵怎么初始化？**
   → A 用高斯初始化，B 初始化为零，保证训练开始时增量为零，输出不变。
3. **QLoRA 和 LoRA 的区别？**
   → QLoRA 在 LoRA 基础上对基座模型做 4-bit 量化，进一步降低显存，适合单卡训大模型。
4. **什么场景 LoRA 效果不如全量微调？**
   → 需要改变模型底层知识、扩展词汇表或调整嵌入层的任务。
5. **LoRA 部署时合并权重的公式？**
   → W = W₀ + BA，推理前可预先将 BA 加到 W₀ 上，零推理开销。
