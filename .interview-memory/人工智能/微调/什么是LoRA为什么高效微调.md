# 什么是LoRA为什么高效微调

## 速记卡片
`秩 | 冻秩少参快 | 冻结+低秩=高效`

## 原始知识
- **LoRA** = Low-Rank Adaptation（低秩适配）
- **冻结**预训练权重 W0，不改原始知识
- 引入低秩矩阵 **B (d×r)** 和 **A (r×k)**，r ≪ d,k
- **W = W0 + BA**，只训 BA
- 参数量从 d×k 降到 **r×(d+k)**，减少 **99%** 以上
- **显存省**：不存 W0 的梯度/优化器状态
- **训练快**：反向传播只过小矩阵
- **推理快**：BA 可合并回 W0，**零额外延迟**
- 属于 **PEFT**（参数高效微调）

## 顺口溜
**冻秩少参快**

## 单字锚点 + 防崩推导
- **锚点字**：**秩**
- **扩展口诀**：**秩低冻原合**
- **为什么选它**："秩"是 LoRA 区别于其他 PEFT 方法（Adapter、Prompt Tuning）的灵魂——低秩直接修改权重。
- **防崩推导**：
  1. **秩低** → r 远小于模型维度 → BA 参数量极少（少参）
  2. **冻原** → W0 冻结不动 → 不存原权重梯度 → 显存大幅降低（省）
  3. **合** → 推理时 W = W0 + BA 预合并 → 无额外计算 → 速度快（快）

## 一字一画面
- **秩** —— 像**两根细铁丝捏泥人**：不用重捏整个泥人（全量微调），只弯两根细铁丝（低秩矩阵 B、A）插在泥人上调整姿态。（本质：低秩近似，极少自由度捕捉主要变化）
- **低** —— 像**矮门槛**：进入门槛低，微调所需参数量极少。（本质：r=8~64 即可有效）
- **冻** —— 像**冰箱冻住蛋糕底**：底层蛋糕（预训练权重）冻住不动，保持原味。（本质：防灾难性遗忘）
- **原** —— 像**原厂发动机**：保留原厂配置完整。（本质：W0 不变，通用能力不丢失）
- **合** —— 像**合并车道**：两根细管并回主干道，不增加收费站。（本质：推理零开销）

## 面试话术
"LoRA 的核心思路是**冻秩少参快**。冻结预训练权重 W0，用两个低秩矩阵 B 和 A 的乘积来近似权重增量。秩 r 很小，参数量不到原模型 1%，所以训练省显存、迭代快。最关键的是推理时 BA 可以直接合并回原始权重，零额外延迟，部署完全无损。"

## 防混补丁
- 容易和 **Adapter**、**全量微调**搞混。
- 补丁：**全量雕冰山，Adapter插旁路，LoRA低秩装小舵**。LoRA 直接修改权重矩阵，Adapter 插入额外 FFN 层，全量微调改所有参数。

## 追问预演

| 追问 | 简要答案 |
|------|----------|
| 为什么低秩就能拟合权重更新？ | 实验表明预训练模型的权重更新具有低本征秩，r=4~64 即可捕捉有效变化。 |
| 和 Adapter 的本质区别？ | LoRA 旁路低秩矩阵直接改权重，推理可合并；Adapter 插入新层增加深度。 |
| 高效体现在哪些维度？ | 参数量少、显存省、训练快、推理零开销、可多任务插拔切换。 |
| r 怎么选？ | 一般 8/16/32，任务复杂或矩阵大可取 64/128。 |
