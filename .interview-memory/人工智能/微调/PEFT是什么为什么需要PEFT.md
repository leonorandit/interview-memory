# PEFT 是什么？为什么需要 PEFT？

## 速记卡片
`省 | 全调贵，PEFT省 | 省资源`

## 原始知识
- **PEFT**：Parameter-Efficient Fine-Tuning，参数高效微调
- **大模型参数量巨大**：GPT-3 1750亿参数，全量微调需大量资源
- **全量微调成本高**：显存爆炸、训练时间长、存储开销大
- **PEFT价值**：只调少量参数，达到全量微调90%+效果
- **适用场景**：消费级GPU、多任务、快速迭代
- **易混点**：PEFT不是减少模型大小，是减少可训练参数

## 顺口溜
**全调贵，PEFT省** —— *全掉贵，泼福特省*

## 单字锚点 + 防崩推导
- **锚点字**：**省**
- **扩展口诀**：**省参省存省时间**
- **为什么选它**：PEFT的全部价值浓缩在一个"省"字——省钱、省卡、省时间。
- **防崩推导**：
  - 省 → 省钱/省卡 → 参数量大导致全量微调贵 → 只调少量参数
  - 省 → 省时间 → 训练快 → 收敛周期短
  - 省 → 省存储 → 每个任务只需存少量适配参数 → 多任务切换轻量
  - 反过来：不省 → 全量微调 → 1750亿参数全动 → 显存爆炸

## 一字一画面
- **省** —— 像**超市精打细算的阿姨**：只买今天做饭需要的菜，不买整个菜市场。（本质：只动必要的参数，不动整个模型）
- **贵** —— 像**市中心整栋写字楼出租**：租金天价，小公司租不起。（本质：全量微调需要A100多卡集群，成本极高）
- **泼**（PEFT谐音"泼"）—— 像**泼水只泼花盆**：精准灌溉，不浪费一滴水。（本质：针对性调整，不影响整体模型能力）

## 面试话术
- 问定义 → "PEFT全称Parameter-Efficient Fine-Tuning，核心就一个字：**省**。大模型参数量动辄百亿千亿，全量微调贵到普通团队玩不起。PEFT的思路是只微调一小部分参数或者插入少量适配模块，既保留预训练知识，又能在下游任务达到接近全量微调的效果。"
- 问价值 → "实际落地中，PEFT让我们能在单卡甚至消费级GPU上做微调，而且每个任务只需要存一个几MB的适配文件，不需要为每个任务存一份完整模型，**省卡、省时间、省存储**三省合一。"

## 防混补丁
- 容易和"模型压缩/量化"搞混。PEFT是**减少训练时需要更新的参数**，量化是**减少推理时参数的精度**。补丁：**PEFT少训参，量化少算位**。

## 追问预演
1. **Q：PEFT和全量微调效果差多少？**
   A：多数任务差距在5%以内，部分任务甚至反超，因为全量微调可能过拟合或灾难性遗忘。
2. **Q：PEFT会导致灾难性遗忘吗？**
   A：大幅降低，因为冻结了预训练参数，保留了原始能力。
3. **Q：什么场景下PEFT不如全量微调？**
   A：下游任务和预训练分布差异极大，或数据量非常充足时，全量微调上限更高。
