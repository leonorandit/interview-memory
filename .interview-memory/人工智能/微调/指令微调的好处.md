## 原始知识

- **指令遵循能力**：模型能按用户意图执行
- **对齐人类意图**：输出符合人类期望
- **激发涌现能力**：In-context Learning 增强
- **统一输入输出格式**：标准化交互方式
- **减少提示工程依赖**：不用写复杂 prompt
- **多任务通用**：一个模型处理多种任务
- **对话能力**：支持多轮连贯交互

## 单字锚点 + 防崩推导

**锚点字：听**

- 扩展口诀：**听**懂指令，**听**话办事
- 为什么选它：指令微调的核心就是让模型"听懂人话"，"听"字最形象
- 推导链：
  1. 听 → 听什么？→ 听用户的指令/意图
  2. 听 → 听懂了能干嘛？→ 按指令执行（指令遵循）
  3. 听 → 听谁的话？→ 听人类的话（对齐人类意图）
  4. 听 → 听多了会怎样？→ 激发更多能力（In-context Learning 增强）
  5. 听 → 听的方式？→ 统一格式，减少提示工程

## 一字一画面

**听** —— 像训练有素的导盲犬，主人说"左转"就左转，说"坐下"就坐下，不是只会自说自话。（本质：模型从自说自话变成按用户指令行动。）

**格** —— 像西装革履的管家，说话得体、举止有度，不再是野人。（本质：对齐人类价值观和表达习惯，输出更文明得体。）

**涌** —— 像打通任督二脉，本来只会 basic 功夫，突然就使出了高级招式。（本质：指令微调能激发预训练阶段 latent 的能力，如更好的上下文学习。）

**省** —— 像傻瓜相机，不用调光圈快门，按一下就能出片。（本质：用户不需要写复杂 prompt，模型自己理解意图。）

## 面试话术

"指令微调最大的好处是让模型真正'听懂人话'。一是提升了指令遵循能力，用户说干嘛就干嘛；二是对齐了人类意图，输出更符合期望；三是能激发涌现能力，比如 In-context Learning 会变得更强。另外还有统一了输入输出格式、减少提示工程依赖这些工程上的好处。"

## 防混补丁

容易和"预训练"的好处搞混？一句补丁：

> **预训练让模型会说话，SFT 让模型听人话。**

（预训练：掌握语言知识和世界知识；SFT：学会按指令行动和对齐人类偏好。）

## 追问预演

1. **Q：SFT 和 RLHF 都能对齐人类意图，有什么区别？**
   A：SFT 是通过示范教模型"该怎么做"，RLHF 是通过反馈教模型"什么更好"，SFT 是打基础，RLHF 是精调偏好。

2. **Q：为什么说 SFT 能激发涌现能力？**
   A：预训练模型已经有潜在能力，但不知道怎么被调用，SFT 通过指令格式把能力和场景对齐，相当于给能力配上了开关。

3. **Q：SFT 数据量少模型会不会过拟合？**
   A：确实可能，所以数据质量比数量重要，而且指令要多样化，防止模型只记住固定模板。
