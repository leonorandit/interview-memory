# LangChain 的 Agent 执行流程是怎样的？

## 速记卡片
`执 | 执观察，思行动 | ReAct 流程`

## 原始知识
- ReAct 模式：Reasoning + Acting
- 步骤：观察(输入) → 思考(推理) → 行动(调用工具) → 观察(结果) → ...循环
- 终止条件：得出最终答案或达到最大步数
- **加粗易混点**：Agent 不是单次调用，是多轮决策循环
- **追问点**：怎么防止无限循环？

## 顺口溜
执观察，思行动

## 单字锚点 + 防崩推导
- **锚点字**：**执**
- **扩展口诀**：**执循环**
- **为什么选它**："执"是执行的核心，Agent 的核心价值在于自主执行多步任务
- **防崩推导**：
  - 执 → 执行 → 观察环境
  - 执 → 思考策略 → 选择工具行动
  - 执 → 观察结果 → 判断完成否 → 循环或终止

## 一字一画面
**执** —— 像 **无人机搜救**：无人机观察地形（Observation）→ 飞手思考往哪飞（Thought）→ 操控杆转向（Action）→ 再观察新画面 → 直到找到目标或电量耗尽。（本质：感知-决策-执行的闭环循环）

## 面试话术
- "LangChain Agent 的执行流程可以用**执观察，思行动**来概括。它遵循 ReAct 模式：先观察用户问题和环境，然后思考该用什么工具，执行工具调用，再看工具返回的结果，循环往复直到得出最终答案。这不是一次 LLM 调用，而是一个多轮决策循环。"

## 防混补丁
- 容易和"Chain"搞混。Chain 是固定流程，Agent 是动态决策。补丁：**Chain 是高铁按站点跑，Agent 是出租车看导航走**。

## 追问预演
- **Q: Agent 怎么防止无限循环？** A: 设置 max_iterations，或设计终止条件判断。
- **Q: Agent 和 Plan-and-Execute 区别？** A: ReAct 是边想边做，Plan-and-Execute 是先规划全部步骤再执行。
