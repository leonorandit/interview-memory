# 如何评估 LangChain RAG 应用的效果？

## 速记卡片
`效 | 效三维，准全信 | 评估指标`

## 原始知识
- 检索评估：Recall、Precision、MRR、NDCG
- 生成评估：Faithfulness（忠实度）、Answer Relevance（答案相关度）、Context Precision
- 端到端：RAGAS 框架
- **加粗易混点**：不能只看生成 fluency，要看是否基于检索内容
- **追问点**：怎么降低幻觉？

## 顺口溜
效三维，准全信

## 单字锚点 + 防崩推导
- **锚点字**：**效**
- **扩展口诀**：**效三准**
- **为什么选它**："效"是效果的直接对应，也是面试官问的核心
- **防崩推导**：
  - 效 → 效果 → 三个维度
  - 效 → 准确性（答对否）→ 全面性（漏没漏）→ 忠实性（幻觉否）
  - 效 → RAGAS 量化

## 一字一画面
**效** —— 像 **三角尺量角**：一把三角尺的三个角同时卡住文档边（准确性）、覆盖范围（全面性）、不歪曲（忠实性），缺一就量不准。（本质：多维度交叉验证）

## 面试话术
- "评估 RAG 效果我用口诀**效三维，准全信**。准确性看答案对不对，全面性看有没有漏关键信息，忠实性看是否胡编幻觉。实际会用 RAGAS 框架自动打分，也会做人工抽检，不能只看出句流畅，得追溯到检索片段验证。"

## 防混补丁
- 容易和"传统检索评估"搞混。传统只看召回率，RAG 还要评估生成质量。补丁：**检索是找书，RAG 是写读后感**。

## 追问预演
- **Q: 怎么降低幻觉？** A: Rerank 精排、检索增强提示、让模型引用来源、Faithfulness 指标监控。
- **Q: 没有标注数据怎么评估？** A: 用 LLM-as-a-Judge，让更强的模型当裁判打分。
