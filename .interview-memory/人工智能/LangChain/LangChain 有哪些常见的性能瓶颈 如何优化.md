## 原始知识

- 瓶颈二：重复调用 → Prompt/LLM 缓存

- **加粗易混点**：缓存键默认是完整 Prompt，改动标点即失效

## 面试话术
LangChain 常见的性能瓶颈集中在 LLM 调用本身慢、重复请求浪费 token、向量检索拖后腿这几块。优化思路很实际：调用层走异步并发，重复走 Prompt 缓存或 LLM 缓存，检索层控制 Top-K 和索引粒度，长对话做摘要压缩，必要时直接上流式输出提升体感。

## 防混补丁
- `LangChainCache` 默认内存缓存，重启失效，生产建议接 Redis
- 异步用 `ainvoke` / `abatch`，不是简单套 `asyncio`
- 流式靠 `astream`，前端要配合 SSE 接收

## 追问预演
- Q：缓存命中率低怎么办？A：规范化 Prompt 模板，抽变量做缓存键
- Q：批量调用怎么控制并发数？A：用 `abatch` 的 `max_concurrency` 参数
