## 速记卡片
`路 | 条件判断分岔路 | Router链里动态入`

## 原始知识
- `RouterChain`：LLM 决策走哪个子链
- `RunnableBranch`：LCEL 里的 if-else
- 先定义 destination + 描述，再让 LLM 做路由
- 动态路由 = 运行时才决定下一步
- **加粗易混点**：Router 的决策也是一次 LLM 调用，有延迟和费用

## 顺口溜
条件判断分岔路

## 单字锚点 + 防崩推导
- **锚点字**：路
- **扩展口诀**：Router 指明路，Branch 做分岔
- **为什么选它**："路" = 路由 = 分支 = 动态选路，直观
- **防崩推导**：路由/分支 → 选路 → 路

## 一字一画面
**路** —— 像 **[黄色Y型路口]**：[5秒画面] 黄色 Y 型路口，交警（LLM）根据车牌（输入内容）打手势，左边去数学链，右边去代码链，中间还可掉头（fallback）（本质：动态路由就是让 LLM 或规则引擎在运行时选择下游处理链）

## 面试话术
条件分支和动态路由在 LangChain 里主要有两条路：如果用传统 Chain，就用 `RouterChain`，让 LLM 根据输入描述选一个子链；如果用 LCEL，就用 `RunnableBranch`，像写 if-else 一样把条件分支串进管道。核心都是先定义好各个分支的能力描述，再由 LLM 或自定义函数做决策。

## 防混补丁
- `RouterChain` 适合子链很多且描述清晰的场景
- `RunnableBranch` 适合条件少、逻辑硬的场景
- 路由决策错了会进错链，务必加 default/fallback

## 追问预演
- Q：路由错误怎么兜底？A：配置 default_chain，或在上层做重试
- Q：除了 LLM 决策，能用规则路由吗？A：能，写自定义函数返回 destination 即可
