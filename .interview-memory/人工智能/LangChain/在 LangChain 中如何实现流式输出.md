# 在 LangChain 中如何实现流式输出？

## 速记卡片
`流 | 流分段，逐字吐 | 实时输出`

## 原始知识
- 实现：使用模型的 stream 参数
- LangChain：通过 Runnable.stream() 或回调函数处理 chunk
- 适用场景：聊天 UI 实时显示、长文本生成
- **加粗易混点**：流式不是更快完成，是更快看到
- **追问点**：怎么和 Chain 结合？

## 顺口溜
流分段，逐字吐

## 单字锚点 + 防崩推导
- **锚点字**：**流**
- **扩展口诀**：**流逐现**
- **为什么选它**："流"是流式输出的核心字，也形象表示数据像水流一样连续
- **防崩推导**：
  - 流 → 数据流 → 分段传输
  - 流 → 逐 chunk 接收 → 实时渲染
  - 流 → 用户体验好

## 一字一画面
**流** —— 像 **水龙头接水**：不用等整桶水接满再喝，打开龙头就一口一口喝，每一口都是新鲜水 chunk。（本质：增量交付降低等待焦虑）

## 面试话术
- "LangChain 里实现流式输出的口诀是**流分段，逐字吐**。调用模型时开启 stream 模式，通过 Runnable 的 stream 方法或回调函数逐 chunk 接收 token，前端实时渲染。这样用户不用干等，长文本生成时体验提升很明显。"

## 防混补丁
- 容易和"异步"搞混。流式是数据传输方式，异步是执行方式，可以结合但不是一回事。补丁：**流是水管细，异步是多线程**。

## 追问预演
- **Q: 流式输出怎么和 Chain 结合？** A: 用 RunnableSequence 的 stream 方法，或自定义 StreamingCallbackHandler。
- **Q: 怎么知道流结束了？** A: 监听 on_llm_end 回调，或迭代器自然终止。
