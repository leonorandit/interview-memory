## 速记卡片
`调 | 调函三步定乾坤 | 定义→绑定→执行`

## 原始知识
- 定义工具函数：@tool 装饰器或 StructuredTool
- 描述 Schema：name、description、args_schema
- 绑定到模型：bind_tools() 或 Agent
- 模型决定调用：输出 ToolCall
- 执行并回传：ToolMessage 返回结果
- **易混点**：Function Call 是模型行为，不是代码直接调用

## 顺口溜
定函绑模再执行

## 单字锚点 + 防崩推导
- **锚点字**：调
- **扩展口诀**：调函三步定乾坤，定义绑定再执行
- **为什么选它**：Function Calling 的核心动作是模型"调"用外部函数
- **防崩推导**：问函数调用实现 → 想到"调" → 调什么？→ 外部工具 → 怎么调？→ 定义→绑定→模型决定→执行

## 一字一画面
**调** —— 像**遥控器换台**：你列好频道清单（定义工具），遥控器配对电视（绑定模型），电视自己选台播放（模型决定调用），画面出来再反馈给你（执行回传）（本质：模型掌握选择权，人只提供菜单和执行器）

## 面试话术
实现 Function Calling 分三步：先用 @tool 装饰器定义好函数的 Schema 和描述，再用 bind_tools 绑定到模型，最后模型根据用户输入决定调哪个函数，执行完把结果用 ToolMessage 塞回去继续对话。

## 防混补丁
- bind_tools 是 LangChain 方法，不同模型语法不同
- 模型可能调错函数，要做参数校验
- ToolMessage 必须回传，否则链断掉
- ReAct Agent 和直接 bind_tools 是两条路

## 追问预演
Q：模型不调函数怎么办？
A：检查工具描述是否清晰，模型是否支持 Function Calling，必要时用 ReAct Agent 显式推理。
