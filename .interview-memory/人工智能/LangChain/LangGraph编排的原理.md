# LangGraph 编排的原理

## 速记卡片
`编 | 节点边，状态传 | 图编排原理`

## 原始知识
- LangGraph编排核心：
  1. **节点**（Node）- 执行单元，每个节点是一个函数
  2. **边**（Edge）- 节点之间的连接，决定执行流向
  3. **状态**（State）- 全局状态对象，节点之间传递数据
  4. **条件边**（Conditional Edge）- 根据状态决定走哪条边
- 执行流程：
  - 初始化状态 → 进入起始节点 → 执行节点函数 → 更新状态 → 判断下一条边 → 进入下一个节点 → ... → 到达终点
- **易混点**：LangGraph vs 工作流引擎
- **追问点**：状态怎么管理？怎么实现循环？

## 顺口溜
**节点边，状态传**

## 单字锚点 + 防崩推导
- **锚点字**：**编**
- **扩展口诀**：**节点边，状态传**
- **为什么选它**：面试题问"编排原理"，"编"是核心动作。
- **防崩推导**：
  - 编（编排）→ LangGraph的编排机制
  - 节点 → 执行单元
  - 边 → 连接和流向
  - 状态 → 全局状态传递
  - 传 → 节点间通过状态传递数据

## 一字一画面
- **节点** —— 像**工厂工位**：每个工位做一道工序。（本质：执行单元）
- **边** —— 像**传送带**：把半成品从一个工位传到下一个。（本质：执行流向）
- **状态** —— 像**随身文件夹**：带着资料走到哪看到哪。（本质：全局状态传递）
- **条件边** —— 像**分拣机**：根据包裹大小决定走哪条道。（本质：条件分支）

## 面试话术
- 问编排原理 → "LangGraph编排原理口诀是**节点边，状态传**。**节点**——每个执行单元是一个函数节点；**边**——节点之间的连接决定执行流向；**状态**——全局状态对象在节点间传递；**传**——上游节点更新状态，下游节点读取状态。执行时，LangGraph按图结构从起始节点出发，沿着边遍历，遇到条件边就根据状态判断方向，支持循环和并行，非常灵活。"

## 防混补丁
- 容易和"DAG"搞混。LangGraph支持循环，DAG不支持。补丁：**DAG是单行道，LangGraph是环路**。"
