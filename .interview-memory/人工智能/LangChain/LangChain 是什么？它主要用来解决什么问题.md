# LangChain 是什么？它主要用来解决什么问题？

## 速记卡片
`链 | 链组件，串流程 | 框架定位`

## 原始知识
- 定位：LLM 应用开发框架
- 核心组件：Model、Prompt、Chain、Agent、Memory、Document Loader、VectorStore
- 解决问题：模型切换、提示管理、工具编排、上下文记忆、RAG 流水线
- **加粗易混点**：不是模型本身，是胶水框架
- **追问点**：和 LlamaIndex 区别？

## 顺口溜
链组件，串流程

## 单字锚点 + 防崩推导
- **锚点字**：**链**
- **扩展口诀**：**链串组**
- **为什么选它**："链"是 LangChain 的品牌核心，也是其设计哲学
- **防崩推导**：
  - 链 → 链条 → 连接组件
  - 链 → Prompt + Model + Parser 串成 Chain
  - 链 → Agent 加入工具决策 → Memory 加入长期记忆 → 形成完整应用

## 一字一画面
**链** —— 像 **自行车链条**：一个个链节（组件）咬合在一起，把脚踏板（用户输入）的动力传递到后轮（模型输出），缺一节就断，松了就会掉链。（本质：组件间的松耦合编排）

## 面试话术
- "LangChain 的口诀是**链组件，串流程**。它是个 LLM 应用开发框架，不是模型本身。核心价值是把 Prompt、模型、解析器、工具、记忆这些组件像链条一样串起来，解决模型切换、工具调用、上下文记忆这些重复劳动，让我们专注业务逻辑。"

## 防混补丁
- 容易和"LlamaIndex"搞混。LangChain 重流程编排，LlamaIndex 重数据索引和检索。补丁：**链是导演排戏，LlamaIndex 是图书管理员**。

## 追问预演
- **Q: LangChain Expression Language (LCEL) 是什么？** A: 用管道符 | 组合 Runnable 的声明式语法，替代传统 Chain 类。
- **Q: 现在还用 LangChain 吗？** A: 复杂编排场景仍有用，简单调用可直接用模型 SDK 或 OpenAI API。
