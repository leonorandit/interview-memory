# A2A协议和MCP协议的区别

## 速记卡片
`代 | 代协工调 | A2A管协作，MCP管工具`

## 原始知识
- **A2A**：Google推出的Agent-to-Agent开放协议，解决多Agent间的发现、认证、协作
- **MCP**：Anthropic推出的Model Context Protocol，解决模型与外部工具/数据源的标准化连接
- **A2A层次**：Agent ↔ Agent（协作层）
- **MCP层次**：Model ↔ Tool/Context（工具层）
- **A2A核心流程**：找（发现）→ 证（认证）→ 派（派任务）→ 做（执行）→ 交（交付）
- **MCP核心原语**：Resources（资源）+ Prompts（提示）+ Tools（工具）
- **传输协议**：均基于HTTP+JSON，A2A用SSE/Webhook推送状态，MCP用stdio/SSE
- **关系**：互补非替代。A2A Agent可通过MCP调用工具
- **易混点**：两者不是竞争关系；A2A ≠ Function Calling
- **追问点**：为什么需要A2A？A2A基于什么协议？两者如何叠加使用？

## 顺口溜
**代协工调**

## 单字锚点 + 防崩推导
- **锚点字**：**代**
- **扩展口诀**：**代协工调**
- **为什么选它**：A2A的核心是"Agent（代理）"之间的协作，"代"字直接锚定Agent身份，区别于MCP的"工具"属性。
- **防崩推导**：
  - 代（Agent）→ A2A是Agent-to-Agent，多个智能体之间协作
  - 协 → 协商任务、交换结果、状态同步（A2A的核心价值）
  - 工（工具）→ MCP是Model-Tool连接，解决工具调用标准化
  - 调（调用）→ 模型通过MCP统一调用外部工具/数据源
  - 关系 → A2A Agent内部可以通过MCP调工具，两者叠加

## 一字一画面
- **代** —— 像**外卖小哥接龙**：一个骑手（Agent A）接到大单，分派给另一个骑手（Agent B）送饮料，两人互相确认身份、各自汇报进度。（Agent之间对等协作）
- **协** —— 像**团队协作群聊**：大家拉个群，发任务、传文件、同步进度、确认交付。（协商与通信）
- **工** —— 像**统一工具箱**：里面有手机、充电宝、保温箱，统一规格谁都能用。（工具标准化）
- **调** —— 像**用遥控器开空调**：按一下，空调就启动了，不用关心空调内部电路。（工具调用）

## 面试话术
- 问区别 → "口诀是**代协工调**。A2A是Agent-to-Agent Protocol，解决多个智能体之间的协作通信，像几个同事拉群对接任务；MCP是Model Context Protocol，解决模型与外部工具/数据的标准化连接，像统一的遥控器能控制各种电器。两者是互补关系：A2A管Agent之间怎么协作，MCP管Agent内部怎么调工具。实际架构中，A2A上的Agent可以通过MCP调用工具，两者叠加使用。"
- 问为什么需要A2A → "MCP只解决Agent和工具的连接问题，但不管多个Agent之间怎么发现、怎么协商任务、怎么交换结果。A2A填补的就是这个'Agent协作层'的标准化空白。"

## 防混补丁
- 容易和"Function Calling"搞混。Function Calling是单个Agent内部调用工具，A2A是多个Agent互相协作。补丁：**Function Call是自己干活，A2A是找人一起干活**。
- 容易把A2A和MCP当竞争关系。补丁：**A2A管协作，MCP管工具，叠加用**。
- 容易和"传统API"搞混。传统API是一次性请求-响应；A2A是长时任务协作，支持流式状态更新。补丁：**API是点外卖，A2A是外包项目**。

## 追问预演
1. **Q: 两者是什么关系？**
   A: 互补关系。A2A解决Agent间协作标准化，MCP解决Agent与工具连接标准化。一个Agent既可以通过A2A与其他Agent协作，又可以通过MCP调用外部工具。
2. **Q: 为什么需要A2A，直接用MCP不行吗？**
   A: MCP只管模型调工具，不管多Agent之间的发现、任务协商、身份认证、状态同步。A2A填补的是Agent协作层的空白。
3. **Q: A2A基于什么传输协议？**
   A: HTTP+JSON，状态更新用SSE（Server-Sent Events）或Webhook。
4. **Q: MCP基于什么传输协议？**
   A: stdio（本地进程通信）或SSE（远程服务通信）。
5. **Q: 实际架构中两者如何叠加？**
   A: Agent A 通过 A2A 把任务派给 Agent B；Agent B 在执行过程中，通过 MCP 调用数据库查询工具、代码执行工具等；最终结果通过 A2A 交回给 Agent A。
