# 说说 GloVE 技术，怎样进行训练？有哪些应用场景？相比 Word2Vec 有哪些优缺点

## 速记卡片
`矩 | 共现矩阵开根除，全局统计比局部 | 矩阵分解+统计优势`

## 原始知识
- **核心思想**：利用全局词-词共现矩阵，而非局部滑动窗口
- **训练目标**：log(X_ij) ≈ w_i·w_j + b_i + b_j，加权最小二乘
- **共现矩阵**：X统计整语料中词i和词j在窗口内共同出现次数
- **应用场景**：词类比、文本分类、命名实体识别
- **vs Word2Vec**：GloVe全局统计更稳定，Word2Vec局部采样更灵活
- **加粗易混点**：GloVe不是纯计数，是带权矩阵分解；Word2Vec也不是纯神经网络，是隐式矩阵分解

## 顺口溜
共现矩，开根除，全局稳

## 单字锚点 + 防崩推导
- **锚点字**：**矩**
- **扩展口诀**：**共现矩阵开根除**
- **为什么选它**：GloVe的一切始于"共现矩阵"，矩是矩阵的核心代称，信息密度最高。
- **防崩推导**：
  - 矩 → 共现矩阵 → 整语料统计词对出现次数
  - 矩 → 不是原始计数 → 取log、加权、f(X) = min((X/X_max)^α, 1)
  - 矩 → 分解出词向量 → w_i和w_j的内积要逼近log(X_ij)
  - 全局 vs 局部 → 矩看全语料，Word2Vec看每个窗口

## 一字一画面
- **矩** —— 像**巨大的Excel表格**：行和列都是词汇，每个格子填的是两词共现次数，GloVe就是对这个表格做降维。（本质：全局统计的浓缩）
- **开根除** —— 像**把大数字压成气温图**：原始共现次数几万几万，取log后变成温和的数字，颜色梯度才均匀。（本质：对数压缩量纲）
- **全局** —— 像**航拍全城市**：Word2Vec是街景车一路拍，GloVe是卫星一次拍完全城。（本质：统计视角差异）

## 面试话术
- "GloVe的核心是**共现矩阵**。它先统计整个语料里任意两个词在窗口内共同出现的次数，构成一个大矩阵；然后让词向量的内积逼近共现次数的log值，用加权最小二乘来训练。相比Word2Vec，GloVe利用全局统计信息，对高频词的表达更稳定，训练更快；Word2Vec则是局部滑动窗口采样，更适合大规模流式语料。"

## 防混补丁
- 容易说"GloVe是矩阵分解，Word2Vec是神经网络"。补丁：**两者都是学向量，GloVe显式建矩阵，Word2Vec隐式做分解；本质殊途同归**。

## 追问预演
- Q：GloVe的加权函数f(X)为什么长那样？
  - A：既要给高频词足够权重，又不能让它垄断；低频词噪声大，要压低权重。
- Q：GloVe和LSA有什么区别？
  - A：LSA直接对共现矩阵做SVD，GloVe是带权目标函数优化，更灵活。
- Q：什么场景选GloVe不选Word2Vec？
  - A：语料不大、需要稳定词类比任务时，GloVe通常更好。
