## 速记卡片
`门 | 三门一态控长期 | 遗忘门/输入门/输出门 + 候选态`

## 原始知识
- **遗忘门 f_t**：决定丢多少旧记忆
- **输入门 i_t**：决定存多少新信息
- **候选态 C̃_t**：生成新候选记忆
- **单元状态 C_t**：长期记忆载体，线性传播
- **输出门 o_t**：决定露多少给隐藏态
- **隐藏态 h_t**：短期输出，带非线性

## 顺口溜
三门一态控长期，遗忘输入再输出。

## 单字锚点 + 防崩推导
- **锚点字**：门
- **扩展口诀**：三门一态控长期，sigmoid 当门卫
- **为什么选它**：LSTM 的灵魂就是三个"门控"机制，用"门"字可以串起所有组件——忘记什么、记住什么、输出什么，全靠门来决定
- **防崩推导**：
  1. 想到"门"→三个门：遗忘门、输入门、输出门
  2. 遗忘门：sigmoid(W_f·[h_{t-1},x_t]+b_f)→0 到 1，决定旧记忆 C_{t-1} 保留多少
  3. 输入门：sigmoid(W_i·[h_{t-1},x_t]+b_i)→决定新候选 C̃_t 写入多少
  4. 候选态：tanh(W_C·[h_{t-1},x_t]+b_C)→生成新候选记忆
  5. 更新：C_t = f_t ⊙ C_{t-1} + i_t ⊙ C̃_t（旧记忆+新信息）
  6. 输出门：sigmoid(W_o·[h_{t-1},x_t]+b_o)→决定 C_t 经过 tanh 后输出多少作为 h_t

## 一字一画面
**门** —— 像**三扇红色旋转门并排立在灰色长廊中**：第一扇（遗忘门）把旧文件碎纸投入焚烧口；第二扇（输入门）把新文件递入档案柜；第三扇（输出门）只把档案摘要递出窗外给行人看（h_t）。长廊深处，档案柜（C_t）永远完整保留。（本质：门控决定信息流，C_t 线性传递防梯度消失）

## 面试话术
"LSTM 单元主要由三个门和一个候选态组成。遗忘门决定上一时刻的单元状态保留多少；输入门和候选态配合，决定当前输入生成的新记忆写入多少；输出门则控制当前单元状态有多少能转化为隐藏状态输出。这三个门都是 sigmoid 激活，输出 0 到 1 之间的值，实现精细的信息控制。"

## 防混补丁
- **f_t / i_t / o_t 激活**：都是 sigmoid，不是 tanh
- **C̃_t 激活**：是 tanh，不是 sigmoid
- **C_t 更新公式**：先乘 f_t，再加 i_t ⊙ C̃_t，不是直接替换
- **h_t 与 C_t**：h_t = o_t ⊙ tanh(C_t)，C_t 是长期记忆，h_t 是短期输出
- **防崩**：门控用 sigmoid（0~1），候选态/输出用 tanh（-1~1），分开记

## 追问预演
Q：为什么用 sigmoid 做门控？
A：sigmoid 输出在 0 到 1 之间，天然适合表示"通过比例"。

Q：LSTM 怎么缓解梯度消失？
A：单元状态 C_t 的传递是线性加法（f_t ⊙ C_{t-1} + ...），没有反复矩阵乘法，梯度可以更稳定地回传。

Q：三个门的输入为什么都包含 h_{t-1} 和 x_t？
A：因为门控决策需要综合上一时刻的短时输出和当前输入，实现时序依赖。
