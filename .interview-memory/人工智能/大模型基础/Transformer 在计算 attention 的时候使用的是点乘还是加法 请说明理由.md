# Transformer 在计算 attention 的时候使用的是点乘还是加法 请说明理由

## 速记卡片
`点 | 点乘对，加法贵 | 矩阵乘法·高效·内积`

## 原始知识
- 使用Scaled Dot-Product Attention
- Q和K做矩阵乘法（点乘）
- 原因：矩阵乘法可用高度优化的BLAS库，GPU并行友好
- 点乘关注度量有良好理论性质（内积衡量相似度）
- 加法注意力（Bahdanau）在RNN时代使用，需额外参数矩阵
- 点乘计算更快、参数量更少
- **易混点**：Transformer用点乘，但还有其他注意力变体

## 顺口溜
点乘对，加法贵

## 单字锚点 + 防崩推导
- **锚点字**：**点**
- **扩展口诀**：**点乘快**
- **为什么选它**："点"直接对应"点乘"，是Attention计算方式的核心。
- **防崩推导**：
  - 点 → 点乘 → Scaled Dot-Product
  - 乘 → 矩阵乘法 → GPU高度优化
  - 对 → 正确选择 → 工程+理论都支持
  - 加 → 加法注意力 → Bahdanau老方案
  - 法 → 方法 → 需要额外参数矩阵W
  - 贵 → 计算昂贵 → 速度慢、参数量大

## 一字一画面
- **点** —— 像**点钞机**：两张矩阵对齐，点一下快速计算相似度。（点乘操作）
- **乘** —— 像**流水线组装**：GPU擅长批量矩阵乘法，一排排同时算。（并行加速）
- **对** —— 像**选对工具**：用螺丝刀比用手拧快。（工程优化）
- **加** —— 像**加法器**：需要额外零件（参数矩阵）才能工作。（额外开销）
- **法** —— 像**老方法**：RNN时代的手工算法，过时了。（历史方案）
- **贵** —— 像**进口零件**：多了参数矩阵，显存和计算都更贵。（成本高）

## 面试话术
- "Transformer用**点乘对，加法贵**。具体是Scaled Dot-Product Attention，Q和K做矩阵乘法。因为矩阵乘法可以用GPU高度并行加速，计算快、参数量少；而加法注意力需要额外参数矩阵，计算成本高。点乘内积天然衡量向量相似度，理论和工程上都更优。"

## 防混补丁
- 容易和Bahdanau注意力搞混。补丁：**Bahdanau是RNN的加法老黄历，Transformer是点乘新时代**。

## 追问预演
- **Q：为什么要除以√d_k？** A：防止点乘结果过大导致Softmax梯度消失，缩放后数值更稳定。
- **Q：点乘和加法的公式区别是什么？** A：点乘是(QK^T)/√d；加法是v^T tanh(W_q·Q + W_k·K)，加法需要额外可学习参数W。
