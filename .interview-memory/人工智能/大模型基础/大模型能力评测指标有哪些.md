# 大模型能力评测指标有哪些？

## 速记卡片
`六 | 考数理码长安 | 六大维度`

## 原始知识
- **考**（知识考查）：MMLU（英文多任务）、C-Eval / CMMLU（中文多任务）、TriviaQA / NaturalQuestions（事实问答）
- **数**（数学推理）：GSM8K（小学数学应用题）、MATH（竞赛级数学）、SVAMP
- **理**（逻辑/复杂推理）：BBH（Big-Bench Hard，多步复杂推理）、GPQA（研究生级别推理）、ARC（科学推理）
- **码**（代码生成）：HumanEval / MBPP（函数级代码）、SWE-bench（真实软件工程）、LiveCodeBench
- **长**（长文本理解）：LongBench / RULER（长上下文综合评测）、Needle in a Haystack（大海捞针）、L-Eval
- **安**（安全与对齐）：TruthfulQA（事实性/防幻觉）、MT-Bench / AlpacaEval（指令遵循与对话）、toxicity / bias 检测、Red Teaming
- 补充：Perplexity（语言建模基础指标）、Arena ELO（人类偏好对战投票）
- **易混点**：训练指标（Loss/PPL）vs 能力评测指标；MMLU（英文）vs C-Eval（中文）
- **追问点**：各指标侧重点、基座模型 vs 对话模型评测差异、人工评估 vs 自动评估

## 单字锚点 + 防崩推导
- **锚点字**：**六**
- **扩展口诀**：**六维考数理码长安**
- **为什么选它**：题目问"有哪些指标"，"六"给出数量锚点，防止漏项。
- **防崩推导**：
  - 六 → 六大评测维度
  - 考 → 知识考查（MMLU, C-Eval）
  - 数 → 数学推理（GSM8K, MATH）
  - 理 → 逻辑推理（BBH, GPQA）
  - 码 → 代码生成（HumanEval, MBPP）
  - 长 → 长文本理解（LongBench, Needle）
  - 安 → 安全对齐（TruthfulQA, MT-Bench）

## 一字一画面
- **六** —— 像**六边形战士**：六个维度全面拉满，缺一不可。（本质：六大评测维度）
- **考** —— 像**高考**：多科目综合考试，考知识广度与深度。（本质：MMLU / C-Eval 知识评测）
- **数** —— 像**数学竞赛**：一步一步推导，不能跳步，答案唯一。（本质：GSM8K / MATH 数学推理）
- **理** —— 像**侦探推理**：复杂逻辑链，多步思考，拨开迷雾。（本质：BBH / GPQA 复杂推理）
- **码** —— 像**编程比赛**：写代码跑测试用例，Pass 才算数。（本质：HumanEval / MBPP 代码生成）
- **长** —— 像**长篇小说阅读**：看完十万字还能准确找到某个细节。（本质：LongBench / Needle 长文本）
- **安** —— 像**安检门**：过滤有害内容，确保价值观对齐。（本质：TruthfulQA / MT-Bench 安全对齐）

## 面试话术
- 问评测指标 → "大模型能力评测我分为六个维度，口诀是**考数理码长安**。**考**——知识考查，如 MMLU、C-Eval；**数**——数学推理，如 GSM8K、MATH；**理**——复杂逻辑推理，如 BBH、GPQA；**码**——代码生成，如 HumanEval；**长**——长文本理解，如 LongBench、大海捞针；**安**——安全对齐，如 TruthfulQA、MT-Bench。基座模型重点看知识、推理和代码；对话模型重点看 MT-Bench 和安全性。"
- 问怎么选 → "不能只看单一指标，要像六边形战士一样全面。PPL 低不代表能力强，最终还要结合人工评估和 Arena ELO 人类对战投票。"

## 防混补丁
- 容易把"训练指标"和"能力评测"搞混。训练看 Loss / Perplexity，能力看下游 Benchmark。补丁：**训练看Loss，能力看考题**。
- 容易把 MMLU 和 C-Eval 搞混。MMLU 是英文多任务，C-Eval 是中文多任务。补丁：**MMLU英文考，C-Eval中文考**。
- 容易只看自动指标忽略安全。补丁：**能力再强，价值观不对也是0分**。

## 追问预演
1. **MMLU 是什么？为什么重要？**
   → Massive Multitask Language Understanding，涵盖 57 个学科的多选题评测，是衡量模型知识广度的标准基准，也是各大模型排行榜的核心指标。
2. **GSM8K 和 MATH 有什么区别？**
   → GSM8K 是小学数学应用题，侧重基础算术和简单推理；MATH 是竞赛级数学，涉及代数、几何、数论等，难度更高。
3. **HumanEval 怎么评测代码能力？**
   → 给定函数签名和 docstring，模型生成代码，用单元测试 pass 率（Pass@k）评估。
4. **Needle in a Haystack 测什么？**
   → 在极长文本中随机插入一个关键信息（needle），测试模型能否在问答中准确检索到该信息，评估长上下文记忆能力。
5. **MT-Bench 和 AlpacaEval 的区别？**
   → MT-Bench 用 GPT-4 打分评估多轮对话和指令遵循能力；AlpacaEval 用自动对比胜率评估单轮指令遵循。两者都是模型对战评估体系。
6. **基座模型和对话模型评测重点有什么不同？**
   → 基座模型（Base）重点看 MMLU、GSM8K、BBH 等知识和推理指标；对话模型（Chat/Instruct）重点看 MT-Bench、AlpacaEval、安全性指标。
