# 温度值top-p-top-k怎么设置

## 速记卡片
`温 | 温低稳高飘，pk跟着调 | 参数联动设置`

## 原始知识
- **温（Temperature）**：控制概率分布的"尖锐程度"，全局随机性旋钮
  - 低温度（0~0.3）：分布 sharpen，输出稳定、准确、可预期
  - 高温度（0.8~1.0）：分布 flatten，输出飘逸、有创意、多样化
- **pk跟着调**：Top-P 和 Top-K 的设置跟着 temperature 的调子走
  - 低温时 → pk 也小（top-p=0.1~0.3, top-k=1~10）→ 候选池小 → 精确
  - 高温时 → pk 也大（top-p=0.9~1.0, top-k=50~100）→ 候选池大 → 发散
- **三者执行顺序**：先 Top-K 硬性截断 → 再 Top-P 动态截断 → 最后 Temperature 重采样
- **典型设置**：
  - 代码/数学/分类：temperature=0.1~0.3, top-p=0.1~0.3, top-k=1~10
  - 通用对话/摘要：temperature=0.5~0.7, top-p=0.7~0.9, top-k=20~50
  - 创意写作/头脑风暴：temperature=0.8~1.0, top-p=0.9~1.0, top-k=50~100
- **易混点**：三个参数互斥、T=0 和 K=1 完全一样
- **追问点**：执行顺序、为什么代码用低温、实际工程中怎么调

## 顺口溜
**温低稳高飘，pk跟着调**

## 单字锚点 + 防崩推导
- **锚点字**：**温**
- **扩展口诀**：**温低稳高飘，pk跟着调**
- **为什么选它**：Temperature 是总指挥，"温"字定了，pk 跟着走就行。
- **防崩推导**：
  - 温 → temperature → 控制全局随机性 → 像调音台主旋钮
  - 低 → 低温度（0~0.3）→ 分布变尖 → 输出稳定准确 → 适合代码/数学/分类
  - 稳 → 稳定可控
  - 高 → 高温度（0.8~1.0）→ 分布变平 → 输出飘逸创意 → 适合头脑风暴/创意写作
  - 飘 → 飘逸多变
  - pk → top-p + top-k → 控制候选词池大小 → 像筛子
  - 跟着调 → pk 跟着 temperature 的调子走 → 低温配小 pk，高温配大 pk

## 一字一画面
- **温** —— 像**调音台主旋钮**：拧小声音集中，拧大声音发散。（temperature 控制全局随机性）
- **低** —— 像**烧开水**：温度低时水面平静，气泡均匀可控。（低温度输出稳定）
- **稳** —— 像**老司机开车**：路线笔直，不跑偏。（代码/数学场景）
- **高** —— 像**煮火锅**：温度高时翻滚沸腾，食材上下翻飞。（高温度输出多样）
- **飘** —— 像**freestyle 说唱**：即兴发挥，每遍都不一样。（创意写作）
- **pk** —— 像**筛子**：筛孔小只留下大颗粒，筛孔大小颗粒都能过。（top-p/top-k 筛选候选词）
- **调** —— 像**乐队合奏**：temperature 是指挥定调，pk 是乐手跟着节奏走。（pk 配合 temperature）

## 面试话术
- 问设置 → "温度值和 top-p/top-k 的设置我总结为**温低稳高飘，pk跟着调**。**温**是 temperature，像调音台旋钮，低温度输出稳定准确，高温度输出飘逸有创意。**pk跟着调**是指 top-p 和 top-k 跟着 temperature 的调子走——温度低时 pk 也小（top-p=0.1, top-k=1~10），保证精确；温度高时 pk 也大（top-p=0.9, top-k=50~100），让模型发散。三者执行顺序是先 K 截断，再 P 截断，最后 temperature 重采样。"
- 问本质 → "三个参数是组合拳：**温重塑概率分布的形状，PK 截断候选词的范围**。调参时先定 temperature 的基调，再用 pk 精细控制。"

## 防混补丁
- 容易以为"三个参数互相独立"。实际上通常先 top-k 截断，再 top-p 截断，最后 temperature 重采样。**补丁：先k后p再温，层层过滤。**
- 容易把 temperature=0 和 greedy decoding 搞混。temperature=0 会让概率最高的词概率趋近1，接近贪心但不完全相同。**补丁：温零趋贪，k一即贪。**

## 追问预演
1. **top-p 和 top-k 有什么区别？** 答：top-k 硬性保留前k个词，top-p 按累积概率动态截断。top-k=10 可能保留概率差很远的词，top-p=0.9 可能只保留2-3个词。
2. **为什么代码生成要设 temperature=0.1？** 答：代码语法严格，一个字符错误就可能导致编译失败，低温提高确定性。
3. **三个参数的执行顺序？** 答：先 top-k 截断候选池，再 top-p 动态截断，最后 temperature 对剩余候选做 softmax 重采样。
4. **实际工程中怎么调？** 答：先固定 top-k=50，调 temperature 和 top-p 做 A/B 测试。推荐从 (0.7, 0.9, 50) 开始，根据场景向高创或低准偏移。
