# 为什么要除以 √d_k？

## 速记卡片
`除 | 方大软除稳 | Scale操作原理`

## 原始知识
- **方**（方差）：假设 Q 和 K 的每个元素独立同分布，均值为 0，方差为 1
- **大**（变大）：Q·K^T 的每个元素是 d_k 个独立随机变量的和，方差为 d_k，标准差为 √d_k
- **软**（softmax 饱和）：如果不除，点积值尺度大 → softmax 输入进入极端区域 → 梯度极小（梯度消失）
- **除**（除以 √d_k）：缩放操作，将方差从 d_k 归一化为 1
- **稳**（训练稳定）：softmax 输入保持在合理范围，梯度正常流动，数值稳定
- **为什么不是除以 d_k**：除以 d_k 会让方差变为 1/d_k，太小，softmax 输出过于平缓，区分度不够
- **数学推导**：q_i ~ N(0,1), k_j ~ N(0,1) 独立 → Q·K^T ~ N(0, d_k) → (Q·K^T)/√d_k ~ N(0,1)
- **工程意义**：d_k 越大，点积"碰撞机会"越多，值越容易爆炸，Scale 是必要归一化
- **易混点**：Scale 和 LayerNorm 的区别；除以 d_k vs √d_k
- **追问点**：数学推导、工程实验验证、不同 d_k 的影响

## 顺口溜
**方大软除稳**

## 单字锚点 + 防崩推导
- **锚点字**：**除**
- **扩展口诀**：**除方大软稳**
- **为什么选它**：题目直接问"为什么要除以"，"除"字精准扣题。
- **防崩推导**：
  - 除 → 除以 √d_k（Scale 操作）
  - 方 → 方差（Q·K^T 方差随 d_k 增大）
  - 大 → d_k 越大，方差越大（Var = d_k）
  - 软 → softmax 饱和，梯度消失
  - 稳 → 除后方差归一化为 1，训练稳定

## 一字一画面
- **除** —— 像**厨师调味加水**：盐放多了，加点水稀释回正常咸度。（本质：除以 √d_k 做缩放归一化）
- **方** —— 像**气球随尺寸膨胀**：气球越大，内部气压方差越大，越容易爆。（本质：d_k 增大 → 方差线性增大）
- **大** —— 像**聚光灯过曝**：亮度太高，照片全白，看不清细节。（本质：点积值过大）
- **软** —— 像**油门踩死卡转速**：发动机红区卡死，再踩油门转速也不升，没动力。（本质：softmax 饱和区梯度趋近于 0）
- **稳** —— 像**稳压器输出**：输入电压波动，稳压器恒稳输出 220V。（本质：方差归一化为 1，数值尺度稳定）

## 面试话术
- 问为什么 → "除以 √d_k 的原因口诀是**方大软除稳**。当 d_k 很大时，Q·K^T 点积的方差会随 d_k 线性增大，导致数值很大，softmax 进入饱和区、梯度消失。除以 √d_k 后，方差被归一化为 1，softmax 输入保持在合理范围，梯度正常流动，训练更稳定。除以 d_k 会让方差太小，softmax 过于平缓，所以 √d_k 刚刚好。"
- 问数学推导 → "假设 Q 和 K 的元素独立同分布 N(0,1)，则 Q·K^T 的每个元素是 d_k 个独立 N(0,1) 之和，服从 N(0, d_k)。除以 √d_k 后标准正态化，这是概率论的大数定律直接结果。"

## 防混补丁
- 容易和 LayerNorm 搞混。Scale 只针对 Attention 分数做方差归一化；LayerNorm 对整个层输出做均值方差归一化。补丁：**Scale管注意力分数，LN管整层输出**。
- 容易问"为什么不除以 d_k"。除以 d_k 方差变为 1/d_k，softmax 过于平缓，注意力权重趋近均匀分布，丧失区分能力。补丁：**除d_k太平淡，除根号d_k刚刚好**。

## 追问预演
1. **不除会怎样？能训练出来吗？**
   → 小 d_k（如 64）可能还能收敛，但大 d_k（如 512、1024）时点积值爆炸，softmax 梯度几乎为 0，训练极慢或不收敛。实验上 d_k > 128 不除就明显不稳定。
2. **从概率角度严格推导一下？**
   → 设 q_i, k_j ~ N(0,1) i.i.d.，则 (QK^T)_ij = Σ_m q_im · k_jm。每项 q_im·k_jm 期望为 0，方差为 1（因为 E[X^2]=1, E[Y^2]=1, X,Y 独立）。d_k 项求和，方差相加得 d_k，所以 (QK^T)_ij ~ N(0, d_k)。除以 √d_k 后 ~ N(0,1)。
3. **实际工程中 d_k 一般多大？**
   → 常见 64、128。GPT-3 中 d_k = 128，√d_k ≈ 11.3。LLaMA2-70B 中 head_dim 也是 128。
4. **这个操作和 Attention 里的其他归一化（如 LayerNorm）是什么关系？**
   → Scale（除以 √d_k）解决的是 Attention 分数的内在方差问题；LayerNorm 解决的是层输出分布漂移问题。两者互补，Scale 在 Attention 内部，LN 在子层之后。
5. **有没有实验验证过不除的效果？**
   → 原始 Transformer 论文中有消融实验，不除时 BLEU 分数显著下降。后续工作（如 Attention Is All You Need、Scaling Laws）都验证了 Scale 的必要性。
6. **如果用加法注意力（Bahdanau）还需要除吗？**
   → 加法注意力用 v^T tanh(W_q Q + W_k K)，tanh 天然把值压缩到 (-1,1)，不需要额外 Scale。但计算成本高，Transformer 选择点乘 + Scale 的组合兼顾效率和稳定性。
