# 长上下文窗口扩展与外推方法

## 速记卡片
`外 | 外爆内糊，NTK混，Yarn调温 | 外推vs内插`

## 原始知识
- **问题根源**：大模型训练成本高，通常只在短文本（4K/8K/32K）上预训练，推理时需处理更长序列
- **瓶颈**：位置编码（尤其是 RoPE）在超出训练长度后失效
- **外推**：直接处理更长序列，不修改位置编码 → Attention score 分布外移 → **爆炸/崩溃**
- **内插**：将长位置编号缩放进训练范围 → 相对距离被压缩 → **区分度下降/糊掉**
- **NTK-aware**：高频分量外推，低频分量内插，分频处理
- **YaRN**：NTK 改进 + 注意力温度缩放，稳定 softmax 分布
- **易混点**：外推不是"向外扩展"的意思，而是直接超范围使用
- **追问点**：为什么 RoPE 直接外推会失效？NTK 的数学原理？

## 顺口溜
**外爆内糊，NTK混，Yarn调温**

## 单字锚点 + 防崩推导
- **锚点字**：**外**
- **扩展口诀**：**外爆内糊**
- **为什么选它**：一切长上下文扩展的争论都围绕"要不要直接超出训练长度"，"外"是矛盾的起点。
- **防崩推导**：
  - 外 → 超出训练长度硬上 → Attention 点积分布外移 → **爆**（爆炸/崩溃）
  - 反过来：不硬上 → 把位置缩进来 → 相对距离变近 → **糊**（区分度下降）
  - 折中 → NTK → 高频外推 + 低频内插 → **混**（混合策略）
  - 再优化 → Yarn → 加温度系数除法 → **温**（调温稳定 softmax）

## 一字一画面
- **外** —— 像**弹簧拉过头**：训练时只拉到 1 米，现在硬拉 2 米，弹簧变形失控。（外推超出训练分布）
- **爆** —— 像**气球吹太大**：注意力点积值远超训练范围，softmax 极端化，输出崩溃。（外推导致 attention 爆炸）
- **内** —— 像**地图缩印**：把 2 米的地图缩印到 1 米纸上，字都挤在一起看不清。（内插压缩位置编码）
- **糊** —— 像**照片压缩过度**：相邻位置的编码变得太像，模型分不清谁是谁。（内插损失位置精度）
