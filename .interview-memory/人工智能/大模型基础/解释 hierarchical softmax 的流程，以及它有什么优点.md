# 解释 hierarchical softmax 的流程，以及它有什么优点

## 速记卡片
`树 | 霍夫曼树分层算，概率连乘走枝干 | O(logV)加速`

## 原始知识
- **核心思想**：把扁平softmax变成二叉树路径分类
- **霍夫曼树**：高频词离根近，低频词在深处，最小化平均路径
- **流程**：每个非叶节点是一个二分类器（sigmoid），决定走左还是右
- **概率计算**：从根到目标词路径上所有二分类概率连乘
- **优点**：训练复杂度从O(V)降到O(logV)，特别适合大词表
- **加粗易混点**：层次softmax只加速训练，不影响推理时的扁平softmax

## 顺口溜
霍夫曼树，分层走，logV快

## 单字锚点 + 防崩推导
- **锚点字**：**树**
- **扩展口诀**：**霍夫曼树分层走**
- **为什么选它**：hierarchical softmax的全部机制都建立在"树"结构上，树是骨架。
- **防崩推导**：
  - 树 → 霍夫曼树 → 高频词短路径、低频词深路径
  - 树 → 每个内部节点一次二分类 → sigmoid决定左/右
  - 树 → 目标词概率 = 路径上各节点概率连乘
  - 树 → 复杂度O(logV) → 大词表训练加速明显

## 一字一画面
- **树** —— 像**图书馆分类架**：根是总入口，第一层分"文学/科学"，再分"小说/诗歌"，一直分到某本书。高频书放在低层矮架上，伸手就够。（本质：层次化概率分解）
- **分层走** —— 像**二十问猜词游戏**："是动物吗？"→"会飞吗？"→"是鸟类吗？"，每个问题排除一半，log步猜到答案。（本质：二分检索）
- **logV快** —— 像**坐电梯而非爬楼梯**：10万词表，softmax要走10万阶，层次softmax只走~17层。（本质：计算复杂度降低）

## 面试话术
- "层次softmax的核心是**树**。它先根据词频建一棵霍夫曼树，高频词离根近。训练时从根开始，每个内部节点做一次二分类决定走左边还是右边，直到叶子节点就是目标词。目标词的概率等于路径上所有二分类概率的乘积。这样就把O(V)的softmax计算降到了O(logV)，词表越大收益越明显。"

## 防混补丁
- 容易以为层次softmax也加速推理。补丁：**层次softmax只用于训练加速，推理时仍可用普通softmax；负采样也仅训练用**。

## 追问预演
- Q：为什么用霍夫曼树而不是平衡二叉树？
  - A：霍夫曼树让高频词路径最短，最小化平均训练时间；平衡树对所有词一视同仁，不最优。
- Q：层次softmax和负采样哪个更好？
  - A：看场景；词表极大、词频差异大时层次softmax优；词表中等时常用负采样更简单。
- Q：每个内部节点的参数怎么更新？
  - A：只更新从根到目标词路径上的节点参数，其他节点不涉及本轮梯度。
