# 模型量化是什么？INT4/INT8 原理？

## 速记卡片
`缩 | 缩格填，八稳四狠，护权补偿 | 量化原理`

## 原始知识
- **量化本质**：将 FP32/FP16 浮点权重映射到低精度整数（INT8/INT4），减存加速
- **映射公式**：`q = round((r - zero_point) / scale)`
  - **scale**：缩放因子（量程 / 255 或 15）
  - **zero_point**：零点偏移（非对称量化用）
- **INT8**：256 个离散值，较稳，精度损失小
- **INT4**：16 个离散值，压缩狠，信息损失大，需特殊处理
- **PTQ（后训练量化）**：训后直接转，简单快速
- **QAT（量化感知训练）**：训练时模拟低精度，精度好但成本高
- **GPTQ**：逐层贪婪量化，利用 Hessian 矩阵补偿误差
- **AWQ**：保护 1% 显著权重（salient weights），对输出影响大的保持高精度
- **GGUF**：llama.cpp 格式，混合精度（如 Q4_K_M = attention 高精 + FFN 低精）
- **易混点**：量化 ≠ 蒸馏，量化 ≠ 剪枝
- **追问点**：为什么量化能加速？INT4 为什么还能用？QAT vs PTQ？

## 顺口溜
**缩格填，八稳四狠，护权补偿**

## 单字锚点 + 防崩推导
- **锚点字**：**缩**
- **扩展口诀**：**缩格填，浮变整**
- **为什么选它**：量化的核心动作就是"缩"——缩小精度、缩小体积、缩小带宽。
- **防崩推导**：
  - 缩 → 缩小精度 → 浮点变整数
  - 缩 → 格子不够 → 算 scale + zero_point → 填进格子
  - 八 → INT8 → 256 格 → 稳
  - 四 → INT4 → 16 格 → 狠 → 要分组 / 护权 / 补偿
  - 护权 → AWQ 保护 1% 显著权重
  - 补偿 → GPTQ 逐层 Hessian 补偿

## 一字一画面
- **缩** —— 像**把大象塞进行李箱**：先称体重算比例（scale），偏胖的减掉平均值（zero_point），再按格子塞进去。（量化映射）
- **格** —— 像**连续的温度计被掰成刻度尺**：FP32 是水银柱任意位置，INT8 只有 256 道刻度线，INT4 只剩 16 道。（离散化）
- **八** —— 像**八车道高速公路**：路宽车多（256 级），通行稳当，一般模型直接跑。（INT8 稳定）
- **四** —— 像**独木桥过车队**：只有 16 个位置，必须精挑细选（AWQ 保护 VIP），分批过桥（分组量化），掉下去就精度崩塌。（INT4 极限）
- **护** —— 像**护送重要人物走专机**：AWQ 发现 1% 显著权重，不让它们挤独木桥。（保护显著权重）
- **补** —— 像**修桥补路**：GPTQ 量化完一层后发现误差，用 Hessian 矩阵信息把路修平。（误差补偿）

## 面试话术
- 问量化原理 → "模型量化就是把浮点权重压缩成低精度整数，核心口诀是**缩格填，八稳四狠，护权补偿**。INT8 有 256 个刻度，比较稳定，训后直接 PTQ 就行；INT4 只有 16 个刻度，压缩狠但信息损失大，需要分组量化，或者用 AWQ 保护显著权重、GPTQ 逐层补偿误差。"
- 问加速原因 → "两点：一是低精度整数运算比浮点运算更快，二是模型体积缩小后内存带宽压力降低，缓存命中率提高，IO 不再是瓶颈。"

## 防混补丁
- 容易和"模型蒸馏""模型剪枝"搞混。
- 补丁：**蒸馏是拜师学艺（小模型学大模型），剪枝是砍掉枝叶（去掉不重要的神经元），量化是减肥瘦身（同模型砍精度）。**

## 追问预演
1. **为什么量化能加速推理？**
   → 整数运算指令比浮点快；模型体积减半，内存带宽不再是瓶颈，缓存利用率高。

2. **INT4 只有 16 个值，为什么还能用？**
   → 靠分组量化（per-channel / per-group）降低每组动态范围；AWQ 保护 1% 显著权重；GPTQ 用 Hessian 逐层补偿；GGUF 采用混合精度策略。

3. **GPTQ 和 AWQ 的核心区别？**
   → GPTQ 是逐层贪婪量化 + Hessian 误差补偿，关注"怎么量得更准"；AWQ 是基于激活感知保护显著权重，关注"哪些权重不该被量"。

4. **QAT 和 PTQ 怎么选？**
   → 追求速度和简单选 PTQ（训后直接转）；追求精度且能承受训练成本选 QAT（训练时模拟低精度）。

5. **GGUF 的 Q4_K_M 是什么意思？**
   → Q4 = 4bit 基线，K = K-quants 混合精度策略，M = medium，表示 attention 等敏感层用更高精度，FFN 用更低精度。

6. **对称量化 vs 非对称量化？**
   → 对称：zero_point = 0，适合分布对称的权重；非对称：有 zero_point 偏移，适合 ReLU 激活后全为正数的分布。
