# 什么是张量并行、流水线并行

## 速记卡片
`切 | 张横流纵，多通有泡 | 横纵切分`

## 原始知识
- **张量并行（TP）**：横向切分，切**层内**参数（矩阵行/列）
- TP：同层不同GPU各算一部分，forward/backward后需 all-reduce 汇总
- TP：**通信量大**且频繁，依赖高带宽（NVLink），适合**节点内**
- **流水线并行（PP）**：纵向切分，切**层间**（不同stage给不同GPU）
- PP：数据像水流过各GPU，只需传递激活值，**通信量小**
- PP：存在**流水线气泡**（bubble），前后GPU可能空等
- PP：适合**节点间**，常配合 micro-batching / 1F1B 缓解气泡
- **易混点**：两者都属于模型并行，但切分维度正交
- **追问点**：什么场景下TP和PP会联合使用？

## 顺口溜
**张横流纵，多通有泡**

## 单字锚点 + 防崩推导
- **锚点字**：**切**
- **扩展口诀**：**切横纵**
- **为什么选它**：两种并行的本质分水岭就是"怎么切"——横切层内 vs 纵切层间。
- **防崩推导**：
  - 切 → 怎么切？
  - **横**切 → 张量并行 → 切层内（矩阵行/列）→ 同层不同GPU各算一块 → 算完要汇总 → **通信多**（all-reduce）
  - **纵**切 → 流水线并行 → 切层间（前N层给GPU0，后N层给GPU1）→ 数据流水传递 → 前后GPU可能空等 → **有气泡**
  - 横切靠带宽 → 节点内（NVLink）；纵切靠调度 → 节点间 + micro-batch

## 一字一画面
- **横** —— 像**横切披萨**：一个披萨横着切成几扇，每扇都有饼底到芝士的全部层（层内全切），大家同时啃，但每啃一口都要互相通报进度（all-reduce通信多）。（本质：层内切分，通信密集）
- **纵** —— 像**纵切三明治**：第一层生菜给A、第二层火腿给B、第三层面包给C，一个人啃完传给下一个（层间流水），中间有人干等着没事做（流水线气泡）。（本质：层间切分，流水传递）
- **多** —— 像**开会对齐**：横切后每个step所有人要集体举手表决（all-reduce），会议室门都被踩烂了。（本质：高频同步，带宽敏感）
- **泡** —— 像**工厂空转**：流水线前端缺料停工、后端等成品也停工，中间的闲置时间就是气泡。（本质：并行度无法拉满）

## 面试话术
1. 问区别 → "核心区别就一个字：**切**。张量并行是**横切**层内，把一层矩阵拆开给多卡算，通信多所以放节点内；流水线并行是**纵切**层间，把不同层分给不同卡，像工厂流水线，通信少但有气泡。口诀叫**张横流纵，多通有泡**。"
2. 问怎么用 → "实际大模型训练常**三层叠加**：节点内TP（NVLink扛住all-reduce），节点间PP（流水线传激活），最外层再套DP（数据并行扩batch）。比如GPT-3就是这么训的。"

## 防混补丁
- 容易和"数据并行"搞混。补丁：**数复全模，横纵切** —— 数据并行是每个GPU复制完整模型，只是数据不同；TP/PP是把模型本身切开，是模型并行。

## 追问预演
- **Q：张量并行为什么必须放在节点内？**
  A：因为每层forward/backward都要做all-reduce，通信量巨大且高频。节点内NVLink带宽动辄几百GB/s，节点间IB/以太网带宽差一个数量级，放节点外会被通信拖死。
- **Q：流水线气泡怎么缓解？**
  A：micro-batching（把batch拆成多个micro-batch流水线填充）、1F1B（PipeDream，一个forward紧跟一个backward减少显存）、interleaved pipeline（交错多个stage）等。
- **Q：什么场景下TP和PP会联合使用？**
  A：超大规模模型（千亿/万亿参数）。单节点8卡TP切一层还不够，必须跨节点PP切多层，再外层数据并行扩规模。这就是3D并行的经典范式。
