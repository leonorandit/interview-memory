# ReAct 是什么

## 速记卡片
`交 | 推实交，环迭代 | 推理行动`

## 原始知识
- ReAct（Reasoning + Acting）：推理和行动交替进行的Agent模式
- 核心循环：
  1. **推**（Thought/推理）- 分析当前状态，决定下一步
  2. **实**（Action/行动）- 执行工具调用
  3. **交**（交替）- 推理和行动交替进行
  4. **环**（循环）- 反复执行直到完成
  5. **迭代**（迭代）- 每轮根据观察结果调整策略
- 观察（Observation）：行动后获取反馈，进入下一轮推理
- **易混点**：ReAct vs CoT
- **追问点**：ReAct优势？什么时候不好用？

## 顺口溜
**推实交，环迭代**

## 单字锚点 + 防崩推导
- **锚点字**：**交**
- **扩展口诀**：**推实交，环迭代**
- **为什么选它**：ReAct的核心是"推理和行动交替"，"交"是灵魂。
- **防崩推导**：
  - 交（交替）→ 推理和行动交替
  - 推（推理）→ Thought，分析思考
  - 实（行动）→ Action，执行工具
  - 环（循环）→ 反复执行
  - 迭代 → 每轮调整策略

## 一字一画面
- **推** —— 像**军师想对策**：先想清楚怎么打。（本质：推理Thought）
- **实** —— 像**将军去打仗**：按对策执行。（本质：行动Action）
- **交** —— 像**轮番出牌**：你出一张我出一张。（本质：交替进行）
- **环** —— 像**循环赛**：打一轮看结果，再打下一轮。（本质：循环迭代）

## 面试话术
- 问ReAct → "ReAct是Reasoning+Acting的Agent模式，口诀是**推实交，环迭代**。**推**——推理（Thought），分析当前状态决定下一步；**实**——行动（Action），调用工具执行；**交**——推理和行动交替进行；**环**——反复循环；**迭代**——每轮根据观察结果（Observation）调整策略。比如问'北京今天天气'，Agent先推理'需要查天气'→行动'调用天气API'→观察'返回晴天25度'→推理'任务完成'→输出答案。"

## 防混补丁
- 容易和"CoT"搞混。CoT只推理不行动，ReAct推理+行动。补丁：**CoT是想一想，ReAct是想一做一**。"
