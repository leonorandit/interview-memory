## 速记卡片
`焦 | 聚焦权重破长局 | 动态加权，长距离依赖+可解释性`

## 原始知识
- **核心思想**：Query 查 Key，加权聚合 Value
- **权重动态**：每步权重随输入变化
- **突破长距**：任意两 token 直接交互
- **并行计算**：矩阵运算一次搞定
- **多头机制**：多子空间并行聚焦
- **可解释性**：权重矩阵直观展示关注关系

## 顺口溜
Query 查 Key，加权取 Value，多头并行破长局。

## 单字锚点 + 防崩推导
- **锚点字**：焦
- **扩展口诀**：聚焦权重破长局，Query Key Value 三元组
- **为什么选它**：注意力机制的本质就是"聚焦"——让模型自动决定该把注意力"焦点"放在输入序列的哪些位置上
- **防崩推导**：
  1. 想到"焦"→焦点、聚焦、焦距
  2. 怎么聚焦？→计算 Query 和 Key 的相似度（点积/缩放点积）
  3. 相似度归一化→softmax 得到注意力权重分布
  4. 权重乘 Value→加权求和得到输出
  5. 为什么破长局？→传统 RNN/CNN 距离远的信息难交互，注意力让任意位置直接连边
  6. 多头？→多个 Q/K/V 投影，不同头关注不同语义关系

## 一字一画面
**焦** —— 像**舞台上的追光灯扫过一排演员**：黑暗观众席中，一束白色强光（Query）从左到右快速扫过；照到第三位红衣演员（Key 匹配度高）时，灯光骤然变亮并定格，其余演员隐入暗影；镜头只把这位演员的特写（Value 加权）投射到银幕中央。（本质：动态选择信息源，无视空间距离）

## 面试话术
"注意力机制的核心就是让模型‘聚焦’到输入中最相关的部分。具体来说，它把输入映射成 Query、Key、Value 三组向量，用 Query 和 Key 的相似度算出权重，再用权重对 Value 做加权聚合。这样模型在处理每个位置时，都能直接‘看’到序列中任意距离的相关信息，既解决了长距离依赖问题，又天然具有可解释性——我们可以通过注意力权重矩阵看到模型到底关注了哪里。"

## 防混补丁
- **Attention vs 全连接**：全连接权重固定，Attention 权重随输入动态变化
- **Self-Attention vs Cross-Attention**：前者 Q/K/V 同源，后者 Q 来自解码器、K/V 来自编码器
- **缩放点积**：除以 √d_k 防 softmax 梯度消失，不是可有可无
- **注意力 ≠ 记忆网络**：注意力是软寻址，记忆网络是显式外部存储
- **防崩**：Q 查 K 得权重，权重乘 V 得输出，三步绝不乱

## 追问预演
Q：注意力机制和全连接层有什么本质区别？
A：全连接的权重是参数固定的，注意力的权重是输入的函数，随样本动态变化。

Q：为什么要用多头注意力而不是单头？
A：单头只能在一个语义子空间做聚焦，多头可以在不同子空间并行捕捉不同类型的依赖关系，比如语法关系和指代关系。

Q：Transformer 中自注意力的时间复杂度是多少？
A：O(n²·d)，n 是序列长度，d 是维度；这也是长序列优化的核心瓶颈。
