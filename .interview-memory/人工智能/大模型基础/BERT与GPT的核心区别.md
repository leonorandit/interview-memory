# BERT与GPT的核心区别

## 速记卡片
`双 | 双掩理，单因生 | BERT vs GPT 核心差异`

## 原始知识
- **双（双向）**：BERT 是 Transformer Encoder，双向看上下文，同时利用左右信息
- **掩（Masked LM）**：BERT 预训练用 Masked Language Model，随机 mask 15% 的词然后预测，像完形填空
- **理（理解任务）**：BERT 擅长理解类任务——文本分类、NER、问答、句子相似度
- **单（单向）**：GPT 是 Transformer Decoder，自回归单向（left-to-right），只能看左边的词
- **因（Causal LM）**：GPT 预训练用 Causal / Autoregressive Language Model，预测下一个词
- **生（生成任务）**：GPT 擅长生成类任务——文本续写、对话、代码生成
- **架构本质**：BERT = Encoder，GPT = Decoder；自注意力掩码策略不同——BERT 无掩码，GPT 用上三角掩码防偷看
- **易混点**：BERT 也能生成、GPT 也能做分类；自注意力本身不区分双向/单向
- **追问点**：为什么 GPT 不能双向、RoBERTa 改进、T5 的 Encoder-Decoder 设计

## 顺口溜
**双掩理，单因生**

## 单字锚点 + 防崩推导
- **锚点字**：**双**
- **扩展口诀**：**双掩理，单因生**
- **为什么选它**：BERT vs GPT 的本质分野就是"双向还是单向"，"双"字一出来，对比框架就立住了。
- **防崩推导**：
  - 双 → BERT 双向 → Transformer Encoder → 同时看上下文 → 自注意力无掩码
  - 掩 → Masked LM → 随机 mask 词 → 完形填空式预训练
  - 理 → 理解任务 → 分类/NER/QA/相似度 → 下游接分类头
  - 单 → GPT 单向 → Transformer Decoder → 只能看左边 → 自回归
  - 因 → Causal LM → 预测下一个词 → 逐字生成
  - 生 → 生成任务 → 续写/对话/代码 → 自回归天然匹配序列生成

## 一字一画面
- **双** —— 像**侦探看监控**：前后左右画面同时看，找线索。（BERT 双向看上下文）
- **掩** —— 像**完形填空**：把文章里几个词抠掉，让你根据上下文猜。（Masked LM）
- **理** —— 像**阅读理解**：读完后做选择题、判断题。（理解类任务）
- **单** —— 像**打字接龙**：只能看已经打出来的字，猜下一个字。（GPT 单向自回归）
- **因** —— 像**因果链**：因为前面说了 A，所以后面接 B。（Causal LM）
- **生** —— 像**故事接龙**：一个人说一句，下一个人接着编。（生成类任务）

## 面试话术
- 问区别 → "BERT 和 GPT 的核心区别我总结为**双掩理，单因生**。**双**是 BERT 双向看上下文，像侦探同时看前后监控；**掩**是 Masked LM 预训练，像完形填空；**理**是适合理解任务，比如分类、NER、问答。**单**是 GPT 单向自左向右，像打字接龙；**因**是 Causal LM 预测下一个词；**生**是适合生成任务，比如文本续写、对话生成。本质上是 Encoder 和 Decoder 两种架构路线的分野。"
- 问架构 → "BERT 用 Encoder，自注意力不加掩码，所以能双向；GPT 用 Decoder，自注意力加上三角掩码，防止'偷看'未来词，所以只能单向。这是由预训练目标决定的。"

## 防混补丁
- 容易以为"BERT 不能生成"。实际上 BERT 也可以做生成（比如用掩码迭代预测），只是效果远不如 GPT。**补丁：BERT 能生但差，GPT 专精生成。**
- 容易把"双向"和"自注意力"搞混。自注意力机制本身不区分双向/单向，关键是位置编码和掩码策略。**补丁：自注是枪，掩码是保险栓。**

## 追问预演
1. **为什么 GPT 不能双向？** 答：GPT 的自回归预训练目标是预测下一个词，如果双向，模型就能"偷看"答案，任务就 trivial 了。BERT 的 MLM 同时 mask 多个词，所以双向不会泄露目标。
2. **RoBERTa 相比 BERT 改了什么？** 答：去掉了 NSP 任务，用更大批次、更多数据、动态掩码训练，效果提升明显。
3. **GPT 为什么能聊天而 BERT 不能？** 答：GPT 的自回归训练天然匹配对话的"一问一答"序列生成模式；BERT 的双向结构没有序列生成的自回归能力。
4. **T5 的 Encoder-Decoder 和这两者什么关系？** 答：T5 结合了两边，Encoder 做双向理解，Decoder 做单向生成，适合做翻译、摘要等 Seq2Seq 任务。
