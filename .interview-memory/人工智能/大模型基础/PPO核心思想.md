# PPO核心思想

## 速记卡片
`裁 | PPO裁比率，稳定不剧变 | 限制更新幅度`

## 原始知识
- 属于**策略梯度**方法，解决传统 PG 方差大、样本效率低
- 核心：**裁剪 Surrogate Objective**，限制新旧策略比率
- 用一阶优化替代 TRPO 的二阶约束，**实现简单、计算轻**
- 比率公式：r_t = π_new(a|s) / π_old(a|s)，裁剪到 [1-ε, 1+ε]
- 兼顾探索与稳定，避免策略**突变崩盘**
- **易混点**：PPO 不是 off-policy，是 on-policy 的改进
- **追问点**：RLHF 第三阶段为什么选 PPO？

## 顺口溜
**PPO裁比率，稳定不剧变**

## 单字锚点 + 防崩推导
- **锚点字**：**裁**
- **扩展口诀**：**裁剪比率稳**
- **为什么选它**：PPO 的灵魂操作就是"裁剪"，没有裁剪就没有近端约束。
- **防崩推导**：
  - 裁 → 裁剪目标函数 → 不让策略跳太远
  - 比率 → 新旧策略概率比 r_t
  - 稳 → 梯度小步走 → 性能不崩盘

## 一字一画面
- **裁** —— 像**裁纸刀修边**：把策略更新的毛边裁掉，只允许在规定范围内变动。（裁剪目标函数）
- **比** —— 像**天平称重**：新旧策略概率比率 r_t，中心是 1，偏了就被拉回来。（重要性采样比率）
- **稳** —— 像**慢火炖汤**：小火慢炖，汤不会溢出来。（稳定训练，不崩盘）

## 面试话术
- 问核心思想 → "PPO 的核心是**裁剪**。它通过限制新旧策略的比率 r_t，把更新幅度控制在 [1-ε, 1+ε] 之间，既保证了策略改进，又避免了像传统策略梯度那样更新过猛导致性能崩溃。相比 TRPO，PPO 用一阶优化就能实现类似效果，实现简单，计算量小。"
- 问 RLHF 为什么用 PPO → "因为 RLHF 里奖励模型打分不稳定，PPO 的裁剪机制能防止模型为了刷分而走火入魔，保证更新稳定。"

## 防混补丁
- 容易和 **TRPO** 搞混。TRPO 用 KL 散度做二阶约束，计算复杂；PPO 直接裁剪，一阶搞定。补丁：**TRPO算KL，PPO直接裁**。
- 容易以为 PPO 是 off-policy。补丁：**PPO是on-policy的裁剪版**，数据用过即废，不能反复吃。

## 追问预演
- **PPO 和 TRPO 的本质区别？** TRPO 用 Fisher 信息矩阵做二阶 KL 约束，计算量大；PPO 用 clip 做一阶近似，简单高效。
- **ε 一般取多少？** 通常 0.1 或 0.2。
- **RLHF 里 PPO 怎么工作的？** 策略模型生成回答，奖励模型打分，PPO 根据奖励更新策略，裁剪防止模型偏离太远。
- **PPO 的劣势？** 仍是 on-policy，样本效率不如 off-policy 方法（如 SAC）。
