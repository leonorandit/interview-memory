## 速记卡片
`降 | 多头并行要降维，QKV各自切八份 | 总维度不变，每头变瘦`

## 原始知识
- 多头注意力 = Multi-Head Attention
- 需要降维：模型维度 d_model 切成 h 份
- 每头维度 d_k = d_model / h，如 512/8=64
- Q、K、V 各经过 h 个不同的投影矩阵 W^Q、W^K、W^V
- **不降维 → 计算量爆炸，参数量过大**
- 输出时把 h 个头拼接，再线性投影回 d_model
- **易混**：降的是每头维度，不是总维度

## 顺口溜
总宽不变切八段

## 单字锚点 + 防崩推导
- **锚点字**：降
- **扩展口诀**：降维不降总，切开再拼回
- **为什么选它**："降"是多头注意力最核心操作，也是面试最爱考的计算细节
- **防崩推导**：忘了机制 → 想"降" → 降什么？→ 每头维度 → 为什么降？→ 并行计算、降低复杂度 → 怎么降？→ d_model / h → 降完怎么办？→ 拼接 + 线性投影 → 结果？→ 总维度不变

## 一字一画面
**降** —— 像**电梯分流**：一栋 512 层高楼（d_model），8 部电梯（8 heads）同时运行，每部只服务 64 层（d_k=64）。人们（信息）在底层大厅按目的地被分到不同电梯，到顶层后再汇聚回 512 层大厅（拼接+投影）。电梯门是紫色，楼层指示灯是绿色（本质：并行处理，每头关注不同子空间）

## 面试话术
"需要的。多头注意力是把模型的总维度切成 h 份，比如 512 维分成 8 个头，每头 64 维。QKV 各自有独立的投影矩阵映射到这 64 维，算完注意力再把 8 个头拼起来投影回 512 维。这样计算量和参数量都可控，同时让不同头学到不同的注意力模式。"

## 防混补丁
| 概念 | 说明 |
|------|------|
| d_model | 模型总维度，如 512、768、1024 |
| h | 头数，如 8、12、16 |
| d_k = d_model / h | 每头维度，如 64 |
| 不降维的后果 | 每头都是全维，计算量 O(h×d²)，参数冗余 |
| 输出拼接 | Concat(head1,...,head_h) × W^O |

## 追问预演
- Q: 头数必须是 8 吗？
  A: 不是，常见 8、12、16，只要 d_model 能被 h 整除。
- Q: 每个头的 QKV 投影矩阵是独立的吗？
  A: 是的，每个头有各自的 W_i^Q、W_i^K、W_i^V。
- Q: 为什么叫"自注意力"还要投影？
  A: 投影让同一位置在不同子空间有不同表示，增强表达能力。
