# 大模型的位置编码是干什么用的？sin/cos、RoPE、ALiBi 有什么区别？

## 速记卡片
`弦 | 弦三变，贴转斜 | 绝对vs相对vs偏置`

## 原始知识
- **作用**：Transformer 自注意力是**位置无关**的，打乱 token 顺序不影响计算。位置编码注入位置信息，让模型知道词语顺序。
- **sin/cos**：原始 Transformer 提出，正弦/余弦函数生成，**加到输入嵌入**上，**绝对位置**编码。理论可外推，实际一般。
- **RoPE**：通过**旋转矩阵**将位置信息注入 Q/K 向量，**相对位置**编码。注意力内积自然体现相对距离，外推性好。**LLaMA、Qwen、ChatGLM** 在用。
- **ALiBi**：不在嵌入层加位置编码，直接在**注意力分数**上加负的**线性偏置**（斜率 × 距离），**相对位置**。训练稳定，外推性最好。**BLOOM** 在用。
- **追问点**：三种编码的注入位置差异、RoPE 旋转的具体实现、外推性优劣原因

## 顺口溜
**弦三变，贴转斜**

## 单字锚点 + 防崩推导
- **锚点字**：**弦**
- **扩展口诀**：**弦三变，贴转斜**
- **为什么选它**："弦"是正弦/位置编码的浓缩，"三变"直接点出三种变体，一听就知道是位置编码的三种形态。
- **防崩推导**：
  - 弦 → 正弦/位置编码 → 解决自注意力位置无关
  - 三变 → 三种变体
  - 贴 → sin/cos → 正弦函数 → 贴到输入嵌入 → 绝对位置
  - 转 → RoPE → 旋转矩阵 → Q/K 向量旋转 → 相对位置 → LLaMA/Qwen
  - 斜 → ALiBi → 线性偏置 → 注意力分数减分 → 越远分越低 → 外推最好 → BLOOM

## 一字一画面
- **弦** —— 像 **琴弦的振动波形**：不同位置拨弦，波形不同，听众（模型）能分辨音高（位置）。（本质：用周期性函数编码位置信息）
- **贴** —— 像 **额头贴号码牌**：每个观众入场时贴上排号，直接叠加在人身上。（本质：sin/cos 绝对位置编码，嵌入层相加）
- **转** —— 像 **旋转木马**：观众坐在木马上转，两人能不能对视（注意力）取决于相对角度，不是绝对位置。（本质：RoPE 旋转 Q/K，注意力内积体现相对距离）
- **斜** —— 像 **滑梯扣分**：两人距离越远，滑下去扣的分越多，越远越难建立联系。（本质：ALiBi 在注意力分数上加负的线性偏置）

## 面试话术
- 问作用 → "位置编码解决 Transformer 自注意力的**位置无关**问题。没有它，打乱词语顺序模型算出来结果一样。"
- 问三种区别 → "我总结为**弦三变，贴转斜**。sin/cos 是绝对位置，把正弦标签贴到输入嵌入上；RoPE 通过旋转 Q/K 向量编码相对位置，目前 **LLaMA、Qwen** 等主流模型在用；ALiBi 是在注意力分数上拉一条负斜线，越远扣分越多，外推性最好，**BLOOM** 在用。"

## 防混补丁
容易把三种编码的**注入位置**搞混。

## 追问预演
1. **为什么需要位置编码？**
   → 自注意力机制是位置无关的，对输入做置换不影响输出。位置编码注入位置信息，让模型感知序列顺序。

2. **RoPE 为什么比 sin/cos 好？**
   → sin/cos 是绝对位置，只能编码固定位置；RoPE 是相对位置编码，注意力内积直接反映 token 间的相对距离，更符合语言建模需求，且外推性更好。

3. **ALiBi 为什么外推性最好？**
   → ALiBi 不做显式位置嵌入，只在注意力分数上加一个与距离成正比的负偏置。训练时模型学会适应这种固定模式，遇到更长序列时偏置规则不变，因此泛化到长序列最稳定。

4. **三种编码的注入位置有什么区别？**
   → sin/cos 在**输入嵌入层**相加；RoPE 在注意力计算的 **Q/K 向量**上乘旋转矩阵；ALiBi 在**注意力分数**（softmax 之前）加线性偏置。

5. **现在的主流大模型用哪种位置编码？**
   → LLaMA 系列、Qwen、ChatGLM、Baichuan 等主要用 **RoPE**；BLOOM 用 **ALiBi**；早期 GPT 使用可学习的位置嵌入（Learned Positional Embedding）。

6. **RoPE 的旋转具体怎么实现？**
   → 将 Q/K 向量的每两个维度视为一个复数，乘以与位置相关的旋转因子 e^(i·m·θ_j)，相当于在二维平面上旋转 m·θ_j 角度。多个二维旋转组合成高维旋转矩阵。
