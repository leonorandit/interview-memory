## 速记卡片
`后 | 后台干活人不候 | 异步并行提效率`

## 原始知识
- **Background Agent**：后台自主运行 Agent
- **特征**：无需人实时盯着，异步执行
- **改变一**：从同步对话变异步任务
- **改变二**：从单轮到多轮长时运行
- **改变三**：人机协作变机机协作
- **加粗易混点**：不是简单的**后台脚本**，而是有决策能力的 Agent

## 顺口溜
后台 Agent 跑，程序员睡觉。

## 单字锚点 + 防崩推导
- **锚点字**：后
- **扩展口诀**：后方坐镇指挥，前线自动执行
- **为什么选它**："后"直接对应 Background，且自带"不在前台"的意象
- **防崩推导**：后 → 后台 → 不阻塞用户 → 异步执行 → 长时任务 + 多 Agent 并行

## 一字一画面
**后** —— 像 **深夜自动扫地机器人**：主人已睡（人离开），它在黑暗中默默规划路线、避开障碍、清扫全屋，天亮前归位充电（本质：Background Agent 在用户离线时自主完成复杂任务链）

## 面试话术
Background Agent 就是能在后台自主运行的智能体，不需要你一句一句跟它对话。它把 AI 编程从"你问我答"的同步模式，变成了"布置任务后它自己搞"的异步模式，特别适合长时任务、定时巡检和多步骤自动化。

## 防混补丁
- Background Agent ≠ Cron 脚本：前者有决策和容错能力
- 需要状态持久化：否则重启就失忆

## 追问预演
- Q：怎么知道它跑完了？A：回调通知、Webhook 或轮询状态接口
- Q：出错了怎么办？A：重试策略 + 人工介入机制
