# KV Cache是什么为什么推理必备

## 速记卡片
`存 | KV存前步，推理省复算 | 显存换速度`

## 原始知识
- 自注意力中每个 token 产 Q、K、V，当前 token 需与**所有历史 token** 做 attention
- 不缓存：每步重新计算全部历史 K、V，复杂度 **O(n²)**
- KV Cache：**缓存历史 token 的 K、V**，每步只算当前 token，拼接后做 attention
- 效果：计算量从平方降为线性，**推理速度大幅提升**
- 代价：显存随序列长度**线性增长**
- **易混点**：KV Cache vs Prompt Caching
- **追问点**：GQA/MQA 怎么优化 KV Cache 显存？

## 顺口溜
**KV存前步，推理省复算**

## 单字锚点 + 防崩推导
- **锚点字**：**存**
- **扩展口诀**：**存前省复算**
- **为什么选它**：KV Cache 的本质就是"存"，没有存就没有加速。
- **防崩推导**：
  - 存 → 缓存前面 token 的 K、V
  - 前 → 只存历史，不存未来（自回归）
  - 省 → 省去重复计算
  - 复 → 复用历史结果
  - 算 → 推理计算量从 O
