# Transformer 模型训练完成后，如何评估其性能和效果

## 速记卡片
`测 | 测准鲁，多真速 | 多维度评估`

## 原始知识
- 困惑度PPL：语言模型核心指标
- BLEU/ROUGE：生成质量
- 下游任务准确率：分类、抽取等
- 人工评估：流畅度、相关性、安全性
- 推理速度：吞吐量、延迟
- 鲁棒性：对抗样本、长文本、多轮对话
- 真实性：幻觉检测、事实一致性
- **易混点**：PPL低不等于生成质量好

## 顺口溜
测准鲁，多真速

## 单字锚点 + 防崩推导
- **锚点字**：**测**
- **扩展口诀**：**测多维**
- **为什么选它**："测"是评估的直接动作字，听到"评估"立即想到"测"。
- **防崩推导**：
  - 测 → 评估 → 不能只看一个数
  - 准 → 准确率/PPL/BLEU → 自动指标
  - 鲁 → 鲁棒性 → 抗干扰能力
  - 多 → 多任务/多维度 → 不能只看一种任务
  - 真 → 真实性/幻觉 → 事实对不对
  - 速 → 推理速度 → 线上吞吐和延迟

## 一字一画面
- **测** —— 像**体检报告**：不止量身高，还要查血压、验血、拍片。（全面评估）
- **准** —— 像**打靶环数**：看命中多少环（PPL、BLEU、ROUGE）。（自动指标）
- **鲁** —— 像**抗震测试**：房子不仅要站得住，还要扛得住地震。（鲁棒性）
- **多** —— 像**全能运动会**：不能只会跑步，游泳、跳高都要测。（多任务）
- **真** —— 像**测谎仪**：说得天花乱坠，事实对不对？（防幻觉）
- **速** —— 像**百米赛跑**：不光到终点，还要计时。（推理性能）

## 面试话术
- "评估不能只看一个指标，我概括为**测准鲁，多真速**。自动指标看PPL、BLEU、ROUGE；下游任务看具体准确率；还要测鲁棒性、多轮对话能力、事实一致性防幻觉；最后线上必须看推理速度和吞吐量。PPL低不代表生成质量高，人工评估必不可少。"

## 防混补丁
- 容易只看PPL。补丁：**PPL是语文分数，生成质量是作文水平，两者不完全等同**。

## 追问预演
- **Q：PPL低为什么生成质量不一定好？** A：PPL衡量概率建模能力，但生成质量还涉及多样性、相关性、事实性，模式崩溃的模型PPL可能很低。
- **Q：人工评估和自动指标怎么配合？** A：自动指标快速迭代筛选，人工评估最终把关；上线前必须做A/B测试和人工抽检。
