## 速记卡片
`批 | 大数据分批次，分布式并行处理 | 批+分布+梯度累积`

## 原始知识
- 大数据处理三板斧：批训练、分布式、梯度累积
- Batch：数据分 batch，平衡显存和稳定性
- Gradient Accumulation：小 batch 模拟大 batch
- 分布式：数据并行（DP/DDP）、模型并行（MP/TP）、流水线并行（PP）
- **数据并行最常用，每个 GPU 算一个子集**
- 混合精度训练（AMP）省显存、加速
- **易混**：batch size 和 learning rate 要联动调整

## 顺口溜
分批并行进，累积梯度稳

## 单字锚点 + 防崩推导
- **锚点字**：批
- **扩展口诀**：一批二并三累积，四混合五检查点
- **为什么选它**："批"是处理大数据最直接的手段，也是面试高频词
- **防崩推导**：忘了方法 → 想"批" → 批什么？→ 数据 → 批不下怎么办？→ 分布式 → 显存不够怎么办？→ 梯度累积 / 混合精度 → 模型太大怎么办？→ 模型并行 / 流水线并行 → 训练中断怎么办？→ Checkpoint 断点续训

## 一字一画面
**批** —— 像**工厂流水线分包裹**：巨大的仓库（数据集）里，红色机械臂把包裹（样本）分成一个个绿色托盘（batch），8 条传送带（8 GPUs）并行运转，每条带末端都有一个银色漏斗（梯度累积）把小批梯度攒成大份再更新。天花板上有蓝色节能灯（混合精度），一闪一闪表示 FP16 计算中（本质：通过分片和并行化将大数据吞吐适配到有限算力）

## 面试话术
"处理大数据主要靠三方面：一是分 batch 训练，显存放不下就用梯度累积模拟大 batch；二是分布式，数据并行让多卡同时算不同子集，模型太大就加模型并行或流水线并行；三是混合精度训练省显存。还会定期存 checkpoint 防止训练中断。"

## 防混补丁
| 技术 | 解决的问题 |
|------|-----------|
| Data Parallel | 数据太大，单机算不完 |
| Model Parallel | 模型太大，单卡放不下 |
| Pipeline Parallel | 层数太多，串行太慢 |
| Gradient Accumulation | 显存小，想要大 batch |
| Mixed Precision | 省显存、加速计算 |
| Checkpoint | 容错、断点续训 |

## 追问预演
- Q: 数据并行和模型并行怎么选？
  A: 模型能放进单卡 → 数据并行；模型太大 → 模型并行+数据并行。
- Q: 梯度累积为什么能模拟大 batch？
  A: 多步小 batch 的梯度累加后再更新，等效于一次大 batch 的梯度。
- Q: 混合精度训练会有精度损失吗？
  A: 一般不明显，因为保留了 FP32 主权重，只有前反向用 FP16。
