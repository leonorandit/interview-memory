# Transformer 模型的性能瓶颈在哪

## 速记卡片
`颈 | 颈长方，内参激 | 长度平方·内存·参数量`

## 原始知识
- Self-Attention复杂度O(n²)，序列长度平方增长
- FFN计算量大，参数量占模型2/3
- 内存带宽瓶颈：显存读写速度跟不上计算
- 参数量大，加载和传输慢
- 激活值存储占用高
- KV Cache推理时随长度线性增长
- **易混点**：训练瓶颈和推理瓶颈不同

## 顺口溜
颈长方，内参激

## 单字锚点 + 防崩推导
- **锚点字**：**颈**
- **扩展口诀**：**颈卡长**
- **为什么选它**："颈"直接对应"瓶颈"，是最直观的记忆钩子。
- **防崩推导**：
  - 颈 → 瓶颈 → 哪里卡住了
  - 长 → 序列长度 → 长文本灾难
  - 方 → 平方复杂度O(n²) → Attention随长度爆炸
  - 内 → 内存带宽 → GPU算得快但显存读写慢
  - 参 → 参数量巨大 → 加载慢、传输慢
  - 激 → 激活值存储 → 每层前向传播产生大量中间结果

## 一字一画面
- **颈** —— 像**窄脖子花瓶**：肚子大，口小，东西进出慢。（瓶颈本质）
- **长** —— 像**长蛇阵**：人越多，队尾传到队首的消息量呈平方爆炸。（序列长度）
- **方** —— 像**广场握手**：10人握手45次，100人握手4950次，人数翻倍次数翻四倍。（O(n²)）
- **内** —— 像**小水管灌大池子**：GPU算得快，但显存读写像细水管，供不上。（内存带宽）
- **参** —— 像**超重货车**：参数就是货物，车越大起步越慢。（参数量）
- **激** —— 像**工厂废料**：每加工一层都产生大量激活废料，堆满仓库。（激活值）

## 面试话术
- "Transformer瓶颈可以概括为**颈长方，内参激**。第一是序列长度的平方复杂度，长文本时Attention计算量爆炸；第二是内存带宽瓶颈，GPU算得快但显存读写跟不上；第三是参数量和激活值存储大。推理时还有KV Cache随长度线性增长的问题。"

## 防混补丁
- 训练和推理瓶颈容易混。补丁：**训时梯度占大头，推时KV卡脖子**。

## 追问预演
- **Q：具体怎么计算Attention的复杂度？** A：Q·K^T是(n,d)·(d,n)=O(n²d)，n是序列长度，d是维度；当n很大时，n²项主导。
- **Q：FFN为什么也占很多计算？** A：FFN把维度从d扩到4d再缩回，参数量占模型约2/3，每次前向都要过两遍大矩阵。
