# 大模型量化是什么？INT8/INT4/AWQ/GPTQ 怎么选？

## 速记卡片
`量 | 量八稳，四极限，G逐层，A保精 | 通用vs极限vs精度`

## 原始知识
- **量化本质**：将 FP32/FP16 高精度权重映射到 INT8/INT4 低精度整数，**压缩显存**并**加速计算**
- **INT8**：8位整数量化，最**通用稳妥**，精度损失小（通常 <1%），推理延迟降低约 2-4 倍，几乎所有框架支持
- **INT4**：4位整数量化，**极限压缩**，显存降至 1/4 甚至更少，但需要 per-group / per-block 量化防止精度崩溃，适合边缘端部署
- **GPTQ**：基于**近似二阶信息**的逐层权重量化，用 OBS（Optimal Brain Surgeon）思想逐个量化权重并更新剩余权重。GPU 推理生态成熟（vLLM、AutoGPTQ），但只关注**权重**本身
- **AWQ**：Activation-aware Weight Quantization，**激活感知**量化，根据激活值大小识别并保护对输出影响大的"重要权重通道"，精度优于 GPTQ，更稳定。适用于对精度要求高的生产场景
- **选型链**：求稳选 INT8 → 极限压缩选 INT4 → 兼顾精度与速度选 AWQ → 成熟生态选 GPTQ
- **追问点**：QLoRA 和量化的关系、量化为什么能加速、分组量化的必要性

## 单字锚点 + 防崩推导
- **锚点字**：**量**
- **扩展口诀**：**量八稳，四极限，G逐层，A保精**
- **为什么选它**："量"是量化的总纲，"八四GA"四种方案的缩写，数字+字母组合节奏感强。
- **防崩推导**：
  - 量 → 量化 → 降精度 → 省显存 → 加速推理
  - 八 → INT8 → 8位 → 精度损失最小 → 最稳 → 通用首选
  - 四 → INT4 → 4位 → 显存砍到 1/4 → 极限压缩 → 边缘部署
  - G → GPTQ → 逐层处理 → OBS 二阶近似 → 成熟生态
  - A → AWQ → 激活感知 → 保护重要通道 → 精度最好

## 一字一画面
- **量** —— 像 **视频压缩**：4K 原片压成 720P，文件小了播放更流畅，但画质有损失。（本质：降精度换效率）
- **八** —— 像 **八车道变双车道**：虽然窄了，但两车并排走完全没问题，通行效率依然高。（本质：INT8 稳，精度损失可忽略）
- **四** —— 像 **四车道变单车道**：路极窄，只能一辆车过，但为了把整条高速塞进山里（边缘设备），不得不压。（本质：INT4 极限压缩，牺牲精度换空间）
- **G** —— 像 **逐层拆楼**：工程师一层一层拆，每拆一层就重新计算承重，让剩余楼层保持稳定。成熟但只看建筑结构本身。（本质：GPTQ 逐层量化，只看权重）
- **A** —— 像 **保龄球只打红瓶**：观察发现某些瓶子（权重通道）倒了会让整排塌，于是重点保护这些关键瓶，其余的随便打。（本质：AWQ 激活感知，保护对输出影响大的通道）

## 面试话术
- 问量化是什么 → "大模型量化是把 FP16/FP32 的浮点权重压缩成 INT8/INT4 等低精度整数，**省显存、加速推理**，代价是轻微精度损失。"
- 问四种怎么选 → "我总结为**量八稳，四极限，G逐层，A保精**。INT8 最稳通用首选；INT4 极限压缩适合边缘端；GPTQ 逐层量化生态成熟；AWQ 激活感知精度最好，生产环境首选。"

## 防混补丁
容易和 **模型剪枝** 搞混。
- **量化** 是砍每个参数的**精度**（32位变8位）
- **剪枝** 是砍**参数数量**（把不重要的权重直接归零）

**补丁口诀**：**量化砍比特，剪枝砍人头**

## 追问预演
1. **量化为什么能加速推理？**
   → 两点：一是低精度整数运算比浮点运算更快，硬件支持更好；二是显存占用降低，带宽压力减小，数据搬运更快。

2. **GPTQ 和 AWQ 的核心区别是什么？**
   → GPTQ 只关注**权重**本身的分布，用二阶信息逐层量化；AWQ 还观察**激活值**分布，识别并保护对输出影响大的权重通道，因此精度更高更稳定。

3. **INT4 为什么需要分组（per-group）量化？**
   → 4 位动态范围只有 [-8, 7]，如果按整个通道（per-channel）统一缩放，数值分布差异大的层会严重失真。per-group 让每个小块有自己的缩放因子，粒度更细，精度得以保留。

4. **QLoRA 和模型量化有什么关系？**
   → QLoRA 是"量化 + LoRA"的组合。先把基座模型量化为 INT4（用 NormalFloat4），然后在 INT4 量化权重之上插入可训练的 LoRA 适配器，实现低显存微调。

5. **量化后的模型还能继续微调吗？**
   → 直接微调量化权重很困难（梯度不连续），通常做法是用 QLoRA 或类似方案，冻结量化后的基座，只训练新增的适配器层。

6. **生产环境推荐哪种量化方案？**
   → 如果追求极致性价比且对精度容忍度高，用 GPTQ/INT4；如果精度要求高，推荐 AWQ（INT4）或 INT8；通用场景 INT8 最稳妥。
