# 大模型的长短期记忆机制

## 速记卡片
`记 | 短会长，库存取 | 记忆机制`

## 原始知识
- **短期记忆**：对话历史直接塞进 Context Window，随会话累积，满了截断或摘要
- **长期记忆-向量库**：历史对话 Embedding 存入向量数据库，需要时语义检索召回
- **长期记忆-结构化存储**：知识图谱、键值对（用户偏好、事实）持久化到数据库
- **长期记忆-外部存储**：SQLite/Redis/MongoDB 存聊天记录，按用户/会话隔离
- **记忆总结**：定期对历史对话做摘要压缩，防止上下文膨胀
- **使用方式**：检索注入（RAG）、摘要注入 System Prompt、分层记忆架构
- **追问点**：窗口满了怎么办？多用户怎么隔离？记忆和 RAG 什么关系？

## 一字一画面
- **短** —— 像**便签纸**：临时记几件事，用完就扔，桌上只能放几张。（本质：短期记忆/对话历史，窗口有限）
- **会** —— 像**当前桌前的谈话**：你和对方面对面说的话，说完这轮就翻篇。（本质：当前会话上下文）
- **长** —— 像**笔记本**：重要信息记下来，下次见面还能翻出来看。（本质：长期记忆，跨会话持久化）
- **库** —— 像**档案室**：按主题分类存放，找的时候按关键词检索。（本质：向量数据库存储 Embedding）
- **存** —— 像**记账本**：用户偏好、关键事实一笔笔记下，不丢。（本质：结构化持久化存储）
- **取** —— 像**查档案**：需要时按关键词找到相关记录，摆在桌上参考。（本质：记忆检索注入 Prompt）

## 面试话术
- **问机制** → "大模型的记忆机制口诀是**短会长，库存取**。**短**——短期记忆是对话历史，直接存在 Context Window 里，窗口满了就截断或摘要；**长**——长期记忆需要跨会话持久化；**库**——用向量数据库存储历史对话的 Embedding，语义检索召回；**存**——关键信息、用户偏好写入结构化数据库；**取**——需要时检索相关记忆注入 Prompt。"
- **问使用** → "实际使用时分三层：工作记忆（当前对话，短期）、参考记忆（向量库检索，长期）、语义记忆（知识图谱/数据库，结构化）。窗口满了就做摘要压缩，或者把早期对话向量化存库，需要时再取。"

## 防混补丁
- **容易觉得"窗口越大越好"**。其实大窗口成本高、注意力稀释。
- **补丁**：**窗口是便签纸，向量库是档案馆**。
- **容易把记忆和 RAG 搞混**。记忆是 RAG 的一种应用场景，但 RAG 更广泛。
- **补丁**：**记忆是记日记，RAG 是查百科**。

## 追问预演
| 追问 | 简要答案 |
|------|----------|
| 上下文窗口满了怎么办？ | 截断（保留最近N轮）、摘要压缩（把早期对话总结成一段话）、滑动窗口 |
| 多用户场景怎么隔离？ | 用户ID+会话ID做分区，向量库按用户过滤，数据库按用户分表/分Key |
| 记忆怎么更新？ | 增量写入新对话；定期合并去重；关键事实变更时覆盖旧值 |
| 记忆和 RAG 什么关系？ | 记忆是 RAG 的子集——RAG 查外部知识，记忆查个人历史；技术实现一样（Embedding+向量库） |
| 怎么防止记忆膨胀？ | 定期摘要压缩、设置记忆 TTL（过期清理）、只保留高价值信息（用户偏好、关键事实） |
| 有哪些开源记忆框架？ | MemGPT、LangChain Memory（ConversationBufferMemory、VectorStoreRetrieverMemory）、AutoGPT 的记忆模块 |

## 手撕代码
```python
# 伪代码：分层记忆架构

class MemoryLayer:
    def __init__(self, vector_db, kv_store):
        self.short_term = []          # 工作记忆：当前对话历史
        self.vector_db = vector_db    # 参考记忆：向量库
        self.kv_store = kv_store      # 语义记忆：键值对存储
    
    def add_turn(self, user_msg, assistant_msg):
        self.short_term.append({user: user_msg, assistant: assistant_msg})
        # 窗口满了做摘要或存入向量库
        if len(self.short_term) > 10:
            summary = self.summarize(self.short_term[:5])
            self.vector_db.add(summary, user_id=u1)
            self.short_term = self.short_term[5:]
    
    def retrieve(self, query, user_id):
        # 1. 取短期记忆（最近N轮）
        context = self.short_term[-5:]
        # 2. 从向量库召回相关长期记忆
        long_term = self.vector_db.search(query, user_id=user_id, top_k=3)
        # 3. 取结构化偏好
        prefs = self.kv_store.get(user_id)
        return context + long_term + prefs
```
