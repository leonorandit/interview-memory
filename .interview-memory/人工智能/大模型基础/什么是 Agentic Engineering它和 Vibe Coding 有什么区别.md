## 速记卡片
`匠 | 匠人搭架构，感性 freestyle | 工程化 vs 直觉流`

## 原始知识
- **Agentic Engineering**：系统化构建 Agent
- **核心特征**：流程拆解、工具编排、状态管理
- **Vibe Coding**：凭直觉写代码
- **核心特征**：Prompt 驱动、即兴迭代、结果导向
- **本质区别**：工程化思维 vs 感性思维
- **加粗易混点**：两者不是对立，可以**混搭使用**

## 顺口溜
匠人画图纸，Vibe 随手涂。

## 单字锚点 + 防崩推导
- **锚点字**：匠
- **扩展口诀**：匠人磨刀不误工，Vibe 提笔就开工
- **为什么选它**："匠"字自带工程化、体系化的意象，与 Vibe 的随意感形成鲜明对比
- **防崩推导**：匠 → 工匠 → 需要设计图纸 → Agentic Engineering 需要架构设计；Vibe → 氛围 → 跟着感觉走

## 一字一画面
**匠** —— 像 **老木匠在刨木板**：墨线弹得笔直，刨花卷成统一的螺旋，每道工序有章可循（本质：Agentic Engineering 是系统化、可复现的工程方法）

## 面试话术
Agentic Engineering 是把 AI Agent 当工程系统来建，强调流程拆解、工具链编排和状态管理；Vibe Coding 更像是跟着感觉走，用 Prompt 即兴驱动代码生成。简单说就是一个是"搭脚手架盖楼"，一个是"随手涂鸦出作品"。

## 防混补丁
- Agentic 不排斥 Vibe：快速原型阶段可以 Vibe，生产环境必须 Agentic
- Vibe 不是乱写：直觉背后其实是对模型能力的隐性理解

## 追问预演
- Q：Agentic 的核心挑战是什么？A：状态管理和错误恢复
- Q：Vibe Coding 适合什么场景？A：探索性编程、MVP 验证
