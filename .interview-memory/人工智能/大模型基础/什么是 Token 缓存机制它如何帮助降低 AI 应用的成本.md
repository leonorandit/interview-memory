## 速记卡片
`缓 | 重复 prompt 缓存免 | 命中即打折`

## 原始知识
- **Token 缓存**：复用已计算的上下文
- **机制**：KV Cache 存储注意力中间结果
- **降低成本的原理**：重复输入不需重算
- **缓存命中**：相同前缀只算一次
- **典型场景**：多轮对话、批量相同 Prompt
- **加粗易混点**：缓存**只省输入不算输出**

## 顺口溜
缓存一命中，费用打五折。

## 单字锚点 + 防崩推导
- **锚点字**：缓
- **扩展口诀**：缓兵之计省算力，重复 prompt 不再算
- **为什么选它**："缓"谐音"省"的缓兵之计，且直接对应 Cache 的中文意涵
- **防崩推导**：缓 → 缓存 → 存起来复用 → 避免重复计算 → 省输入 Token 费用

## 一字一画面
**缓** —— 像 **图书馆的复印机缓存**：第一个学生复印完整本书（全量计算），后面学生只要同一本书，机器直接从缓存托盘里吐出来，速度快而且只收半价（本质：KV Cache 把已算好的注意力状态存起来复用）

## 面试话术
Token 缓存机制就是把模型已经算过的注意力状态存下来，下次遇到相同的输入前缀直接复用。比如多轮对话里，历史消息不用每次都重新算，这样输入 Token 的费用就能大幅降低，有的平台缓存命中能打到五折甚至更低。

## 防混补丁
- 缓存只省输入费，输出该多少还是多少
- 缓存有 TTL，不是永久有效
- 前缀必须完全匹配才算命中

## 追问预演
