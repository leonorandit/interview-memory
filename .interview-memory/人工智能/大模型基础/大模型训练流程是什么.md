# 大模型训练流程是什么

## 速记卡片
`洗 | 洗预调奖对评 | 数据→预训练→SFT→RM→RLHF→部署`

## 原始知识
- **数据清洗**：去重（MinHash/SimHash）、过滤低质量（规则+模型打分）、隐私脱敏、语种平衡、格式化
- **预训练**：海量无标注文本，自回归/自监督学习，学语言规律、世界知识。消耗算力最大，7B 模型需数千 GPU·天
- **SFT（监督微调）**：高质量 instruction-response 对数据，有监督学习让模型"听话"。数据量小（通常 10K~1M），但质量要求极高
- **奖励模型（Reward Model）**：人工标注偏好数据（A 比 B 好），训练打分模型学会评估回答质量
- **RLHF 对齐**：PPO/DPO/ORPO 等强化学习算法，让策略模型生成高奖励分数的回答，对齐人类偏好
- **评估部署**：自动指标（perplexity、BLEU、ROUGE）+ 人工评测（ helpfulness、harmlessness、honesty），验证通过后部署上线
- **易混点**：SFT 不是预训练的延续，而是独立阶段；RLHF 不是必须，但大模型普遍需要
- **追问点**：为什么预训练后还要 SFT？RLHF 和 DPO 区别？数据清洗怎么做？

## 顺口溜
**洗预调奖对评**

## 单字锚点 + 防崩推导
- **锚点字**：**洗**
- **扩展口诀**：**洗预调奖对评**
- **为什么选它**：大模型训练从"洗"开始——数据质量决定模型上限。先洗数据，再逐步加工。
- **防崩推导**：
  - 洗 → 数据清洗（去重、过滤、脱敏）
  - 预 → 预训练（海量无标注，自监督学语言）
  - 调 → SFT（高质量指令数据，有监督微调）
  - 奖 → 奖励模型（人工标偏好，训练打分器）
  - 对 → RLHF/DPO 对齐（强化学习，对齐人类偏好）
  - 评 → 评估部署（自动指标+人工评测，上线）

## 一字一画面
- **洗** —— 像**洗菜淘米**：把脏的、烂的、重复的挑出去，留下干净的食材。（数据清洗：去重、过滤低质量、隐私脱敏）
- **预** —— 像**读万卷书**：不针对考试，先海量阅读建立语感。（预训练：TB 级无标注文本，自回归预测下一个 token，学语言规律和世界知识）
- **调** —— 像**考前刷真题**：有标准答案的题，反复练到会。（SFT：高质量 instruction-response 对，让模型学会按指令格式回答）
- **奖** —— 像**老师改卷打分**：A 卷比 B 卷好，老师学会打分标准。（奖励模型：人工标注偏好数据，训练一个会打分的"裁判"）
- **对** —— 像**学生按分数排名**：高分答案多产出，低分答案少产出。（RLHF：用奖励模型的分数指导策略模型，PPO/DPO 优化生成偏好）
- **评** —— 像**模拟考试+正式高考**：先测评再上线。（评估：自动指标+人工评测，3H 原则 helpfulness/harmlessness/honesty）

## 面试话术
- 问训练流程 → "大模型训练流程口诀是**洗预调奖对评**。第一步数据清洗，去重过滤低质量；第二步预训练，海量无标注文本自监督学习语言规律；第三步 SFT 监督微调，用高质量指令-回答对有监督学习让模型'听话'；第四步训练奖励模型，人工标注偏好数据训练打分器；第五步 RLHF/DPO 对齐，强化学习让人类偏好指导模型；第六步评估部署，自动指标+人工评测验证后上线。"
- 问 RLHF → "RLHF 是对齐阶段的核心。先让人工标注偏好数据（A 比 B 好），训练奖励模型学会打分；再用 PPO 等强化学习算法，让模型生成高分回答，避免'说胡话'或'有害内容'。DPO 则省去奖励模型，直接用偏好数据优化策略，更简洁。"

## 防混补丁
- 容易把 SFT 当成预训练的一部分。补丁：**预训练学'接话'，SFT 学'听话'**，是两个独立阶段。
- 容易以为 RLHF 是必须的。补丁：**RLHF 不是必须**，但主流大模型（GPT、Claude、Llama）都用它来提升有用性和安全性。

## 追问预演
1. **为什么预训练后还要 SFT？**
   → 预训练只学"接话"（预测下一个 token），不会按指令格式回答。SFT 用高质量指令数据教模型"听话"——学会按用户意图、格式、风格生成。
2. **RLHF 和 DPO 区别？**
   → RLHF 需要单独训练奖励模型 + PPO 强化学习，流程复杂、训练不稳定；DPO 直接把偏好数据转化为分类损失，省去奖励模型，训练更稳定更简单。
3. **预训练数据怎么清洗？**
   → 去重（MinHash/SimHash 去近似重复）、质量过滤（规则+模型打分去低质量）、隐私脱敏（去 PII）、语种平衡、格式化（统一文本结构）。
4. **SFT 数据质量为什么比数量重要？**
   → SFT 阶段数据量小（通常 10K~1M），模型容易过拟合。一条错误数据可能让模型学会错误模式，所以清洗和人工校验至关重要。
5. **奖励模型为什么需要人工标注？**
   → 模型本身不知道"哪个回答更好"，需要人类偏好来定义"好"的标准（有用、无害、诚实）。奖励模型学习人类偏好后，才能自动给生成结果打分。
