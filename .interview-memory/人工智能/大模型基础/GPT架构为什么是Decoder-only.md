# GPT 架构为什么是 Decoder-only

## 速记卡片
`生 | 生右统，可扩展 | 生成vs理解vs翻译`

## 原始知识
- **自回归生成**：Decoder-only 天然适合自回归生成，一次预测一个 token，从左到右生成文本
- **训练效率**：因果掩码（Causal Masking）只计算单向注意力，不需要双向编码，训练更简单高效
- **统一架构**：所有 NLP 任务都可以转化为"给定前文，生成下文"的生成问题（分类、翻译、问答、摘要）
- **可扩展性**：结构简洁（一个 Transformer 块重复堆叠），容易扩展到大规模（GPT-3 175B，GPT-4 1.8T）
- **涌现能力**：大规模 Decoder-only 模型展现出强大的 few-shot/zero-shot 能力，无需微调即可适应新任务
- **对比 Encoder-only（BERT）**：双向编码适合理解任务（分类、抽取），但生成时需要额外解码器
- **对比 Encoder-Decoder（T5/BART）**：适合 seq2seq（翻译、摘要），但结构复杂（两个组件），不如 Decoder-only 统一
- **易混点**：Decoder-only 不是不能做理解任务，而是通过生成方式做理解
- **追问点**：Decoder-only 和 BERT 的区别？T5 为什么没 GPT 流行？推理效率优势？

## 顺口溜
**生右统，可扩展**

## 单字锚点 + 防崩推导
- **锚点字**：**生**
- **扩展口诀**：**生右统扩**
- **为什么选它**：GPT 的核心使命是"生成"，Decoder-only 的每个设计都围绕生成展开。
- **防崩推导**：
  - 生 → 自回归生成（适合文本生成，一次预测一个 token）
  - 右 → 从左到右（因果掩码，单向注意力，计算高效）
  - 统 → 统一架构（所有任务变成生成任务，一把钥匙开所有锁）
  - 扩 → 可扩展性（结构简洁，容易堆叠到大规模，涌现能力）

## 一字一画面
- **生** —— 像**写小说**：一个字一个字往下写，不能先写结尾再回头改开头。（自回归生成：适合文本生成，天然流畅）
- **右** —— 像**单向车道**：只准往右开，不用看后视镜，速度快。（因果掩码：只看左边历史，计算高效，不需要双向编码）
- **统** —— 像**万能钥匙**：一把钥匙开所有锁，不用换钥匙。（统一架构：分类、翻译、问答、摘要都变成"生成下文"）
- **扩** —— 像**搭积木**：一块一块往上堆，越高越强。（可扩展性：结构简洁，GPT-3 175B、GPT-4 1.8T，规模越大能力越强）

## 面试话术
- 问为什么 Decoder-only → "GPT 选择 Decoder-only 的口诀是**生右统扩**。自回归生成适合文本生成；从左到右单向注意力计算高效；统一架构让所有任务都变成生成任务；结构简洁容易扩展到大规模。相比之下，BERT 是双向编码适合理解但不适合生成，T5 是 Encoder-Decoder 适合翻译但结构复杂。"
- 问统一性 → "Decoder-only 的核心优势是统一性——任何任务都可以变成'给定前文，生成下文'。分类任务生成标签，翻译任务生成译文，抽取任务生成答案。这种统一让预训练不需要特别标注，海量无标注文本就能训练。"

## 防混补丁
- 容易以为 Decoder-only 只能做生成，不能做理解。补丁：**Decoder-only 通过生成方式做理解**——比如分类时生成" positive"或" negative"标签。
- 容易把 GPT 和 BERT 的适用场景搞混。补丁：**BERT 看两边理解强，GPT 看左边生成强**。

## 追问预演
1. **Decoder-only 和 BERT 的双向编码有什么区别？**
   → Decoder-only 使用因果掩码（Causal Mask），每个位置只能看到左边历史 token，适合自回归生成；BERT 使用双向注意力，可以看到全部 token，适合理解任务如分类、抽取。
2. **T5 也是生成模型，为什么 GPT 更流行？**
   → T5 是 Encoder-Decoder，需要两个组件，结构复杂，扩展性不如 Decoder-only；GPT 一个组件统一所有任务，结构简洁更容易堆叠到大规模（175B+），且涌现出强大的 few-shot 能力。
3. **GPT 怎么通过生成做分类任务？**
   → 把分类变成生成标签文本。例如输入"这部电影很好看，情感是___"，模型生成" positive"。这样所有任务统一为生成问题。
4. **Decoder-only 的推理效率为什么高？**
   → 自回归生成时，KV Cache 可以缓存历史 token 的 K/V，下一步只算新 token 的注意力，不需要像双向编码那样重新计算全部注意力矩阵。
5. **为什么 GPT-3 能涌现 few-shot 能力？**
   → 大规模（175B 参数）+ Decoder-only 自回归 + 海量预训练数据（300B tokens），让模型学会了从上下文推断模式，看到几个示例就能模仿任务格式。
