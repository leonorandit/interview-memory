# 什么是大模型项目的分词器？原理是什么？

## 速记卡片
`拆 | 切拆并，子词生 | 文本→Token`

## 原始知识
- **定义**：分词器（Tokenizer）是大模型处理文本的前置组件，将原始文本转换为模型可理解的 token ID 序列
- **核心目标**：压缩文本表示、平衡词表大小与序列长度、消除 OOV（未登录词）
- **BPE**：GPT 系列使用，从字符开始迭代合并频率最高的子词对
- **WordPiece**：BERT 使用，基于语言模型概率最大化选择合并对
- **SentencePiece**：T5/LLaMA 使用，基于 BPE 或 Unigram，不依赖空格和语言预处理
- **Unigram**：从大量子词候选出发，迭代删除低概率子词，保留最优子集
- **关键概念**：词表（Vocabulary）、Merge Rules、Subword、Byte Fallback（字节回退）
- **易混点**：Tokenizer 不是模型的一部分，是独立预处理组件
- **追问点**：为什么中文也需要 BPE？Token 和字的区别？

## 顺口溜
**切拆并，子词生**

## 单字锚点 + 防崩推导
- **锚点字**：**拆**
- **扩展口诀**：**拆字并，查表换**
- **为什么选它**："拆"是分词器的灵魂动作——遇到不认识的词，先拆成小块。
- **防崩推导**：
  - 拆 → 拆什么？→ 文本拆成字符/字节
  - 拆 → 为什么拆？→ 处理未知词、压缩词表
  - 并 → 怎么并？→ 高频子词对合并（BPE 核心）
  - 查表换 → 最终做什么？→ 查词表映射为 token ID

## 一字一画面
- **切** —— 像**菜刀切面条**：一长条句子咔嚓切成小段。（按规则初分文本）
- **拆** —— 像**拆手表**：遇到生僻词，拆成齿轮、发条等通用零件。（字符级拆解，消除 OOV）
- **并** —— 像**拼乐高**：两个小块老是挨着出现，就粘成一块新零件。（BPE 高频合并）
- **子** —— 像**子母扣**：子词就是小扣子，任意组合都能扣上。（Subword 组合表示任意词）
- **词** —— 像**字典厚书**：词表就是翻译手册，查一个字对应一个数字编号。（Vocabulary 映射）
- **生** —— 像**工厂流水线**：一头进原始文本，一头出整齐的数字序列。（生成 token ID 序列）

## 面试话术
- 问定义 → "分词器是大模型的'翻译官'，负责把人类文本翻译成模型能懂的数字。核心动作就三个：**切拆并**。切是初分文本，拆是把未知词拆成字符，并是把高频出现的字符对合并成子词。最终查词表换成 token ID。"
- 问原理 → "最典型的是 BPE，从字符开始，统计训练语料里最高频的字符对，合并成新符号，反复迭代直到词表大小达标。这样既能控制词表规模，又能表示任意新词，解决 OOV 问题。"
- 问对比 → "GPT 用 BPE，BERT 用 WordPiece，LLaMA 用 SentencePiece。BPE 看频率，WordPiece 看概率，SentencePiece 不依赖空格，更适合多语言。"

## 防混补丁
- 容易和"模型结构"搞混。**分词器不是神经网络层，是预处理工具**。补丁：**Tokenizer 在模型外，Embedding 在模型内**。

## 追问预演
1. **为什么中文也需要 BPE，不是已经有字了吗？**  
   → 中文字虽然独立，但常用组合（如"机器学习"）作为整体 token 可以大幅减少序列长度，提升效率。且 BPE 能处理罕见字、错别字、新词组合。

2. **Token 和字的区别？一个字一定是一个 token 吗？**  
   → 不一定。常见字可能是一个 token，罕见字可能被拆成多个 byte-level token，多字词可能合并成一个 token。平均来说 1 个 token 约 0.75 个英文单词或 0.5 个汉字。

3. **词表大小怎么选？越大越好吗？**  
   → 不是。词表大 → 序列短但 embedding 层参数爆炸、OOV 少
