# 你觉得可以怎样缓解这个性能瓶颈

## 速记卡片
`缓 | 缓压剪，量混并 | 压缩·量化·并行`

## 原始知识
- 模型压缩：剪枝、知识蒸馏
- 量化：INT8/INT4/FP8，减少显存占用
- 混合精度训练：FP16/BF16 + FP32
- 并行策略：数据并行、模型并行、流水线并行、张量并行
- 高效Attention：FlashAttention、Sparse Attention、Linear Attention
- KV Cache优化：PageAttention、KV Cache量化
- MOE：稀疏激活，用更少计算量达到更大模型效果
- **易混点**：不同优化手段针对不同瓶颈

## 顺口溜
缓压剪，量混并

## 单字锚点 + 防崩推导
- **锚点字**：**缓**
- **扩展口诀**：**缓三招**
- **为什么选它**："缓"直接对应"缓解"，是问题的动作核心。
- **防崩推导**：
  - 缓 → 缓解 → 性能瓶颈怎么破
  - 压 → 压缩/蒸馏 → 模型变小
  - 剪 → 剪枝 → 剪掉冗余权重
  - 量 → 量化 → INT8/INT4降低精度省空间
  - 混 → 混合精度 → FP16算FP32稳
  - 并 → 并行 → 分布式多卡一起算

## 一字一画面
- **缓** —— 像**缓速带**：让过快的资源消耗慢下来。（缓解本质）
- **压** —— 像**压缩饼干**：同样的营养，体积更小。（知识蒸馏）
- **剪** —— 像**修剪树枝**：剪掉不重要的枝丫，主干不变。（模型剪枝）
- **量** —— 像**用整数代替小数**：INT8是粗略估计，但省空间。（量化）
- **混** —— 像**混血儿**：FP16干活快，FP32兜底准。（混合精度）
- **并** —— 像**多人搬砖**：数据并行是多个人搬同一批砖，模型并行是一个人搬大砖分几次。（并行策略）

## 面试话术
- "缓解瓶颈可以从**缓压剪，量混并**六个方向入手。算法层用剪枝、蒸馏压缩模型；工程层做INT4/INT8量化、混合精度训练；分布式层用数据并行、张量并行、流水线并行；Attention层用FlashAttention、KV Cache优化。实际落地要看瓶颈具体在哪，是计算、显存还是通信。"

## 防混补丁
- 容易把所有优化混为一谈。补丁：**显存不够找量剪，速度不够找并闪，模型太大找压并**。

## 追问预演
- **Q：FlashAttention为什么快？** A：通过分块计算和重计算，减少HBM读写次数，让Attention计算变成IO-aware。
- **Q：量化和剪枝哪个更适合推理优化？** A：量化更适合推理，直接降低显存和计算位宽；剪枝更适合模型压缩部署到边缘设备。
