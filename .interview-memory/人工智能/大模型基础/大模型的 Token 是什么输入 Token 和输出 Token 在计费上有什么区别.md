## 速记卡片
`计 | 输入输出分开计 | 计费单元不同价`

## 原始知识
- **Token**：模型处理的最小单元
- **1 Token ≈ 0.75 个英文词**
- **中文更费 Token**：1 字 ≈ 1-2 Token
- **输入 Token**：Prompt 部分
- **输出 Token**：生成内容部分
- **计费差异**：输出通常比输入贵
- **加粗易混点**：不同模型**单价差异巨大**

## 顺口溜
进便宜，出贵气。

## 单字锚点 + 防崩推导
- **锚点字**：计
- **扩展口诀**：计算器按两遍，进出价格不相见
- **为什么选它**："计"直指计费，是这道题的落脚点，且谐音"记"，方便记忆
- **防崩推导**：计 → 计费 → 输入 vs 输出 → 输出贵因为生成更耗算力

## 一字一画面
**计** —— 像 **老式机械计数器**：左边转盘记录投进去的 Token（Prompt），每进一个咔哒一声；右边转盘记录吐出来的 Token（生成），每出一个咔哒两声，声音更沉更贵（本质：输出 Token 消耗更多计算资源）

## 面试话术
Token 就是大模型处理信息的最小片段，一个英文单词大概拆成 1.3 个 Token，中文更吃亏。计费上输入 Token 和输出 Token 是分开算的，而且输出通常更贵，因为生成过程要一步一步采样，算力消耗更大。

## 防混补丁
- 输入输出单价不同，别混在一起算
- 缓存命中可以省输入费，但输出费省不了

## 追问预演
- Q：怎么估算 Token 数？A：用 tiktoken 或官方工具，经验法则 100 Token ≈ 75 词
- Q：长上下文为什么贵？A：输入 Token 线性增长，注意力计算平方增长
