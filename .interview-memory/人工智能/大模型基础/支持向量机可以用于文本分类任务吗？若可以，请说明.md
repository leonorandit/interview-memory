# 支持向量机可以用于文本分类任务吗？若可以，请说明

## 速记卡片
`边 | SVM找边最大化，高维核函分词沙 | 间隔最大化分类`

## 原始知识
- **可以**：SVM是文本分类的经典基线方法
- **流程**：文本 → 分词 → TF-IDF向量化 → SVM训练 → 分类
- **核技巧**：线性核速度快，RBF核处理非线性，文本高维通常线性核就够
- **间隔最大化**：找最优超平面，让支持向量到边界的距离最大
- **优势**：小样本表现好，泛化强，高维稀疏数据鲁棒
- **加粗易混点**：SVM不是深度学习方法，和BERT是互补而非替代

## 顺口溜
找边界，最大化，高维稀疏不怕炸

## 单字锚点 + 防崩推导
- **锚点字**：**边**
- **扩展口诀**：**SVM找边界**
- **为什么选它**：SVM的核心数学思想就是"最大化间隔（margin）"，"边"是边界/间隔的简称，信息密度最高。
- **防崩推导**：
  - 边 → 边界/间隔 → SVM找最优分类边界
  - 边 → 支持向量定边界 → 只有边界附近的点决定模型
  - 边 → 文本用TF-IDF → 高维稀疏向量 → SVM对高维友好
  - 边 → 核函数 → 低维不可分映射到高维找边

## 一字一画面
- **边** —— 像**两队人中间画三八线**：SVM不是随便画线，而是找让两边人马离界线都最远的那个位置。（本质：间隔最大化）
- **支持向量** —— 像**两军对垒的前排士兵**：只有最靠近边界的前排士兵决定战线在哪，后排人再多也不影响。（本质：稀疏解）
- **核** —— 像**把纸折成纸飞机**：二维纸上的两个点本来隔很远，折成三维飞机后可能就挨着了。（本质：核技巧映射）

## 面试话术
- "当然可以。SVM在深度学习之前是文本分类的主流方法。流程很简单：文本分词后用TF-IDF转成向量，然后SVM找最优分类边界。它的核心优势是**间隔最大化**，只让边界附近的支持向量决定模型，对高维稀疏的文本数据特别友好。实际项目中我常用线性核SVM做快速基线，和深度学习模型对比。"

## 防混补丁
- 容易觉得SVM过时没用。补丁：**SVM在小样本、可解释性要求高、无GPU场景仍是利器；和BERT不是非此即彼**。

## 追问预演
- Q：SVM和朴素贝叶斯哪个做文本分类更好？
  - A：通常SVM更好，尤其是特征维度高时；贝叶斯假设特征独立，现实中不成立。
- Q：为什么文本分类常用线性核？
  - A：TF-IDF维度已经很高，数据在高维空间里往往线性可分，RBF反而容易过拟合。
- Q：SVM能处理多分类吗？
  - A：可以，用One-vs-One或One-vs-Rest策略组合多个二分类SVM。
