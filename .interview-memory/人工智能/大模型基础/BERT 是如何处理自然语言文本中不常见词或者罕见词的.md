# BERT 是如何处理自然语言文本中不常见词或者罕见词的

## 速记卡片
`拆 | BERT拆字用BPE，罕见词变子词凑 | 子词切分兜底`

## 原始知识
- **BPE算法**：Byte Pair Encoding，从字符级逐步合并高频子词对
- **WordPiece**：BERT实际使用，类似BPE，基于语言模型概率选合并
- **UNK陷阱**：传统词表对罕见词直接映射<UNK>，信息丢失
- **拆字逻辑**：unhappiness → un + ##happiness → 遇到罕见词拆成熟悉子词
- **OOV解决**：任何词都能拆成字符或子词，不存在完全OOV
- **加粗易混点**：中文也是按字或子词切分，不是按词；BERT不直接认整词

## 顺口溜
拆字BPE，罕见变子词

## 单字锚点 + 防崩推导
- **锚点字**：**拆**
- **扩展口诀**：**拆字为子词**
- **为什么选它**：BERT处理罕见词的本质就是"拆"——拆成更小的、见过的子词。
- **防崩推导**：
  - 拆 → 罕见词不认识 → 拆成熟悉子词
  - 拆 → 用WordPiece/BPE → 从字符开始合并最高频子词对
  - 拆 → 例：antidisestablishmentarianism → anti + dis + establish + ment + arian + ism
  - 拆 → 彻底拆到字符级兜底 → 绝不出现UNK

## 一字一画面
- **拆** —— 像**乐高积木拆大块**：一个没见过的大模型飞船，拆开来发现是几个常见的小积木块拼的。（本质：组合泛化）
- **BPE** —— 像**不断合并双胞胎**：从单个字母开始，看到哪对字母总一起出现就粘成一块，反复粘到词表大小为止。（本质：自下而上聚类）
- **子词** —— 像**偏旁部首**：中文里"dragon"不认识，但"龙"这个部首常见；英文里unhappiness不认识，但un-和-ness是常见零件。（本质：形态学还原）

## 面试话术
- "BERT处理罕见词的核心就一个字：**拆**。它用WordPiece算法，把词拆成子词（subword）。比如一个罕见词可能被拆成'un'、'##happy'、'##ness'，每个子词都在训练时见过。就算拆到最极端，还能拆成单个字符兜底，所以BERT理论上不会出现OOV问题。这比传统词向量直接用<UNK>要聪明得多。"

## 防混补丁
- 容易把WordPiece和BPE混为一谈。补丁：**BPE看频率，WordPiece看概率；BERT用WordPiece，GPT用BPE**。

## 追问预演
- Q：WordPiece和BPE具体区别？
  - A：BPE选合并后频率最高的对；WordPiece选合并后语言模型概率增益最大的对。
- Q：中文BERT怎么拆？
  - A：可以按字切分，也可以用更细粒度的子词；常见做法是单字为基本token，部分高频词整词保留。
- Q：拆成子词会不会丢失整词语义？
  - A：会部分丢失，但Transformer高层会通过注意力重组子词信息，实际影响有限。
