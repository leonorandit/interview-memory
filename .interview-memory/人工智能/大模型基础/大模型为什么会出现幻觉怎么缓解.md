# 大模型为什么会出现幻觉？怎么缓解？

## 速记卡片
`幻 | 幻数编回，RAG治 | 成因与缓解`

## 原始知识
- **定义**：模型生成看似合理但不符合事实或输入的内容
- **原因-数据**：训练数据含错误、噪声、过期信息，知识有截止日期
- **原因-模式补全**：模型倾向生成流畅文本，不确定时也会"编造"
- **原因-自回归**：逐token预测，错误累积，无法回头修正
- **原因-过度迎合**：RLHF 对齐后过于迎合用户意图，牺牲事实性
- **缓解-RAG**：检索增强，给模型外部知识源，减少凭空编造
- **缓解-对齐/微调**：事实性微调、RLHF 降低幻觉奖励
- **缓解-Prompt工程**：要求先思考、引用来源、承认不知道
- **缓解-后处理**：事实核查、多模型投票、一致性检查
- **追问点**：RAG 能完全消除幻觉吗？怎么评估幻觉程度？

## 顺口溜
**幻数编回，RAG治**

## 单字锚点 + 防崩推导
- **锚点字**：**幻**
- **扩展口诀**：**幻数编回，RAG治**
- **为什么选它**："幻"直接对应问题核心——幻觉。先想"为什么会幻"，再推"怎么治"。
- **防崩推导**：
  - 幻 → 幻觉 → 为什么会出现？
  - 数 → 数据问题（过期、错误、噪声、知识截止）
  - 编 → 模式补全 → 模型为了流畅会编造不确定的内容
  - 回 → 自回归 → 逐token生成，错了只能继续编，像接龙游戏
  - R → RAG → 检索增强，给外部知识，开卷考试
  - A → Alignment/对齐 → 事实性微调、RLHF 调奖励
  - G → 工具/校验 → 搜索引擎、事实核查、多模型投票

## 一字一画面
- **幻** —— 像**魔术师变鸽子**：看起来是真的，其实是手法，观众信以为真。（本质：生成内容看似合理但非事实）
- **数** —— 像**厨师用过期食材**：原料本身有问题，菜再好也是坏的，客户吃了拉肚子。（本质：训练数据过时、错误、有噪声）
- **编** —— 像**小说作者编情节**：为了故事连贯，即使不知道细节也会现编一段，读者看不出来。（本质：模式补全，过度自信）
- **回** —— 像**接龙游戏说错话**：第一个人说错词，后面的人只能顺着错词继续接，越接越离谱。（本质：自回归逐token生成，错误累积无法修正）
- **R** —— 像**考试允许开卷查资料**：不会就翻书，不用瞎编。（本质：RAG 检索增强，提供外部事实）
- **A** —— 像**教练纠正运动员动作**：做对了给奖励，做错了给惩罚，慢慢改掉坏毛病。（本质：RLHF 对齐训练，降低幻觉奖励）
- **G** —— 像**论文发表前找同行评审**：多个专家交叉验证，发现错误打回修改。（本质：事实核查、多模型投票、一致性检查）

## 面试话术
- **问原因** → "大模型幻觉的核心口诀是**幻数编回**。一是因为训练数据本身有问题，过期、错误、噪声；二是模型为了生成流畅文本，会模式补全、编造不确定的内容；三是自回归架构决定了它逐token生成，错了只能继续编，像接龙游戏越编越偏。"
- **问缓解** → "缓解策略对应**RAG治**。RAG 检索增强让模型开卷考试；RLHF 和事实性微调从训练层面降低幻觉；Prompt 工程要求模型引用来源、先思考再回答；线上还可以做多模型投票和事实核查。但注意，幻觉只能缓解，无法完全消除。"

## 防混补丁
- **容易和"模型不确定时拒答"搞混**。有些模型即使不知道也会强行回答，这是幻觉；好的做法是承认不知道。
- **补丁**：**不知道就承认，硬编就是幻**。
- **容易以为 RAG 能完全消除幻觉**。RAG 只是减少凭空编造，检索本身也可能出错，或模型忽略检索内容。
- **补丁**：**RAG 是开卷考试，但书也可能印错，学生也可能看错**。

## 追问预演
| 追问 | 简要答案 |
|------|----------|
| RAG 能完全消除幻觉吗？ | 不能。检索文档可能有误，模型可能忽略检索内容继续编造（Contextual Hallucination） |
| 怎么评估幻觉程度？ | 自动指标：FactScore、HALUE；人工评估：事实性打分；对比 ground truth |
| 为什么 RLHF 后幻觉反而可能增加？ | 对齐训练过度优化"有帮助、无害"，模型变得过于迎合用户，牺牲事实性 |
| 长文本里幻觉更严重吗？ | 是。长文本注意力分散，模型容易忽略中间内容（Lost in the Middle），编造细节 |
| 温度参数和幻觉有关吗？ | 有关。Temperature 越高，采样越随机，幻觉倾向增加；降低 Temperature 可减少编造 |
