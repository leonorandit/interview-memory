# 相对位置编码 vs 绝对位置编码？

## 速记卡片
`绝 | 绝号固，相移距 | 绝对vs相对`

## 原始知识
- **绝对位置编码**：每个位置有固定编号/向量，如正弦/余弦编码、可学习位置嵌入
- **相对位置编码**：编码位置间的相对距离，具有平移不变性，如 RoPE、ALiBi、T5
- **核心差异**：绝对 PE 依赖训练时的最大长度，外推性差；相对 PE 只关注距离，外推性更好
- **RoPE**：通过旋转矩阵将相对距离内积到 QK 注意力分数中
- **ALiBi**：直接在注意力分数上按距离加负偏置，越远越弱
- **追问点**：为什么 LLaMA 用 RoPE？长度外推性差什么意思？

## 顺口溜
**绝号固，相移距**

## 单字锚点 + 防崩推导
- **锚点字**：**绝**
- **扩展口诀**：**绝号固，相移距**
- **为什么选它**：面试题从"绝对位置编码"出发，先想"门牌号"就能区分两者。
- **防崩推导**：
  - 绝 → 绝对位置编码 → 每个位置有固定编号
  - 号 → 位置号，如第 1 个词、第 2 个词
  - 固 → 固定不变，训练时确定，不能扩展
  - 相 → 相对位置编码
  - 移 → 平移不变性，编码只与 i-j 有关
  - 距 → 距离，相对距离不受序列长度限制

## 一字一画面
- **绝** —— 像**门牌号**：101、102、103，每个房间固定编号，搬家后新房间没有门牌。（本质：绝对位置，每个位置唯一编码）
- **号** —— 像**座位号**：演唱会座位按号排，但场馆扩大后后排没号了。（本质：训练时的最大位置决定能力上限）
- **固** —— 像**刻章**：每个位置编码训练完就固定，不会变。（本质：正弦/余弦或可学习嵌入，固定不变）
- **相** —— 像**两人间距**：无论你在哪条街，你离我 3 米就是 3 米。（本质：相对距离与绝对位置无关）
- **移** —— 像**尺子量距离**：把尺子平移到任何地方，刻度差不变。（本质：平移不变性）
- **距** —— 像**弹簧连接**：两个词之间的"弹簧"长度是固定的，不随整体移动改变。（本质：只编码相对距离）

## 面试话术
- **问区别** → "绝对和相对位置编码的口诀是**绝号固，相移距**。绝对 PE 像门牌号，每个位置有固定编号，如正弦编码或可学习嵌入；相对 PE 像两人间距，只编码相对距离，具有平移不变性。关键差异在于长度外推性：绝对 PE 训练时没见过更长序列就处理不了；相对 PE 如 RoPE、ALiBi 可以更好地外推到更长文本。"
- **问选型** → "现在大模型基本都用相对 PE。GPT-4、LLaMA 用 RoPE，通过旋转矩阵把相对距离内积到注意力里；ALiBi 更简单，直接在注意力分数上按距离加负偏置。两者都比原始正弦编码的外推性强得多。"

## 防混补丁
- **容易把 RoPE 和绝对 PE 搞混**。RoPE 是相对编码，只是通过旋转矩阵实现。
- **补丁**：**RoPE 是相对编码穿了个旋转马甲**。
- **容易觉得"可学习位置嵌入"是相对 PE**。BERT 的位置嵌入是绝对 PE，因为每个位置有独立可学习向量。
- **补丁**：**BERT 位置嵌入是门牌号，只是门牌号可以训练调整**。

## 追问预演
| 追问 | 简要答案 |
|------|----------|
| 正弦编码为什么是绝对 PE？ | 每个位置有唯一确定的编码向量，与周围位置无关 |
| RoPE 为什么是相对 PE？ | 通过旋转矩阵将相对距离内积到 QK 注意力分数中，满足平移不变性 f(q,m)f(k,n)=g(q,k,m-n) |
| ALiBi 怎么实现相对 PE？ | 直接在注意力分数上按距离加负偏置 bias=-m·\|i-j\|，越远的词注意力越弱 |
| 为什么 LLaMA 用 RoPE 不用正弦？ | RoPE 外推性更好，且与注意力机制天然融合，支持更长上下文 |
| 长度外推性差什么意思？ | 训练时最大序列长度 2048，推理时超过 2048 模型就懵，因为没见过更大门牌号 |
| 相对 PE 能无限外推吗？ | 不能无限，但比绝对 PE 强很多。实际中超过训练长度几倍仍能工作，如 LLaMA 32K 训练支持 128K 推理 |

## 手撕代码
```python
# 伪代码：RoPE 旋转位置编码
import torch
import math

def apply_rope(x, seq_len, dim):
    """
    x: [batch, seq_len, num_heads, head_dim]
    """
    # 生成旋转角度
    freqs = 1.0 / (10000 ** (torch.arange(0, dim, 2).float() / dim))
    t = torch.arange(seq_len)
    angles = torch.outer(t, freqs)  # [seq_len, dim/2]
    
    # 计算旋转矩阵的 cos 和 sin
    cos = torch.cos(angles)
    sin = torch.sin(angles)
    
    # 将 x 每两个维度一组，进行旋转
    x1, x2 = x[..., ::2], x[..., 1::2]
    x_rotated = torch.stack([x1 * cos - x2 * sin, x1 * sin + x2 * cos], dim=-1)
    return x_rotated.flatten(-2)

# 核心思想：通过旋转，让 q_i 和 k_j 的内积只与 (i-j) 有关
```
