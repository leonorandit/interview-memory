# Word2Vec 如何获取词向量？如何评估该训练得到的词向量的好坏？

## 速记卡片
`窗 | 窗滑动，负采样 | 预测练向量`

## 原始知识
- **获取方式**：通过预测任务训练，而非直接统计
- **CBOW**：上下文词预测中心词
- **Skip-gram**：中心词预测上下文词
- **滑动窗口**：在句子上滑动，生成(目标,上下文)训练对
- **负采样（Negative Sampling）**：只更新少量负样本，加速训练
- **层次Softmax**：用霍夫曼树加速，高频词路径短
- **评估-内部**：词类比（king-man+woman≈queen）
- **评估-外部**：下游任务效果（分类、NER等）
- **可视化**：t-SNE降维看聚类
- **易混点**：Word2Vec不是神经网络语言模型（NNLM）

## 顺口溜
**窗滑动，负采样**

## 单字锚点 + 防崩推导
- **锚点字**：**窗**
- **扩展口诀**：**窗滑练，类比评**
- **为什么选它**："窗"=滑动窗口，是Word2Vec训练机制的核心画面
- **防崩推导**：
  - 窗 → 滑动窗口 → 在句子上滑 → 生成(目标,上下文)对
  - 滑 → 滑动产生训练数据
  - 练 → 用CBOW或Skip-gram预测训练
  - 负 → 负采样 → 加速收敛
  - 评 → 评估：词类比（内部）+ 下游任务（外部）

## 一字一画面
- **窗** —— 像**火车窗口看风景**：窗口滑过一段风景（上下文），拍下中间那个景点（中心词）。（本质：局部上下文）
- **滑** —— 像**扫描仪从左扫到右**：每滑一步就产生一组训练样本。（本质：滑动采样）
- **负** —— 像**考试排除错误选项**：不用看所有选项，排除几个错的就够了。（本质：负采样加速）
- **评** —— 像**词汇版的智力测验**：king - man + woman = queen。（本质：类比评估）

## 面试话术
- "Word2Vec获取词向量靠**窗滑动，负采样**。用滑动窗口在文本上扫，生成目标词和上下文词的配对，然后用CBOW或Skip-gram做预测训练。为了加速，用负采样只更新少量负例，而不是遍历整个词表。"
- "评估分内外两种：内部评估做词类比实验，比如king-man+woman是不是等于queen；外部评估看下游任务，比如拿词向量做文本分类，准确率有没有提升。"

## 防混补丁
- 容易和GloVe搞混。GloVe是基于全局统计共现矩阵，Word2Vec是基于局部窗口预测。补丁：**GloVe看全局统计，Word2Vec看窗口预测**。

## 追问预演
- **问**：负采样为什么能加速？
  - **答**：原始Softmax要算词汇表所有词的概率，负采样只算1个正样本+k个负样本，从O(V)降到O(k)。
- **问**：Word2Vec的词向量有偏置吗？
  - **答**：有，比如性别偏见（医生-男人+女人=护士），这是训练数据带来的。
