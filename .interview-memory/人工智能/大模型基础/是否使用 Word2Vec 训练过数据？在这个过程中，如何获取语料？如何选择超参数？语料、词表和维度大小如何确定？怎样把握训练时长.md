# 是否使用 Word2Vec 训练过数据？在这个过程中，如何获取语料？如何选择超参数？语料、词表和维度大小如何确定？怎样把握训练时长

## 速记卡片
`训 | 训前洗，参靠调，早停看损失曲线降 | 经验+验证驱动`

## 原始知识
- **语料获取**：公开语料（Wiki/新闻）+ 业务语料（日志/评论）+ 爬虫
- **清洗流程**：去重、去HTML、统一编码、分句、去停用词（可选）
- **词表确定**：按频排序截断，通常5万-50万，最小频阈值过滤噪音
- **维度选择**：小语料50-100维，大语料200-300维，任务复杂可更高
- **超参数**：窗口大小（通常5）、负采样数（5-20）、最小词频（5-100）
- **训练时长**：看验证集类比任务准确率或内部一致性，早停
- **加粗易混点**：不是越大越好，维度超过语料支撑会过拟合

## 顺口溜
洗语料，定词表，调窗维，早停保

## 单字锚点 + 防崩推导
- **锚点字**：**训**
- **扩展口诀**：**训前洗料调参早停**
- **为什么选它**：整个题目围绕"训练"过程展开，"训"是统摄字，后面的流程都是训的前中后。
- **防崩推导**：
  - 训 → 前 → 获取语料（公开+业务）→ 清洗（去重去噪）
  - 训 → 中 → 定词表（频序截断）→ 选维度（语料量决定，宁小勿大）
  - 训 → 中 → 调参（窗口5、负采样5-20、min_count 5-100）
  - 训 → 后 → 看损失/验证指标 → 平台期早停，防过拟合

## 一字一画面
- **训** —— 像**厨师练刀工**：先买新鲜食材（语料），反复清洗切配（预处理），再调整火候刀法（超参数），最后尝味道决定停火（早停）。（本质：迭代优化过程）
- **洗** —— 像**淘金筛沙子**：把金粉（有效文本）从泥沙（HTML/乱码/重复）里筛出来。（本质：数据清洗）
- **早停** —— 像**煮面看火候**：面条煮到刚好弹牙就关火，再煮就烂了。（本质：防止过拟合）

## 面试话术
- "我训练过Word2Vec。流程是**训前洗料调参早停**。语料一般混用公开数据（Wiki、新闻）和业务数据；清洗要去了重、去HTML、统一编码。词表按频排序截断，通常几万到几十万。维度方面，小语料用100维以内，大语料200-300维。窗口设5，负采样5-20个。训练时监控词类比任务的准确率，损失曲线平台了就早停，避免过拟合。"

## 防混补丁
- 容易追求大维度大词表。补丁：**语料不够时，大词表+大维度=过拟合噪音；宁可用小模型喂饱，不用大模型饿着**。

## 追问预演
- Q：业务语料和公开语料比例？
  - A：看任务，通用任务公开为主；领域任务业务为主，可混少量公开预训练。
- Q：min_count设多少合适？
  - A：小语料5，大语料50-100；设太高会丢专有名词，设太低噪音多。
- Q：怎么判断训练好了？
  - A：内部指标看损失收敛；外部指标做词类比（king-man+woman≈queen）或下游任务验证。
