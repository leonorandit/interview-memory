# 大模型是怎么训练出来的

## 速记卡片
`预 | 洗预调对评 | 五步炼模`

## 原始知识
- **洗**：数据清洗与预处理，去重、去噪、过滤低质量文本、Tokenization
- **预**：预训练（Pre-training），自监督学习，Next Token Prediction，海量无标注数据，建立知识底座
- **调**：SFT监督微调（Supervised Fine-Tuning），高质量指令数据，教模型对话与遵循指令
- **对**：对齐（Alignment），RLHF/DPO，让模型输出符合人类价值观和偏好
- **评**：评估与迭代，能力评测、安全性评测、红队测试，持续优化
- **核心目标函数**：预训练阶段因果语言建模（Causal LM），交叉熵损失
- **追问点**：预训练占多少算力？RLHF和DPO区别？数据清洗为什么重要？
- **易混点**：SFT vs RLHF vs 预训练的目标差异

## 顺口溜
**洗预调对评**

## 单字锚点 + 防崩推导
- **锚点字**：**预**
- **扩展口诀**：**洗预调对评**
- **为什么选它**：预训练是大模型训练的核心与底座，占最多算力和数据，是整个流程的"主心骨"。
- **防崩推导**：
  - 预（预训练）→ 核心阶段，自监督学习，Next Token Prediction，读遍互联网
  - 预之前要**洗**→ 海量数据清洗、去重、Tokenization，占工程大头
  - 预之后要**调**→ SFT微调，用指令数据教模型怎么对话
  - 调之后要**对**→ RLHF/DPO对齐人类偏好，安全+有用+诚实
  - 对之后要**评**→ 全面评估、红队测试、迭代优化

## 一字一画面
- **洗** —— 像**淘金工人洗沙子**：从海量泥沙（原始互联网数据）中洗掉石头和杂质，留下金砂（高质量训练语料）。（数据清洗与预处理）
- **预** —— 像**学生读遍图书馆**：不问问题，先把全人类的书都读一遍，建立知识底座和语言规律。（预训练：自监督学习）
- **调** —— 像**家教一对一辅导**：老师给出问题和标准答案（指令对），学生模仿学习怎么回答。（SFT监督微调）
- **对** —— 像**面试礼仪培训**：不仅要有知识，还要学会讨人喜欢、遵守社会规范。（RLHF/DPO人类偏好对齐）
- **评** —— 像**模拟考试+复盘**：考完试打分，发现弱项再针对性补习。（评估与迭代）

## 面试话术
- 问训练流程 → "大模型训练可以概括为五步，口诀是**洗预调对评**。第一步'洗'——海量原始数据清洗、去重、分词；第二步'预'——预训练阶段用自监督学习做Next Token Prediction，让模型读遍互联网建立知识底座；第三步'调'——SFT监督微调，用高质量指令数据教模型对话；第四步'对'——RLHF或DPO对齐人类偏好和价值观；第五步'评'——全面评估模型能力、安全性，迭代优化。其中预训练最吃算力，对齐阶段最决定用户体验。"
- 追问各阶段差异 → "预训练是'读书'，SFT是'学说话'，RLHF是'学讨喜'。预训练用无标注数据，SFT用指令对，RLHF用偏好对比数据。"

## 防混补丁
- 容易漏掉"数据清洗"，以为直接从网上扒数据就训练。补丁：**洗是地基，预是建楼**。
- 容易把SFT和RLHF搞混。SFT教模型"怎么回答"，RLHF教模型"回答得更像人喜欢的"。补丁：**SFT教说话，RLHF教讨喜**。
- 容易以为预训练后就直接用。补丁：**预训有知识，微调会对话，对齐更安全**。

## 追问预演
1. **Q: 预训练用的什么目标函数？**
   A: 自回归语言建模（Causal Language Modeling），即给定前文预测下一个token，最小化交叉熵损失。
2. **Q: SFT数据怎么来？**
   A: 人工标注的指令-回答对（Instruction-Response），或从高质量文档中自动构造，也可借助更强模型蒸馏生成。
3. **Q: RLHF和DPO的区别？**
   A: RLHF需要训练单独的Reward Model，再用PPO等强化学习算法优化；DPO直接利用偏好数据（chosen/rejected）优化策略模型，无需显式Reward Model，更简洁高效。
4. **Q: 为什么需要RLHF/DPO？**
   A: 预训练+SFT后的模型虽然能回答问题，但可能产生有害、偏见、不符合人类偏好的内容。对齐阶段让模型输出更安全、有用、诚实。
5. **Q: 训练大模型需要多少数据？**
   A: 现代LLM预训练通常需要万亿级token（1T+），来自网页、书籍、代码、论文等多源数据。数据质量比数量更重要。
6. **Q: 数据清洗具体做什么？**
   A: 去重（MinHash/LSH）、过滤低质量文本（ perplexity 阈值、规则过滤）、去隐私敏感信息、多语言平衡、Tokenization（BPE/SentencePiece）。
