# 什么是自回归属性（autoregressive property）

## 速记卡片
`归 | 归字逐，依前推 | 逐字依赖前文`

## 原始知识
- 从左到右逐token生成
- 当前输出依赖之前所有输出
- 条件概率连乘：P(x_t|x_1...x_{t-1})
- 不能并行生成，必须串行
- **易混点**：自回归 vs 自编码（BERT是双向，非自回归）

## 顺口溜
归字逐，依前推

## 单字锚点 + 防崩推导
- **锚点字**：**归**
- **扩展口诀**：**归逐前，一序推**
- **为什么选它**："归"字直接对应"回归"，是self-regressive的核心标识。
- **防崩推导**：
  - 归 → 回归/返回 → 回到前面生成的内容
  - 字 → 逐字生成 → 一个token一个token出
  - 逐 → 逐个 → 不能跳，不能并行
  - 依 → 依赖 → 必须依赖前文
  - 前 → 前面所有token → 历史信息
  - 推 → 预测下一个 → 条件概率

## 一字一画面
- **归** —— 像**接力赛跑**：一棒跑完交下一棒，不能同时跑。（串行生成）
- **字** —— 像**成语接龙**：前一个词决定下一个词说什么。（逐token）
- **逐** —— 像**排队过独木桥**：只能一个一个过，没法并排走。（无法并行）
- **依** —— 像**搭积木**：下面每一块都支撑上面的块，抽掉一块全塌。（强依赖）
- **前** —— 像**倒下的多米诺**：只能从前向后倒，不能反过来。（单向因果）
- **推** —— 像**推箱子游戏**：每推一步，后面的箱子才能动。（条件概率）

## 面试话术
- "自回归就是**归字逐，依前推**。生成时从左到右逐token预测，每个新token都依赖之前生成的所有token，像接力赛跑一棒接一棒。GPT就是典型自回归，而BERT是自编码双向的，这是本质区别。"

## 防混补丁
- 容易和"自编码"搞混。补丁：**回归是往前看，编码是左右看**。

## 追问预演
- **Q：自回归为什么不能并行生成？** A：因为第t个token的条件概率依赖于前面t-1个已生成token，必须等前面生成完才能算下一个。
- **Q：非自回归模型有哪些？** A：BERT是自编码双向的；NAT（Non-Autoregressive Translation）通过一次前向并行生成，但质量通常不如自回归。
