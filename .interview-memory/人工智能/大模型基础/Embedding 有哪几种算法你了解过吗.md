# Embedding 有哪几种算法你了解过吗？

## 场景触发词
Embedding 算法、词向量算法、Word2Vec、GloVe、FastText、ELMo、BERT、Sentence-BERT、Embedding 演进

## 速记卡片
`算 | 嵌词共子双深句 | 三代七算法`

## 原始知识

- **第一代：静态词向量**
  - **Word2Vec（词）**
    - CBOW：用上下文预测中间词
    - Skip-gram：用中间词预测上下文
    - 训练方式：局部滑动窗口 + 负采样/层次 softmax
    - 特点：静态一词一义，无法处理一词多义
  - **GloVe（共）**
    - 基于全局词-词共现矩阵分解
    - 融合全局统计信息和局部上下文
    - 特点：训练比 Word2Vec 稳定，效果相当
  - **FastText（子）**
    - 引入子词（subword/n-gram）信息
    - 把词拆成字符 n-gram，共享子词向量
    - 特点：能处理 OOV（未登录词），对形态丰富语言效果好

- **第二代：动态上下文嵌入**
  - **ELMo（双）**
    - 双向 LSTM，逐层提取上下文特征
    - 同一个词在不同句子中有不同向量
    - 特点：首次解决一词多义，但模型浅、计算慢
  - **BERT（深）**
    - 深层 Transformer + Masked LM + NSP
    - 双向深层注意力，上下文理解极强
    - 特点：效果远超 ELMo，但计算量大，输出是词级而非句子级

- **第三代：句子级语义嵌入**
  - **Sentence-BERT（句）**
    - 基于 Siamese 网络 + 对比学习
    - 直接输出句子向量，可用余弦相似度快速比较
    - 特点：解决 BERT 做句子相似度需要拼接输入的计算瓶颈
  - **通用预训练模型**
    - OpenAI text-embedding、BGE、E5、GTE
    - 大规模语料预训练，直接用于语义搜索
    - 底层多为 BERT 架构变体

- **演进主线**
  - 静态 → 动态 → 句子级
  - 局部上下文 → 全局统计 → 双向深层 → 对比学习
  - 词级 → 句子级 → 文档级

## 一字一画面
- **算** —— 像 **宝石镶嵌工艺**：[金色工作台] 不同朝代有不同镶嵌技法，从手工到机器到激光。（Embedding 算法演进）
- **词** —— 像 **猜词游戏**：根据前后词猜中间词（CBOW），或根据中间词猜前后词（Skip-gram）。（Word2Vec）
- **共** —— 像 **统计词频表**：整本书翻一遍，统计每对词一起出现多少次，做成大表格。（GloVe 共现矩阵）
- **子** —— 像 **乐高积木**：不认识整个模型，但认识每块小积木，拼起来就能认。（FastText 子词）
- **双** —— 像 **双眼看东西**：左眼往前看，右眼往后看，合起来视野完整。（ELMo 双向 LSTM）
- **深** —— 像 **深海探测器**：一层一层往下潜，每层看到不同的语义特征。（BERT 深层 Transformer）
- **句** —— 像 **句子合影**：不是给每个词单独拍照，而是给整句话拍一张合影。（Sentence-BERT）

## 面试话术
- 问有哪些算法 → "Embedding 算法经历了三代演进，口诀是**嵌词共子双深句**。第一代静态词向量：**词**——Word2Vec，用 CBOW 或 Skip-gram 预测上下文；**共**——GloVe，基于全局共现矩阵；**子**——FastText，引入子词信息解决 OOV。第二代动态上下文：**双**——ELMo，双向 LSTM 让词向量随上下文变化；**深**——BERT，深层 Transformer + Masked LM。第三代句子级：**句**——Sentence-BERT，用对比学习直接生成句子向量，适合语义搜索。现代 RAG 一般用 BGE、OpenAI 等通用预训练模型，底层都是 BERT 架构的变体。"
- 问演进趋势 → "从静态到动态，从词级到句子级，从局部上下文到全局统计再到深层双向注意力。核心突破是 ELMo 解决了一词多义，BERT 把上下文理解做到极致，Sentence-BERT 让句子相似度计算从 O(n²) 降到 O(n)。"

## 防混补丁
- 容易把"算法"和"模型"搞混。Word2Vec/GloVe/FastText 是算法思路，BERT 是模型架构，BGE 是具体预训练模型。补丁：**Word2Vec/GloVe 是算法思路，BERT 是模型架构，BGE 是具体模型**。
- 容易认为 BERT 直接输出句子向量。实际上 BERT 输出词向量，做句子相似度需要拼接输入，计算量大。补丁：**BERT 出词向量，Sentence-BERT 出句向量**。

## 追问预演
1. **问**：Word2Vec 和 GloVe 的本质区别？
   **答**：Word2Vec 是局部滑动窗口预测（CBOW/Skip-gram），靠预测任务训练；GloVe 是全局共现矩阵分解，靠统计信息训练。Word2Vec 善于捕捉局部语义，GloVe 善于利用全局统计。
2. **问**：FastText 为什么能处理 OOV？
   **答**：FastText 把词拆成 n-gram 子词（如 "apple" → "app", "ppl", "ple"），即使没见过某个词，也能通过子词组合推断向量。
3. **问**：ELMo 和 BERT 都是上下文相关，有什么区别？
   **答**：ELMo 基于双向 LSTM，模型浅（通常2-3层），特征拼接；BERT 基于 Transformer，模型深（12-24层），自注意力机制更强。BERT 的上下文理解能力远超 ELMo。
4. **问**：Sentence-BERT 和普通 BERT 有什么区别？
   **答**：普通 BERT 输出词向量，做句子相似度需要把两句话拼在一起输入，计算量是 O(n²)；Sentence-BERT 用 Siamese 网络 + 对比学习，直接输出句子向量，用余弦相似度比较，计算量降到 O(n)。
5. **问**：为什么现代 RAG 不用 Word2Vec 而用 BERT 类模型？
   **答**：Word2Vec 是静态词向量，一词一义，无法处理多义词；BERT 类模型是动态上下文嵌入，能根据语境理解词义，语义表示能力更强，检索精度更高。
