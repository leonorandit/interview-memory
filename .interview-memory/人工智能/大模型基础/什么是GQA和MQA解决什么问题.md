# 什么是GQA/MQA？解决什么问题？

## 速记卡片
`共 | 多头共K省带宽 | MHA独，MQA全共，GQA组共`

## 原始知识
- **MHA**：每个头独立Q、K、V，效果最好但KV Cache最大
- **MQA**：所有头共享一组K、V，Cache最小但可能损失效果
- **GQA**：头分若干组，组内共享K、V，效果与效率折中
- **解决问题**：推理时KV Cache内存爆炸、带宽瓶颈
- **易混点**：训练时不用GQA/MQA也能跑，推理自回归生成才是瓶颈
- **追问点**：LLaMA2/3用GQA，PaLM用MQA，GPT早期用MHA

## 顺口溜
**多头共K省带宽**

## 单字锚点 + 防崩推导
- **锚点字**：**共**
- **扩展口诀**：**共K省带宽**
- **为什么选它**：GQA/MQA的灵魂是"共享KV"，共享才省Cache
- **防崩推导**：
  - 共 → K、V被多个Q共享 → KV Cache变为头数分之一
  - 省 → Cache小了 → 内存带宽压力降了
  - 多Q → 每个头保留独立Q → 表达能力还在
  - MQA全共 → 最省但效果差
  - GQA组共 → 折中方案（LLaMA3用）

## 一字一画面
- **共** —— 像**全班共用一个图书馆**：不用每人建图书馆，省地方（省KV Cache）。（本质：共享KV投影）
- **K** —— 像**图书管理员**：Q是学生提问，K是管理员，V是书。多个学生问同一个管理员，Cache只需存一份。（本质：KV Cache压缩）
- **省** —— 像**快递合并发货**：100个包裹并一车发，运费（带宽）大幅降低。（本质：减少内存带宽压力）
- **组** —— 像**宿舍楼每层一个宿管**：一层（一组）共用一个宿管，比全校一个（MQA）服务好，比每宿舍一个（MHA）省钱。（本质：GQA折中策略）

## 面试话术
> "核心思路是**多头共K省带宽**。MHA每个注意力头有独立Q、K、V，推理时KV Cache随序列线性增长，内存带宽成瓶颈。MQA让所有头共享一组K、V，Cache最小；GQA折中，头分组、组内共享K、V。LLaMA3就用GQA，在效果和速度间取平衡。"

## 防混补丁
容易和"多头注意力MHA"搞混。MHA是标配，GQA/MQA是推理优化版。

**补丁**：**MHA是标配房，GQA是合租屋，MQA是集体宿舍**。

## 追问预演
1. **为什么推理时KV Cache会爆炸？**
   - 自回归生成每步都要把历史K、V存下来，序列越长Cache越大，访问带宽成为瓶颈。
2. **MQA和GQA效果真的比MHA差吗？**
   - 论文显示GQA接近MHA，MQA略差但可接受。实际LLaMA2-70B用GQA效果几乎无损。
3. **除了改注意力，还有什么降低Cache的方法？**
   - 量化KV Cache（INT8/INT4）、PageAttention（vLLM）、滑动窗口注意力。
4. **训练时也要用GQA吗？**
   - 是，从头训练就确定结构，不能训完MHA再改。
