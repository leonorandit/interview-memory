# 大模型的结构化输出与 GPT Structured Outputs

## 速记卡片
`格 | 定格绑，程序用 | 结构化输出`

## 原始知识
- 结构化输出：让LLM按指定格式（JSON、XML、YAML）输出，便于程序解析
- 为什么需要：
  - LLM原生输出是自然语言，程序难以直接解析
  - 需要提取实体、分类结果、函数参数等结构化数据
- GPT Structured Outputs：OpenAI API特性，通过JSON Schema严格约束输出格式
- 实现方式：Function Calling、JSON Mode、Response Format、Pydantic约束
- **易混点**：结构化输出 vs Function Calling
- **追问点**：怎么保证输出格式正确？Schema怎么写？

## 顺口溜
**定格绑，程序用**

## 单字锚点 + 防崩推导
- **锚点字**：**格**
- **扩展口诀**：**定格绑，程序用**
- **为什么选它**：结构化输出的核心是"格式"，"格"是灵魂。
- **防崩推导**：
  - 格（格式）→ 指定输出格式（JSON/XML）
  - 定（固定）→ 固定格式，不自由发挥
  - 绑（绑定）→ 用Schema/Pydantic绑定结构
  - 程序（程序解析）→ 程序能直接解析使用
  - 用（应用）→ API对接、数据提取

## 一字一画面
- **格** —— 像**填表格**：按格子填，不能乱写。（本质：固定格式输出）
- **定** —— 像**模具**：倒出来的形状都一样。（本质：Schema约束）
- **绑** —— 像**安全带**：绑紧了才不会跑偏。（本质：严格格式绑定）
- **程序** —— 像**机器读卡**：只有涂对格子机器才认。（本质：程序可解析）

## 面试话术
- 问结构化输出 → "结构化输出是让LLM按固定格式（如JSON）输出，口诀是**定格绑，程序用**。**定**——固定格式；**格**——JSON/XML等结构化格式；**绑**——用JSON Schema或Pydantic绑定结构；**程序用**——程序能直接解析，不用再做NLP提取。OpenAI的Structured Outputs功能就是通过JSON Schema严格约束模型输出，保证格式100%正确。"
- 问和Function Calling区别 → "Function Calling是结构化输出的一种应用——输出函数调用的参数。结构化输出是更通用的能力，可以是任何JSON格式。补丁：**Function Calling是打电话，结构化输出是写便条**。"

## 防混补丁
- 容易和"Function Calling"搞混。Function Calling是结构化输出的特例（输出函数参数）。补丁：**Function Calling是打电话，结构化输出是写便条**。

## 手撕代码（Pydantic约束输出）

```python
from pydantic import BaseModel
from openai import OpenAI

class UserInfo(BaseModel):
    name: str
    age: int
    city: str

client = OpenAI()
response = client.beta.chat.completions.parse(
    model="gpt-4o",
    messages=[{"role": "user", "content": "提取：张三，25岁，住北京"}],
    response_format=UserInfo
)
result = response.choices[0].message.parsed
# result.name = "张三", result.age = 25, result.city = "北京"
```
