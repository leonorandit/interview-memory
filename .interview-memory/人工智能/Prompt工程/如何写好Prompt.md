# 如何写好Prompt

## 速记卡片
`角 | 角背示，格限练 | Prompt六要素`

## 原始知识
- **角（角色设定）**：给模型明确身份，如"你是一位资深Java工程师"，输出风格和专业度立刻对齐
- **背（背景上下文）**：提供充分的情境信息，如项目技术栈、业务场景、历史数据，减少模型猜测
- **示（Few-shot示例）**：给1-3个输入输出样例，让模型模仿风格和格式；质>量，3个通常足够
- **格（输出格式）**：规定结构——JSON、Markdown表格、代码块、分步骤列表，降低解析成本
- **限（约束条件）**：长度限制、风格要求、禁止事项；复杂任务加"请一步一步想"做思维链（CoT）
- **练（迭代优化）**：根据输出反复调整Prompt，做A/B对比；Prompt是实验科学，不是作文比赛
- **System + User 分工**：System Prompt 设定全局角色和行为规范，User Prompt 传具体任务和输入数据
- **易混点**：Prompt越长越好、一次就能写好
- **追问点**：Few-shot vs Zero-shot、CoT什么时候用、Prompt注入怎么防

## 顺口溜
**角背示，格限练**

## 单字锚点 + 防崩推导
- **锚点字**：**角**
- **扩展口诀**：**角背示，格限练**
- **为什么选它**：写好Prompt的第一步是"选角"——给模型一个身份，身份定了，语气和专业度才对味。
- **防崩推导**：
  - 角 → 角色设定 → 明确身份（资深架构师/产品经理/医生）→ 输出风格对齐
  - 背 → 背景上下文 → 项目技术栈、业务场景、历史数据 → 减少模型猜测和幻觉
  - 示 → Few-shot示例 → 1-3个输入输出样例 → 让模型模仿格式和风格 → 质>量
  - 格 → 输出格式 → JSON/Markdown表格/代码块 → 降低下游解析成本
  - 限 → 约束条件 → 长度、风格、禁止项 → 复杂任务加"请一步一步想"（CoT）
  - 练 → 迭代优化 → 看输出 → 调Prompt → A/B对比 → 持续打磨

## 一字一画面
- **角** —— 像**导演选角**："你来演一个资深架构师。"（角色设定）
- **背** —— 像**剧本背景介绍**："故事发生在2024年的互联网公司，技术栈是Spring Boot 3.0..."（提供上下文）
- **示** —— 像**示范表演**：导演先演一遍，演员跟着学。（Few-shot示例）
- **格** —— 像**镜头规格**："这段用特写，那段用全景，最后出字幕。"（输出格式）
- **限** —— 像**拍摄注意事项**："不要露商标，台词别超过50字，一步一步来。"（约束+CoT）
- **练** —— 像**排练打磨**：拍完看回放，不行再来一条。（迭代优化）

## 面试话术
- 问经验 → "Prompt工程实践经验我总结为**角背示，格限练**六个字。**角**是角色设定，给模型一个明确身份；**背**是提供充分的背景上下文；**示**是Few-shot示例，给1-3个样例让模型模仿；**格**是规定输出格式，比如JSON或Markdown表格；**限**是设定约束，包括长度、风格、禁止项，复杂任务还要加'请一步一步想'做思维链；**练**是迭代优化，根据输出反复调Prompt，做A/B对比。实际项目中，System Prompt用来设定全局行为，User Prompt用来传具体任务，两者配合效果最好。"
- 问本质 → "Prompt工程的本质不是'话术包装'，而是**信息压缩与意图对齐**——用最少、最精确的token，把人的意图无损传递给模型。"

## 防混补丁
- 容易以为"Prompt越长越好"。实际上要精确而不是冗长。**补丁：Prompt要精确不要堆砌。**
- 容易忽略"迭代"。实际上写好Prompt是实验科学，不是一次性的。**补丁：Prompt是实验科学，不是作文比赛。**

## 追问预演
1. **Few-shot和Zero-shot什么区别？** 答：Zero-shot不给示例直接下指令；Few-shot给1-3个示例让模型模仿。示例不是越多越好，3个通常足够，多了反而增加噪声。
2. **思维链（CoT）什么时候用？** 答：复杂推理任务（数学、逻辑、多步分析）用CoT效果提升明显；简单任务不需要，反而增加token消耗和延迟。
3. **System Prompt和User Prompt怎么分工？** 答：System设定全局角色和行为规范（如"你是一位严谨的工程师，回答要简洁"），User传具体任务和输入数据（如"请优化这段SQL"）。
4. **Prompt注入怎么防？** 答：输入校验、敏感词过滤、输出后处理、沙箱执行、用户输入与系统指令用分隔符隔离。
