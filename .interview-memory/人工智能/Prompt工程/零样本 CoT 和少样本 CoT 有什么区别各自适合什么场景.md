## 速记卡片
`零 | 零例靠引导，少例靠示范 | 有无示例，是核心差异`

## 原始知识
- CoT = Chain-of-Thought，思维链
- 零样本 CoT：不加示例，只在 Prompt 写"请逐步思考"
- 少样本 CoT：给 1~5 个带推理过程的示例
- 零样本适合：通用任务、快速尝试、示例难构造
- 少样本适合：复杂推理、格式固定、需要稳定输出
- **易混**：零样本不是无 Prompt，而是无示例

## 顺口溜
零引导，少示范

## 单字锚点 + 防崩推导
- **锚点字**：零
- **扩展口诀**：零个例子靠嘴说，少量例子靠手做
- **为什么选它**："零"是最大差异点，有无示例一目了然
- **防崩推导**：忘了区别 → 想"零" → 零代表什么？→ 零个示例 → 那另一个呢？→ 少量示例 → 零样本怎么做？→ 加引导词"let's think step by step" → 少样本怎么做？→ 给推理示例 → 各自场景？→ 通用/快速 vs 复杂/稳定

## 一字一画面
**零** —— 像**空碗倒汤**：一只白瓷碗（零样本）里什么都没有，但碗边贴着红色标签"请一步一步来"；另一只碗（少样本）里已盛了三勺金黄蛋花汤（示例），第四勺倒进去时自动跟着蛋花 pattern（本质：有无先验示例决定引导方式）

## 面试话术
"零样本 CoT 就是不给例子，只在提示词里加一句'请逐步思考'，适合快速尝试通用任务。少样本 CoT 是给几个带推理过程的例子让模型照猫画虎，适合复杂推理或者需要固定输出格式的场景。"

## 防混补丁
| 维度 | 零样本 CoT | 少样本 CoT |
|------|-----------|-----------|
| 示例数量 | 0 | 1~5 |
| 触发方式 | "逐步思考"等引导词 | 示例中的推理过程 |
| 成本 | 低（省 Token） | 高（示例占 Token） |
| 稳定性 | 较低 | 较高 |
| 适用场景 | 简单推理、快速验证 | 复杂数学、代码、格式固定 |

## 追问预演
- Q: 为什么少样本 CoT 比零样本稳定？
  A: 示例提供了明确的推理模式和输出格式，减少模型自由发挥。
- Q: 示例越多越好吗？
  A: 不是，通常 1~5 个足够，太多会占 Token 且可能过拟合示例模式。
