## 速记卡片
`隔 | 三引号隔边界，角色任务不越界 | 结构化 = 防串味`

## 原始知识
- 分隔符作用：区分角色、任务、示例、上下文
- 常用符号：```、"""、<>、XML 标签、###
- 防止模型把指令当内容，或把内容当指令
- **不用分隔符 → 角色/任务/示例混淆**
- 使用场景：多轮对话、多文档输入、角色扮演
- **易混**：分隔符不是越多越好，要清晰一致

## 顺口溜
分隔如围墙，各区域不撞

## 单字锚点 + 防崩推导
- **锚点字**：隔
- **扩展口诀**：隔出三区——你是谁、做什么、参考啥
- **为什么选它**："隔"直接对应分隔符的本质功能——物理隔离
- **防崩推导**：忘了作用 → 想"隔" → 隔什么？→ 隔角色、隔任务、隔示例 → 用什么隔？→ ``` / """ / XML → 不隔会怎样？→ 模型把指令当正文，或把正文当指令

## 一字一画面
**隔** —— 像**透明玻璃隔断**：一间大办公室被蓝色玻璃（```）隔成三个格子，左边格子坐着穿西装的"角色"，中间格子是穿工装的"任务"，右边格子堆着"参考资料"，三格互不穿透，光线却还能透过去（本质：语义边界清晰，信息仍能流通）

## 面试话术
"分隔符就像给 Prompt 划区域，用三引号或者 XML 标签把角色设定、任务指令和参考内容隔开，防止模型把任务描述当成要处理的内容，或者把示例当成新指令。"

## 防混补丁
| 分隔符 | 适用场景 |
|--------|---------|
| ``` | 代码块、长文本 |
| """ | 引用内容、多行字符串 |
| <tag></tag> | 结构化输入、多文档 |
| ### | 分段、章节划分 |
| 注意 | 不要和任务内容中的符号冲突 |

## 追问预演
- Q: 分隔符和 Markdown 语法冲突怎么办？
  A: 用不常见的符号组合，或用 XML 标签更稳妥。
- Q: 为什么有时候加了分隔符效果反而差？
  A: 可能分隔符太多导致结构复杂，或和训练数据中的分布不一致。
