# 什么是上下文窗口 Context Window？它有什么限制？

## 速记卡片
`窗 | 窗有边，超则断 | 长度限制`

## 原始知识
- 上下文窗口 = 模型一次可见的 token 范围
- 包含输入 + 输出总长度
- 超出则截断或报错
- 滑动窗口可缓解长文本
- **加粗易混点**：窗口大 ≠ 记忆长，是短期工作记忆
- **追问点**：长文本怎么处理？

## 顺口溜
窗有边，超则断

## 单字锚点 + 防崩推导
- **锚点字**：**窗**
- **扩展口诀**：**窗有边**
- **为什么选它**：上下文窗口的"窗"字直接对应，像窗户一样有固定大小
- **防崩推导**：
  - 窗 → 有边框 → 有最大 token 数
  - 窗 → 超了尺寸 → 截断或报错
  - 窗 → 看到头尾忘中间 → 需要 RAG/摘要/滑动窗口

## 一字一画面
**窗** —— 像 **飞机舷窗**：圆形玻璃框住有限风景，窗外云海再美，超出窗框就看不见。（本质：模型视野有硬边界）

## 面试话术
- "上下文窗口就像飞机舷窗，只能看到固定范围的风景。它限制了一次输入加输出的总 token 数，超长文档会被截断，导致模型'看到头尾忘中间'。实际应用中我们用 RAG、摘要、滑动窗口来绕过这个限制。"

## 防混补丁
- 容易和"模型参数规模"搞混。窗口是短期记忆容量，参数量是长期知识储备。补丁：**窗是眼界，参是脑容量**。

## 追问预演
- **Q: 长文本怎么处理？** A: RAG 检索增强、文本摘要、滑动窗口、LongRope 位置编码外推。
- **Q: 不同模型的窗口大小？** A: GPT-4 128k，Claude 3 200k，国产模型普遍 32k-128k。
