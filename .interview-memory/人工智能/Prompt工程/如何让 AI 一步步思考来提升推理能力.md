# 如何让 AI 一步步思考来提升推理能力？

## 速记卡片
`步 | 步拆解，链推理 | CoT 思维链`

## 原始知识
- Chain-of-Thought：显式写出中间步骤
- Few-shot-CoT：给带推理过程的示例
- Zero-shot-CoT："Let's think step by step"
- 自动 CoT：Auto-CoT 采样多样本生成推理链
- **加粗易混点**：CoT 只对推理任务有效，创意生成不一定需要
- **追问点**：CoT 和 ReAct 区别？

## 顺口溜
步拆解，链推理

## 单字锚点 + 防崩推导
- **锚点字**：**步**
- **扩展口诀**：**步带链**
- **为什么选它**："步"是核心动作，一步步是 CoT 的精髓
- **防崩推导**：
  - 步 → 拆解问题 → 中间步骤
  - 步 → 链式推理 → 每一步可验证
  - 步 → 最终答案更准确

## 一字一画面
**步** —— 像 **跳房子格子**：孩子必须一格一格跳，不能从起点直飞终点，每格脚印就是推理步骤。（本质：显式中间态防跳步出错）

## 面试话术
- "提升推理能力的秘诀是**步拆解，链推理**。在 prompt 里加入 'Let's think step by step'，或者给几个带详细推导过程的示例。这样 AI 会把大难题拆成小步骤，像解数学题一样写出中间过程，最终答案准确率能提升 40% 以上。"

## 防混补丁
- 容易和"ReAct"搞混。CoT 只思考不行动，ReAct 是思考加行动加工具调用。补丁：**CoT 动脑，ReAct 动手**。

## 追问预演
- **Q: 所有任务都需要 CoT 吗？** A: 否，简单分类、创意生成不需要，数学/逻辑/代码需要。
- **Q: CoT 对模型有要求吗？** A: 小模型效果差，通常 100B 以上参数才能稳定涌现 CoT 能力。
