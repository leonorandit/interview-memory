# 如何为不同领域设计专用提示词？比如编程、创作、数据分析

## 速记卡片
`域 | 域不同，角色换 | 领域定制`

## 原始知识
- 编程：强调代码规范、测试用例、语言版本
- 创作：强调风格、受众、字数、情绪
- 数据分析：强调数据格式、统计方法、可视化要求
- 通用套路：角色 + 背景 + 任务 + 约束 + 输出格式
- **加粗易混点**：不是堆砌关键词，是结构化约束
- **追问点**：通用模板是什么？

## 顺口溜
域不同，角色换

## 单字锚点 + 防崩推导
- **锚点字**：**域**
- **扩展口诀**：**域定角**
- **为什么选它**："域"是领域，先定领域再定角色，是设计专用提示词的第一步
- **防崩推导**：
  - 域 → 确定领域
  - 域 → 切换专家角色 → 调整术语
  - 域 → 定制输出格式 → 加入领域约束

## 一字一画面
**域** —— 像 **手术室换衣**：医生进手术室换绿大褂（编程领域穿代码规范服），作家进书房穿睡衣（创作领域穿文艺范），不能穿错场合。（本质：环境决定行为规范）

## 面试话术
- "设计专用提示词，核心是**域不同，角色换**。编程领域让 AI 扮演资深工程师，强调代码规范和边界测试；创作领域让 AI 扮演专栏作家，强调受众和情绪；数据分析领域强调统计方法和可视化格式。通用模板是'角色 + 背景 + 任务 + 约束 + 输出格式'。"

## 防混补丁
- 容易和"prompt 模板"搞混。模板是通用框架，领域专用是在框架内填充专业约束。补丁：**模板是骨架，域是血肉**。

## 追问预演
- **Q: 怎么验证提示词效果？** A: A/B 测试、人工评估、用 eval set 跑分。
- **Q: 多领域混合怎么办？** A: 用 system prompt 锁定主领域，user prompt 补充子任务约束。
