# 如何让 AI 进行自我反思和迭代优化？

## 速记卡片
`反 | 反查错，再改对 | 迭代优化`

## 原始知识
- 让模型 critique 自己的输出
- 多轮迭代：生成 → 评审 → 修正
- Self-reflection / Reflexion 框架
- 可引入外部反馈（单元测试、人工评分）
- **加粗易混点**：不是简单重试，是显式列出错误再修正
- **追问点**：怎么防止假反思？

## 顺口溜
反查错，再改对

## 单字锚点 + 防崩推导
- **锚点字**：**反**
- **扩展口诀**：**反查改**
- **为什么选它**："反"是反思的核心动作，也是反思的第一步
- **防崩推导**：
  - 反 → 反过来检查 → 发现错误
  - 反 → 列出问题清单 → 逐条修正
  - 反 → 再验证 → 直到通过标准

## 一字一画面
**反** —— 像 **镜子照脸**：先化妆（生成），再照镜子找瑕疵（反思），补妆（修正），直到满意。（本质：闭环质检）

## 面试话术
- "核心口诀是**反查错，再改对**。先让 AI 生成第一版答案，然后让它以评审者身份挑剔自己的错误，列出问题清单，最后基于清单修正输出。通常三轮迭代就能大幅提升质量，这在代码审查和写作场景中特别有效。"

## 防混补丁
- 容易和"多轮对话"搞混。普通多轮是用户追问，反思是 AI 自己 critique。补丁：**用户追是外驱，AI 反是内省**。

## 追问预演
- **Q: 怎么让 AI 真的反思而不是套话？** A: 给评分标准、强制输出错误列表、温度调低、引入外部验证（如单元测试）。
- **Q: 迭代次数多少合适？** A: 通常 2-3 轮，设质量阈值提前终止。
