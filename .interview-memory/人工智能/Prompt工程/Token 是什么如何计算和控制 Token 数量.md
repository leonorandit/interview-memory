## 速记卡片
`计 | Token计数用BPE，拆分合并要算计 | 文本→子词→整数`

## 原始知识
- Token 是最小文本单位
- 英文按子词拆分，中文按字或词
- BPE 算法合并高频对
- **1 Token ≠ 1 字**，中文约 1~2 字/Token
- 计费按 Token 数，输入输出都算
- 控制方法：精简提示、设 max_tokens、分块处理
- **易混**：字符数 vs Token 数，差 2~4 倍

## 顺口溜
BPE 合并计 Token

## 单字锚点 + 防崩推导
- **锚点字**：计
- **扩展口诀**：计费、计算、计数量，字字不离计

- **防崩推导**：忘了细节 → 想"计" → 计算什么？→ 怎么算？→ BPE / 子词拆分 → 怎么控制？→ 限长/精简/分块

## 防混补丁
| 概念 | 区别 |
|------|------|
| Token vs 字符 | 英文 1 词 ≈ 1~2 Token，中文 1 字 ≈ 1~2 Token |
| Token vs 词 | Token 是子词，可能只含半个单词 |
| 计费方式 | 通常 input + output 合计收费 |

## 追问预演
- Q: 中文一个词语大概多少 Token？
  A: 常见词 1~2 Token，生僻字可能拆成 3~4 个。
- Q: 如何估算一段文本的 Token 数？
  A: 用 tiktoken 库，或粗略按英文 4 字符 ≈ 1 Token、中文 1 字 ≈ 1~2 Token 估算。
