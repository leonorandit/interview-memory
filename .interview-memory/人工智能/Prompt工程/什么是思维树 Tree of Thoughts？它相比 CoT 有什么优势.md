# 什么是思维树 Tree of Thoughts？它相比 CoT 有什么优势？

## 速记卡片
`树 | 树分叉，多路搜 | 单链死，树回头`

## 原始知识
- **CoT（思维链）**：线性单路径推理，一步一步往下走
- **ToT（思维树）**：多分支探索，可以回溯、评估、剪枝
- **CoT缺点**：一旦某步错了，后面全错，无法回头
- **ToT优势**：多路径并行探索，评估后选最优分支，支持回溯
- **ToT步骤**：生成候选→评估得分→选最优→继续展开
- **代价**：调用次数多，成本高
- **易混点**：ToT不是简单枚举，是有策略的搜索

## 顺口溜
**树分叉，多路搜**

## 单字锚点 + 防崩推导
- **锚点字**：**树**
- **扩展口诀**：**树多回优**
- **为什么选它**："树"直接对应Tree of Thoughts，画面感最强
- **防崩推导**：
  - 树 → 树状结构 → 有分叉 → 多路径并行
  - 多 → 多个候选思维同时生成
  - 回 → 可以回溯，走错路退回
  - 优 → 评估后选最优分支
  - CoT是单链 → 一条路走到黑 → 错一步全崩

## 一字一画面
- **树** —— 像**迷宫中的分叉路口**：面前三条路，每条都先试一步。（本质：分支探索）
- **多** —— 像**同时派三支侦察队**：分别探路，回来报告哪条最好走。（本质：并行候选）
- **回** —— 像**走迷宫撞墙退回**：此路不通，回到上一个路口换条路。（本质：回溯机制）
- **优** —— 像**评委打分选冠军**：三支侦察队汇报后，选得分最高的继续深入。（本质：评估剪枝）

## 面试话术
- "ToT相比CoT，核心区别是**树分叉，多路搜**。CoT像一条直线往下走，一步错步步错；ToT像树一样在每个节点生成分支，评估后选最优，还能回溯。优势是复杂推理题准确率更高，缺点是调用成本也更高。"
- "比如24点游戏，CoT可能第一步选错数字后面全错，ToT会同时试几种组合，评估哪条更接近目标，再深入。"

## 防混补丁
- 容易和"多轮对话"搞混。多轮是时间上的延续，ToT是空间上的分支。补丁：**多轮是纵向聊，ToT是横向扩**。

## 追问预演
- **问**：ToT的评估函数怎么设计？
  - **答**：可以用模型自评（"这个思路 promising 吗？"），也可以写规则评分，或者让模型输出置信度。
- **问**：ToT和Agent的ReAct有什么区别？
  - **答**：ReAct是观察-思考-行动的循环，ToT是思维空间的多路径搜索，两者可以结合，ReAct负责行动，ToT负责思考分支。
