# 什么是系统提示词 System Prompt？它和用户提示词有什么区别？

## 速记卡片
`系 | 系导演，用演员 |  backstage vs 前台`

## 原始知识
- **System Prompt**：设定AI角色、能力边界、全局规则
- **User Prompt**：用户具体的问题或任务
- **区别1**：系统提示是"背景设定"，用户提示是"当天戏份"
- **区别2**：系统提示通常不变，用户提示每次不同
- **区别3**：系统提示权重更高，影响整体风格
- **易混点**：System Prompt不是"隐藏指令"，模型可能泄露
- **追问点**：多轮对话中系统提示是否会被覆盖

## 顺口溜
**系导演，用演员**

## 单字锚点 + 防崩推导
- **锚点字**：**系**
- **扩展口诀**：**系定调，用提问**
- **为什么选它**："系"=系统=导演，是整个对话的幕后掌控者
- **防崩推导**：
  - 系 → 系统 → 导演 → 定角色、定规则、定风格
  - 导 → 导演不露面 →  backstage，全局把控
  - 演 → 演员前台表演 → 用户提示，具体执行
  - 用 → 用户提问 → 每次不同，当天任务

## 一字一画面
- **系** —— 像**电影导演坐在监视器后**：不露面，但决定整个片子的基调和规则。（本质：全局设定）
- **导** —— 像**剧本扉页的角色设定表**：写着"主角性格冷静、不说脏话"。（本质：角色边界）
- **演** —— 像**演员在镜头前念台词**：导演说"演悲伤"，演员演具体哪场哭戏。（本质：具体执行）
- **用** —— 像**观众点单**：今天要看喜剧还是悲剧，每天不一样。（本质：用户意图）

## 面试话术
- "System Prompt就像**导演**，决定AI演什么角色、用什么风格；User Prompt就像**演员当天的戏份**，是用户具体的问题。系统提示权重更高，通常隐藏在后，用户提示是每一次的前台交互。"
- "实际应用里，系统提示一般写角色设定和全局约束，比如'你是一位严谨的代码审查员'，用户提示才是具体的代码片段。"

## 防混补丁
- 容易和"隐藏指令注入"搞混。系统提示是官方机制，不是攻击手段。补丁：**系统提示是明牌导演，隐藏注入是偷偷改剧本**。

## 追问预演
- **问**：多轮对话后系统提示会被遗忘吗？
  - **答**：上下文窗口足够时不会，但超长对话可能被压缩或截断，关键约束要在用户提示里重复强调。
- **问**：系统提示能不能完全防止越狱？
  - **答**：不能，System Prompt只是建议性权重，不是绝对护栏，敏感场景要配内容过滤层。
