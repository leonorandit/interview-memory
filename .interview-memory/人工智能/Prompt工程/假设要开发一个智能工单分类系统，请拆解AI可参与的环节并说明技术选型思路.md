## 原始知识

- **分类**打标（BERT 类模型 / 大模型 Few-shot / 微调 Fine-tuning）
- **优先级**判定（规则引擎 + 模型融合打分）
- **相似**工单合并（向量检索相似度 + ANN 召回）
- **自动**回复建议（大模型生成话术 / 模板匹配）
- **兜底**规则（置信度低转人工 / 敏感词规则拦截）
- **技术**选型：数据少选 Few-shot，数据多选微调 BERT，相似度用向量检索，核心业务加规则引擎兜底

## 顺口溜

## 单字锚点 + 防崩推导

- **锚点字**：拆
- **扩展口诀**：拆分派跟，四环节
- **为什么选它**："拆" = 拆解 = 把传统工单处理流程拆成四个 AI 可参与的环节
- **推导链**：
  1. 拆 → 拆解流程
  2. 拆解 → 四环节流水线：分（分类打标）、派（优先级判定 = 派单）、跟（相似合并 = 跟进历史）、回（自动回复建议）
  3. 技术选型对齐：分类用 **BERT**（快 + 准）或 **大模型 Few-shot**（样本少）或 **微调**（样本多）；相似合并用 **向量检索**；优先级用 **规则 + 模型融合**；回复用 **大模型生成**
  4. 关键环节：**规则引擎兜底**（防 AI 抽风）

## 一字一画面

**拆** —— 像【拆快递包裹的四连拍】：第一步分类（看面单写品类），第二步分优先级（急件插红旗），第三步比对相似（是不是重复下单），第四步自动回复（塞张说明书）（本质：把人工流水线拆成 AI 节点）。

## 面试话术

"智能工单系统就是把人工处理流程拆解给 AI，先自动分类打标，再判定优先级派单，接着用向量检索合并相似工单，最后大模型生成回复建议。技术选型上，分类用 BERT 或 Few-shot，相似度用向量检索，最后一定要加规则引擎兜底，不能让 AI 直接决定退款或赔偿。"

## 防混补丁

容易和**全自动无人值守**搞混？一句补丁：**拆字记环节，AI 辅助人决策，规则兜底不能弃。**

## 追问预演

- **Q：为什么分类用 BERT 而不是大模型？**
  - A：BERT 推理快、成本低，大模型准但贵；数据量小选 Few-shot，数据量足选微调 BERT。
- **Q：相似合并怎么做？**
  - A：工单文本向量化，ANN 检索 TopK，设定相似度阈值，超过阈值自动合并或提示关联。
- **Q：规则兜底一般设哪些？**
  - A：敏感词拦截、高金额强制人工、置信度低于阈值转人工、 SLA 超时告警。
