# 如何选择和设计 Few-shot 示例以提升效果？

## 速记卡片
`例 | 例要少，质要高 | 示例设计`

## 原始知识
- 数量：通常 3-5 个足够
- 质量：覆盖边界 case、代表性
- 多样性：正反例、不同角度
- 格式：输入输出格式一致
- 顺序：重要示例放后面（近因效应）
- **加粗易混点**：不是越多越好，垃圾示例会带偏模型
- **追问点**：示例顺序有影响吗？

## 顺口溜
例要少，质要高

## 单字锚点 + 防崩推导
- **锚点字**：**例**
- **扩展口诀**：**例精全**
- **为什么选它**："例"是 few-shot 的核心，也是面试关键词
- **防崩推导**：
  - 例 → 示例 → 少量精选
  - 例 → 覆盖边界 → 正反兼备
  - 例 → 格式统一 → 模型模仿

## 一字一画面
**例** —— 像 **教学标本**：老师只拿 3 个典型蝴蝶标本（少而精），不是搬整箱昆虫（多而杂），每个都带标签说明特征。（本质：示范学习的质量 > 数量）

## 面试话术
- "设计 few-shot 示例的口诀是**例要少，质要高**。一般 3 到 5 个就够了，关键是覆盖边界情况和典型模式，格式必须统一。如果示例质量差，模型会精准模仿错误，所以宁缺毋滥。"

## 防混补丁
- 容易和"fine-tuning"搞混。few-shot 是 in-context learning 不改动模型，微调是改权重。补丁：**例是临时抄作业，调是长期改习惯**。

## 追问预演
- **Q: 示例顺序有影响吗？** A: 有，最近示例权重更高，重要/难 case 放后面。
- **Q: 示例从哪来？** A: 训练集采样、人工编写、Auto-CoT 自动生成后筛选。
