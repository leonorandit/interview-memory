## 原始知识

- **角色定义**：让AI扮演资深前端工程师，明确身份和水平
- **框架约束**：指定**Vue3 + TypeScript** + Composition API / `<script setup>`
- **具体功能需求**：表单字段、输入类型、交互逻辑描述清楚
- **验证规则**：必填、格式、长度、正则等规则逐条列出
- **错误提示**：每条验证对应具体的错误文案
- **输出格式**：要求完整组件代码 + 注释 + 使用示例

## 顺口溜

> 框角验，三要素：Vue3框死技术栈，角色定好调角度，验证规则逐条验。

## 单字锚点 + 防崩推导

- **锚点字**：框
- **扩展口诀**：框角验，三要素
- **为什么选它**：题目核心是“写出包含以下要素的Prompt”，Prompt工程本质是给AI画一个**框**，约束它只能在框内输出，“框”字点出Prompt的灵魂是约束与限定。
- **推导链**：
  1. 框 → **框架约束** → 技术栈框死 → **Vue3 + TypeScript**
  2. 框 → 角色**角**度 → 让AI扮演什么角色？→ 资深前端工程师，决定代码质量和风格
  3. 框 → 验证规则逐条**验** → 必填、格式、长度、正则、错误提示
  4. 框 → 三要素 → 框架（技术栈）、角色（角度）、验证（规则），三者缺一不可
  5. 框 → 输出还要锁什么？→ 输出格式要求（完整代码 + 注释 + 示例）

## 一字一画面

- **框** —— 像建筑脚手架：黄色钢管搭成四方形框架，工人在框内砌砖，砖块绝不会飞出框外（本质：Prompt的约束边界）
- **角** —— 像导演喊Action：黑色鸭舌帽导演举着扩音器，对演员喊“你要演一个十年的老前端！”（本质：角色定义决定输出视角）
- **验** —— 像机场安检门：红色警报灯闪烁，行李逐件过X光机，不合格就亮红灯拦截（本质：验证规则逐条过滤输入）

## 面试话术

“要写一个高质量的Prompt，我会先框死技术栈，比如明确说‘用Vue3和TypeScript，script setup语法’；然后定角色，比如‘你是一位资深前端工程师’；再把功能需求和验证规则逐条列清楚，包括每个字段的校验逻辑和对应的错误提示；最后要求输出完整可运行的组件代码加上使用示例。这样AI的输出质量和可用性会高很多。”

## 防混补丁

- **容易搞混**：只写功能需求，不写角色和格式要求，导致输出风格飘忽、代码不完整。
- **补丁口诀**：没有角色的Prompt像没导演的剧组，没有格式的Prompt像没排版的试卷。

## 追问预演

1. **问**：如果AI生成的代码有bug，Prompt里怎么引导它自查？
   **答**：可以在Prompt末尾加一步“在输出代码前，先逐条检查验证规则是否都实现了，并列出检查清单”。
2. **问**：Prompt里需要指定组件库吗？比如Element Plus？
   **答**：建议指定，否则AI可能混用不同组件库的写法。如果不指定UI库，也要说明“使用原生HTML表单元素”。
3. **问**：怎么让AI输出的代码直接可用，不需要大量修改？
   **答**：要求输出“完整的单文件组件（SFC）”，包含template、script、style三个块，并且给出父组件调用该组件的示例代码。
