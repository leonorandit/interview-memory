## 原始知识

- **饿汉式**（类加载创建 / 线程安全 / 可能浪费）
- **懒汉式**（延迟加载 / 需 synchronized 或 DCL）
- **静态内部类**（延迟 + 线程安全 / **推荐**）
- **枚举**（最简洁 / **防反射攻击** / 推荐）
- **DCL 双重检查**（volatile 防指令重排序）
- 反射可破坏普通单例
- 序列化需 readResolve

## 面试话术

"单例有五种常见实现。饿汉式简单但可能浪费内存；懒汉式要加锁或 DCL，DCL 记得加 volatile 防止指令重排。实际项目中推荐静态内部类，延迟加载又线程安全。枚举最简洁还能防反射攻击。"

## 追问预演

**Q: 为什么枚举能防反射？**  
A: 反射调用枚举构造器会抛异常，JVM 层面禁止。

**Q: 静态内部类为什么线程安全？**  
A: 类加载由 JVM 保证线程安全，内部类被引用时才加载。

**Q: DCL 为什么要 volatile？**  
A: 防止指令重排序，避免拿到未构造完成的对象。
