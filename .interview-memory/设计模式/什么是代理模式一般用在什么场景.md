# 什么是代理模式？一般用在什么场景？

## 速记卡片
`代 | 代增控，三场景 | 代vs装饰`

## 原始知识
- 为对象提供**代理控制访问**
- **静态代理** / **动态代理**（JDK动态代理/CGLIB）
- 场景：AOP、远程代理、懒加载、权限控制
- **易混点**：与装饰器模式区别
- **追问点**：JDK动态代理原理、CGLIB区别

## 顺口溜
**代增控，三场景**

## 单字锚点 + 防崩推导
- **锚点字**：**代**
- **扩展口诀**：**代增控，三场景**
- **为什么选它**：代理模式的本质就是"代"——代替原对象出面，在访问前后来点操作。
- **防崩推导**：
  - 代 → 代替 → 客户端不直接找目标 → 找代理对象
  - 代 → 增强 → 代理在目标前后加逻辑 → 日志、事务、权限
  - 代 → 控制 → 决定让不让访问 → 远程代理、权限代理、懒加载
  - 代 vs 装饰器 → 代控制访问，装饰增强功能；代理和目标关系通常编译期定，装饰更灵活

## 一字一画面
- **代** —— 像**明星经纪人**：粉丝（客户端）不直接找明星（目标对象），先找经纪人（代理），经纪人先验资、排期、签合同，再安排见明星。（本质：代理代替目标接见请求，前后加控制逻辑）
- **增** —— 像**外卖打包**：餐馆做好菜（目标逻辑），外卖员套保温袋、贴订单（增强），再送给顾客。（本质：在目标方法前后附加行为，如日志、事务）
- **控** —— 像**小区门禁**：访客不能直接进业主家，先在门卫登记、刷二维码，权限够了才放行。（本质：控制对目标对象的访问）

## 面试话术
- "代理模式就是**代增控，三场景**。给目标对象找个替身，客户端先访问替身，替身在调用目标前后做一些增强或控制。比如Spring AOP就是代理模式，在方法前后自动加事务或日志。"
- "代理分静态代理和动态代理，JDK动态代理基于接口生成代理类，CGLIB基于继承生成子类。Spring默认用JDK，没有接口时切CGLIB。"

## 防混补丁
- 容易和**装饰器模式**搞混。代理控制访问，装饰增强功能；代理通常和目标一一对应，装饰可以层层叠加。补丁：**代理看大门，装饰贴墙纸**。

## 追问预演
- **Q：JDK动态代理和CGLIB的区别？**
  - A：JDK动态代理要求目标类实现接口，通过反射生成接口实现类；CGLIB通过ASM字节码技术生成目标类的子类，不需要接口。JDK代理只能代理接口方法，CGLIB可以代理类方法（但无法代理final方法）。
- **Q：动态代理的原理是什么？**
  - A：JDK动态代理核心在Proxy类和InvocationHandler。运行时动态生成字节码加载到JVM，所有代理方法调用都转发到InvocationHandler的invoke方法。
- **Q：代理模式和适配器模式的区别？**
  - A：代理是同一个接口，控制访问；适配器是接口不兼容，转接成另一个接口。
