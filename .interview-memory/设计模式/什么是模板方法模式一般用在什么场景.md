# 什么是模板方法模式？一般用在什么场景？

## 速记卡片
`模 | 模骨架，子填肉 | 模vs策略`

## 原始知识
- 父类定义**算法骨架**
- 子类**重写特定步骤**
- **钩子方法**控制流程
- 场景：JDBC模板、Spring回调、算法框架
- **易混点**：与策略模式区别
- **追问点**：钩子方法作用、好莱坞原则

## 顺口溜
**模骨架，子填肉**

## 单字锚点 + 防崩推导
- **锚点字**：**模**
- **扩展口诀**：**模骨架，子填肉**
- **为什么选它**：模板方法的核心就是"模板"定框架，像填空题。
- **防崩推导**：
  - 模 → 模板 → 先搭骨架 → 父类把流程定死
  - 模 → 填空 → 子类填具体实现 → 重写抽象方法
  - 模 → 钩子 → 模板里留开关 → 子类决定某些步骤执不执行
  - 模 vs 策略 → 模是"一条流水线，换工人"；策略是"换整条流水线"

## 一字一画面
- **模** —— 像**3D打印骨架**：机器先把整体框架打好（父类骨架），然后人工往里面填肌肉和器官（子类实现）。（本质：骨架固定，内容可变）
- **骨** —— 像**炒菜流程**：热油→下料→翻炒→调味→出锅，流程不能变，但下什么料、调什么味各自发挥。（本质：算法步骤固定，具体实现可变）
- **肉** —— 像**填空题试卷**：题目和答题框（模板）印好了，学生往里填答案。（本质：子类只负责填内容）

## 面试话术
- "模板方法模式就是**模骨架，子填肉**。父类把算法的大流程定好，像炒菜必须先热油再翻炒最后出锅，但下什么料、怎么调味由子类决定。Spring的JdbcTemplate就是典型，它把获取连接、关闭资源包好了，只让你填SQL怎么执行。"
- "它体现了好莱坞原则：'别调用我们，我们调用你'。父类控制整体流程，在合适时机回调子类的方法。"

## 防混补丁
- 容易和**策略模式**搞混。模板方法是继承复用骨架，策略是组合替换整个算法。补丁：**模是填空，策是换卷**。

## 追问预演
- **Q：模板方法和策略模式的区别？**
  - A：模板方法是白箱复用，通过继承共享骨架；策略模式是黑箱复用，通过组合替换整个算法。模板方法改局部，策略模式改整体。
- **Q：什么是钩子方法？**
  - A：父类提供一个默认实现或空实现的方法，子类可选择性重写，用来控制流程中某些步骤是否执行。
- **Q：好莱坞原则是什么？**
  - A：Don't call us, we'll call you。高层组件控制调用流程，低层组件被回调。
