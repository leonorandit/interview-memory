## 原始知识

- **标记-清除**（Mark-Sweep）：标记存活对象，清除未标记，**碎片多**
- **复制算法**（Copying）：新生代Eden→S0/S1，存活对象复制到另一块，**无碎片但浪费一半空间**
- **标记-整理**（Mark-Compact）：标记后把存活对象往一端移动，清理边界外内存，**无碎片但STW长**
- **分代收集**：结合以上算法按代选择（新生代复制、老年代标记-整理/清除）
- **增量收集**：降低STW，垃圾回收和用户线程交替执行

标完清掉碎成渣，

## 单字锚点 + 防崩推导

**锚点字：标**

- **扩展口诀**：标复分，三算法
- **为什么选它**："标"是垃圾回收的第一步——标记存活对象，所有算法都以"标记"为起点，是最核心的动作。
- **推导链**：
  1. 标 = 标记-清除（Mark-Sweep）= 先标记谁活着，再清除死的 = 简单但碎片多
  2. 复 = 复制（Copying）= 把活着的复制到新地方 = 新生代Eden→Survivor = 无碎片但浪费一半空间
  3. 分 = 分代收集 = 按代选算法 = 新生代用复制，老年代用标记-整理或标记-清除
  4. 三算法 = 标记-清除、复制、标记-整理（最基础的三个）
  5. 标记-整理 = 标 + 整理（Compact）= 标记后把存活对象挤到一边 = 无碎片但移动对象耗时、STW长
  6. 增量收集 = 垃圾回收和用户线程交替 = 降低单次STW = 系统吞吐量下降

## 一字一画面

**标** —— 像[贴小红花]：老师先给好学生贴上红花（标记存活），然后把没花的学生请出教室（清除垃圾）。走后桌椅七零八落（内存碎片）（本质：先标记后清除，产生碎片）。

**复** —— 像[搬家只带走行李]：两间房子，把要用的东西搬到隔壁空房，这间全部清空。虽然整齐，但始终有一间房空着（本质：复制存活对象，浪费一半空间，无碎片）。

**分** —— 像[新老员工分区管理]：新员工流动大（新生代），搬家制（复制）；老员工稳定（老年代），整理制（标记-整理），因地制宜（本质：分代收集，不同代用不同算法）。

**整** —— 像[地铁早高峰挤车厢]：所有人往车头方向紧紧靠拢，车尾空出一大片一次性清扫。虽然挤在一起很耗时（STW长），但车厢没有零散空位（本质：标记后压缩移动，无碎片，但移动对象耗时长）。

## 面试话术

"Java垃圾回收算法主要是'标复分'三种。标记-清除就是先标记存活对象再清除垃圾，但会留下碎片；复制算法把存活对象复制到另一块内存，适合新生代，无碎片但浪费一半空间；标记-整理是标记后把存活对象往一端移动，适合老年代，无碎片但STW比较长。实际JVM用的是分代收集，新生代复制、老年代标记-整理。"

## 追问预演

1. **标记-清除和标记-整理的区别？**
   清除只清理垃圾，不移动存活对象，有碎片；整理会把存活对象挤到一边，无碎片但STW更长。

2. **为什么新生代用复制算法？**
   新生代对象存活率低（大部分朝生夕死），复制成本低，且Eden:Survivor默认8:1:1，只浪费10%空间。

3. **增量收集的代价是什么？**
   降低单次STW，但总垃圾回收时间变长，系统吞吐量下降。

4. **CMS用的是哪种算法？**
   CMS老年代用标记-清除，所以有碎片；G1整体是标记-整理+复制（Region内复制）。
