## 速记卡片

堆 | 堆代方线直，五OOM | 堆/元空间/虚拟机栈/本地方法栈/直接内存

---

## 原始知识

- **堆溢出**：Java heap space，对象过多无法GC
- **元空间/方法区溢出**：Metaspace/PermGen，类加载过多
- **虚拟机栈溢出**：StackOverflowError，递归太深/线程过多
- **本地方法栈溢出**：Native方法调用栈溢出
- **直接内存溢出**：Direct buffer memory，NIO堆外内存分配过多

---

## 顺口溜

堆代方线直，五OOM，堆上对象挤爆窝，类多栈深直存错。

---

## 单字锚点 + 防崩推导

- **锚点字**：堆
- **扩展口诀**：堆代方线直，五OOM
- **为什么选它**：堆是 OOM 最常见、最典型的发生地，一提到 OOM 大家首先想到的就是堆溢出
- **推导链**：
  1. 堆 → Java heap space → 对象太多 GC 不掉
  2. 代 → 元空间/方法区（PermGen/Metaspace）→ 类加载器泄漏、动态类生成过多
  3. 方 → 虚拟机栈（VM Stack）→ 方形像栈帧一层层叠高 → StackOverflowError
  4. 线 → 本地方法栈（Native Stack）→ 线程调用本地方法 → 像一根线连到外部
  5. 直 → 直接内存（Direct Memory）→ NIO 堆外内存，不走 JVM 堆，直接分配
  6. 五OOM = 堆 + 代 + 方 + 线 + 直 = 5种

---

## 一字一画面

- **堆** —— 像一座堆满垃圾的山：褐色的垃圾山越堆越高，山顶冒着黑烟，压路机（GC）根本推不动，最终山体崩塌（本质：堆内存对象过多，GC无法回收，内存耗尽）
- **代** —— 像代代相传的族谱越印越厚：一台印刷机疯狂印刷家族族谱，纸张（类文件）塞满整个仓库（Metaspace），仓库门被撑爆（本质：类元信息加载过多，方法区/元空间溢出）
- **方** —— 像一摞方形饭盒无限叠高：食堂窗口前，方形饭盒一层一层往上叠，叠到天花板还在叠，最后整摞倒塌砸下来（本质：栈帧无限入栈，虚拟机栈深度溢出）
- **线** —— 像一根线穿过针眼后打结：绣花针拖着线穿过布匹，线在针眼处反复缠绕打结，越缠越紧最终断掉（本质：本地方法调用栈出现问题，线程执行Native方法时栈溢出）
- **直** —— 像一条直路绕过收费站直接堵车：汽车不走高速公路（JVM堆），直接走乡间小路（堆外内存），结果小路也挤满车动弹不得（本质：直接内存绕过JVM堆分配，但物理内存有限，分配过多导致OOM）

## 面试话术

1. "JVM产生OOM主要有五种情况，我记作'堆代方线直'：堆内存溢出、元空间溢出、虚拟机栈溢出、本地方法栈溢出，还有直接内存溢出。"
2. "最常见的是堆溢出，一般是内存泄漏或者大对象分配导致GC跟不上；栈溢出通常是递归没写终止条件或者线程创建太多。"
3. "直接内存溢出比较隐蔽，因为不归JVM堆管，排查的时候要注意NIO的ByteBuffer有没有及时释放。"

## 防混补丁

容易和 **StackOverflowError 不算 OOM** 搞混？

**补丁口诀**：方线两栈错，OOM是Error，栈深爆Stack，堆满报Heap。

（补充：严格来说 StackOverflowError 和 OOM 都属于 Error，但日常面试中五类都算作内存相关问题。）

## 追问预演

1. **问**：元空间和方法区有什么区别？
   **答**：JDK8之前叫永久代（PermGen），在JVM内存中；JDK8之后改为元空间（Metaspace），使用本地内存，默认只受物理内存限制。

2. **问**：直接内存为什么不受JVM堆大小限制？
   **答**：直接内存通过 Unsafe.allocateMemory 或 ByteBuffer.allocateDirect 分配，不走JVM堆，但受 -XX:MaxDirectMemorySize 参数限制。

3. **问**：如何排查堆内存溢出？
   **答**：开启 -XX:+HeapDumpOnOutOfMemoryError，用MAT或VisualVM分析dump文件，找大对象和GC Roots引用链。

4. **问**：线程过多会导致哪种OOM？
   **答**：线程过多可能导致虚拟机栈OOM（无法创建新的Native线程），因为每个线程都有独立栈空间，线程数受物理内存限制。
