# JVM 由哪些部分组成？

## 速记卡片

| 锚点字 | 口诀 | 核心差异 |
|--------|------|----------|
| 堆 | 堆方计，五组成 | 堆存对象，方存类，计指执行 |

## 原始知识

- **类加载子系统**：ClassLoader（加载/验证/准备/解析/初始化）
- **运行时数据区**：堆/方法区/虚拟机栈/本地方法栈/程序计数器
- **执行引擎**：解释器/JIT编译器/GC垃圾收集器
- **本地方法接口**：JNI（Java Native Interface）
- **本地方法库**：Native方法的具体实现（C/C++编写）

## 单字锚点 + 防崩推导

**锚点字**：堆

**扩展口诀**：堆方计，五组成

**为什么选它**：堆是运行时数据区中最大、最常被问的部分，也是OOM最常发生的地方，"堆"字最容易联想整个JVM结构。

**推导链**：
1. 堆 → 运行时数据区的一部分，存对象实例
2. 方 → 方法区（Method Area），存类信息、常量、静态变量（JDK8后叫元空间Metaspace）
3. 计 → 执行引擎（计算机要执行代码），包含解释器、JIT编译器、GC
4. 五组成 → ①类加载子系统 ②运行时数据区（5小块） ③执行引擎 ④本地方法接口（JNI） ⑤本地方法库
5. 运行时数据区细化 → 堆（对象）、方法区（类）、虚拟机栈（栈帧）、本地方法栈（Native方法）、程序计数器（行号指示器）
6. 执行引擎细化 → 解释器（逐行解释）、JIT（热点代码编译）、GC（垃圾回收）

## 一字一画面

**堆** —— 像**一座黄色垃圾山堆满纸箱**：每个纸箱是一个对象实例，山越大GC越累，山塌了就是OOM（本质：所有对象实例和数组都在堆上分配）

**方** —— 像**一块灰色方碑刻满咒语**：碑上是类的模板信息、常量池、静态变量，JDK8以前碑在永久代，现在在本地内存的元空间（本质：方法区存储类元数据，是堆的逻辑部分但独立管理）

**计** —— 像**一台银色发动机引擎轰鸣**：一边有慢慢转的涡轮（解释器），一边有高速喷射的涡轮（JIT），底下连着排气管排出废料（GC）（本质：执行引擎把字节码翻译成机器码并执行）

## 面试话术

"JVM主要分五大组成部分：类加载子系统负责把.class文件加载到内存；运行时数据区包括堆、方法区、虚拟机栈、本地方法栈和程序计数器；执行引擎负责解释执行或JIT编译字节码，还有垃圾收集器；另外就是本地方法接口和本地方法库，用来调用C/C++的Native方法。"

## 防混补丁

容易和JDK/JRE搞混？一句补丁："JDK是工具箱，JRE是运行时，JVM是运行时的心脏。"

## 追问预演

**Q：堆和栈的区别？**
A：堆存对象，线程共享；栈存栈帧（局部变量/操作数栈），线程私有。

**Q：程序计数器为什么是私有的？**
A：因为CPU在多线程间切换，每个线程需要记录自己执行到哪一行。

**Q：元空间和永久代的区别？**
A：永久代在堆内存，元空间在本地内存，避免了永久代OOM问题。

**Q：JIT编译器什么时候触发？**
A：方法调用计数器和回边计数器达到阈值（热点代码），触发即时编译。

**Q：哪些区域会OOM？**
A：堆、方法区（元空间）、虚拟机栈（StackOverflowError/OOM）、直接内存。
