## 原始知识

- **堆**：线程共享，新生代（Eden/Survivor0/Survivor1）+ 老年代
- **方法区/元空间**：类信息、常量池、静态变量（JDK8前PermGen，JDK8+元空间）
- **虚拟机栈**：线程私有、栈帧、局部变量、操作数栈
- **本地方法栈**：Native方法
- **程序计数器**：当前执行字节码行号，唯一不OOM区域
- **直接内存**：NIO堆外内存，不受JVM GC管理

## 单字锚点 + 防崩推导

**锚点字：堆**

- **扩展口诀**：堆方计栈，五区域
- **为什么选它**：堆是JVM中最大、最核心的内存区域，几乎所有对象都在这里分配，"堆"字本身就有"堆积如山"的画面感。
- **推导链**：
  1. 堆 = 堆内存 = 线程共享 = 对象实例存放地 = 新生代（Eden + S0 + S1）+ 老年代
  2. 方 = 方法区/元空间 = 类元数据、常量池、静态变量 = JDK8后叫Metaspace（本地内存）
  3. 计 = 程序计数器 = 记字节码行号 = 线程私有 = **唯一不会OOM**
  4. 栈 = 虚拟机栈 + 本地方法栈 = 线程私有 = 栈帧（局部变量、操作数栈、动态链接、返回地址）
  5. 五区域 = 堆、方法区、虚拟机栈、本地方法栈、程序计数器（JVM规范定义的5个运行时数据区）
  6. 直接内存 = 堆外内存（NIO的DirectByteBuffer），不属于JVM规范，但实际存在且重要

## 一字一画面

**堆** —— 像[一座巨型仓库]：所有快递包裹（对象）先堆在这里，新到的放新生代货架（Eden），经常取用的晋级到老年代大货架（本质：对象分配和GC分代的核心场所）。

**方** —— 像[图书馆分类卡片柜]：每个类都有一张卡片，记录类名、方法表、常量池，静态变量也贴在这里（本质：存储类元数据和静态信息）。

**计** —— 像[剧场座位排号器]：每个观众（线程）手里一个小屏幕，显示自己看到第几排第几座，剧场再小也不会溢出（本质：程序计数器记录字节码地址，极小而安全，不OOM）。

**栈** —— 像[一叠餐盘]：每个线程一叠盘子，每个方法调用压一个盘（栈帧），方法结束弹掉一个，盘里放局部变量和操作数（本质：后进先出，方法执行的内存模型）。

## 面试话术

"JVM内存区域可以记成'堆方计栈'。堆是线程共享的，放对象实例，分新生代和老年代；方法区放类信息、常量、静态变量；程序计数器记字节码行号，线程私有且不会OOM；虚拟机栈和本地方法栈也是线程私有，存栈帧和局部变量。"

## 防混补丁

容易和Java内存模型（JMM）搞混？
**补丁口诀**：内存区域是运行时地盘，JMM是并发规范，主内存工作内存别混谈。

## 追问预演

1. **堆和栈的区别？**
   堆线程共享存对象，栈线程私有存栈帧；堆GC管理，栈自动释放。

2. **方法区和元空间的区别？**
   JDK8前叫PermGen（堆内），JDK8后叫Metaspace（本地内存），不再受固定大小限制。

3. **什么情况下栈会溢出？**
   无限递归或方法调用过深会导致StackOverflowError；如果栈内存动态扩展失败会报OutOfMemoryError。

4. **直接内存有什么用？**
   NIO的DirectByteBuffer使用直接内存，避免Java堆和Native堆之间来回复制，提高IO效率，但不受JVM GC直接管理。
