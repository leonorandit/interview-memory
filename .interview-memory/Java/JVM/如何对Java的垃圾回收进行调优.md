# 如何对 Java 的垃圾回收进行调优

## 速记卡片

| 锚点字 | 口诀 | 核心差异 |
|--------|------|----------|
| 选 | 选参代，五调优 | 先选对 GC，再调参，再改代码 |

## 原始知识

- **选** GC 收集器（Serial / Parallel / CMS / G1 / ZGC / Shenandoah）
- **调** 堆内存大小（**-Xms** / **-Xmx** / -Xmn）
- **参** 数微调（**停顿时间**目标 / **吞吐量**目标）
- **代** 码优化（**避免大对象** / 对象池 / 减少临时对象）
- **监** 控日志（**-Xloggc** / GCViewer 分析）
- **防** 泄漏（**内存泄漏**排查 / MAT 分析堆转储）

## 顺口溜

选参代，五调优  
（选 GC，调参数，改代码，看日志，防泄漏）

## 单字锚点 + 防崩推导

> **锚点字：选**
>
> **扩展口诀**：选参代，五调优
>
> **为什么选它**：GC 调优第一步永远是"选"对收集器，选错了后面全白搭。
>
> **推导链**：
> - 选 → 选 GC 收集器（Serial/Parallel/CMS/**G1**/**ZGC**/Shenandoah）
> - 选完 → 调**参**数（-Xms/-Xmx 防扩容、-Xmn 控年轻代、停顿与吞吐量目标）
> - 调参后 → 改**代**码（避免大对象、对象池、减少临时对象）
> - 改完 → 看**监**控（GC 日志、GCViewer 可视化分析）
> - 最后 → **防**泄漏（MAT 分析堆转储，揪出内存泄漏）

## 一字一画面

**选** —— 像[挑兵器]：将军站在兵器架前，先**挑**一把趁手的刀（选 GC），再磨锋利（调参），再练招式（代码优化），再看刀光剑影的记录（监控日志），最后检查刀鞘有无裂缝（防泄漏）。（本质：GC 调优是有优先级的系统工程）

## 面试话术

"GC 调优我总结为五步走：第一步**选**对收集器，比如低延迟场景选 ZGC，平衡选 G1；第二步调**参**数，-Xms 和 -Xmx 设成一样避免运行时扩缩容；第三步从**代**码层面减少临时对象和避免大对象；第四步靠 GC 日志做**监**控分析；最后排查**防**止内存泄漏。口诀就是'选参代，五调优'。"

## 防混补丁

容易和"JVM 参数调优"搞混？一句补丁：  
**GC 调优是 JVM 调优的子集，前者只管垃圾回收，后者还管编译、线程、内存模型。**

## 追问预演

1. **问：G1 和 ZGC 怎么选？**  
   答：G1 目标停顿 200ms 以内，适合大堆（几十 G）；ZGC 目标停顿 10ms 以内，适合超大堆（TB 级）。

2. **问：-Xms 和 -Xmx 为什么要设一样？**  
   答：避免运行时堆内存动态扩缩容带来的 Full GC 停顿。

3. **问：大对象为什么会影响 GC？**  
   答：大对象直接进入老年代，容易触发 Full GC，且复制成本高。
