## 一字一画面

**高** —— 像一栋银色摩天大楼：电梯不是逐层停，而是智能调度把上千人同时送到不同楼层（本质：NIO事件驱动让单线程管理十万级连接，不阻塞）。

## 面试话术

Netty主要用在高并发网络通信场景。最常见的有RPC框架比如Dubbo和gRPC的底层通信、即时通讯IM的WebSocket长连接、游戏服务器这种需要支撑十万级并发的场景，还有大数据传输、API网关以及消息队列客户端。它底层基于NIO事件驱动模型，用Reactor线程模式来支撑高并发。

## 防混补丁

容易和Tomcat搞混？一句补丁：**Netty是通用NIO框架，Tomcat是Servlet容器**。

## 追问预演

| 追问 | 简要答案 |
|------|----------|
| Netty和Tomcat有什么区别？ | Tomcat是Servlet容器专门处理HTTP，Netty是通用NIO框架，不限协议。 |
| Netty为什么高性能？ | NIO非阻塞+Reactor线程模型+零拷贝+ByteBuf内存池。 |
| Netty的线程模型是怎样的？ | 主从Reactor：Boss线程accept连接，Worker线程处理IO读写。 |
