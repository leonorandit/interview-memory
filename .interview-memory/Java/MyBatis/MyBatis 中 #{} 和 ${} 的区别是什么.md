## 速记卡片

| 锚点字 | 口诀 | 核心差异 |
|--------|------|----------|
| 井 | 井预占，美直拼 | #{}预编译占位，${}直接拼接 |

## 原始知识

- **#{}**：预编译处理，SQL中用`?`占位符替换
- **${}**：直接字符串替换，**Statement拼接**
- **#{} 防SQL注入**，自动加单引号
- **${} 有注入风险**，不自动加引号
- #{} 底层用 PreparedStatement
- ${} 底层用 Statement
- ${} 适用**动态表名/列名/排序字段**
- #{} 适用传参值场景

## 顺口溜

> 井预占，美直拼，
> 防注入，用井号，
> 表名列，美元套。

## 单字锚点 + 防崩推导

- **锚点字**：井（#{}）
- **扩展口诀**：井预占，美直拼
- **为什么选它**：# 像井字，一口井把参数"吞进去"占位，先占坑后填值
- **推导链**：
  1. 看到 "井" → 想到 #{} → 预编译占位 → PreparedStatement
  2. 有 "预占" → 预编译 + 占位符 ? → 参数先占坑，运行时替换
  3. 对仗 "美直拼" → ${} 美刀符号 → 直接字符串拼接
  4. "预占" 安全（防注入），"直拼" 危险（可注入）
  5. 井号自动加引号，美元不加引号 → 所以表名/列名/排序只能用 ${}

## 一字一画面

**井** —— 像一口深井：参数像石头丢进井里，井口盖着"?"的盖子，石头落底才揭晓（本质：先占位，后填充，外界看不到真实石头）

**美** —— 像美元钞票赤裸裸摊开：参数直接贴上去，所有人一眼看穿（本质：字符串直接暴露拼接，无防护）

## 面试话术

- "#{} 用的是预编译，参数用问号占位，执行时才替换，所以能防 SQL 注入；${} 是直接字符串拼接，有注入风险。"
- "如果要做动态表名、列名或者排序字段，只能选 ${}，因为这些地方加单引号会语法报错。"

## 防混补丁

- **容易和什么搞混**：以为 ${} 也能防注入，或者以为 #{} 能用于表名。
- **补丁口诀**：表名列，美元套；传参数，井号罩。

## 追问预演

| 追问 | 简要答案 |
|------|----------|
| ${} 有什么合法使用场景？ | 动态表名、列名、排序字段（这些加引号会语法错误） |
| #{} 为什么能防 SQL注入？ | 参数作为值绑定到 ?，不是 SQL 文本的一部分 |
| 底层分别用什么对象执行？ | #{}→PreparedStatement，${}→Statement |
| 两者能混用吗？ | 可以，按场景选择，敏感参数一律 #{} |
