# 什么是 AOP？

## 速记卡片

| 锚点字 | 口诀 | 核心差异 |
|--------|------|----------|
| 切 | 切通注，三概念 | OOP纵向，AOP横向 |

## 原始知识

- **AOP**：面向切面编程（Aspect Oriented Programming）
- **Aspect**：切面，横切关注点的模块化
- **JoinPoint**：连接点，程序执行过程中的点
- **Pointcut**：切入点，匹配连接点的表达式
- **Advice**：通知，切面在特定点执行的动作
- **Weaving**：织入，将切面应用到目标对象
- **应用场景**：日志、事务、权限、监控
- **与OOP关系**：OOP纵向扩展，AOP横向扩展

## 单字锚点 + 防崩推导

**锚点字**：切

**扩展口诀**：切通注，三概念

**为什么选它**：AOP的核心是"切面"，"切"字直接对应 Aspect（切面），是AOP的灵魂动作。

**推导链**：
1. 切 → 切面（Aspect）→ 面向切面编程
2. 通 → 通知（Advice）→ 切面要干什么（Before/After/Around）
3. 注 → 织入（Weaving）→ 如何把切面放进业务代码
4. 三概念 → JoinPoint（能切的地方）、Pointcut（切哪里）、Advice（切完干啥）
5. 应用场景 → 日志、事务、权限、监控都是横切关注点
6. 与OOP区别 → OOP纵向继承，AOP横向拦截

## 一字一画面

**切** —— 像**一把橙色美工刀切蛋糕**：横着一刀划过三层奶油，奶油是业务层，刀痕是日志/事务/权限（本质：横切关注点与业务逻辑分离）

**织** —— 像**金色绣娘穿针引线**：在织好的布（业务代码）上绣花纹（切面），不拆布只添花（本质：运行时或编译时将增强逻辑编织进目标）

## 面试话术

"AOP就是面向切面编程，核心思想是把日志、事务这些横切关注点从业务代码里抽出来。打个比方，OOP是纵向的继承树，AOP是横向切一刀，在这一层统一处理通用逻辑。Spring AOP主要用动态代理实现，有JDK动态代理和CGLIB两种方式。"

## 防混补丁

容易和OOP搞混？一句补丁："OOP纵向造对象，AOP横向切一刀。"

## 追问预演

**Q：Spring AOP的实现原理是什么？**
A：基于动态代理，有接口用JDK动态代理，没接口用CGLIB子类代理。

**Q：Advice有哪些类型？**
A：Before、AfterReturning、AfterThrowing、After、Around五种。

**Q：AOP和拦截器有什么区别？**
A：AOP更通用，基于方法拦截；拦截器是Web层面的，基于请求拦截。
