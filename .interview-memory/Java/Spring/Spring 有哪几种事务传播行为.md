# Spring 有哪几种事务传播行为？

## 速记卡片
`传 | 传新嵌，七行为 | 传播七剑`

## 原始知识
- **REQUIRED**：**默认**，有则加入，无则新建
- **SUPPORTS**：有则用，**无则非事务**
- **MANDATORY**：**必须有**，否则抛异常
- **REQUIRES_NEW**：**挂起**当前，新建独立
- **NOT_SUPPORTED**：挂起当前，**非事务**执行
- **NEVER**：**必须无**，否则抛异常
- **NESTED**：**嵌套**事务，savepoint 回滚点

## 顺口溜
**传新嵌，七行为**

## 单字锚点 + 防崩推导
- **锚点字**：**传**
- **扩展口诀**：**传新嵌，七行为**
- **为什么选它**：题目核心词就是"传播行为"，"传"字直接扣题，像传球一样决定事务怎么传下去。
- **防崩推导**：
  - 传 → 传播行为 → 方法 A 调方法 B，B 怎么继承或拒绝 A 的事务
  - 七行为 → 七个级别，记三大类：
    - **跟着走**（传）：REQUIRED（默认，跟着）、SUPPORTS（跟着，没事务拉倒）、MANDATORY（必须跟着，不然报错）
    - **自己走**（新）：REQUIRES_NEW（挂起旧的，开新的）、NOT_SUPPORTED（挂起旧的，不跟事务）、NEVER（必须没事务，不然报错）
    - **套着走**（嵌）：NESTED（嵌套，里层崩只回滚到 savepoint）
  - 只记"传新嵌"三个字：传（跟着）、新（独立）、嵌（嵌套）
  - 每类再细分：跟着的有 REQUIRED / SUPPORTS / MANDATORY；独立的有 REQUIRES_NEW / NOT_SUPPORTED / NEVER；嵌套只有 NESTED

## 一字一画面
- **传** —— 像**绿色接力棒**：前面的人举着绿色接力棒跑，后面的人接过来继续跑；没人举就自己新发一根。（本质：REQUIRED，默认传播，融入现有事务或新建）
- **新** —— 像**红色新火把**：前面的人举着火把，后面的人把自己的火把灭了（挂起），另外点燃一根全新的红色火把，互不影响。（本质：REQUIRES_NEW，挂起当前事务，开启全新事务）
- **嵌** —— 像**俄罗斯套娃打结**：大套娃里面装小套娃，小套娃碎了只换小套娃，大套娃还能完整保留。（本质：NESTED，嵌套事务通过 savepoint 实现局部回滚）
- **随** —— 像**透明影子**：有光就现形（有事务就加入），没光就消失（没事务就以非事务运行）。（本质：SUPPORTS）
- **逼** —— 像**黑色警棍指着**：必须接棒！不接就一棍打过来（抛异常）。（本质：MANDATORY，强制要求存在事务）
- **拒** —— 像**蓝色拒收印章**：看到接力棒就盖"拒收"，把对方的棒没收（挂起），自己空手跑。（本质：NOT_SUPPORTED，非事务执行）
- **禁** —— 像**红色禁行牌**：此处严禁接力棒出现！看到就报警（抛异常）。（本质：NEVER，禁止存在事务）

## 面试话术
- "Spring 事务传播行为有七种，口诀是**传新嵌，七行为**。默认是 REQUIRED，有事务就加入，没有就新建。最常用的是 REQUIRED 和 REQUIRES_NEW，前者融入现有事务，后者挂起旧的开新事务。NESTED 是嵌套事务，底层用 savepoint 实现局部回滚。"
- "实际业务里，外层 service 用 REQUIRED，内层需要独立提交的用 REQUIRES_NEW，比如记录日志不能因为主业务回滚而丢失。"

## 防混补丁
- 容易把 NESTED 和 REQUIRES_NEW 搞混。NESTED 是**套娃里的套娃，外层管得了内层**；REQUIRES_NEW 是**两根独立火把，外层管不了内层**。补丁：**嵌套是套娃，新建是两根火把**。

## 追问预演
1. **REQUIRED 和 NESTED 区别？** → REQUIRED 是同一个事务，内层崩外层一起回滚；NESTED 内层崩可只回滚到 savepoint，外层还能提交。
2. **REQUIRES_NEW 在什么场景用？** → 记录审计日志、发送通知，必须与主业务解耦。
3. **Spring 事务默认传播行为是什么？** → REQUIRED。
4. **NOT_SUPPORTED 和 NEVER 区别？** → NOT_SUPPORTED 是挂起当前以非事务执行；NEVER 是严格要求当前无事务，有就抛异常。
5. **NESTED 底层怎么实现？** → JDBC savepoint，要求底层数据库支持。
