# 你了解的 Spring 都用到哪些设计模式？

## 速记卡片
`模 | 模工单代，策工观 | 六大设计模式`

## 原始知识
- **单**例：Bean **默认**单例作用域
- **工**厂：BeanFactory / ApplicationContext 造 Bean
- **代**理：AOP **动态**代理（JDK / CGLIB）
- **策**略：Resource / InstantiationStrategy 多实现切换
- **模**板方法：JdbcTemplate / RestTemplate 固定骨架
- **观**察者：ApplicationEvent **监听**广播

## 顺口溜
**模工单代，策工观**

## 单字锚点 + 防崩推导
- **锚点字**：**模**
- **扩展口诀**：**模工单代，策工观**
- **为什么选它**：题目问的就是"设计模式"，"模"字直接扣题，一想"模"就联想到 Spring 里无处不在的模式。
- **防崩推导**：
  - 模 → 设计模式 → Spring 是设计模式教科书
  - 单 → 单例：Bean 默认就是单例，最显眼
  - 工 → 工厂：BeanFactory 名字里带 Factory
  - 代 → 代理：AOP 就是动态代理
  - 策 → 策略：不同场景换不同实现
  - 模 → 模板方法：JdbcTemplate 名字里带 Template
  - 观 → 观察者：事件监听，发布-订阅
  - 口诀前四字"模工单代"是四个最核心、面试最常问的；后三字"策工观"是扩展加分项

## 一字一画面
- **单** —— 像**银色孤独奖杯**：仓库里只放一座银色奖杯，所有人来参观都指向同一个，不会克隆第二座。（本质：Spring 默认单例作用域，一个 Bean 只存在一个实例）
- **工** —— 像**橙色自动贩卖机**：投币（配置文件）进去，橙色机器自动吐出对应饮料（Bean），不用自己造。（本质：BeanFactory 负责对象的创建和管理）
- **代** —— 像**透明替身演员**：观众看到的是明星脸（代理对象），实际干活的是替身（目标对象），透明玻璃隔在中间做增强。（本质：AOP 动态代理，在目标方法前后加逻辑）
- **策** —— 像**多档旋钮开关**：一拧到 A 档用文件资源，一拧到 B 档用网络资源，骨架不变，实现切换。（本质：策略模式，Resource / InstantiationStrategy 多实现）
- **模** —— 像**蓝色乐高说明书**：搭积木的步骤骨架固定（连接数据库 → 执行 SQL → 处理结果 → 释放资源），但中间哪块积木由你填。（本质：模板方法模式，JdbcTemplate 固定流程，回调变细节）
- **观** —— 像**红色大喇叭+耳朵群**：红色喇叭一喊"订单创建了！"，一群耳朵（监听器）同时竖起接收，各自去发短信、扣库存、记日志。（本质：观察者模式，ApplicationEvent 广播机制）

## 面试话术
- "Spring 是设计模式的集大成者，我记了一个口诀**模工单代，策工观**。最核心的是单例模式，Bean 默认单例；工厂模式，BeanFactory 负责创建对象；代理模式，AOP 用 JDK 动态代理或 CGLIB 做增强。另外策略模式比如 Resource 的不同实现，模板方法比如 JdbcTemplate 固定骨架回调细节，观察者模式比如 ApplicationEvent 事件监听。"
- "回答的时候先抛口诀，再挑两三个展开，面试官会觉得你既有体系又有深度。"

## 防混补丁
- 容易把"代理模式"和"装饰器模式"搞混。代理是**控制访问**，装饰器是**叠加功能**。Spring AOP 是代理。补丁：**代理替你去谈，装饰器给你加装备**。

## 追问预演
1. **单例 Bean 是线程安全的吗？** → 不一定，Spring 只保证单例，不保证状态安全；无状态 Bean 天然安全，有状态需自行同步。
2. **BeanFactory 和 FactoryBean 区别？** → BeanFactory 是工厂容器，FactoryBean 是一个特殊的 Bean，用来生成其他 Bean。
3. **JDK 动态代理和 CGLIB 区别？** → JDK 基于接口，CGLIB 基于继承；目标类没接口时用 CGLIB。
4. **ApplicationEvent 和 MQ 有什么区别？** → ApplicationEvent 是进程内广播，MQ 是跨进程异步解耦。
