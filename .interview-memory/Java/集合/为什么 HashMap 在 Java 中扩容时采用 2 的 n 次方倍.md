## 速记卡片

| 锚点字 | 口诀 | 核心差异 |
|--------|------|----------|
| 按 | 按位与，三好处 | 位运算快、位置二选一、尾插防死循环 |

## 原始知识

- **位运算替代取模**：hash % n → **hash & (n-1)**，性能高
- **位置只二选一**：扩容后元素位置 = 原位置 或 原位置+旧容量
- **减少rehash**：hash & oldCap 判断，无需重新计算hash
- **尾插法配合**：JDK1.8尾插+2倍扩容，**避免死循环**

## 顺口溜

按位与，三好处，
取模变慢它变速，
原位移或加旧屋，
尾插不怕环成虎。

## 单字锚点 + 防崩推导

- **锚点字**：按
- **扩展口诀**：按位与，三好处
- **为什么选它**："按"直接点出核心运算——**按位与 &**，这是2的n次方能带来一切好处的根源
- **推导链**：
  1. 按 → 按位与（hash & (n-1)）→ 替代hash % n → CPU指令更快
  2. n是2的幂 → n-1低位全1 → 按位与等价取模 → 结果均匀分布
  3. 扩容2倍 → newCap = oldCap * 2 → hash & oldCap 结果只可能是0或oldCap
  4. 为0 → 留在原位；为oldCap → 移到原位置+oldCap
  5. JDK1.8尾插法 → 链表不会倒序 → 并发扩容不会成环

## 一字一画面

- **按** —— 像安检闸机：刷卡（hash）& 通道号（n-1），秒过不用算（本质：位运算替代除法）
- **二** —— 像双开门：左边原房间，右边新房间，只选其一（本质：hash & oldCap 判断0或1）
- **尾** —— 像排队尾进尾出：前面的人不动，新人排后面，不会乱序成环（本质：尾插法避免倒序死循环）

## 面试话术

"HashMap容量用2的n次方，核心就是一个'按'字——按位与。hash & (n-1)直接替代取模运算，CPU层面快得多。扩容的时候，元素要么留在原地，要么移到原位置加旧容量，判断条件就是hash & oldCap是0还是1。"

## 防混补丁

- **容易和HashTable搞混**：HashTable默认11，取模运算；HashMap改2的幂为位运算优化
- **补丁口诀**：HashTable模十一，HashMap按位与

## 追问预演

| 追问 | 简要答案 |
|------|----------|
| 为什么不用取模用位运算？ | 位运算单指令周期，取模需要除法，慢10倍以上 |
| hash & oldCap怎么判断位置？ | 结果0→原位置；结果oldCap→原位置+oldCap |
| JDK1.7为什么会死循环？ | 头插法导致链表倒序，并发扩容时指针环成环 |
| 尾插法怎么避免死循环？ | 保持链表顺序，扩容后相对位置不变，不会交叉 |
| 容量如果不是2的幂会怎样？ | hash & (n-1) 分布不均，某些桶爆满，某些为空 |
