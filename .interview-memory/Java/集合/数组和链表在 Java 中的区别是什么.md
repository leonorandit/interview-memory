## 原始知识

- **数组**：**连续内存**、随机访问O(1)、插入删除O(n)、**固定大小/扩容成本高**
- **链表**：**离散内存**、随机访问O(n)、插入删除O(1)、**动态扩容灵活**
- **ArrayList vs LinkedList**：前者底层数组，后者底层双向链表

## 单字锚点 + 防崩推导

- **锚点字**：连
- **扩展口诀**：连插删，数查快
- **为什么选它**："连"代表**链表**，链表的核心优势（插入删除O(1)）和核心劣势（访问O(n)）与数组形成最鲜明的对比，面试官最关注的就是这个差异
- **推导链**：
  1. 连 → 链表 → 节点离散存储，指针相连
  2. 插删 → 插入删除O(1) → 改指针即可，不用搬元素
  3. 数 → 数组 → 连续内存，下标直达
  4. 查快 → 随机访问O(1) → 地址公式：base + index * size
  5. 数组插删慢 → 中间插入要搬移后面所有元素 → O(n)
  6. 链表查慢 → 必须从头遍历找位置 → O(n)

## 一字一画面

- **连** —— 像串珠子：珠子（节点）散落桌上，用线（指针）串起来，中间加一颗只用剪线重穿（本质：指针重定向，无需搬移）
- **数** —— 像电影院座位：座位号连续排好，报3号直接走到第3个座位（本质：连续内存+偏移量计算）
- **插** —— 像插队买票：链表像有人递条子插队，改个指向就行；数组像人人紧挨，中间插一个要全往后挪（本质：是否涉及数据搬移）

## 面试话术

"数组和链表最核心的区别，我总结为'连插删，数查快'。链表插入删除是O(1)，因为只改指针；数组查询是O(1)，因为连续内存直接算地址。反过来，链表查询慢，数组插删慢。"

## 防混补丁

- **容易和ArrayList/LinkedList搞混**：ArrayList不完全是数组（支持自动扩容），LinkedList是双向链表
- **补丁口诀**：ArrayList底层是数组，LinkedList指针连双向

## 追问预演

| 追问 | 简要答案 |
|------|----------|
| ArrayList扩容机制？ | 默认容量10，扩容1.5倍，Arrays.copyOf搬移元素 |
| 为什么LinkedList插入快？ | 找到位置后，只改前驱后继指针，O(1) |
| 什么场景选ArrayList？ | 查询多、随机访问多、尾部增删多 |
| 什么场景选LinkedList？ | 频繁中间插入删除、实现队列/栈 |
| 内存占用谁更大？ | LinkedList更大，每个节点要存两个指针 |
