## 原始知识

- CAS是**原子操作**，无锁实现线程安全
- 三个参数：**内存位置V**、**期望值A**、**新值B**
- 比较V与A，**相等则更新为B**，否则不操作
- 底层由**CPU指令cmpxchg**保证原子性
- 存在**ABA问题**：A→B→A，用**版本号**解决
- **自旋开销**：忙等循环消耗CPU
- 只能保证**单个变量**原子性

## 顺口溜

比原新，三参数，
原子换值不慌张。
ABA有陷阱，版本号来防，
自旋烧CPU，单变量才刚。

## 单字锚点 + 防崩推导

**锚点字：比**

- **扩展口诀**：比原新，三参数
- **为什么选它**：CAS核心动作就是"比较"，Compare首义
- **推导链**：
  1. 比 → 比较并交换（Compare-And-Swap）
  2. 比什么 → 内存值V vs 期望值A
  3. 比完呢 → 相等换新值B，不相等不操作
  4. 谁保证 → CPU原子指令cmpxchg
  5. 有啥坑 → ABA问题，加版本号
  6. 还有啥 → 自旋开销大，只适单变量

## 一字一画面

- **比** —— 像[天平秤]：两盘各放V和A，指针咔哒对齐（相等）才放行，不齐就挡住（本质：条件判断是原子动作的核心）
- **原** —— 像[旧钥匙]：黄铜色，齿纹磨损，只有完全匹配锁孔才能拧开（本质：期望值是准入凭证）
- **新** —— 像[红漆印章]：啪一声盖下去，白纸瞬间变红（本质：更新操作一旦条件满足立刻生效）
- **三** —— 像[三脚凳]：V、A、B三条腿缺一不可，缺一条就坐不稳（本质：三参数缺一不可）

## 面试话术

"CAS就是Compare-And-Swap，它是一种无锁的原子操作。核心逻辑是用CPU的cmpxchg指令，比较内存里的当前值和期望值，如果一样就换成新值，不一样就失败重试。优点是线程安全还不阻塞，缺点是会有ABA问题和自旋开销。"

## 防混补丁

容易和`synchronized`搞混？一句补丁：
> CAS是无锁乐观比，synchronized是加锁悲观堵。

## 追问预演

| 追问 | 简要答案 |
|------|----------|
| CAS和synchronized怎么选？ | 竞争少用CAS，竞争大用锁 |
| ABA问题具体怎么解决？ | AtomicStampedReference加版本号 |
| CAS底层怎么保证原子性？ | CPU指令cmpxchg，多核加lock前缀 |
| 什么是自旋？ | 循环重试直到成功，消耗CPU |
| CAS能同时操作两个变量吗？ | 不能直接，需用AtomicReference封装对象 |
