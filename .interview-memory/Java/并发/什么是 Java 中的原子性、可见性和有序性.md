## 原始知识

- **原子性**：操作不可中断，**要么全做要么不做**
- 原子性保证：**synchronized / Lock / Atomic 类**
- **可见性**：一个线程修改，另一个线程**立即可见**
- 可见性保证：**volatile / synchronized / final**
- **有序性**：指令**执行顺序**符合预期
- 有序性保证：**Happens-Before 规则 / volatile 禁止重排序**

## 顺口溜

> 原可见，三特性，
> 原子完整不可断，
> 可见立即可传播，
> 有序不乱靠 HB。

## 单字锚点 + 防崩推导

- **锚点字**：原（原子性）
- **扩展口诀**：原可见，三特性
- **为什么选它**：原子性是最"小"的概念，不可分割，像锚点一样稳固
- **推导链**：
  1. 看到 "原" → 原子性（Atomic）→ 最小单位，不可再分
  2. "不可再分" → 要么全成功，要么全失败 → i++ 不是原子性（读-改-写三步）
  3. 对仗 "可见" → 可见性 → 修改对其他线程**立即可见**
  4. "三特性" → 第三个是有序性 → 指令执行**不乱序**
  5. 分别对应解决方案：原子性（锁/Atomic）、可见性（volatile/sync/final）、有序性（HB规则/volatile）

## 一字一画面

**原** —— 像一颗金色实心钢珠：从高处落下，要么完整弹起，要么完整静止，绝不会裂成两半（本质：操作不可分割，中间状态不可被观测）

**见** —— 像一面即时同步的镜子：左边人举左手，右边镜子里立刻也举左手，零延迟（本质：修改值刷回主内存，其他线程立刻读取新值）

**序** —— 像一列严格按号入座的绿皮火车：1号车厢必须先于2号，不能跳着来（本质：编译器和CPU不能乱重排指令，保证程序语义正确）

## 面试话术

- "原子性就是说一个操作不能被线程调度机制打断，比如 synchronized 包裹的代码块就是原子的。"
- "可见性解决的是一个线程改了值，另一个线程能不能马上读到；volatile 就是干这个的。"
- "有序性是指令不能乱重排，JMM 的 Happens-Before 规则给我们规定了哪些操作必须先发生。"

## 防混补丁

- **容易和什么搞混**：把 volatile 当成万能的，以为它能保证原子性。
- **补丁口诀**：volatile 管见序，不管原子数；要原子，上锁或 Atomic。

## 追问预演

| 追问 | 简要答案 |
|------|----------|
| i++ 是原子操作吗？ | 不是，是读-改-写三步，多线程下会丢数据 |
| volatile 能保证原子性吗？ | 不能，只能保证可见性和禁止重排序 |
| synchronized 同时保证哪三个特性？ | 原子性、可见性、有序性（独占锁顺带都解决了） |
| Happens-Before 有几条规则？ | 8条，如程序次序、锁规则、volatile规则、线程启动/终止等 |
