## 原始知识

- **JMM** 定义线程与**主内存**的抽象关系
- 每个线程有**私有工作内存**，共享数据在主内存
- **三大特性规范**：原子性、可见性、有序性
- **Happens-Before 规则**：8条规则保证有序性和可见性
- **volatile**：可见性 + 禁止重排序
- **synchronized**：原子性 + 可见性 + 有序性
- **final**：引用不可变 + 构造函数 HB 可见

## 顺口溜

> 模主工，三规范，
> 线程私有主存共，
> HB 八条定先后，
> volatile sync 来助攻。

## 单字锚点 + 防崩推导

- **锚点字**：模（内存模型）
- **扩展口诀**：模主工，三规范
- **为什么选它**：JMM 是 Java Memory Model 的缩写，"模"是核心关键字，一切并发问题的抽象层
- **推导链**：
  1. 看到 "模" → JMM（Java Memory Model）→ Java 内存模型
  2. "主" → 主内存（Main Memory）→ 所有线程共享的数据存储区
  3. "工" → 工作内存（Working Memory）→ 每个线程私有的缓存副本
  4. "三规范" → 对三大特性的规范：原子性、可见性、有序性
  5. 落地手段：Happens-Before 8条规则 + volatile / synchronized / final 的内存语义

## 一字一画面

**模** —— 像一本蓝色规则手册：所有线程都按这本手册读写数据，手册规定了什么时候必须把缓存写回主存（本质：抽象规范，屏蔽硬件差异，统一并发行为）

**主** —— 像中央银行的金库：所有钱（共享变量）存在这里，每个人（线程）只能去柜台（工作内存）取副本（本质：单一数据源，保证数据一致性基准）

**工** —— 像每个人手里的钱包：从银行取钱后先放钱包，花完再存回银行，但如果不及时存，别人看到的就是旧余额（本质：线程私有缓存，导致可见性问题）

## 面试话术

- "JMM 是 Java 内存模型，它规定了线程怎么和主内存交互：每个线程有自己的工作内存，修改完共享变量要刷回主内存，其他线程才能读到。"
- "JMM 主要规范了三大特性：原子性、可见性、有序性，通过 Happens-Before 的八条规则来保证，比如 volatile 变量规则、锁规则这些。"

## 防混补丁

- **容易和什么搞混**：把 JMM 和 JVM 内存结构（堆/栈/方法区）搞混。
- **补丁口诀**：JMM 是规范（抽象），JVM 内存结构是实现（具体）；一个管并发语义，一个管内存分配。

## 追问预演

| 追问 | 简要答案 |
|------|----------|
| JMM 和 JVM 内存结构有什么区别？ | JMM 是抽象规范（线程/主存交互），JVM 内存结构是具体实现（堆/栈/方法区） |
| Happens-Before 有哪 8 条规则？ | 程序次序、管程锁定、volatile、线程启动、线程终止、线程中断、对象终结、传递性 |
| 工作内存是什么？ | 线程私有的缓存区，对应 CPU 寄存器 + 高速缓存的抽象 |
| 为什么需要 JMM？ | 屏蔽不同硬件和操作系统的内存访问差异，提供统一的并发编程规范 |
