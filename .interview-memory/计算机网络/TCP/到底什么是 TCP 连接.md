## 速记卡片

| 锚点字 | 口诀 | 核心差异 |

## 原始知识

- **虚拟连接**：TCP连接不是物理线，是**逻辑状态**
- **四元组唯一标识**：源IP / 源端口 / 目的IP / 目的端口
- **连接状态机**：LISTEN / SYN_SENT / SYN_RECV / **ESTABLISHED**
- **三要素维护连接**：**序列号Seq** / 确认号Ack / **窗口大小**Window
- **Socket是端点**：应用层通过Socket API操作连接
- **连接本质**：通信**双方共同维护**的状态信息

## 顺口溜

四元组，定连接，态序窗，虚似真。

## 单字锚点 + 防崩推导

**锚点字：四**

- **扩展口诀**：四态序，三要素
- **为什么选它**：四元组是TCP连接的唯一标识，数字"四"最具体
- **推导链**：
  1. 四 -> 四元组
  2. 哪四个？-> 源IP、源端口、目的IP、目的端口
  3. 四元组定了就完事？-> 还要维护状态
  4. 什么状态？-> LISTEN/SYN_SENT/SYN_RECV/ESTABLISHED
  5. 连接靠什么可靠传输？-> 三要素：序列号、确认号、窗口大小
  6. 对应用层暴露什么？-> Socket作为端点
  7. 本质是什么？-> 双方共同维护的状态，不是物理连接

## 一字一画面

**四** —— 像快递单上的四个必填字段：寄件地址、寄件电话、收件地址、收件电话，四个信息对上了包裹才能准确送达（本质：四元组唯一标识一条TCP连接，缺一不可）。

## 面试话术

TCP连接其实不是一根物理电线，而是通信双方共同维护的一套状态信息。它通过四元组——源IP、源端口、目的IP、目的端口——来唯一标识。连接建立后靠序列号、确认号和窗口大小这三个要素来保证可靠传输，应用层通过Socket来操作这个连接。

## 防混补丁

容易和UDP搞混？一句补丁：**TCP有连接有状态有握手，UDP无连接不管丢包只管发**。

## 追问预演

| 追问 | 简要答案 |
|------|----------|
| TCP三次握手为什么不是两次？ | 两次无法确认客户端的接收能力，且可能产生历史重复连接的失效请求被误处理。 |
| 四次挥手为什么多一次？ | 因为TCP全双工，双方各需关闭自己的发送通道，ACK和FIN不能合并发送。 |
| 一台服务器最多能撑多少TCP连接？ | 理论受限于文件描述符和内存，四元组唯一，客户端IP和端口组合足够多时可百万级。 |
