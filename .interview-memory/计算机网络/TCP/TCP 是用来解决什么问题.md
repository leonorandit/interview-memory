# TCP 是用来解决什么问题？

## 速记卡片

| 锚点字 | 口诀 | 核心差异 |
|--------|------|----------|
| **靠** | 靠序流，三问题 | TCP 让传输可靠、有序、不拥塞 |

## 原始知识

- **IP 尽力而为**：网络层不保证到达、不保证顺序、不保证不丢包
- **三大问题**：**丢包**、**乱序**、**重复**
- **确认应答**：ACK 机制确认收到
- **超时重传**：没收到 ACK 就重发
- **滑动窗口**：**流量控制**，接收方告诉发送方窗口大小
- **拥塞控制**：**拥塞窗口**，网络堵了就减速
- **面向连接**：先 **三次握手** 建立连接再传数据

## 顺口溜

**靠序流，三问题**
> 可靠有序流量控，丢乱重复全搞定

## 单字锚点 + 防崩推导

- **锚点字**：**靠**
- **扩展口诀**：靠序流，三问题
- **为什么选它**：TCP 的全称是 Transmission Control Protocol，核心是 **Control**，而"靠"就是"可靠传输"的简称，也是大家日常夸 TCP 的口语——"这连接真靠谱"
- **推导链**：
  1. 靠 → 可靠传输 → 确认应答 + 超时重传
  2. 靠 → 有序 → 序列号 Seq + 确认号 Ack，乱序的包排序重组
  3. 靠 → 流量控制 → 滑动窗口（接收窗口 rwnd），接收方缓冲满了就喊停
  4. 靠 → 拥塞控制 → 拥塞窗口 cwnd，网络堵了就慢启动、拥塞避免
  5. 三问题 → 丢包、乱序、重复 → 都是 IP 不靠谱带来的

## 一字一画面

- **靠** —— 像**快递签收单**：每发一个包裹都要对方签字回传，没回签就再发一次，直到对方说"收到了"（本质：确认应答 + 超时重传）
- **序** —— 像**扑克牌编号**：发到手的牌是 3、7、2、5，TCP 按编号排成 2、3、5、7 再交给应用层（本质：序列号保证有序）
- **流** —— 像**水坝闸门**：下游水库满了就把闸门关小，空了再开大，防止洪水或干涸（本质：流量控制 + 拥塞控制双窗口）

## 面试话术

> "TCP 主要解决网络层 IP 协议的不可靠问题。IP 是尽力而为，不保证到达、不保证顺序，TCP 通过确认应答、超时重传、序列号排序、滑动窗口流量控制和拥塞控制，把不可靠的网络变成可靠的传输。总结就是'靠序流'三个字——可靠、有序、流量控制。"

## 防混补丁

- **容易和什么搞混**：把 TCP 和 UDP 的"效率"搞反，觉得 TCP 更快
- **补丁口诀**：TCP 靠但慢，UDP 快但飘

## 追问预演

| 追问 | 简要答案 |
|------|----------|
| TCP 和 UDP 的区别？ | TCP 面向连接、可靠、有序、流量控制；UDP 无连接、快、不保证可靠 |
| 滑动窗口和拥塞窗口的区别？ | 滑动窗口是接收方限制（rwnd），拥塞窗口是发送方根据网络状况自限（cwnd），取 min(cwnd, rwnd) |
| 三次握手能不能改成两次？ | 不能，两次无法确认双方的收发能力都正常，且可能产生历史连接误判 |
| 什么是 TIME_WAIT？ | 主动关闭方最后等 2MSL，确保最后的 ACK 对方收到，并清掉迟到的包 |
