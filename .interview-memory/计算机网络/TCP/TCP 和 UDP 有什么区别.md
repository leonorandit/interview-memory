# TCP 和 UDP 有什么区别？

## 速记卡片
`连 | 连可拥，快不管 | TCP连vsUDP断`

## 原始知识
- TCP：**面向连接**、可靠传输、字节流
- TCP：**拥塞控制**、首部20字节、三次握手四次挥手
- UDP：**无连接**、不可靠、数据报
- UDP：无拥塞控制、**首部8字节**、快
- TCP用：**HTTP/文件传输**
- UDP用：**直播/DNS/游戏**

## 顺口溜
**连可拥，快不管**

## 单字锚点 + 防崩推导
- **锚点字**：**连**
- **扩展口诀**：**连保稳，断求快**
- **为什么选它**："连"是TCP的灵魂特征，一切机制围绕连接建立和维护；UDP的核心是"不连"。
- **防崩推导**：
  - 连 → 要建连接 → 三次握手/四次挥手
  - 连 → 怕丢包 → ACK确认+重传 → 可靠
  - 连 → 怕网络堵 → 拥塞控制+流量控制 → 稳
  - 反推UDP：不连 → 不确认 → 快；不连 → 不控制拥塞 → 更快；不管 → 首部小到8字节 → 开销极小
  - 所以TCP稳但慢，UDP快但糙

## 一字一画面
- **连** —— 像**打电话**：先拨号建立连接（三次握手），对方接了才说话，说完确认"听懂了没"（ACK），最后挂电话（四次挥手）。（本质：面向连接）
- **可** —— 像**快递保价**：丢了重发，乱序重组，保证必达。（本质：可靠传输）
- **拥** —— 像**红色限速高速**：车多自动减速，绿灯了才加速，防止撞车堵死整条路。（本质：拥塞控制）
- **快** —— 像**绿色弹射起步的赛车**：不等人、不等确认，一脚油门直接冲。（本质：无连接、低延迟）
- **不** —— 像**黑色大喇叭广播**：一个人喊一群人听，不点名，不确认谁漏听。（本质：不可靠、多播高效）
- **管** —— 像**放飞的信鸽**：飞出去就不管了，飞到算运气，飞丢拉倒，所以轻量。（本质：无状态、无控制）

## 面试话术
- "TCP和UDP的核心区别，我用六个字概括：**连可拥，快不管**。TCP是面向连接的，像打电话要先拨号；有ACK确认和重传所以可靠；还有拥塞控制防止网络堵。UDP正好相反，无连接、不确认、不管拥塞，所以快。TCP用于HTTP、文件传输这种要稳的；UDP用于直播、DNS、游戏这种要快的场景。"
- "记住'连'字就够了：TCP一切特性都因为'连'，UDP一切特性都因为'不连'。"

## 防混补丁
- 容易和"TCP/IP模型"搞混。TCP是传输层协议，TCP/IP是四层网络体系结构。
- 补丁：**TCP是车道，TCP/IP是地图**

## 追问预演
1. **TCP三次握手为什么不是两次？**
   - 两次无法确认客户端的接收能力，也不能防止历史重复连接请求（已失效的连接请求报文突然到达服务端）导致资源浪费。三次是双方收发能力都确认的最小次数。

2. **UDP不可靠，那怎么实现可靠传输？**
   - 在应用层自己实现确认和重传机制，比如QUIC协议基于UDP实现了类似TCP的可靠性，同时保留了低延迟特性。

3. **什么时候选TCP什么时候选UDP？**
   - 要可靠、数据完整、顺序一致选TCP（HTTP、文件传输、邮件）；要实时性、低延迟、能接受少量丢包选UDP（直播、DNS、在线游戏、语音通话）。

4. **TCP和UDP可以共用端口吗？**
   - 可以。TCP和UDP的端口号空间是独立的，同一个数字端口号可以被TCP和UDP同时使用而不冲突。

5. **TCP粘包问题怎么解决？**
   - 固定长度、分隔符、Length+Content三种方式，本质上都是给无边界的字节流加消息边界。（可自然过渡到粘包知识点）
