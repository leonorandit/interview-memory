# TCP/IP 四层模型是什么？

## 速记卡片
`传 | 传网链应，四层上楼 | 四层模型`

## 原始知识
- **应用层**：HTTP/FTP/DNS/SMTP，面向用户
- **传输层**：TCP/UDP，端到端通信
- **网络层**：IP/ICMP/ARP，寻址和路由
- **链路层**：以太网/WiFi/PPP，物理传输
- **易混点**：与OSI七层模型的对应关系
- **追问点**：每层典型协议和设备

## 顺口溜
**传网链应，四层上楼**

## 单字锚点 + 防崩推导
- **锚点字**：**传**
- **扩展口诀**：**传网链应**
- **为什么选它**：TCP/IP模型的灵魂是"传输层"，四层模型从下到上依次是链路→网络→传输→应用，口诀按应用层视角倒序排列（传网链应），更顺口。
- **防崩推导**：
  - 传 → 传输层 → TCP/UDP → 端到端 → 端口
  - 网 → 网络层 → IP → 寻址路由 → 路由器
  - 链 → 链路层 → 以太网/MAC → 帧传输 → 交换机
  - 应 → 应用层 → HTTP/FTP/DNS → 面向用户

## 一字一画面
- **传** —— 像**快递员送包裹**：负责从A地送到B地，不管中间走哪条路。（端到端传输）
- **网** —— 像**导航地图**：决定包裹从哪条路走最快。（路由选择）
- **链** —— 像**高速公路**：真正承载车辆行驶的物理道路。（物理传输）
- **应** —— 像**手机APP**：用户直接使用的界面。（用户接口）

## 面试话术
- 问四层模型 → "TCP/IP四层模型口诀是**传网链应，四层上楼**。应用层有HTTP/FTP/DNS面向用户；传输层TCP/UDP负责端到端；网络层IP负责寻址路由；链路层以太网负责物理传输。"
- 问与OSI区别 → "OSI是七层，TCP/IP是四层。TCP/IP把OSI的会话层、表示层合并到应用层，把数据链路层和物理层合并到链路层。"

## 防混补丁
- 容易和OSI七层搞混。补丁：**OSI七层更细，TCP/IP四层更实用**。记住TCP/IP没有会话层和表示层。

## 追问预演
1. **每层典型设备是什么？**
   - 链路层：交换机、网卡；网络层：路由器；传输层：防火墙（四层防火墙）。
2. **ARP协议属于哪一层？**
   - 网络层，但依赖链路层广播实现。
3. **TCP/IP模型和OSI模型哪个更实用？**
   - TCP/IP更实用，是互联网实际使用的模型；OSI是理论参考模型。
