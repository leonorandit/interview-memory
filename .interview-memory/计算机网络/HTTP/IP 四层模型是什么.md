## 速记卡片

传 | 传网链应，四模型 | 传输层 / 网络层 / 链路层 / 应用层

## 原始知识

- **应用层**：HTTP、FTP、DNS、SMTP、SSH
- **传输层**：TCP、UDP、端口寻址
- **网络层**：IP、ICMP、ARP、路由选择
- **链路层**：以太网、WiFi、PPP、MAC地址
- **与OSI对应**：应用层≈OSI上三层，传输层≈传输层，网络层≈网络层，链路层≈OSI下两层
- **数据封装**：每层加头部，向下传递
- **核心思想**：端到端原则，底层尽力传输

## 顺口溜

传网链应，四层上楼

## 单字锚点 + 防崩推导

- **锚点字**：传
- **扩展口诀**：传网链应，四模型
- **为什么选它**：传输层是TCP/IP模型的核心枢纽
- **推导链**：
  1. 传 → 传输层（TCP/UDP，端到端通信，端口区分应用）
  2. 传往上 → 应用层（HTTP/FTP/DNS，直接为用户服务）
  3. 传往下 → 网络层（IP/ICMP/ARP，负责寻址和路由）
  4. 网往下 → 链路层（以太网/WiFi/PPP，负责相邻节点传输）
  5. 四层顺序（从下到上）：链 → 网 → 传 → 应
  6. 对应OSI：链≈物理+数据链路，网≈网络，传≈传输，应≈会话+表示+应用

## 一字一画面

传 —— 像快递中转站：黄色货车（链路层）把包裹运到区域分拣中心（网络层），红色专车（传输层）负责点对点直送，最后蓝色快递员（应用层）把盒子交到用户手上（本质：分层解耦，每层只关心自己的职责）

## 面试话术

"TCP/IP四层模型从下到上是链路层、网络层、传输层、应用层。链路层负责相邻节点传输比如以太网；网络层负责寻址和路由比如IP；传输层负责端到端通信比如TCP和UDP；应用层直接为用户提供服务比如HTTP和DNS。"

## 防混补丁

容易和OSI七层搞混？记住：TCP/IP四层是把OSI上三层合并成应用层，下两层合并成链路层，中间传输层和网络层对应不变。

## 追问预演

- **问**：四层和OSI七层有什么区别？
  **答**：TCP/IP更实用、先有协议后有模型；OSI更理论化、先有模型后有协议。TCP/IP应用层=OSI应用+表示+会话，链路层=OSI物理+数据链路。
- **问**：每层代表性协议？
  **答**：应用层HTTP/DNS/FTP，传输层TCP/UDP，网络层IP/ICMP/ARP，链路层以太网/WiFi。
- **问**：为什么要分层？
  **答**：各层职责独立，上层不用关心下层实现，便于标准化和替换。
