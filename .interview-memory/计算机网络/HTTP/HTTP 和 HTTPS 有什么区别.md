# HTTP 和 HTTPS 有什么区别？

## 速记卡片

| 锚点字 | 口诀 | 核心差异 |
|--------|------|----------|
| **证** | 证密端，四区别 | HTTPS 比 HTTP 多了证书、加密、端口变、性能差 |

## 原始知识

- **HTTPS = HTTP + SSL/TLS**，在 HTTP 之下加了安全层
- **安全性**：HTTP **明文传输**，HTTPS **加密传输**
- **端口**：HTTP 走 **80**，HTTPS 走 **443**
- **证书**：HTTPS 需 **CA 证书**，HTTP **不需要**
- **性能**：HTTPS **握手慢**，有 **加密开销**
- **SEO**：搜索引擎对 **HTTPS 更友好**

## 顺口溜

**证密端，四区别**
> 证书加密端口变，性能略差更安全

## 单字锚点 + 防崩推导

- **锚点字**：**证**
- **扩展口诀**：证密端，四区别
- **为什么选它**："证" 是 HTTPS 最显眼的标志，浏览器地址栏那个小锁就是"证"字现身
- **推导链**：
  1. 证 → CA 证书 → HTTPS 才有，HTTP 没有
  2. 证 → 加密 → SSL/TLS 握手 → 密文传输
  3. 证 → 端口 → 443（HTTPS）vs 80（HTTP）
  4. 证 → 性能 → 握手多两次 RTT，加密解密有开销
  5. 证 → SEO → Google 优先推带"证"的网站

## 一字一画面

- **证** —— 像**一把银色小锁挂在浏览器地址栏**：咔嚓一声锁住数据包，黑客伸手只能抓到乱码（本质：身份认证 + 加密通道）
- **密** —— 像**透明信封变黑色保险箱**：HTTP 是玻璃信封人人可读，HTTPS 是钢板保险箱只有收件人能开（本质：TLS 对称+非对称加密）
- **端** —— 像**两条车道分流**：80 号车道敞开跑，443 号车道安检严（本质：IANA 端口分配约定）

## 面试话术

> "HTTP 和 HTTPS 最核心的区别，我总结为'证密端'三个字——HTTPS 有 CA 证书、通信加密、端口是 443，而 HTTP 明文传、走 80 端口、不需要证书。另外 HTTPS 因为 TLS 握手会有一定性能开销，但现在硬件优化后影响已经很小了。"

## 防混补丁

- **容易和什么搞混**：把 HTTPS 当成"HTTP 的另一个版本"，其实不是，是 HTTP + SSL/TLS 的**叠加**
- **补丁口诀**：HTTPS 不是升级版，是 HTTP 穿铠甲

## 追问预演

| 追问 | 简要答案 |
|------|----------|
| TLS 握手过程是怎样的？ | ClientHello → ServerHello + 证书 → 密钥交换 → Finished，总共 2-RTT |
| 为什么需要 CA 证书？ | 防止中间人攻击，证书链验证服务器身份 |
| HTTPS 一定安全吗？ | 不一定，如果证书被伪造、或网站本身有漏洞，HTTPS 也救不了 |
| HTTP/2 必须基于 HTTPS 吗？ | 不是必须，但主流浏览器只支持 TLS 上的 HTTP/2 |
