## 速记卡片

| 锚点字 | 口诀 | 核心差异 |
|--------|------|----------|
| 三 | 三快连，五区别 | HTTP/3 基于 QUIC/UDP，0-RTT 快连 / 连接迁移 / 无队头阻塞 |

## 原始知识

- HTTP/2 **基于 TCP**
- HTTP/2 有 **二进制分帧 / 多路复用 / HPACK 头部压缩**
- HTTP/3 **基于 QUIC**（UDP 之上可靠传输）
- HTTP/3 **解决 TCP 队头阻塞**
- QUIC 在 **应用层实现流控制**
- **连接迁移**（QUIC 连接 ID 不变）
- **0-RTT 握手**（更快建立连接）
- HTTP/3 **加密默认 TLS1.3**

## 顺口溜

HTTP3，QUIC 跑，  
零 RTT 握手快，  
连接 ID 不变号，  
队头阻塞全扫掉。

## 单字锚点 + 防崩推导

- **锚点字**：三
- **扩展口诀**：三快连，五区别
- **为什么选它**：HTTP/3 是主角，所有区别都围绕"3"相对于"2"的改进
- **推导链**：三 → HTTP/3 → QUIC → 快（0-RTT） → 连（连接迁移） → 五区别（协议层 / 队头阻塞 / 连接迁移 / 0-RTT / 默认加密）

## 一字一画面

**三** —— 像[蓝色闪电换赛道]：[红色赛车困在拥堵的 TCP 赛道，蓝色闪电从 UDP 草地抄近道，车头挂着固定编号牌，瞬间到达终点]（本质：HTTP/3 走 QUIC 新赛道，连接 ID 不变）

## 面试话术

"HTTP/2 虽然有多路复用，但底层 TCP 一旦丢包就会阻塞所有流。HTTP/3 把传输层换成了 QUIC，基于 UDP，每个流独立控制，解决了队头阻塞。还支持连接迁移，WiFi 切 4G 连接不断，握手也是 0-RTT，更快。"

## 防混补丁

容易和 "HTTP/2 的多路复用" 搞混？一句补丁：**HTTP/2 多路复用但共一条 TCP，HTTP/3 是 QUIC 流各自独立。**

## 追问预演

**Q: QUIC 怎么做到连接迁移？**  
A: 用连接 ID 标识连接，不是 IP + 端口，网络切换不影响。

**Q: HTTP/3 为什么解决队头阻塞？**  
A: QUIC 在应用层实现可靠传输，每个流独立重传，互不影响。

**Q: QUIC 基于 UDP 怎么保证可靠？**  
A: QUIC 自己在应用层实现了重传、拥塞控制、流控，不依赖 TCP。
