# 从网络角度来看，用户从输入网址到网页显示，期间发生了什么？

## 速记卡片
`输 | 输网TCP，HT渲 | 连环五步`

## 原始知识
- **输→域→IP**：DNS解析，域名转IP
- **握同确**：TCP三次握手，SYN→SYN+ACK→ACK
- **请求→响应**：HTTP/HTTPS，发请求收响应
- **DOM→CSSOM→渲染树→布局→绘制**：浏览器渲染五小步
- **易混点**：HTTP是应用层，TCP是传输层
- **追问点**：HTTPS多了SSL/TLS握手；DNS缓存层级

## 顺口溜
**输网TCP，HT渲** —— *输网踢吸皮，爱吃踢渲*

## 单字锚点 + 防崩推导
- **锚点字**：**输**
- **扩展口诀**：**输网踢，爱吃渲**
- **为什么选它**：一切从"输入URL"这个动作开始，是整道题的时间原点。
- **防崩推导**：
  - 输 → 输入网址 → 先找IP → **DNS解析**（输→域→IP）
  - 网 → 找到IP后 → 连网络 → **TCP三次握手**（握同确）
  - 踢(T) → TCP连上 → 发请求 → **HTTP/HTTPS**（请求→响应）
  - 吸(C) → 如HTTPS → 先SSL/TLS握手（证书+密钥）
  - 爱吃(H) → HTTP响应回来 → 浏览器开始**渲染**
  - 渲 → DOM+CSSOM→渲染树→布局→绘制，页面显示

## 一字一画面
- **输** —— 像**在红色地址栏敲字**：手指敲下回车键，网址像箭一样射出去。（触发整个流程）
- **网** —— 像**查电话簿找号码**：域名是本名，IP是电话号码，DNS是114查号台。（DNS解析）
- **踢** —— 像**两人踢毽子传三次**：你踢过去（SYN），我踢回来加确认（SYN+ACK），你再确认（ACK），才开始玩。（TCP三次握手）
- **吃** —— 像**点餐上菜**：浏览器点菜（HTTP请求），厨房做菜，服务员端盘子上来（HTTP响应）。（请求响应）
- **渲** —— 像**画家拼拼图**：先拼骨架（DOM），再拼颜色纸（CSSOM），合起来看整体（渲染树），量尺寸（布局），最后涂色（绘制）。（浏览器渲染）

## 面试话术
- 问全过程 → "口诀是**输网TCP，HT渲**。输入URL后，先DNS解析域名到IP；然后TCP三次握手建立连接；接着发送HTTP请求，服务器返回响应；最后浏览器解析HTML、CSS、JS，构建DOM和渲染树，布局绘制后页面显示出来。如果是HTTPS，TCP之后还有SSL/TLS握手。"
- 问渲染细节 → "渲染环节可以记**DOM→CSSOM→渲染树→布局→绘制**。先解析HTML生成DOM树，解析CSS生成CSSOM，合并成渲染树，计算元素位置和大小后绘制到屏幕上。"

## 防混补丁
- 容易把"TCP三次握手"和"HTTPS握手"搞混。补丁：**先TCP连上路，再HTTPS发钥匙**。TCP是传输层搭通道，HTTPS是应用层发钥匙。

## 追问预演
1. **DNS解析具体过程？** → 先查浏览器缓存→本地hosts→本地DNS服务器→根域名→顶级域名→权威域名，递归+迭代查询。
2. **TCP为什么三次握手？两次不行吗？** → 两次无法确认双方收发能力都正常，三次才能全双工确认，防历史重复连接。
3. **SSL/TLS握手过程？** → Client Hello→Server Hello+证书→验证证书→密钥交换→生成会话密钥→Finished。
4. **渲染过程中遇到JS会阻塞吗？** → 会，JS会阻塞DOM解析，可用defer/async优化。CSS阻塞渲染树构建。
5. **TCP和HTTP分别在OSI哪层？** → TCP在传输层（四层），HTTP在应用层（七层）。
