## 速记卡片

| 锚点字 | 口诀 | 核心差异 |
|--------|------|----------|
| 二 | 二头多，五特性 | HTTP/2二进制分帧、头部压缩、多路复用 |

## 原始知识

- **HTTP/1.0短连接**：每次请求**新建TCP**，用完即关
- **HTTP/1.1持久连接**：复用TCP但**队头阻塞**，支持管线化仍串行
- **HTTP/2二进制分帧**：不再是文本报文，拆成**二进制帧**
- **多路复用**：**一个TCP连接**上并行传输多个请求流
- **头部压缩HPACK**：消除重复头部，**霍夫曼编码**
- **服务器推送**：服务端主动推资源，减少客户端请求
- **流优先级**：不同请求流可设置优先级权重

## 顺口溜

二进分帧多路跑，头压推送优先级高。

## 单字锚点 + 防崩推导

**锚点字：二**

- **扩展口诀**：二头多，五特性
- **为什么选它**：HTTP/2版本号就是"二"，所有特性都围绕它展开
- **推导链**：
  1. 二 -> HTTP/2
  2. 相比1.x最核心的变化？-> 二进制分帧（不再是文本）
  3. 二进制分帧带来了什么？-> 多路复用（一个TCP并行多个请求）
  4. 还有什么优化？-> 头部压缩（HPACK）
  5. 服务端能主动吗？-> 服务器推送
  6. 多个流怎么调度？-> 流优先级
  7. 1.0有什么痛点？-> 短连接每次新建TCP
  8. 1.1改善了什么？-> 持久连接，但队头阻塞还在

## 一字一画面

**二** —— 像双层高速巴士：一层车厢只能坐一个乘客走到底（HTTP/1.x串行），二层车厢把乘客拆成碎片混装进多个座位同时出发，到站再拼起来（本质：HTTP/2二进制分帧后，多个请求流在同一个TCP连接上并行传输，互不阻塞）。

## 面试话术

HTTP/2相比1.x最大的变化是二进制分帧和多路复用。以前HTTP/1.x一个TCP连接上请求只能串行，即使1.1有了持久连接也还是队头阻塞。HTTP/2把请求响应拆成二进制帧，带上流标识符，在一个连接上并行跑。另外还有头部压缩HPACK、服务器推送和流优先级这几个特性。

## 防混补丁

容易和HTTP/1.1搞混？一句补丁：**1.1持久连接但仍串行排队，HTTP/2才是真正并行多路复用**。

## 追问预演

| 追问 | 简要答案 |
|------|----------|
| HTTP/2还有什么问题？ | 虽然应用层多路复用了，但底层TCP仍有队头阻塞，一个丢包全停。 |
| HTTP/3怎么解决的？ | HTTP/3基于QUIC协议，QUIC跑在UDP上，单个流丢包不影响其他流。 |
| HPACK怎么压缩头部？ | 静态表+动态表+霍夫曼编码，用索引代替重复发送完整头部字段。 |
