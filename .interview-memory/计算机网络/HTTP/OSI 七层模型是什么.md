## 速记卡片

物 | 物数网传会表应 | 物理层 / 数据链路层 / 网络层 / 传输层 / 会话层 / 表示层 / 应用层

## 原始知识

- **物理层**：比特流传输，网线、光纤、集线器
- **数据链路层**：帧传输，MAC地址，交换机
- **网络层**：IP、路由，路由器
- **传输层**：TCP/UDP，端口，端到端
- **会话层**：会话管理，建立/维护/终止会话
- **表示层**：加密、压缩、编码转换
- **应用层**：HTTP、FTP、DNS，直接面向用户
- **记忆口诀**：物数网传会表应（ All People Seem To Need Data Processing）

## 顺口溜

物数网传会表应

## 单字锚点 + 防崩推导

- **锚点字**：物
- **扩展口诀**：物数网传，会表应
- **为什么选它**：物理层是最底层，一切通信的基础
- **推导链**：
  1. 物 → 物理层（比特流传输，看得见摸得着的线缆和信号）
  2. 物往上 → 数 → 数据链路层（帧、MAC地址、交换机）
  3. 数往上 → 网 → 网络层（IP、路由、路由器）
  4. 网往上 → 传 → 传输层（TCP/UDP、端口、端到端）
  5. 传往上 → 会 → 会话层（会话管理、建立维护终止）
  6. 会往上 → 表 → 表示层（加密、压缩、编码转换）
  7. 表往上 → 应 → 应用层（HTTP/FTP/DNS，直接给用户用）
  8. 顺序（从下到上）：物 → 数 → 网 → 传 → 会 → 表 → 应

## 一字一画面

物 —— 像海底电缆：漆黑深海中橙色光缆发出微弱脉冲光，一串串0和1化作电火花在铜线中奔跑（本质：所有高层协议的最终载体都是物理信号）

## 面试话术

"OSI七层模型从下到上是物理层、数据链路层、网络层、传输层、会话层、表示层、应用层。口诀是物数网传会表应。物理层传比特流，链路层传帧，网络层负责路由，传输层端到端，会话层管理会话，表示层处理加密压缩，应用层面向用户。"

## 防混补丁

容易把会话层和传输层搞混？记住：传输管端到端连接，会话管应用间的对话状态。

## 追问预演

- **问**：交换机工作在哪一层？路由器呢？
  **答**：交换机工作在数据链路层，路由器工作在网络层。
- **问**：表示层具体做什么？
  **答**：加密解密、压缩解压、编码格式转换比如ASCII和Unicode转换。
- **问**：OSI模型和TCP/IP模型哪个更实用？
  **答**：TCP/IP更实用，是实际互联网的基础；OSI更理论化，但分层思想影响深远。
