# 什么是物理地址，什么是逻辑地址？

## 速记卡片
`虚 | 虚页物，三区别 | 程序看vs硬件看`

## 原始知识
- **逻辑地址**：虚拟地址，程序看到的地址
- **物理地址**：内存条上的真实地址
- **MMU**：内存管理单元，负责转换
- **页表映射**：逻辑页→物理页帧
- **逻辑连续**：地址空间连续
- **物理可不连续**：离散分配
- **易混点**：逻辑地址≠物理地址
- **追问点**：页表太大怎么办？多级页表？

## 顺口溜
**虚页物，三区别** —— *虚页物，三区别*

## 单字锚点 + 防崩推导
- **锚点字**：**虚**
- **扩展口诀**：**虚页物，MMU换**
- **为什么选它**：逻辑地址又称虚拟地址，"虚"是程序视角的本质特征。
- **防崩推导**：
  - 虚 → **逻辑地址=虚拟地址** → 程序看到的地址 → **虚假的、连续的**
  - 页 → 以**页**为单位映射 → 逻辑页号+页内偏移
  - 物 → **物理地址** → 内存硬件上的**真实**地址 → 由MMU转换得到
  - 三区别 →
    - **谁产生**：逻辑地址由CPU生成；物理地址由MMU转换后得到
    - **是否连续**：逻辑地址空间连续；物理地址可不连续（离散分配）
    - **可见性**：程序只能看到逻辑地址；操作系统和MMU才能看到物理地址

## 一字一画面
- **虚** —— 像**电影院的座位号**：票上写着"3排5号"，这是观众看到的统一编号，但实际对应大厅里某个真实椅子。（逻辑地址，程序视角）
- **页** —— 像**翻书找页码**：先看目录找到第几章（页号），再翻到这页找第几行（页内偏移）。（页表映射）
- **物** —— 像**椅子底部的钢印编号**：工厂生产时打的唯一真实编号，对应实际物理位置。（物理地址）
- **MMU** —— 像**检票员**：你拿票（逻辑地址），检票员查系统（页表），告诉你真实椅子在哪（物理地址）。

## 面试话术
- 问区别 → "口诀是**虚页物，三区别**。逻辑地址也叫虚拟地址，是程序看到的地址，连续且统一；物理地址是内存硬件上的真实地址。MMU负责通过页表把逻辑地址转换成物理地址。三区别：逻辑地址由CPU生成，物理地址由MMU转换；逻辑地址空间连续，物理地址可以不连续；程序只认逻辑地址，只有内核和MMU能访问物理地址。"
- 问作用 → "虚拟地址让每个程序以为自己独占内存，方便隔离和重定位；物理地址实现实际的离散存储，提高内存利用率。"

## 防混补丁
- 容易觉得逻辑地址和物理地址数值可能一样。补丁：**程序永远摸不到物理地址**，就像你永远看不到电影院椅子底下的钢印，只能看票上的座位号。

## 追问预演
1. **页表太大怎么办？** → 多级页表、倒排页表、TLB缓存。多级页表只分配用到的页目录，省空间。
2. **TLB是什么？** → 转换检测缓冲区，缓存最近用过的页表项，加速地址转换。TLB miss才去查页表。
3. **缺页中断怎么处理的？** → MMU发现页表项无效，触发缺页中断，OS找空闲页帧，从磁盘加载页面，更新页表，恢复执行。
4. **逻辑地址怎么组成的？** → 逻辑页号（高位）+ 页内偏移（低位）。页号查页表得物理页帧号，拼接偏移得物理地址。
5. **为什么需要虚拟内存？** → 隔离进程地址空间、实现内存超额分配（swap）、简化编程模型、方便共享内存。
