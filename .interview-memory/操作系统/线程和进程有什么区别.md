# 线程和进程有什么区别？

## 速记卡片
`轻 | 轻共独，三区别 | 调度vs资源单位`

## 原始知识
- **进程**：资源分配的**基本单位**
- **独立地址空间**，有PCB，资源隔离
- **线程**：CPU调度的**基本单位**
- **共享进程资源**，独立栈和寄存器
- **切换开销**：进程大（换页表），线程小
- **通信方式**：进程IPC，线程直接读写共享内存
- **崩溃影响**：进程隔离不波及，线程崩可能拖垮进程
- **易混点**：谁有独立地址空间
- **追问点**：线程栈多大？线程切换具体换什么？

## 顺口溜
**轻共独，三区别** —— *轻共读，三区别*

## 单字锚点 + 防崩推导
- **锚点字**：**轻**
- **扩展口诀**：**轻共独，三不同**
- **为什么选它**：线程叫"轻量级进程"，一个"轻"字带出和进程的本质差异。
- **防崩推导**：
  - 轻 → **轻**量级 → 线程比进程**轻** → 为什么轻？
  - 轻 → 不自带大量资源 → **共享**进程资源
  - 共 → 共享代码段、数据段、堆、打开的文件
  - 独 → 但线程也有**独**立的：栈、寄存器、程序计数器
  - 三区别 → 轻带出三个核心差异：
    - **切换开销**：进程大（页表/缓存失效），线程小
    - **通信方式**：进程需IPC（管道/消息队列/共享内存），线程直接读写共享内存
    - **崩溃影响**：进程隔离安全，线程崩可能拖垮整个进程

## 一字一画面
- **轻** —— 像**羽毛飘在手掌上**：几乎感觉不到重量，随手就能抛接。（线程轻量级）
- **共** —— 像**合租公寓**：客厅、厨房、卫生间共用，但每人一间卧室。（共享进程资源）
- **独** —— 像**各自床头柜**：每人抽屉里放私人物品，互不干涉。（独立栈和寄存器）
- **换** —— 像**搬家vs换房间**：搬家要整户搬迁（进程切换），换房间只带枕头被子（线程切换）。（切换开销差异）
- **崩** —— 像**公寓着火**：一户着火可能烧整栋楼（线程），别墅着火只烧自家（进程隔离）。（崩溃影响）

## 面试话术
- 问区别 → "口诀是**轻共独，三区别**。进程是资源分配的基本单位，有独立地址空间；线程是CPU调度的基本单位，共享进程资源但有自己的栈和寄存器。三个核心区别：切换开销进程大线程小；通信方式进程用IPC线程直接共享内存；崩溃影响进程隔离线程可能拖垮进程。"
- 问场景 → "需要强隔离用进程，需要高并发切换快用线程。现代服务器一般是多进程+多线程混合。"

## 防混补丁
- 容易觉得线程完全独立。补丁：**线程是进程的打工仔**，没有自己的地址空间，离了进程活不了。

## 追问预演
1. **线程切换具体保存什么？** → 保存/恢复程序计数器、寄存器、栈指针，不换页表。
2. **进程切换为什么开销大？** → 切换页表（TLB失效刷新缓存），切换内核栈，保存恢复更多上下文。
3. **协程和线程区别？** → 协程用户态调度，更轻量，由程序控制切换，不由内核调度。
4. **多线程共享什么？独享什么？** → 共享：代码段、数据段、堆、打开文件描述符；独享：栈、寄存器、程序计数器、线程局部存储TLS。
5. **线程栈一般多大？** → Linux默认8MB（可ulimit调整），Java虚拟机栈默认1MB左右。
