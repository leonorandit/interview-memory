# 说说你知道的几种 I/O 模型

## 速记卡片
`阻 | 阻非多信，五模型 | 同步到异步`

## 原始知识
- **阻塞IO（BIO）**：等数据+拷贝都阻塞
- **非阻塞IO（NIO）**：轮询检查，不阻塞等
- **IO多路复用**：select/poll/epoll，一线程监控多fd
- **信号驱动IO**：SIGIO通知，数据准备好来信号
- **异步IO（AIO）**：全程不阻塞，完成通知
- **同步vs异步**：数据拷贝是否阻塞
- **易混点**：同步=阻塞？不是！
- **追问点**：多路复用为什么高效？epoll原理？

## 顺口溜
**阻非多信，五模型** —— *阻非多信，五魔醒*

## 单字锚点 + 防崩推导
- **锚点字**：**阻**
- **扩展口诀**：**阻非多信异**
- **为什么选它**：阻塞IO是最原始模型，"阻"是理解其他模型的起点。
- **防崩推导**：
  - 阻 → **阻塞IO** → 等数据和拷贝都**阻塞** → 最简单但效率低
  - 非 → **非阻塞IO** → 不等了，**轮询**问内核好了没 → 不阻塞但费CPU
  - 多 → **IO多路复用** → 一个线程**多**路监控 → select/poll/epoll
  - 信 → **信号驱动IO** → 内核发**信号**通知我 → SIGIO
  - 异 → **异步IO** → 全程**异**步不阻塞 → 我发请求后完全不管，内核做完了通知我
  - 同步异步分界线 → 看**数据拷贝阶段**是否阻塞。同步：拷贝时自己等；异步：拷贝完内核通知。

## 一字一画面
- **阻** —— 像**在餐厅傻等**：站着不动等厨师做完，拿到餐才走。（阻塞IO，全程阻塞）
- **非** —— 像**每隔5分钟催一次菜**：不站着傻等，但反复跑厨房问好了没。（非阻塞IO，轮询）
- **多** —— 像**服务员看十桌**：一个服务员同时盯着十桌客人的菜，哪桌好了端哪桌。（IO多路复用）
- **信** —— 像**叫号器震动**：你不用盯着，轮到你了机器叫你去取餐。（信号驱动IO）
- **异** —— 像**外卖送到家门口**：你下单后就睡觉，外卖员做好+送到，敲门叫醒你。（异步IO，全程不阻塞）

## 面试话术
- 问模型 → "五种IO模型，口诀是**阻非多信，五模型**。阻塞IO全程阻塞；非阻塞IO轮询检查；IO多路复用让一个线程监控多个连接；信号驱动IO由内核发信号通知；异步IO全程不阻塞，内核做完才通知。前四种都是同步IO，只有AIO是异步IO，因为同步异步的分界线在数据拷贝阶段是否阻塞。"
- 问应用场景 → "高并发服务端用IO多路复用（epoll），比如Nginx、Redis；异步IO用于高吞吐文件读写，如Windows IOCP。"

## 防混补丁
- 容易把"同步"等同于"阻塞"。补丁：**同步阻塞是两回事**。同步看"谁完成拷贝"，阻塞看"等的时候干不干别的**。多路复用是同步非阻塞，不是异步！

## 追问预演
1. **select、poll、epoll区别？** → select有1024限制，poll链表无限制，epoll红黑树+就绪链表事件驱动。epoll适合高并发。
2. **为什么epoll高效？** → 不需要遍历所有fd，只处理就绪的；没有重复拷贝fd集；支持ET和LT模式。
3. **同步IO和异步IO根本区别？** → 数据拷贝阶段是否由内核完成并通知。同步：用户进程自己拷贝；异步：内核拷贝完通知用户。
4. **Linux有真正的AIO吗？** → Linux原生AIO（io_submit）主要支持文件IO，网络IO真正的异步比较少见，Windows IOCP更成熟。
5. **Redis为什么用epoll？** → 单线程处理大量连接，epoll让单线程高效监控所有socket的就绪状态，避免阻塞和轮询。
