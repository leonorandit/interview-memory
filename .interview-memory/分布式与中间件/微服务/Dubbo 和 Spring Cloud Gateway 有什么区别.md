# Dubbo 和 Spring Cloud Gateway 有什么区别？

## 速记卡片
`网 | 网关调，两定位 | RPC vs 网关`

## 原始知识
- **Dubbo**：RPC框架，服务间远程调用
- **基于TCP/Netty**，高性能二进制通信
- **Gateway**：API网关，统一入口
- **基于HTTP/WebFlux**，路由限流鉴权
- **定位不同**：Dubbo管**服务间通信**，Gateway管**统一接入**
- **可配合**：Gateway接外部，Dubbo调内部
- **易混点**：两者不是替代关系
- **追问点**：Dubbo也用网关吗？用，但功能不同

## 顺口溜
**网关调，两定位** —— *网关调，两定镜*

## 单字锚点 + 防崩推导
- **锚点字**：**网**
- **扩展口诀**：**网关调，定不同**
- **为什么选它**：Gateway名字带"网"，Dubbo是"调"，两字拼出核心差异。
- **防崩推导**：
  - 网 → API**网关** → 统一**入口** → 对外暴露 → 负责**路由/限流/鉴权**
  - 关 → **关**口守门员 → 所有请求先过它
  - 调 → **Dubbo**是**调**用框架 → 服务间**远程调用** → 管内部通信
  - 两定位 → 一个守大门（Gateway），一个跑内网（Dubbo）
  - 不冲突 → 外部请求先过Gateway，再经Dubbo调内部服务

## 一字一画面
- **网** —— 像**小区大门保安亭**：所有外来人车必须过闸机，登记、限流、验身份。（API网关统一入口）
- **关** —— 像**闸门起落杆**：超流量就放下栏杆阻断，没卡就不让进。（限流熔断鉴权）
- **调** —— 像**小区内部对讲机**：住户之间互相呼叫，不走大门，直接内网通话。（RPC服务间调用）
- **定** —— 像**两个岗位牌**：一个写"门卫"，一个写"内勤"，职责不同但都在物业系统里。（定位不同）

## 面试话术
- 问区别 → "核心区别是**定位不同**。Dubbo是RPC框架，解决服务之间的远程调用，基于TCP/Netty，性能高；Spring Cloud Gateway是API网关，是统一入口，负责路由转发、限流熔断、鉴权认证，基于HTTP/WebFlux。它们不是竞争关系，而是**配合使用**：外部请求先到Gateway，再由Gateway通过Dubbo调用内部服务。"
- 问使用场景 → "微服务集群对外暴露一个Gateway地址，内部服务间用Dubbo互相调用。Gateway管南北流量，Dubbo管东西流量。"

## 防混补丁
- 容易觉得Dubbo也能做Gateway的事。补丁：**Dubbo是对讲机，Gateway是大门**。对讲机不能当大门用，大门也不能代替对讲机。

## 追问预演
1. **Dubbo和Feign有什么区别？** → Dubbo是RPC框架（TCP/Netty），Feign是HTTP客户端（HTTP/REST），Dubbo性能更高。
2. **Gateway和Nginx有什么区别？** → Nginx是反向代理/L7负载均衡（C语言高性能），Gateway是微服务网关（Java生态，集成Spring Cloud，支持动态路由、熔断）。
3. **Dubbo的服务治理有哪些？** → 注册发现、负载均衡、容错熔断、服务分组/version、动态配置。
4. **Gateway怎么做限流？** → 基于Redis令牌桶/漏桶算法，或集成Sentinel，按IP/接口/用户维度限流。
5. **Dubbo支持哪些协议？** → 默认Dubbo协议（TCP），还支持Triple（兼容gRPC/HTTP2）、REST、Hessian等。
