# 说说什么是 API 网关？它有什么作用？

## 速记卡片
`关 | 关路限，五作用 | 统一入口六大功能`

## 原始知识
- **API网关**：微服务统一入口
- **路由转发**：请求发到对应服务
- **负载均衡**：分摊到多个实例
- **限流熔断**：防流量冲垮服务
- **鉴权认证**：验身份验权限
- **日志监控**：统一记录和观测
- **协议转换**：HTTP转RPC等
- **常见实现**：Gateway/Kong/Nginx/Zuul
- **易混点**：网关不是负载均衡器
- **追问点**：网关单点故障怎么解？

## 顺口溜
**关路限，五作用** —— *关路现，五作拥*

## 单字锚点 + 防崩推导
- **锚点字**：**关**
- **扩展口诀**：**关路限，日记协**
- **为什么选它**：API网关的核心意象就是"关口"，所有流量必经之关。
- **防崩推导**：
  - 关 → **网关**是统一**关**口 → 所有请求先过这里
  - 路 → **关**口要指**路** → **路由转发**
  - 限 → **关**口要**限**流 → **限流熔断**，防冲垮
  - 五作用 → 指完路、限完流，还要干五件事
    - 日 → **日志监控**
    - 记 → 记录调用的**日记**
    - 协 → **协议转换**
    - 再加鉴权认证、负载均衡
  - 常见实现 → Gateway（Spring）、Kong（OpenResty）、Nginx、Zuul（已过时）

## 一字一画面
- **关** —— 像**古城门楼**：所有商队行人必须穿过城门才能进城，无法绕路。（统一入口）
- **路** —— 像**城门内的指路牌**：东边去集市，西边去衙门，各走各路。（路由转发）
- **限** —— 像**城门吊桥**：敌人太多就拉起吊桥，保护城内安全。（限流熔断）
- **鉴** —— 像**守城士兵查腰牌**：没腰牌或权限不够，一律拦下。（鉴权认证）
- **协** —— 像**翻译官**：外邦说外语，翻译官转成城内官话。（协议转换）

## 面试话术
- 问是什么 → "API网关是微服务架构的**统一入口**，所有外部请求先到网关。"
- 问作用 → "作用记口诀**关路限，五作用**。关是统一入口；路是路由转发；限是限流熔断。另外还有鉴权认证、负载均衡、日志监控、协议转换。常见实现有Spring Cloud Gateway、Kong、Nginx。"
- 问必要性 → "没有网关，每个微服务都要自己处理认证、限流，重复且难维护。网关把这些横切关注点统一收拢。"

## 防混补丁
- 容易把网关当成单纯负载均衡器。补丁：**负载均衡是网关的一个功能，不是全部**。Nginx是负载均衡，Gateway是网关，网关=负载均衡+路由+鉴权+限流+监控。

## 追问预演
1. **网关单点故障怎么解决？** → 集群部署+ keepalived/VIP，或结合Service Mesh去中心化。
2. **Gateway和Zuul的区别？** → Zuul 1.x阻塞IO，Zuul 2.x和Gateway基于Netty非阻塞，Gateway是Spring官方推荐。
3. **限流算法有哪些？** → 令牌桶（允许突发）、漏桶（匀速）、滑动窗口计数。
4. **熔断降级怎么实现？** → 集成Sentinel或Hystrix，失败率超阈值打开熔断，走降级逻辑。
5. **网关性能瓶颈在哪？** → 自身转发损耗、Filter链处理、鉴权查询，可配合Redis缓存令牌。
