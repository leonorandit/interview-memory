# 什么是 Seata？

## 速记卡片

| 锚点字 | 口诀 | 核心差异 |
|--------|------|----------|
| 分 | 分 AT，四模式 | 分布式事务+AT 自动补偿+四模式 |

## 原始知识

- **Seata** 阿里开源
- **分布式事务** 解决方案
- **AT 模式** 自动补偿
- **SQL 解析** 生成反向 SQL
- **TCC 模式** Try-Confirm-Cancel
- **Saga 模式** 长事务状态机
- **XA 模式** 强一致性
- **TC** 事务协调器
- **TM** 事务管理器
- **RM** 资源管理器
- **一阶段** 执行业务 SQL
- **记录 undo_log**
- **二阶段成功** 删 undo_log
- **二阶段失败** 回滚

## 顺口溜

Seata 分事务，AT 最省事；
TCC 手实现，Saga 管长事，XA 强一致。

## 单字锚点 + 防崩推导

- **锚点字**：分
- **扩展口诀**：分 AT，四模式
- **为什么选它**：Seata 解决的核心问题就是“分”布式事务。
- **推导链**：
  1. 分 → 分布式事务：跨多个服务/数据库的事务一致性问题
  2. A → AT 模式：自动补偿，最常用，基于 SQL 解析生成反向 SQL
  3. T → TCC 模式：手动实现 Try-Confirm-Cancel 三接口
  4. 四模式 → AT、TCC、Saga（长事务状态机）、XA（强一致性两阶段提交）
  5. 核心组件：TC 协调全局事务，TM 定义事务边界，RM 管理分支事务
  6. AT 原理：一阶段执行业务 SQL 并记录 undo_log；二阶段成功删 undo_log，失败用 undo_log 回滚

## 一字一画面

- **分** —— 像[分叉路口的交通警察]：一个路口分成四个车道（四个模式），交警 TC 站在中间指挥，司机 TM 报路线，车辆 RM 按指令行驶（本质：分布式事务需要协调多个参与者）
- **A** —— 像[自动倒车]：车开错了路，AT 自动驾驶系统自动按原路倒回出发点，方向盘自动反向打（本质：AT 自动生成反向 SQL 自动回滚）
- **T** —— 像[三段式跳远]：Try 助跑踩板，Confirm 起跳进坑，Cancel 踩空回退，三步都要自己练（本质：TCC 需要业务手动实现三个接口）

## 面试话术

- “Seata 是阿里开源的分布式事务框架，提供了 AT、TCC、Saga、XA 四种模式，最常用的是 AT 模式，它对业务侵入小，通过解析 SQL 自动生成反向补偿 SQL。”
- “Seata 的核心组件有 TC 事务协调器、TM 事务管理器和 RM 资源管理器，AT 模式下第一阶段执行业务 SQL 并记录 undo_log，第二阶段根据结果决定删除还是回滚。”

## 防混补丁

容易和单体事务搞混？**Seata 解决的是跨服务跨库的事务**，AT 的 undo_log 是框架自动生成的，不是数据库的 undo。

## 追问预演

| 追问 | 简要答案 |
|------|----------|
| AT 模式和 XA 有什么区别？ | XA 是强一致性两阶段锁，AT 是最终一致性，一阶段就提交本地事务，二阶段用 undo_log 补偿。 |
| TCC 和 AT 怎么选？ | AT 侵入小适合简单业务，TCC 适合复杂业务但需要自己实现三个接口。 |
| Saga 适用什么场景？ | 长事务业务，比如旅游预订机酒车，按状态机逐步推进。 |
