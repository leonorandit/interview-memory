## 原始知识

- **聚簇索引**：数据与索引**在一起**，主键即聚簇
- **非聚簇索引**：也叫**二级索引**，索引和数据分离
- **普通索引**：加速查询，无唯一约束
- **唯一索引**：列值**必须唯一**
- **联合索引**：多列**组合**索引
- **前缀索引**：对字段**前缀**建索引
- **全文索引**：用于**文本搜索**
- **B+树索引**：InnoDB **默认**索引类型
- **Hash 索引**：仅 **Memory 引擎**支持
- 主键索引是特殊的**聚簇索引**
- 一个表**只能有一个**聚簇索引

## 单字锚点 + 防崩推导

**锚点字**：类

**扩展口诀**：类聚联全，四索引

**为什么选它**："类"就是分类，索引有很多种，记住四大核心类别就能展开推导。

**推导链**：
1. 锚点字"类" → 索引分类
2. 分类 → 类聚联全 → 聚簇、联合、全文、（非聚簇/二级）
3. 聚簇 → 主键索引，数据和索引在一起，一个表只有一个
4. 非聚簇 → 二级索引，包含普通索引、唯一索引
5. 普通索引 → 只加速查询，不限制重复
6. 唯一索引 → 加速查询 + 列值唯一
7. 联合索引 → 多列组合，最左前缀匹配
8. 前缀索引 → 长字段取前缀建索引，省空间
9. 全文索引 → 文本搜索，match against
10. 数据结构 → B+树（默认）、Hash（Memory 引擎）

## 一字一画面

**聚** —— 像**超市货架和价签粘在一起**：价签（索引）直接贴在商品（数据）上，拿到价签就等于拿到商品（本质：聚簇索引叶子节点存储完整数据行）

**联** —— 像**三人组合的密码锁**：三个人（三列）的指纹按顺序按下才能开门，顺序错了就开不了（本质：联合索引按多列顺序组合排序，遵循最左前缀）

**全** —— 像**图书馆的全文检索机**：输入关键词就能在整本书里找到相关内容（本质：全文索引对文本内容分词建立索引，支持模糊搜索）

## 面试话术

"MySQL 的索引类型，从数据结构上分有 B+树索引和 Hash 索引，B+树是默认的。从功能上分，有聚簇索引、非聚簇索引也就是二级索引，二级索引里又分普通索引和唯一索引。另外还有联合索引、前缀索引和全文索引。聚簇索引一个表只能有一个，通常就是主键索引。"

## 防混补丁

容易把**普通索引和唯一索引的作用**搞混？补丁口诀：**普通只加速，唯一加约束，都可有多个，主键仅一个**。

## 追问预演

- **Q**：为什么 InnoDB 用 B+树不用 B 树？
  **A**：B+树叶子节点有序且通过指针相连，范围查询更高效；非叶子节点不存数据，能容纳更多键值，树更矮，IO 更少。
- **Q**：Hash 索引为什么不适合范围查询？
  **A**：Hash 索引是根据 hash 值定位，数据无序存储，无法支持范围扫描和排序。
- **Q**：前缀索引怎么设计长度？
  **A**：要在索引选择性和索引大小之间权衡，通常看不同前缀长度的选择性，接近完整列选择性即可。
