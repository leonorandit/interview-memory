## 速记卡片

| 锚点字 | 口诀 | 核心差异 |
|--------|------|----------|
| 读 | 读重复，幻脏三 | 三种读异常：脏读、不可重复读、幻读 |

## 原始知识

- **脏读**：读到**未提交**数据
- **不可重复读**：同一事务**两次读**结果不同
- 不可重复读由 **update** 导致
- **幻读**：同一事务两次查询**行数不同**
- 幻读由 **insert/delete** 导致
- **读已提交**解决脏读
- **可重复读**解决不可重复读
- **串行化**解决幻读（InnoDB 可重复读+间隙锁也能解决）

## 顺口溜

脏读未提交，重复被更新，
幻读多几行，隔离来搞定。

## 单字锚点 + 防崩推导

**锚点字**：读

**扩展口诀**：读重复，幻脏三

**为什么选它**：三种异常都是"读"的问题，脏读是读了不该读的，不可重复读是读不一致的，幻读是读出幻觉的。

**推导链**：
1. 锚点字"读" → 三种读异常
2. 三种读 → 脏读、不可重复读、幻读
3. 脏读 → 脏，就是脏数据，未提交的 → 读到别人未提交的数据
4. 不可重复读 → 重复读，结果不一样 → 同一事务两次读不同 → update 修改了数据
5. 幻读 → 幻觉，像变魔术多出来或少了 → 同一事务两次查询行数不同 → insert/delete 导致
6. 解决 → 隔离级别层层加码：读已提交 → 可重复读 → 串行化

## 一字一画面

**脏** —— 像**从垃圾桶里捡馒头**：别人还没决定扔不扔，你就捡起来吃了（本质：读到其他事务未提交的脏数据，可能回滚）

**重** —— 像**照镜子两次换了发型**：第一次看是长发，第二次看变短了，别人中间剪了（本质：同一事务内两次查询，数据被其他事务 update 修改）

**幻** —— 像**变魔术数观众**：第一次数10个人，眨眼变成12个，有人偷偷溜进来（本质：同一事务两次统计，数据行被 insert/delete 改变）

## 面试话术

"脏读就是读到了其他事务还没提交的数据；不可重复读是同一个事务里两次读同一个数据，结果不一样，主要是 update 导致的；幻读也是同一个事务两次查询，但是行数变了，是 insert 或者 delete 导致的。这三种异常可以通过提升事务隔离级别来解决。"

## 防混补丁

容易把**不可重复读和幻读**搞混？补丁口诀：**update改的是重复读，insert/delete出的是幻读，行数变了就是幻**。

## 追问预演

- **Q**：InnoDB 在可重复读级别下能完全避免幻读吗？
  **A**：通过 MVCC + 间隙锁（Gap Lock）能很大程度上避免，但快照读和当前读表现不同，严格意义上串行化才能完全避免。
- **Q**：幻读和不可重复读的本质区别是什么？
  **A**：不可重复读是某一行数据内容变了（update），幻读是数据行数变了（insert/delete）。
- **Q**：数据库默认隔离级别是什么？
  **A**：MySQL InnoDB 默认是可重复读（RR）；Oracle、SQL Server 默认是读已提交（RC）。
