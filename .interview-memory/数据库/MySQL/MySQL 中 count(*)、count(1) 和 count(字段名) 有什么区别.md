## 速记卡片

| 锚点字 | 口诀 | 核心差异 |
|--------|------|----------|
| 全 | 全一行，空不算 | count(*)全要，count(1)一样，count(字段)空不要 |

## 原始知识

- **count(*)**：统计**所有行**，包含NULL
- **count(1)**：与count(*)基本一致，MySQL优化后相同
- **count(字段)**：忽略**NULL值**，只统计非空
- **MyISAM**：缓存count(*)，O(1)速度
- **InnoDB**：无缓存，均需遍历，都慢
- **性能排序**：count(*) ≈ count(1) > count(主键) > count(字段)
- **语义差异**：count(*)求总行数，count(字段)求该字段非空数
- **优化器**：count(*)和count(1)都被优化为相同执行计划
- **NULL处理**：count(*)不判NULL，count(字段)判NULL
- **聚合函数**：均属于聚合函数，配合group by使用

## 顺口溜

全要行数用星号，
一加效果一样好。
字段统计空不要，
MyISAM缓存星号快，
InnoDB遍历全都慢。

## 单字锚点 + 防崩推导

**锚点字：全**

- **扩展口诀**：全一行，空不算；MyISAM缓存快，InnoDB都慢。
- **为什么选它**：count(*)是最常用的统计行数方式，"全"字直接表达"全部行都统计"的核心语义，且与count(字段)的"空不算"形成鲜明对比。
- **推导链**：
  1. 全 → count(*) → 统计所有行 → **包含NULL**
  2. 一 → count(1) → 放常量1 → 优化器视为与count(*)相同
  3. 行 → 总行数 → 不管哪列有没有值
  4. 空不算 → count(字段) → 逐行判断该字段 → NULL跳过
  5. 存储引擎差异：
     - MyISAM：表级元信息缓存count(*) → 极快
     - InnoDB：MVCC无缓存 → count(*)也要遍历 → 都慢
  6. 性能：count(*) ≈ count(1) > count(主键) > count(非索引字段)

## 一字一画面

**全** —— 像[红盖章的全班点名册]：[老师拿着全班点名册，不管谁请假谁旷课，名字全打勾，一个不落]（本质：统计总行数，NULL也算一行）

## 面试话术

1. "count(*)和count(1)在MySQL里基本没区别，优化器会把它们处理成一样的执行计划；count(字段)就不一样了，它会跳过NULL值。"
2. "MyISAM会把count(*)缓存起来，所以特别快；但InnoDB因为支持MVCC，不同事务看到的行数可能不同，所以没法缓存，count(*)也要遍历。"
3. "如果就是想统计总行数，无脑用count(*)；如果要统计某个字段有值的有多少条，就用count(字段)。"

## 追问预演

| 追问 | 简要答案 |
|------|----------|
| count(*)为什么比count(字段)快？ | count(*)不判NULL，可能走覆盖索引；count(字段)要回表判NULL |
| MyISAM的count(*)为什么快？ | 表级锁，元数据里缓存了精确行数，直接返回 |
| InnoDB怎么优化count(*)？ | 用计数表、Redis缓存、或者如果只需要近似值用show table status |
| count(主键)和count(*)谁快？ | 通常count(*)更快，优化器可能选择更小的索引 |
| count(1)里的1会被读取吗？ | 不会，优化器会忽略这个常量，和count(*)等价 |
