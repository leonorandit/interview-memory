# Redis 性能瓶颈时如何处理

## 速记卡片

| 锚点字 | 口诀 | 核心差异 |
|--------|------|----------|
| 慢 | 慢命分，五优化 | 慢日志 / 大Key / 命令优化 / 分片集群 / 内存持久化 |

## 原始知识

- **慢查询分析**：**slowlog** 定位慢命令，先找病因
- **大 Key 清理 / 拆分**：String > 10KB、集合元素过多要拆分或清理
- **命令优化**：避免 **O(n) 命令**（KEYS / FLUSHALL / HGETALL 大哈希），用 SCAN 替代 KEYS
- **分片 / 集群扩展**：单机扛不住就水平拆分，Redis Cluster 分槽位
- **内存优化**：压缩编码（ziplist、intset）、**淘汰策略**（LRU / LFU / TTL）
- **持久化优化**：AOF 刷盘策略（everysec / no / always），**RDB _fork 阻塞**
- **主从读写分离**：读多写少场景，从节点分担读压力
- **客户端连接数优化**：连接数过多消耗内存，用连接池、TCP 保活

## 单字锚点 + 防崩推导

- **锚点字**：慢
- **扩展口诀**：慢命分，五优化
- **为什么选它**：性能瓶颈的表现就是"慢"，先发现问题再解决，逻辑链清晰
- **推导链**：
  1. 慢 → 性能瓶颈，先定位哪里慢
  2. 命 → 慢查询日志（slowlog），看命令层面
  3. 分 → 分析结果：要么命令慢，要么数据大，要么并发高
  4. 五优化 → 五大处理方向
  5. 查 → 命令优化（避免 KEYS、大聚合命令）
  6. 大 → 大 Key 拆分 / 清理
  7. 片 → 分片 / 集群扩展
  8. 存 → 内存优化 + 持久化优化
  9. 主 → 主从读写分离
  10. 客 → 客户端连接数优化

## 一字一画面

- **慢** —— 像[红色交通堵塞]：车都堵在路口，先放无人机看哪里堵（slowlog），再决定是拓宽道路（集群）、限流（命令优化）还是拆大货车（大 Key）。（本质：性能问题先诊断再治理）
- **命** —— 像[医生看病历]：slowlog 就是 Redis 的病历本，哪条命令耗时多久记得清清楚楚，先翻它找病因。（本质：慢查询日志是性能诊断第一入口）
- **大** —— 像[一辆超载卡车]：一个 Key 塞了百万元素，Redis 是单线程处理，清理这个大 Key 会阻塞整个服务，要拆成多辆小车分批运。（本质：大 Key 是单线程模型的头号敌人）
- **片** —— 像[超市开分店]：一家店人太多，按商品种类分几家店（分片），顾客去不同店消费，总吞吐量上去了。（本质：水平扩展分散负载）
- **主** —— 像[主编配助手]：老板只负责写稿（写操作），校对复印交给助手（从节点读），效率翻倍。（本质：主从读写分离分担读压力）

## 面试话术

"Redis 慢了我们一般会按这个流程排查：先用 slowlog 看有没有慢命令，再分析有没有大 Key，接着检查是不是用了 KEYS 这种 O(n) 命令。解决方向有几个：命令层面用 SCAN 替代 KEYS、大 Key 做拆分；架构层面做集群分片或者主从读写分离；内存层面调整淘汰策略和持久化配置。"

## 防混补丁

容易和 MySQL 性能优化搞混？记住：Redis 是**单线程模型**，瓶颈往往不在 CPU 而在**大 Key 阻塞**和**网络 IO**，所以大 Key 和命令复杂度是 Redis 独有的排查重点。

## 追问预演

- **Q：slowlog 怎么配置？**
  - A：`slowlog-log-slower-than` 设阈值（微秒），`slowlog-max-len` 设记录条数，默认 128 条。
- **Q：大 Key 怎么发现？**
  - A：`redis-cli --bigkeys` 扫描，或 `MEMORY USAGE key` 查看具体大小。
- **Q：KEYS 和 SCAN 区别？**
  - A：KEYS 会遍历全库并阻塞，O(n)；SCAN 是增量迭代，每次返回一批游标，不阻塞。
- **Q：AOF 三种刷盘策略怎么选？**
  - A：always 最安全但最慢，no 最快但丢数据风险大，**everysec 是平衡默认选项**。
- **Q：Redis Cluster 怎么分片？**
  - A：16384 个槽位，每个节点负责一部分槽，key 做 CRC16 后对 16384 取模决定落哪个槽。
- **Q：主从延迟怎么处理？**
  - A：监控 `info replication` 的 offset 差值，敏感读可以走主节点或强制同步。
