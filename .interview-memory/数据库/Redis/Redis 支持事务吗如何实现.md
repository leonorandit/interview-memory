## 速记卡片

**批** | 批弱乐，三命令 | Redis 事务是批量命令队列，弱原子性无回滚；三命令 = MULTI / EXEC / DISCARD，WATCH 乐观锁

## 原始知识

- **支持事务**，但不保证原子性，**无回滚机制**
- **命令**：MULTI 开启事务、EXEC 执行、DISCARD 取消、WATCH 乐观锁
- **机制**：命令先入队再执行
- **隔离性**：事务执行过程中其他命令不会插入
- **区别**：与关系型数据库事务不同，**无 ACID 保证**

## 顺口溜

批弱乐，三命令，入队执行不回滚

## 单字锚点 + 防崩推导

**锚点字**：批

**扩展口诀**：批弱乐，三命令

**为什么选它**：Redis 事务本质就是"批量"命令打包执行

**推导链**：
1. 批 → 批量执行，把多个命令打包
2. 弱 → 弱原子性，没有回滚，中间命令错了后面的继续执行
3. 乐 → 乐观锁，用 WATCH 监控 key，被改则 EXEC 失败
4. 三命令 → MULTI（开启）、EXEC（执行）、DISCARD（取消）
5. 怎么执行？先入队，再一次性执行

## 一字一画面

**批** —— 像[食堂排队打饭]：[大家先排好队（入队），窗口一开一起打（执行），但打错了不能退（无回滚）]（本质：命令先入队列再批量执行）

## 面试话术

Redis 支持事务，但和 MySQL 那种事务不一样。它是用 MULTI 开启事务，然后命令先入队，最后 EXEC 一起执行。Redis 事务没有回滚机制，中间某条命令失败了后面的还会继续执行，所以原子性是弱的。还可以用 WATCH 做乐观锁。

## 防混补丁

容易和 MySQL 事务搞混？MySQL 有 ACID 和回滚，Redis 事务只是命令打包执行，错了就错了不会回滚。

## 追问预演

**Q：Redis 事务为什么不支持回滚？**
A：作者认为命令错误是编程错误，应该在开发阶段避免，且回滚会增加复杂度。

**Q：WATCH 的作用是什么？**
A：乐观锁，监控 key 是否在事务执行前被其他客户端修改，若修改则 EXEC 返回 nil。

**Q：事务执行过程中其他命令能插入吗？**
A：不能，Redis 是单线程，事务中的命令是连续执行的。
