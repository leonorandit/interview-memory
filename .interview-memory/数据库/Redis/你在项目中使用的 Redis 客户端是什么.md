# 你在项目中使用的 Redis 客户端是什么

## 速记卡片

| 锚点字 | 口诀 | 核心差异 |
|--------|------|----------|
| 客 | 客J红，三选型 | Jedis直连、Lettuce异步、Redisson分布式 |

## 原始知识

- **Jedis**：直连 Redis，**线程不安全**，需配连接池
- **Lettuce**：基于 Netty，**线程安全**，支持异步 / Reactive
- **Redisson**：分布式锁 / 集合 / 队列等**高级功能**
- **Spring Boot 2.x 默认使用 Lettuce**
- 选型看场景：简单直连选 Jedis，高并发异步选 Lettuce，分布式工具选 Redisson

## 顺口溜

客J红，三选型，
直连异步分布式。

## 单字锚点 + 防崩推导

- **锚点字**：客
- **扩展口诀**：客 J 红，三选型
- **为什么选它**：问题问的是"客户端"，"客"字一出，立刻定位到客户端选型
- **推导链**：
  1. 客 → Redis 客户端
  2. J → Jedis（直连、线程不安全、连接池）
  3. 红 → Redisson（分布式高级功能）
  4. 中间缺一个 → Lettuce（Netty、异步、Spring Boot 2.x 默认）
  5. 三选型 → 三个主流客户端

## 一字一画面

- **客** —— 像[酒店前台]：三种服务员迎客，Jedis 是单人接待（忙不过来要排队），Lettuce 是多线程智能前台（同时处理多路），Redisson 是 VIP 管家（提供分布式专属服务）。（本质：客户端是应用与 Redis 之间的服务入口）
- **J** —— 像[单刀直入的快递]：一辆摩托车直送到门，快但一次只能送一单，多了要组建车队（连接池）。（本质：Jedis 是同步直连模型）
- **红** —— 像[红色保险箱]：不仅存取，还能上分布式锁、做分布式队列，是企业级工具箱。（本质：Redisson 封装了分布式场景的高级数据结构）

## 面试话术

"我们项目里主要用过 Jedis 和 Lettuce，Spring Boot 2.x 之后默认就是 Lettuce，它基于 Netty 所以线程安全，还支持异步和 Reactive。如果要做分布式锁或者分布式集合，那会用 Redisson，它对这类场景封装得比较好。"

## 防混补丁

容易和 Redis 数据类型搞混？记住：客是"门外客"，站在应用和 Redis 之间；数据类型是"屋里货"，存在 Redis 里头。

## 追问预演

- **Q：Jedis 为什么线程不安全？**
  - A：Jedis 底层用一个 Socket 连接，多线程共用会互相干扰，必须用连接池每个线程拿独立实例。
- **Q：Lettuce 为什么线程安全？**
  - A：基于 Netty 实现单线程事件循环，多线程共享同一个连接，通过异步非阻塞 IO 处理。
- **Q：你们为什么选 Lettuce 而不是 Jedis？**
  - A：Spring Boot 2.x 默认集成，省去额外配置；异步支持在高并发下性能更好。
- **Q：Redisson 的分布式锁原理是什么？**
  - A：基于 Redis 的 hash 结构加 Lua 脚本保证原子性，支持看门狗自动续期。

## 关联知识点

- Redis 线程模型（单线程 IO 多路复用）
- Netty 异步非阻塞网络框架
- Spring Boot 自动配置原理
- Redis 分布式锁实现方案
