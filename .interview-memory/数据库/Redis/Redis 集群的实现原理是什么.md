## 速记卡片

槽 | 槽分迁，三原理 | 哈希槽分片+Gossip通信+槽迁移

## 原始知识

- Redis Cluster模式
- **16384**个哈希槽
- CRC16(key)%16384分配槽
- 节点间**Gossip**协议通信
- 槽迁移扩容缩容
- 客户端直连任意节点
- **MOVED/ASK**重定向

## 顺口溜

槽分一万六，CRC来指路。  
Gossip传消息，迁移扩缩容。  
客户端直连，MOVED来转向。

## 单字锚点 + 防崩推导

**锚点字**：槽

**扩展口诀**：槽分迁，三原理

**为什么选它**：槽是 Redis Cluster 最核心的概念，所有数据都挂在槽上，没有槽就没有集群分片。

**推导链**：
槽 → Redis Cluster 有 16384 个哈希槽 → 每个 key 经 CRC16 取模落到某个槽 → 槽绑定到具体节点 → 客户端连任意节点，若槽不在本节点则返回 MOVED/ASK 重定向 → 节点间通过 Gossip 协议交换状态 → 槽可在线迁移实现扩容缩容

## 一字一画面

**槽** —— 像赌场筹码分桌：荷官把客人（key）按筹码颜色（CRC16）分到 16384 张桌子（槽），每张桌子固定属于某个庄家（节点），客人走错桌会被指引到正确位置（MOVED/ASK），庄家之间用对讲机互通消息（Gossip），赌桌可以随时搬去别家（槽迁移）。（本质：一致性哈希的数据分片实现）

## 面试话术

Redis Cluster 的实现原理核心就是哈希槽。整个集群有 16384 个槽，每个 key 通过 CRC16 取模决定落在哪个槽上，每个节点负责一部分槽。客户端可以连任意节点，如果 key 不在这个节点上，会返回 MOVED 或者 ASK 重定向。节点之间通过 Gossip 协议交换状态，槽可以在线迁移来实现扩容缩容。

## 防混补丁

容易和一致性哈希搞混？槽是固定分片，一致性哈希是环形动态映射，记住"槽是预制格子，一致性哈希是弹性橡皮筋"。

## 追问预演

1. **为什么是 16384 个槽而不是更多？**
   答：心跳包携带槽信息，16384 个 bit 刚好 2KB，既够用又不占带宽。
2. **MOVED 和 ASK 的区别？**
   答：MOVED 是槽已永久迁移，ASK 是槽正在迁移中的临时重定向。
3. **Gossip 协议的具体工作方式？**
   答：每个节点定期随机选几个节点交换信息，包括节点状态、槽分布、集群配置等。
