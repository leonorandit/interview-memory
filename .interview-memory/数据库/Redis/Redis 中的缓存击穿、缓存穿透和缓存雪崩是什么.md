# Redis 中的缓存击穿、缓存穿透和缓存雪崩是什么？

## 速记卡片

| 锚点字 | 口诀 | 核心差异 |
|--------|------|----------|
| **穿** | 穿透崩，三问题 | 击穿是热点过期，穿透是查空值，雪崩是集体过期 |

## 原始知识

- **缓存击穿**：**热点 key 过期瞬间**，大量请求打到 DB
  - 解法：**互斥锁**（只有一个线程回源）、**永不过期**（逻辑过期）
- **缓存穿透**：查询**不存在的 key**，缓存和 DB 都为空
  - 解法：**布隆过滤器**、**缓存空值**（设置短 TTL）
- **缓存雪崩**：**大量 key 同时过期**，DB 压力暴增
  - 解法：**随机过期时间**、**多级缓存**、**熔断降级**
- **三者区别**：
  - 击穿 = **一个热点 key** 过期
  - 穿透 = key **不存在**
  - 雪崩 = **大量 key** 同时过期

## 顺口溜

**穿透崩，三问题**
> 击穿热点过期了，穿透查空 DB 恼，雪崩一片全倒掉

## 单字锚点 + 防崩推导

- **锚点字**：**穿**
- **扩展口诀**：穿透崩，三问题
- **为什么选它**："穿"是击穿，也是穿透，两个字都以"穿"开头，天然好记；雪崩虽然不搭"穿"字，但"崩"和"穿"押韵，组成"穿透崩"朗朗上口
- **推导链**：
  1. 穿 → **击穿** → 一个热点 key **过期** → 像墙上破了一个洞，请求漏到 DB
  2. 穿 → **穿透** → 查询**不存在的 key** → 像子弹穿过缓存和 DB 两层靶子，直接打穿
  3. 崩 → **雪崩** → **大量 key 同时过期** → 像雪山集体崩塌，DB 瞬间被埋
  4. 击穿解法 → 互斥锁 / 永不过期
  5. 穿透解法 → 布隆过滤器 / 缓存空值
  6. 雪崩解法 → 随机 TTL / 多级缓存 / 熔断

## 一字一画面

- **击** —— 像**城墙上唯一的大门突然坏了**：敌军（请求）全部涌向这个缺口，守军（DB）瞬间被冲垮（本质：单热点 key 失效，集中回源）
- **透** —— 像**用霰弹枪打一张纸**：子弹穿过纸（缓存），又穿过后面的纸（DB），啥也没打到，但墙穿了两个洞（本质：key 不存在，每次请求都直达 DB）
- **崩** —— 像**雪山所有支撑点同时融化**：轰隆隆整片雪山塌下来，山下村庄（DB）被淹没（本质：大量 key 集中失效，瞬时回源压垮 DB）

## 面试话术

> "这三个问题我记为'穿透崩'——击穿是热点 key 过期导致大量请求打到 DB，用互斥锁或永不过期解决；穿透是查询不存在的 key，缓存和 DB 都拦不住，用布隆过滤器或缓存空值解决；雪崩是大量 key 同时过期，用随机过期时间、多级缓存和熔断降级来解决。"

## 防混补丁

- **容易和什么搞混**：击穿和雪崩都涉及 key 过期，但击穿是"一个"，雪崩是"一批"
- **补丁口诀**：击穿是一个洞，雪崩是整面墙塌了

## 追问预演

| 追问 | 简要答案 |
|------|----------|
| 互斥锁怎么实现？ | 用 Redis setnx 抢锁，抢到才查 DB 并回写缓存，其他线程等待或降级 |
| 布隆过滤器有什么缺点？ | 存在误判（可能把不存在的判断为存在），不支持删除 |
| 缓存空值 TTL 设多久？ | 一般设较短，比如 30 秒~5 分钟，防止攻击者随机 key 占满内存 |
| 多级缓存怎么设计？ | L1 本地缓存（Caffeine）→ L2 Redis → L3 DB，每层都有过期策略 |
