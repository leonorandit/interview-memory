# 面试题顺口溜记忆法

## 核心任务
把知识点变成**3秒想起、能直接说**的面试武器。链路：听到题 → 蹦口诀 → 逐字展开 → 流畅表达。

**每次输出后，必须将结果写入 `.interview-memory/` 下的对应分类目录中。**

---

## 知识类型 → 口诀策略

| 类型 | 特征 | 策略 | 例子 |
|------|------|------|------|
| 对比型 | A vs B | 对偶句式 | TCP/UDP，GET/POST |
| 流程型 | 有步骤 | 动作链/故事线 | HTTPS握手，GC |
| 列举型 | 多并列 | 首字提取+谐音 | 设计原则，细胞器 |
| 因果型 | 原理/为什么 | 逻辑压缩 | B+树，索引失效 |
| **连环型** | 多问串联 | 大故事套小口诀 | 输入URL全过程 |

> **连环型处理**：把整个过程编成一个**大故事口诀**，每个环节再用小口诀细化。例如"输网THT渲"（输入URL → DNS → TCP → HTTP → 渲染），每个字展开一个小口诀。

## 输出格式（严格遵循）

### 1. 场景触发词
会触发这个知识点的面试题关键词。

### 2. 速记卡片
一行：`锚点字 | 口诀 | 核心差异`
> 例：`连 | TCP连可拥，UDP快不管 | 连vs断`

### 3. 原始知识
- bullet 列要点（每项≤10字）
- **加粗易混点、追问点**

### 4. 顺口溜
≤7字为优，最多14字。必须能念出声、有节奏、押韵。

### 5. 单字锚点 + 防崩推导
- **锚点字**：信息密度最高的字
- **扩展口诀**：锚点字开头，≤7字
- **为什么选它**：一句话解释钩子力
- **防崩推导**：只记得这个字时，如何一步步推出其他要点？写推导链。

### 6. 一字一画面
格式：**X** —— 像 **[具体事物]**：[5秒画面]（本质：...）
- 画面具体到颜色/动作
- 比喻间形成因果链或空间链

### 7. 面试话术
2~3句可直接背的面试回答。口语化，先抛口诀再展开术语。

### 8. 防混补丁
容易和什么搞混？一句补丁口诀加固。

## 领域知识链路图（必须维护）

### 什么是知识链路图
**知识链路图是同一个领域内所有知识点的全局流程图**，展示知识点之间的**上下游依赖关系**——不是单个文件里的零散关联链接，而是站在领域视角的"知识地图"。

### 为什么要做链路图
- **面试连环追问**：面试官可能从"RAG完整流程"追问到"混合检索"再追问到"Rerank"，链路图让你提前知道追问路径
- **串联复习**：考前扫一眼链路图，整个领域的知识体系一目了然
- **发现盲区**：链路图里的空白节点就是还没覆盖的知识点

### 链路图画法
在每个领域的 `_INDEX.md` 文件末尾，用文本流程图画出该领域所有知识点的串联关系：

```
上游（输入） → 核心流程 → 下游（输出/优化）
     ↓              ↓              ↓
  前置知识      主链路节点      后置优化
```

**规则**：
- 用 `→` 表示上下游依赖（前面的知识点是后面的前置条件）
- 用 `+` 表示并列关系（同一层的不同分支）
- 用括号标注**锚点字口诀**，一眼定位
- 横向是流程顺序，纵向是并列分支

### 示例：RAG领域知识链路图

```
数据层：洗整去重标（数据清洗和预处理）
           ↓
查询层：一扩多召回（查询扩展）
           ↓
流程层：切存查排合生（RAG完整流程）
           ↓
检索层：稀稠混（混合检索）
           ↓
精排层：粗召精排（Rerank）
```

### 示例：A2A/Agent领域知识链路图

```
设计层：找宣管安用（A2A设计原则）
           ↓
协议层：找证派做交（A2A工作原理）
           ↓
框架层：搭编工话部（Google ADK）
```

### 更新规则
- 每新增一个知识点，**必须更新该领域的链路图**
- 如果新知识点无法融入现有链路，**单独列出或开辟新分支**
- 链路图中的节点必须与 `_INDEX.md` 表格中的知识点一一对应

## 好口诀 vs 坏口诀

| 坏口诀 | 为什么坏 | 改成好口诀 |
|--------|----------|------------|
| "TCP有连接，UDP无连接" | 只是原文缩写，没压缩信息 | **TCP连可拥，UDP快不管** |
| "线粒体是动力车间" | 太长，没有锚点字 | **线叶核，内高溶** |
| "HTTPS握手五步很复杂" | 没有画面，无法联想 | **我问你要，你给我证，我发钥匙，一起开门** |

**坏口诀的共同特征**：只是原文缩句、超过14字、没有锚点字、念起来没节奏、画面感为零。

## 示例

### 对比型：TCP vs UDP

**场景触发词**：TCP和UDP区别、TCP为什么可靠、UDP用在哪

**速记卡片**：`连 | TCP连可拥，UDP快不管 | 连vs断`

**原始知识**：
- TCP：面向连接、可靠传输、字节流、拥塞控制、首部20字节
- UDP：无连接、不可靠、数据报、无拥塞控制、首部8字节
- **易混点**：谁快谁慢
- **追问点**：三次握手为什么不是两次？TIME_WAIT？

**顺口溜**：**TCP连可拥，UDP快不管**

**单字锚点 + 防崩推导**：
- **锚点字**：**连**
- **扩展口诀**：**连保稳，断不管**
- **为什么选它**：TCP的灵魂是"连"，UDP的灵魂是"断"。先想"连不连"。
- **防崩推导**：
  - 连 → 要建立连接 → 三次握手/四次挥手
  - 连 → 保证对方在 → ACK确认 → 可靠传输
  - 连 → 怕路堵 → 拥塞控制
  - 反过来：断 → 不管 → 快

**一字一画面**：
- **连** —— 像**打电话**：先拨号建立连接（三次握手），对方接了才说话，说完确认"听懂了没"（ACK），最后挂电话（四次挥手）。（面向连接）
- **保** —— 像**快递保价**：丢了重发，乱序重组。（可靠传输）
- **稳** —— 像**限速高速**：车多自动减速。（拥塞控制）
- **断** —— 像**发短信**：不管对方在不在，发出去就完事。（无连接）
- **不管** —— 像**大喇叭广播**：一个人喊一群人听，不点名，快但不管谁漏听。（不可靠、多播高效）

**面试话术**：
- 问区别 → "核心区别是**连不连**。TCP面向连接，像打电话要先建立连接；UDP无连接，像发短信直接发。TCP有确认、重传、拥塞控制，可靠但慢；UDP快但不可靠。TCP用于HTTP、文件传输；UDP用于直播、DNS、游戏。"

**防混补丁**：
- 容易和"TCP/IP模型"搞混。TCP是传输层协议，TCP/IP是四层体系。补丁：**TCP是车道，TCP/IP是地图**。

### 流程型：HTTPS握手

**场景触发词**：HTTPS握手、HTTPS为什么安全、SSL/TLS握手

**速记卡片**：`证 | 我问你要，你给我证，我发钥匙，一起开门 | 先看证`

**顺口溜**：**我问你要，你给我证，我发钥匙，一起开门**

**单字锚点 + 防崩推导**：
- **锚点字**：**证**
- **扩展口诀**：**先看证，再发钥**
- **为什么选它**：HTTPS一切安全建立在证书上。没有证，后面全崩。
- **防崩推导**：证 → 验证身份（防中间人）→ 拿到公钥 → 安全发预主密钥 → 生成对称会话密钥 → 之后对称加密（快）

**一字一画面**：
- **问** —— 像**进小区问保安**："你们支持什么门禁卡？"（Client Hello，协商加密套件）
- **给** —— 保安**掏工作证**："我是正规保安。"（Server Hello + 证书）
- **证** —— 你**查工作证真伪**：看是不是公安局发的。（验证证书）
- **钥** —— 你**把钥匙装进保安的公用信封**递进去，只有保安能开。（公钥加密预主密钥）
- **开门** —— 双方**各复制一把相同钥匙**，以后用这把钥匙开门。（生成对称会话密钥）

**面试话术**：
- 问握手 → "概括为五步：**我问你要，你给我证，我发钥匙，一起开门**。Client Hello协商算法；Server Hello回证书；客户端验证证书，生成预主密钥用公钥加密发过去；双方生成对称会话密钥；Finished确认后加密通信。"
- 问安全 → "安全来自两点：一是**证书机制**防中间人伪造；二是**非对称加密换对称密钥**，既安全又高性能。"

**防混补丁**：
- 容易和TCP三次握手搞混。TCP三次握手是**传输层连上路**，HTTPS握手是在TCP之上**发钥匙**。补丁：**先TCP连上路，再HTTPS发钥匙**。

### 列举型：Java七大设计原则

**场景触发词**：Java设计原则、SOLID原则、怎么写出好代码

**速记卡片**：`单 | 单开里，接依迪合 → 单车里，接一迪鹤 | 首字串联`

**顺口溜**：**单开里，接依迪合** —— *单车里，接一迪鹤*

**单字锚点 + 防崩推导**：
- **锚点字**：**单**
- **扩展口诀**：**单车里，接一迪鹤**
- **为什么选它**："单"是第一原则，谐音"单车"唤起整个画面。
- **防崩推导**：单（一辆车一件事）→ 开（换坐垫不改车架）→ 里（电动替换普通单车）→ 接（接单只接能送的）→ 依（依赖平台不依赖客户）→ 迪（迪鹤只认驯养员）→ 合（组合优于继承）

**一字一画面**：
- **单** —— 像**单车只骑一个人**：一辆车只干一件事。（单一职责）
- **开** —— 像**单车能换坐垫**：不改车架，加配件升级。（开闭原则）
- **里** —— 像**电动单车替代普通单车**：骑法一样，不能变三轮车让人不会骑。（里氏替换）
- **接** —— 像**接单只接能送的**：别接超重单，整车散架。（接口隔离）
- **依** —— 像**接单依赖平台不依赖客户**：面向抽象编程。（依赖倒置）
- **迪** —— 像**迪鹤只认驯养员**：不认识的不管，只和直接朋友说话。（迪米特法则）
- **合** —— 像**合成双人单车**：两个单车站一起（组合），不是焊死成连体车（继承）。（合成复用）

**面试话术**：
- 问原则 → "七个原则，口诀是**单开里，接依迪合**。比如单一职责，一个类只负责一件事；开闭原则，对扩展开放、对修改关闭。最近项目在XX模块用了接口隔离，把大接口拆小，避免客户端依赖不需要的方法。"

**防混补丁**：
- 容易漏掉"合成复用"，只记得SOLID五个。补丁：**SOLID五 principle，加上迪鹤和合单车是七个**。
